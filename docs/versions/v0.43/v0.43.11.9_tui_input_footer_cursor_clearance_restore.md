# v0.43.11.9 TUI Input/Footer Cursor Clearance Restore

## Restore Purpose

v0.43.11.9 repairs the Schumpeter full-screen TUI bottom layout so the input text and cursor are not clipped or visually crowded by the footer/status line.

## User-Observed Input Clearance Problem

The input field sat too close to the bottom edge. Korean input such as `넌 누구야` appeared squeezed, and the status row below it competed visually with the input border and cursor line.

## v0.43.11.9 Goal

Guarantee input visibility and cursor clearance while preserving the v0.43.11 native bubble UX and navy-white visual direction.

## Bottom Layout Contract

The bottom area is split into three separate regions:

1. `palette-region`
2. `input-region`
3. `footer-status-region`

The main chat remains `height: 1fr`; the input and status rows are reserved separately.

## Command Palette Region

`palette-region` sits above `input-region`. When the slash palette opens, it must not cover the input line.

## Input Region

`input-region` reserves four rows. The input widget itself reserves three rows and keeps a one-row bottom clearance margin before the footer/status region.

## Footer Status Region

`footer-status-region` reserves one row below the input region. It contains `status-bar` and does not share the input widget's vertical space.

## Cursor Clearance

The cursor line must remain fully visible for Korean/CJK input. Input text must not overlap the status row or terminal bottom edge.

## Snapshot Tests

Helper snapshots verify:

- `InputRegion` appears before `FooterStatusRegion`;
- input text remains present;
- status/footer remains present;
- 140x42, 110x32, and 80x24 snapshots remain width-safe;
- no high-risk rendering side effects are introduced.

## Manual Acceptance

Run:

```powershell
$env:CHANTACORE_HOME="$env:LOCALAPPDATA\ChantaCore"
chanta-cli start
```

Inside the TUI:

```text
넌 누구야
/provider
/summary 오늘 입력창과 footer clearance를 테스트하고 있어.
/what-happened
/exit
```

Pass if the Korean input and cursor are fully visible, the input border is not overlapped, the status row is clearly below the input, and the slash palette opens above the input.

## Safety Boundary

This is a visual/layout hotfix only. It does not open workspace read, repo search, shell/git execution, file edit/apply inside Schumpeter, provider tool/function calling, subagents, memory mutation, CORE_MEMORY write, or production certification.

## Withdrawal Conditions

Withdraw this repair if:

- input text is clipped;
- the cursor is partly hidden;
- status/footer overlaps the input border;
- slash palette covers the input;
- narrow terminal resize hides the input;
- rendering introduces runtime side effects.

## v0.44 Gate Recommendation

Do not proceed to v0.44 from this change alone. First complete a manual fullscreen TUI check focused on Korean input, cursor visibility, slash palette position, and footer separation.
