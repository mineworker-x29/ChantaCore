# v0.43.11.8 Native Conversation Bubble UX Restore

## Restore Purpose

v0.43.11.8 repairs the Schumpeter full-screen TUI transcript after the ASCII card hotfix proved visually too console-like. This is a visual and interaction-surface repair only.

## User-Observed Native Bubble Problem

The prior transcript used literal `+`, `-`, `|`, and `=` characters to draw message boxes inside the text stream. That separated messages technically, but it made the chat look like a log or table rather than a native conversation surface. The default Textual header also exposed the internal application class name.

## v0.43.11.8 Goal

Use actual Textual widget hierarchy and TCSS styling for conversation bubbles while preserving the navy-white visual direction. Remove the default Textual header so internal app names are not visible.

## Native Textual Bubble Contract

Conversation messages must be rendered as widgets:

- row container;
- message bubble container;
- speaker label widget;
- message body widget;
- semantic CSS classes per kind.

The message body string contains only message content. It does not contain manual borders, repeated rules, or box-drawing characters.

## Header Removal

The app no longer composes Textual's default `Header` widget. The product identity remains in the left sidebar. The main layout starts directly at the terminal top edge.

## Navy-White Visual Refinement

The theme keeps the dark navy base and white/blue text direction:

- screen background: deep navy;
- sidebar: slightly raised navy;
- message bubbles: restrained navy surfaces;
- assistant accent: white;
- user accent: pale blue;
- status/diagnostic/error accents remain subdued.

## Status Artifact Diagnostic Error Surfaces

Status, artifact, diagnostic, and unavailable/error outputs remain distinct through `.message-card.status`, `.message-card.artifact`, `.message-card.diagnostic`, and `.message-card.error`. They use widget styling rather than text frames.

## Snapshot Tests

Helper snapshots verify:

- no `SchumpeterTextualApp` visible;
- no default top header;
- no ASCII message frames;
- semantic labels present;
- input remains visible;
- width does not overflow at 140x42, 110x32, and 80x24.

## Manual Acceptance

Run:

```powershell
$env:CHANTACORE_HOME="$env:LOCALAPPDATA\ChantaCore"
chanta-cli start
```

Inside the TUI:

```text

/what-happened
저장소 상태 확인해줘
/exit
```

Pass if the transcript looks like native conversation bubbles, not ASCII tables, the default Textual header is gone, and the navy-white tone is preserved.

## Safety Boundary

This release does not open v0.44 capability gates. Rendering must not call providers, submit prompts, run shell or git, read repository/workspace files, invoke tools/functions/subagents, mutate memory, write CORE_MEMORY, or claim production certification.

## Withdrawal Conditions

Withdraw this implementation if:

- any message body includes manual frame characters;
- `SchumpeterTextualApp` appears to the user;
- the default Textual header strip returns;
- transcript widgets stop scrolling;
- input becomes hidden;
- status/artifact/diagnostic/error surfaces become indistinguishable;
- rendering opens any high-risk runtime capability.

## v0.44 Gate Recommendation

Do not proceed to v0.44 from this repair alone. First complete manual visual acceptance for the native bubble UI. If it passes, either perform one restrained v0.43.11.9 polish pass or start a separate v0.44 gate review.
