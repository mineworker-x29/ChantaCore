Only local work notes and memory candidates are open for bounded append in v0.43.2.

## Session Context vs Persistent Memory

Session context is recent conversation and work-session state. It can guide current answers through bounded context selection. It is not persistent memory.

Persistent memory means durable long-term memory such as `CORE_MEMORY.md` or an equivalent long-term store. v0.43.2 does not write persistent memory.

## Business Artifacts vs Local Notes

A business artifact is a structured assistant output, such as summary, memo, todo, decision brief, or handoff note. It can be referenced in the current session. It is not automatically persistent memory.

A local work note is explicitly user-created bounded operational evidence stored under:

`profiles/default-personal/state/work_notes/notes.jsonl`

Local notes are not CORE_MEMORY and not persona/identity memory.

## Feedback vs Memory

Feedback records capture friction, bugs, UX notes, and Process Intelligence review observations. Feedback is improvement evidence. It is not persistent memory and should not be treated as answer context unless explicitly requested.

## Memory Candidates

A memory candidate is a proposed item that might be useful to remember later.

Memory candidates are stored under:

`profiles/default-personal/state/work_notes/memory_candidates.jsonl`

They are not promoted automatically. Promotion requires a future explicit memory gate that is not open in v0.43.2.

## Note Commands

Slash commands:

* `/note <text>` appends a local work note
* `/notes` lists recent local work notes
* `/note last` shows latest note
* `/note from-artifact` saves a summary of the last artifact as a local note
* `/notes search <query>` searches only the local note store

These commands do not call provider, submit prompts, execute shell, invoke subagents, mutate CORE_MEMORY, or write workspace files.

## Memory Boundary Command

`/memory-boundary` explains:

* session context
* business artifacts
* local work notes
* feedback
* memory candidates
* persistent memory
* what is open now
* what remains closed

It is deterministic and does not call provider.

## Context Boundary Command

`/context` shows what ChantaCore can currently use:

* explicit user content
* bounded recent session turns
* last business artifact
* last run/report summaries
* trace summary
* local notes only when explicitly shown or searched
* feedback only as improvement evidence unless explicitly requested

It must state that arbitrary files and repo search are unavailable.
