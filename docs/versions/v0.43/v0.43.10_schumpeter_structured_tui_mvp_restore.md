## v0.43.9 Architecture Baseline

v0.43.9 defined RuntimeAdapter, UIState, component boundaries, renderer boundaries, display-width-safe layout, and deterministic snapshot mode. v0.43.10 extends that contract into a preview TUI without bypassing the adapter.

## v0.43.10 Goal

Open a usable structured TUI preview that can be visually inspected and lightly interacted with, while keeping runtime capability gates closed.

## Structured TUI MVP Scope

The MVP includes sidebar, main chat, transcript, input box, status line, command hints, slash palette fallback, snapshot rendering, and a plain interactive loop. It may use prompt_toolkit when available, but falls back to plain input.

## TUI Entrypoint Policy

Primary command: `chanta-cli tui`.

Aliases: `chanta-cli start --tui` and `chanta-cli tui snapshot`.

`chanta-cli start` remains the stable existing flow and is not replaced by default.

## prompt_toolkit / Plain Fallback Policy

prompt_toolkit is detected lazily. Import failure does not break package import. Plain fallback prints a structured snapshot, accepts line input, routes through the RuntimeAdapter, and exits on `/exit`, EOF, or Ctrl-C.

## RuntimeAdapter Contract

RuntimeAdapter is the only TUI bridge to the existing work-session runtime. Rendering, sidebar, status, palette, and snapshot collection are side-effect-free. User-submitted text goes through the adapter, which delegates to the existing work-session input path.

## TUI App State Model

`V04310TUIAppState` contains runtime snapshot, transcript, sidebar, main panel, input state, palette state, status line, busy/exit flags, and no secret/debug metadata flags. It does not contain API keys, provider base URLs, raw trace dumps, raw safety footers, or production certification claims.

## Sidebar Component

Sidebar renders product identity, project, session, profile/provider/mode, PI monitor, and shortcut commands from state only.

## Main Chat Component

Main chat renders the welcome banner and transcript messages for user, assistant, artifact, diagnostic, status, error, and notice surfaces.

## Input Box Component

Input box renders the placeholder `Ask Schumpeter anything...` and the current line input state.

## Status Bar / PI Monitor

Status line renders compact PI, provider, trace, evidence, safety, and version labels. Details remain available through explicit status surfaces.

## Slash Command Palette

Slash command completion and palette reuse the v0.43.8.2 deterministic registry. Completion inserts text only and never executes before Enter.

## Renderer Boundary

Message, artifact, diagnostic, status, and error rendering are separated. Normal and artifact renderers hide raw metadata. Diagnostics are explicit. Errors are clean and do not show stack traces by default.

## Snapshot Mode

Snapshot mode renders one frame and exits. It supports width 120, width 100, width 80, plain mode, and palette projection.

## Manual UI Acceptance Guide

Open `chanta-cli tui`, confirm Schumpeter branding, sidebar, main chat, input, status, PI monitor, hints, and clean output. Test `/help`, `/status`, `/`, normal Korean text, `/summary`, `/artifact last`, `/what-happened`, `/v044 readiness`, and `/exit`.

## No-Side-Effect Policy

Rendering, sidebar, status, palette, snapshot, and completion must not call providers, submit prompts, run shell/git, search repo/workspace, mutate memory, or write CORE_MEMORY.

## Safety Boundary

Still closed: workspace read, arbitrary file read, repo search, workspace search, git status execution, shell execution, file edit/apply, provider tool calling, function calling, subagents, child sessions, general AgentLoop expansion, autonomous coding, memory mutation by rendering, CORE_MEMORY write, CLI/package/module rename, and production certification.
