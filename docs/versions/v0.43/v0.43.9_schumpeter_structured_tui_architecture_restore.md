## User UX Reference and Goal

The target shape is a modern structured terminal work surface with a left sidebar, main transcript, input box, command hints, and compact status. v0.43.9 creates the testable architecture and snapshot harness only.

## Structured TUI Architecture Principle

Runtime Core is not the TUI shell. RuntimeAdapter is not UIState. UIState is not component code. Components are not renderers. Renderers do not call runtime actions.

The UI is a projection of runtime state.

## Runtime Core vs TUI Shell Boundary

The v0.43.9 TUI package must not submit prompts, call providers, inspect repositories, search workspaces, run shell or git, mutate memory, write CORE_MEMORY, invoke subagents, or open child sessions.

## RuntimeAdapter Contract

`V0439RuntimeAdapterPolicy` keeps provider completion, prompt submission, shell, git, repo search, workspace read, memory mutation, and CORE_MEMORY write closed. Unavailable bounded sources return `unknown` or display-safe fallback values.

`V0439RuntimeSnapshot` contains display labels and side-effect flags only.

## UIState Model

`V0439UIState` is display-only. It includes product name, subtitle, profile, provider label, mode, working directory label, project/session labels, PI/provider/trace/evidence/safety status, messages, command hints, optional command palette state, version label, and render mode.

It does not include secrets, provider base URL, raw trace dumps, raw safety flags, or raw debug metadata by default.

## Component Boundary

Components are pure renderers from UIState or component state. v0.43.9 defines sidebar, header, chat view, input box, status bar, command palette, and code/artifact block components.

## Renderer Boundary

Renderers are separated into message, artifact, diagnostic, status, and error renderers. Default renderers hide raw metadata. Diagnostic/debug surfaces must be explicit.

## Layout Contract

The snapshot supports two-column layout for wide widths, compact stacked layout for narrow widths, and plain mode when Unicode box drawing should be avoided.

## Display Width and Terminal Compatibility

All padding and truncation use terminal display width helpers. ANSI escapes are stripped before width calculation. Korean/CJK double-width characters are handled explicitly.

## Command Palette Integration Contract

The TUI command palette reuses the v0.43.8.2 slash command registry. Palette rendering shows commands but does not execute them, call providers, run shell/git, read repositories, read workspaces, or mutate memory.

## Snapshot Mode Contract

Snapshot mode renders one deterministic frame and exits. It supports width 120, width 100, width 80, plain mode, and optional command palette projection.

Implemented commands:

```powershell
chanta-cli tui snapshot --width 120
chanta-cli tui snapshot --width 100
chanta-cli tui snapshot --width 80
chanta-cli tui snapshot --plain
chanta-cli tui snapshot --width 120 --palette
chanta-cli start --snapshot --width 120
```

## Golden Snapshot Acceptance

Golden snapshots verify Schumpeter branding, sidebar, main chat, input box, command hints, PI monitor, status bar, compact fallback, plain mode, command palette projection, forbidden-string absence, and no side-effect flags.

## No-Side-Effect Policy

`V0439NoSideEffectPolicy` states that rendering may not call providers, submit prompts, run shell, run git, read repositories, open workspace read, mutate memory, or write CORE_MEMORY. `production_certified=False`.

## Forbidden Runtime Calls

v0.43.9 snapshot code must not introduce subprocess execution, `shell=True`, `os.system`, eval/exec, curses, required Textual imports, apply operations, git operations, subagent invocation, broad workspace scanning, provider completion, workspace read, repo search, memory mutation, or CORE_MEMORY writes.

## Still-Closed Capabilities

Still closed: full interactive TUI, workspace read, arbitrary file read, repo search, workspace search, git status execution, shell execution, file edit/apply, provider tool calling, function calling, subagents, general AgentLoop expansion, autonomous coding, memory mutation, CORE_MEMORY write, package/module/CLI rename, and production certification.
