# ChantaCore v0.43.7 Default Conversation Router & Minimal Output Restore

이 문서는 ChantaCore v0.43.7을 새 Codex/GPT 세션에서 복원하기 위한 단일 통합 문서다. 목적은 v0.44로 넘어가기 전에 기본 대화 UX를 고치는 것이다.

v0.43.7은 기능 확장이 아니다. workspace read, repo search, shell, edit/apply, provider tool/function calling, subagent, memory mutation, CORE_MEMORY write, production certification을 열지 않는다.

## User-Observed UX Failure

Observed failure:

* Plain `넌 누구야` rendered as a summary artifact.
* Default output exposed `type: summary`, `grounding:`, `confidence:`, `verification_required:`, `source:`, and raw `safety:` fields.
* `## 핵심 요약` appeared twice.
* Empty sections such as `## 배경 / 맥락` followed by `알 수 없음` were shown.
* Repository status requests were treated like business summary generation rather than capability-boundary status answers.

The root issue is router/renderer leakage. ChantaCore preserved traceability internally, but rendered internal artifact/debug state as normal user-facing text.

## OpenCode Interaction Contract Lesson

The relevant OpenCode lesson is not to copy coding-agent capabilities. The useful lesson is the interaction contract:

* runtime state is internal
* message parts and event state are internal
* tool, permission, status, and debug state are internal unless explicitly requested
* normal chat renders assistant text, not internal objects
* explicit commands use command-specific renderers
* debug/report/trace commands intentionally expose metadata
* stored evidence is not the same thing as rendered text

ChantaCore adopts this contract without opening tool execution, workspace read, repo search, or shell execution.

## v0.43.7 Goal

v0.43.7 must guarantee:

* Plain user text does not become summary artifact by default.
* `넌 누구야` routes to `identity_question`.
* Repository status requests route to `repository_status_request`.
* Minimal default answers hide raw metadata.
* Explicit artifact commands still render structured artifacts.
* Debug/report/trace surfaces still expose metadata when explicitly requested.
* Duplicate headings and empty unknown sections are suppressed.
* v0.44 remains blocked until golden transcript checks pass.

## Conversation Intent Router

Required intent kinds:

* `identity_question`
* `capability_question`
* `runtime_status_question`
* `repository_status_request`
* `general_chat`
* `explicit_artifact_command`
* `evidence_command`
* `grounded_command`
* `pilot_command`
* `debug_or_report_command`
* `unknown`

Routing rule:

Plain text defaults to `general_chat` or a specific minimal/status intent. It must not default to `explicit_artifact_command` unless the user uses a slash artifact command or clearly asks for a summary.

## Renderer Separation

Renderer contract:

* `MinimalConversationRenderer`: identity, capability, runtime status, and general chat.
* `StatusRenderer`: repository-boundary status, v0.44 readiness, capability limitations.
* `ArtifactRenderer`: explicit `/summary`, `/todo`, `/memo`, `/decision`, `/handoff`, `/artifact last`.
* `GroundedArtifactRenderer`: explicit grounded synthesis commands.
* `DiagnosticRenderer`: `/what-happened`, run/report/trace style commands.
* `DebugRenderer`: explicit debug variants such as `/artifact last --debug`.
* `ErrorRenderer`: unavailable capability boundaries.

Internal PI metadata remains stored, but renderer choice determines what is visible.

## Minimal Conversation Renderer

Minimal default output rules:

* no `업무 요약`
* no `type: summary`
* no raw `grounding:`
* no raw `confidence:`
* no raw `verification_required:`
* no raw `source:`
* no raw `safety:`
* no raw `shell=false`
* no raw `production_certified=false`
* no duplicate headings
* no empty unknown-only sections

Identity answer target:

```text
저는 ChantaCore default-personal runtime에서 동작하는 업무 보조 에이전트입니다.
대화 기반 업무 정리, 요약, TODO 추출, 의사결정 정리, 인수인계문 작성, 로컬 노트와 근거 조회를 도와드릴 수 있습니다.
현재는 저장소 직접 읽기, 파일 수정, 셸 실행, repo search는 열려 있지 않습니다.
```

## Artifact Renderer Boundary

Artifact rendering is allowed only for explicit artifact commands or explicit summary requests.

Default artifact output may show a user-facing title and headings such as `업무 요약`, `## 핵심 요약`, and `## 다음 액션`.

Default artifact output still hides:

* raw `type:`
* raw `grounding:`
* raw `confidence:`
* raw `verification_required:`
* raw `source:`
* raw `safety:`
* raw safety flag footers

## Diagnostic / Debug Renderer Boundary

Diagnostic/debug metadata is visible only when the user explicitly requests it:

* `/what-happened`
* `/run-report`
* `/trace`
* `/artifact last --debug`
* debug/report equivalents

These surfaces may show bounded metadata, but must not accidentally wrap output as summary artifacts.

## Metadata Disclosure Policy

Default visibility:

* grounding: hidden
* confidence: hidden
* verification_required: hidden
* source: hidden
* provider: hidden
* run_id/session_id: hidden
* response_parse_status: hidden
* safety flags: hidden
* production_certified: never shown as raw default footer
* evidence ids: shown only for explicit grounded/evidence commands
* trace ids: shown only in debug/report/trace surfaces

Metadata must never appear as raw key-value spam in normal chat.

## Duplicate Heading Suppression

Renderer cleanup must suppress:

* consecutive duplicate headings
* duplicated artifact title
* repeated `type:` labels
* repeated `## 핵심 요약`

Example:

```text
## 핵심 요약

## 핵심 요약
내용
```

renders as:

```text
## 핵심 요약
내용
```

## Unknown Section Suppression

Default rendering removes sections whose content is empty or unknown-only.

Unknown tokens:

* `알 수 없음`
* `없음`
* `N/A`
* `unknown`
* `Unknown`
* `empty`

Unknowns may be preserved for explicit clarify, grounding-check, or debug surfaces when that uncertainty is the actual product.
