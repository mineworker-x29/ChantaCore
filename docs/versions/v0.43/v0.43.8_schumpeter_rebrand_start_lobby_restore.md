# Schumpeter v0.43.8 Rebrand Surface & Start Lobby Restore

## Restore Purpose

이 문서는 Schumpeter v0.43.8을 새 Codex/GPT 세션에서 복원하기 위한 단일 통합 문서다. v0.43.8의 목적은 ChantaCore 구현 계열을 유지하면서 사용자-facing 제품 표면을 Schumpeter로 정리하고, `chanta-cli start` 첫 화면을 업무 세션용 lobby로 다듬는 것이다.

## One-Screen Restore Summary

Schumpeter v0.43.8 rebrands the user-facing ChantaCore runtime surface as Schumpeter and adds an OpenCode-inspired start lobby for business work sessions.

v0.43.8 adds:

* Schumpeter product branding
* start lobby renderer
* PI status bar
* compact provider/profile/mode row
* command hints
* plain/compact/no-logo fallback
* about card
* noisy output guard
* terminal display-width hotfix for Korean/CJK text
* slash command palette / completion fallback
* v0.44 gate recheck

v0.43.8 does not add:

* workspace read
* arbitrary file read
* repo search
* shell/git execution
* file edit/apply
* provider tool calling
* function calling
* subagents
* memory mutation
* CORE_MEMORY write
* CLI/package/module rename
* production certification

## Schumpeter Start Lobby Concept

The start lobby should feel like entering a clean business work session:

* centered product name
* concise subtitle
* input card
* profile/provider/mode row
* short command hints
* compact PI status bar
* no raw debug or safety footer

## OpenCode-inspired Layout Pattern

The relevant OpenCode lesson is the interaction pattern, not coding-agent capability:

* normal surface is clean
* runtime/event state stays internal
* command and debug surfaces are explicit
* status is summarized first and expanded only by command

Schumpeter copies that interaction contract without opening tools, shell, repo search, or workspace read.

## Terminal Render Modes

Supported helper render modes:

* `full`
* `compact`
* `plain`
* `no_logo`

`chanta-cli start --plain`, `--no-logo`, and `--compact` map to those render paths. Plain mode avoids box drawing.

v0.43.8.1 terminal-width hotfix:

* bordered lobby lines use terminal display width, not Python string length
* Korean/CJK characters are counted as wide terminal cells
* ANSI escape sequences are ignored when measuring display width
* the mixed-language placeholder is split into two lines
* long provider/mode rows are shortened or safely truncated before rendering
* narrow terminals use compact/plain fallback instead of a broken box

## Slash Command Palette / Completion UX

v0.43.8.2 adds a deterministic slash command registry and palette surface.

Interaction contract:

* pressing `/` may show interactive suggestions when `prompt_toolkit` is available
* if interactive completion is unavailable, entering `/` shows the same grouped palette
* `/help commands` shows the same command registry
* `/s` filters commands such as `/summary` and `/status`
* `/grounded` filters grounded workflow commands
* completion inserts command text only and does not execute before Enter

Command groups:

* Workflows
* Artifacts
* Notes
* Evidence
* Grounded Workflows
* Pilot / Release
* System

The palette is UI-only. It does not call providers, submit prompts, run shell/git, search repositories, open workspace read, mutate memory, or write CORE_MEMORY.

Palette output must not show raw metadata, raw safety footers, ChantaGrowthKernel, legacy Schumpeter, provider base URLs, API keys, or secrets.
