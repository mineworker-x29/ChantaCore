## v0.43.5 Goal

## Pilot Review Concept

Pilot review evaluates existing bounded work-session evidence: current session state, recent run reports, trace summaries, local notes, business artifacts, evidence retrieval records, grounded synthesis records, feedback records, diagnostic summaries, and safety reports.

It does not search arbitrary files, search repositories, execute shell, call providers by default, submit prompts by default, mutate memory, or write CORE_MEMORY.

## Metric Dimensions

The metric dimensions are work session usability, business artifact usefulness, evidence retrieval quality, grounded synthesis quality, Process Intelligence reviewability, safety boundary integrity, and v0.44 Controlled Workspace Read readiness.

## Scoring Model

Scores use a deterministic 0.0 to 1.0 scale.

Labels:

* `excellent`: score >= 0.85
* `good`: score >= 0.70
* `acceptable`: score >= 0.55
* `weak`: score >= 0.35
* `blocked`: score < 0.35
* `unknown`: explicitly unknown only

`ready_for_v044_design=True` only when there are no blockers and the safety metric is acceptable or better.

## Work Session Usability Review

The review checks start flow clarity, command discoverability, chat output cleanliness, default/debug separation, user-facing wording, and friction count.

## Business Artifact Usefulness Review

The review checks summary usefulness, TODO actionability, memo readability, decision brief quality, handoff usefulness, facts/assumptions/unknowns separation, and next action clarity.

## Evidence Retrieval Review

The review checks source disclosure, recall usefulness, bounded source coverage, no-result guidance, ranking/snippet usefulness, and searched/not-searched clarity.

## Grounded Synthesis Review

The review checks active evidence pack linkage, evidence id citations, unsupported claim marking, `/grounding-check`, `/evidence used`, and absence of invented evidence ids.

## Process Intelligence Reviewability

The review checks trace/run/session linkage, artifact lineage, evidence lineage, feedback linkage, reconstructable process events, and high-risk counts remaining zero.

## Safety Boundary Review

The review confirms shell, file edit/apply, arbitrary file search, repo search, provider tool/function calling, subagents, memory mutation, CORE_MEMORY write, and production certification remain closed.

## v0.44 Controlled Workspace Read Readiness

v0.44 readiness assesses whether bounded workspace read has clear user value, whether local evidence retrieval is insufficient for workspace-specific tasks, and whether safety preconditions are identified.

The recommended v0.44 scope is Controlled Workspace Read Design only: read-only, bounded, allowlisted, source-disclosed workspace read. It must not begin with edit/apply/shell/test/subagent.
