# v0.43.11.11 Help Modal Focus Dismissal Restore

## Restore Purpose
Repair the Schumpeter full-screen TUI Help modal so it behaves as a real modal instead of a passive overlay above the input.

## User-Observed Help Modal Issue
The Help surface could remain open while ordinary text was typed into the main input. The modal also lacked a visible close/navigation hint, making dismissal behavior unclear.

## v0.43.11.11 Goal
Help must own keyboard focus while open, block main input changes, block slash palette opening behind it, and close predictably with `Esc`, `q`, or F1 toggle behavior.

## Help Modal Focus Policy
The Help modal is represented by a focusable `HelpModal` widget. Opening Help disables `#main-input`, closes the slash palette, updates `#help-modal`, removes its hidden class, and focuses the modal.

## Close / Navigation Hint
Every Help rendering ends with:

```text
Esc 닫기  ·  q 닫기  ·  ↑/↓/PgUp/PgDn 스크롤
```

## Input Isolation While Modal Open
While Help is open, ordinary keys are trapped by the modal and do not mutate the main input buffer. The prior input value is preserved and restored as the invariant target.

## Slash Palette Interaction
The slash palette is closed before Help opens. `Ctrl+P`, `/`, and ordinary slash input do not open a palette behind Help.

## Copy Corrections
The Help overview uses the corrected copy:

```text
일반 문장을 입력하거나, slash command로 업무 흐름을 선택할 수 있습니다.
```

## Tests
`tests/test_v04311_11_help_modal_focus_and_dismissal.py` verifies copy, close hints, focus ownership, input isolation, slash palette blocking, dismissal keys, transcript cleanliness, and closed safety flags.

## Manual Acceptance
Run `chanta-cli start`, enter `/help`, type `ss`, confirm the input does not change, press `Esc`, then type `ss` and confirm input works again. Open `/help examples`, close with `q`, then confirm `/` opens the slash palette normally.

## Safety Boundary
This patch is UI-only. It does not call a provider, submit prompts, run shell/git, read repositories or workspaces, mutate memory, write CORE_MEMORY, or claim production certification.

## Withdrawal Conditions
Withdraw this patch if ordinary typing changes `#main-input` while Help is open, if the slash palette opens behind Help, if `Esc` or `q` fails to close Help, or if Help content appears as transcript text.

## v0.44 Gate Recommendation
Do not move to v0.44 until Help modal focus and dismissal behavior passes automated tests and manual TUI acceptance.
