# v0.43.11.2 Non-Obstructive Keyboard Slash Palette Restore

## Restore Purpose

v0.43.11.2 repairs the Schumpeter full-screen TUI slash palette so command suggestions assist typing without covering the input field.

## User-Observed Palette UX Problem

The current slash command popup can appear over the typing/input area. When the user presses `/`, the suggestions obscure what the user is typing. That makes the palette counterproductive and fails the full-screen TUI acceptance standard.

## v0.43.11.2 Goal

The goal is a non-obstructive, keyboard-navigable slash palette:

- `/` opens suggestions.
- Ctrl+P opens suggestions.
- Up/Down moves selection.
- Enter or Tab inserts the selected command.
- Selection never executes the command.
- Esc closes the palette and preserves input.
- Input remains visible at all times.

## Non-Obstructive Palette Contract

The palette is rendered in normal layout flow above the input bar, not as an overlay that covers the input. The input bar remains a persistent region below the palette.

## Input Visibility Invariant

The input box must remain visible while the palette is open. This applies to wide, medium, narrow, filtered, navigated, and resized states. The palette state records `input_remains_visible=True` and `command_executed_by_selection=False`.

## Keyboard Navigation Contract

Keyboard policy:

- `/` opens palette.
- Ctrl+P opens palette.
- Down moves selection down.
- Up moves selection up.
- Tab inserts selection.
- Enter inserts selection when palette is open.
- Esc closes palette.
- Enter executes only when palette is closed.

## Selection vs Execution Rule

Palette selection inserts text only. It does not run the command.

Example:

```text
/
Down
Enter
```

The selected command appears in the input field. The runtime command handler is not invoked until the user submits the completed input after the palette closes.

## Layout and Anchoring Policy

The palette anchor is `above_input`.

Policy:

- `anchor_above_input=True`
- `may_cover_input=False`
- `reserve_input_line=True`
- `reserve_status_line=True`
- `compact_when_narrow=True`
- `scroll_when_overflow=True`
- `production_certified=False`

## Narrow Terminal Behavior

For narrow widths, the palette renders compact command rows, truncates long text, and limits visible items. Overflow is represented as additional items rather than expanding over the input.

## Command Registry Integration

The palette uses the canonical command registry introduced in v0.43.11.1. It does not maintain a second command list and does not execute handlers during filtering or navigation.
