## 5. v0.43.4 Goal

v0.43.4 uses bounded evidence packs to produce business artifacts that preserve evidence lineage, cite local evidence ids, and mark unsupported claims as assumptions or unknowns.

## 6. Evidence-Grounded Synthesis Concept

Evidence-grounded synthesis means a bounded evidence pack is selected, the user explicitly asks for a grounded workflow, the prompt includes only evidence pack content and user instruction, and the result links back to evidence pack and evidence item ids.

It does not mean arbitrary file search, repo search, internet search, shell execution, tool calling, hidden memory access, automatic memory mutation, or certainty beyond available evidence.

## 7. Evidence Use Policy

Provider synthesis requires an explicit grounded command. `/use-evidence`, `/evidence used`, and `/grounding-check` are deterministic and non-provider by default.

The policy requires evidence id citations, unsupported claim marking, bounded sources only, and all high-risk capabilities closed.

## 8. Active Evidence Pack

`/use-evidence <query>` performs bounded retrieval and selects the resulting evidence pack for the current work session. `/use-evidence last` selects the last retrieval evidence pack when available.

Evidence selection does not call provider, submit prompts, execute shell, search repos, search arbitrary files, mutate memory, or write CORE_MEMORY.

## 9. Grounded Synthesis Commands

Grounded commands are:

* `/grounded-summary`
* `/grounded-todo`
* `/grounded-memo`
* `/grounded-decision`
* `/grounded-handoff`

These commands may call the configured or mock provider because the user explicitly requested synthesis.

## 10. Grounded Prompt Contract

The grounded prompt must include runtime identity, evidence-grounding instruction, evidence items with ids/snippets/source kinds, citation instruction, unsupported-claim instruction, forbidden capability claims, and the user instruction.

It must not include arbitrary file content, repo content, or shell output.

## 11. Evidence Citation Format

Allowed local citation formats:

* `[EVID:<evidence_item_id>]`
* `[NOTE:<note_id>]`
* `[ART:<artifact_id>]`
* `[FB:<feedback_id>]`
* `[RUN:<run_id>]`
* `[TRACE:<trace_record_id>]`

Do not use web citations. Do not invent evidence ids.

## 12. Unsupported Claim Policy

If a claim is not supported by selected evidence, label it as an assumption, unknown, or needs verification. Unsupported claims must not be presented as confirmed facts.

## 13. Grounded Artifact Model

Grounded artifacts preserve artifact type, workflow kind, evidence pack id, used evidence item ids, unsupported claim count, session id, run id, provider-generated status, and production certification false.

## 14. Grounding Verification

The grounding check verifies evidence pack presence, citation count, unsupported-claim marking, no arbitrary file search, no repo search, no shell, no tool/function calling, no memory mutation, no CORE_MEMORY write, source disclosure preservation, and process reviewability.

## 15. Evidence Used View

`/evidence used` shows the active or last grounded evidence pack id and used evidence item ids. It is read-only and non-provider.

## 16. Evidence Usage Trace Record

The trace record captures evidence pack id, used evidence item ids, workflow kind, run id, session id, provider/prompt flags, and high-risk false flags.

## 17. PI Review Contract

The PI review record requires evidence lineage preserved, citation policy applied, unsupported claims marked, bounded sources only, high-risk counts zero, and reconstructability as a process event.

## 18. Safety Boundary

Opened: explicit provider-backed synthesis over a bounded local evidence pack.

Closed: arbitrary filesystem search, repo search, workspace search, shell execution, provider tool calling, function calling, subagents, general AgentLoop, automatic memory mutation, CORE_MEMORY write, and production certification.
