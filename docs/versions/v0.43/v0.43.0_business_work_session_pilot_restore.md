## Business Work Session Concept

A work-session flow is user-invoked and bounded:

* the user explicitly sends a slash command or work question
* ChantaCore creates a structured business-readable response
* session/run/trace/report evidence remains available
* no external action is claimed unless actually available
* no shell, file edit, repo search, or subagent action occurs

The valid v0.43.0 modes are:

* general
* work_summary
* meeting_memo
* decision_brief
* todo_extraction
* handoff_note
* process_review
* issue_diagnosis
* unknown

## Start Command Contract

`chanta-cli start` is the canonical business session entrypoint.

Default screen:

```text
ChantaCore Agent
업무 세션을 시작합니다.

무엇을 도와드릴까요?

추천 흐름:
* /summary : 현재 내용을 요약합니다
* /todo : 다음 액션을 뽑습니다
* /memo : 회의/업무 메모로 정리합니다
* /decision : 판단 근거와 결론을 정리합니다
* /handoff : 다른 사람에게 넘길 인수인계문을 만듭니다
* /what-happened : 방금 세션에서 일어난 일을 설명합니다
* /capabilities : 제가 할 수 있는 일과 아직 못 하는 일을 보여줍니다
* /exit : 종료합니다
```

Start must not invoke provider, submit a prompt, execute shell, invoke subagents, mutate memory, or claim production certification.

## Work Session Command Surface

v0.43.0 commands:

* `/new`: starts a new work-session state without deleting prior session data
* `/summary`: creates a business summary through the existing single-turn provider run path
* `/todo`: extracts action, owner, due date, dependency, confidence/unknowns
* `/memo`: creates context, key points, decisions, open questions, next actions
* `/decision`: creates issue, options, evidence, tradeoffs, decision, risks, next actions
* `/handoff`: creates background, current state, what was done, remaining work, risks, next action
* `/what-happened`: deterministic trace/run/session explanation by default
* `/capabilities`: honest capability map
* `/feedback`: routes to bounded feedback note
* `/report`: routes to diagnostic bundle or safe guidance
* `/help`, `/status`, `/provider`, `/history`, `/trace`, `/exit`: safe supporting commands

## What Happened Contract

`/what-happened` explains the current or recent session in user language.

It should answer:

* what command or run happened
* provider used, if known
* response status, if known
* trace/session status
* safety status
* next recommended action

Default behavior is deterministic. It does not call provider or submit prompts unless a future version explicitly opens and documents that design.

## Feedback / Report Integration

`/feedback <text>` routes to the v0.42.6 bounded feedback loop. It may write a feedback record under the selected ChantaCore home, but it must not mutate core memory, workspace files, provider config, or session state beyond the feedback store.

`/report` routes to diagnostic bundle guidance and can use the existing v0.42.6 diagnostic bundle helper. It does not upload externally, execute shell, call provider, or expose secret values.
