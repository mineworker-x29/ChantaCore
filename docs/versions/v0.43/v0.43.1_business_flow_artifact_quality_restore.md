## Business Artifact Model

A business artifact is a structured assistant output inside the work session. It is not a file write and not an external document export.

Artifact types:

* summary
* todo
* memo
* decision_brief
* handoff_note
* clarification_questions
* process_review
* issue_diagnosis
* unknown

Each artifact has sections, grounding classes, confidence, source summary, verification flags, run/session linkage, quality report, PI review record, and safety flags.

## Session Context Pack

The session context pack may use:

* explicit user content after a slash command
* bounded recent session turns
* last business artifact
* last run report
* trace summary

It must not use arbitrary files, repo search, shell output, network access, hidden memory mutation, or external sources.

## Bounded Context Selection Policy

Policy requirements:

* prefer explicit user content
* allow only bounded recent session turns
* allow last artifact, last run report, and trace summary
* block arbitrary file read
* block repo search
* keep max_turns and max_chars bounded

## Summary Flow Standard

Summary artifacts should include:

* 핵심 요약
* 배경 / 맥락
* 중요한 결정 또는 변화
* 다음 액션
* 불확실 / 확인 필요

## Todo Flow Standard

Todo artifacts should include:

* action
* owner, if known
* due date, if known
* dependency, if known
* priority
* confidence or unknowns

## Memo Flow Standard

Memo artifacts should include:

* 제목
* 맥락
* 주요 내용
* 결정 사항
* 미해결 질문
* 다음 액션

## Decision Brief Standard

Decision brief artifacts should include:

* 판단 대상
* 선택지
* 확인된 근거
* 해석
* 장단점 / tradeoff
* 리스크
* 권고안
* 다음 액션
* 불확실 / 확인 필요

## Handoff Note Standard

Handoff note artifacts should include:

* 배경
* 현재 상태
* 지금까지 한 일
* 남은 일
* 리스크 / 주의사항
* 다음 액션
* 참고할 명령 또는 report bundle 안내

## Clarification Flow

`/clarify` should list:

* missing information
* why it matters
* question to ask
* suggested default assumption if the user wants to proceed

It must not claim unavailable facts.

## What Happened Refinement

`/what-happened` should explain:

* 최근 실행된 흐름
* 사용된 provider / mode
* run/session 상태
* response parse status if available
* trace/session/report state
* safety status
* next recommended action

Default behavior should remain deterministic and non-provider-backed.

## Artifact Last / Revise Commands

`/artifact last` shows the last generated business artifact in clean form.

It must not call provider, submit prompt, mutate files, mutate memory, execute shell, or invoke subagents.

`/revise <instruction>` creates a new artifact version from the previous artifact and the user instruction. The original is preserved. The revision may use the existing single-turn provider run path, but it must not edit files or apply changes externally.
