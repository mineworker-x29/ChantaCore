# v0.43.11 Schumpeter Full-Screen TUI Productization Restore

## Restore Purpose

Restore the final v0.43 UI productization context for Schumpeter before the v0.44 gate.

## One-Screen Restore Summary

Schumpeter v0.43.11 replaces the line-oriented preview shell with a real full-screen Textual TUI. It makes chanta-cli start the default product entrypoint, adds a persistent sidebar, scrollable chat view, fixed input and status areas, slash-command popup, help/about modals, responsive layouts, PI status monitoring, snapshot and interaction tests, while preserving classic/plain fallbacks and keeping all high-risk capabilities closed.

## User Feedback and Previous UI Failure

The previous TUI preview had useful routing and help, but still felt like a structured print loop. It did not provide a persistent sidebar, chat viewport, fixed input area, modal help, or popup command palette.

## v0.43.11 Goal

Provide the final full-screen Schumpeter TUI product surface before v0.44 without opening workspace read, repo search, shell/git, file edit/apply, provider tools/functions, subagents, memory mutation, CORE_MEMORY write, or production certification.

## Product Identity

Default product name is Schumpeter. Subtitle is Process Intelligence-native Work Agent. ChantaCore appears only as runtime lineage or project label, not as the product name.

## Textual Technology Decision

Textual is the authorized full-screen TUI framework for v0.43.11. It is declared in `pyproject.toml` through the project dependency list. Textual imports are localized to the TUI full-screen module.

## Entrypoint Migration

`chanta-cli start` now targets the full-screen Textual TUI by default. `chanta-cli tui` and `chanta-cli start --tui` target the same product surface. `--classic` preserves the previous stable line shell. `--plain` preserves the clean non-full-screen fallback.

## Full-Screen Layout

The full-screen app has persistent regions: sidebar, main chat, input bar, and status bar.

## Responsive Layout

Snapshots and layout policy cover wide, medium, and narrow sizes. Narrow layout uses a compact status strip rather than forcing broken columns.

## Theme System

The Textual CSS theme is centralized in `styles/schumpeter.tcss`, with restrained dark colors, panel borders, focus styling, and overlay styling.

## RuntimeAdapter Boundary

Widgets talk through the app controller and RuntimeAdapter. Sidebar, help, about, palette, status refresh, snapshot, and resize do not call provider completion, shell/git, repo/workspace read, or memory mutation.

## Sidebar

Sidebar contains Brand, Project, Session, PI Monitor, and Shortcuts panels. It uses concise human statuses and avoids raw booleans.

## Main Chat View

Main chat uses a scrollable Textual RichLog surface. User, assistant, artifact, status, diagnostic, error, and notice messages remain in transcript without static chrome.

## Input Area

The bottom input area is persistent. Enter submits. Ctrl+C exits. Escape closes overlays. Ctrl+P opens the slash palette.

## Slash Command Palette

The palette is an overlay near the input area. It shows commands and inserts text only; command execution happens only on final submission.

## Help Modal

`/help` and F1 open a help overlay, not a giant transcript dump. Topic help can focus sections from the help information architecture.

## About Modal

`/about` opens a compact modal showing Schumpeter, Process Intelligence-native Work Agent, ChantaCore runtime lineage, and chanta-cli compatibility.

## PI Status Monitor

PI monitor displays PI, Provider, Trace, Evidence, and Safety as concise statuses. Refresh remains display-only.

## Renderer Boundary

Default renderers hide raw grounding/source/safety metadata. Debug details remain explicit surfaces only.

## Busy State

The app tracks a busy flag around submitted turns and avoids duplicate submission while a turn is being handled.

## Snapshot and Visual Test Strategy

Deterministic text snapshots cover 140x42, 110x32, 80x24, palette, help, and plain modes. Textual pilot tests cover mounting, palette, help, about, messages, status, diagnostics, and exit.

## Automated Interaction Tests

`tests/test_v04311_schumpeter_fullscreen_tui.py` includes dependency, entrypoint, layout, sidebar, chat, palette, help, about, snapshot, fake adapter, safety, and documentation tests.

## Manual UI Acceptance

Run `chanta-cli start`, test `/`, Ctrl+P, `/help`, `/status`, Korean normal text, `/summary`, `/what-happened`, `/about`, `/exit`, and resize behavior. Also test `--classic`, `--plain`, and snapshot commands.

## Classic / Plain Fallback

`--classic` preserves the old line shell. `--plain` preserves the clean non-full-screen fallback. Missing Textual prints an actionable message and does not silently route to the old UI.

## Safety Boundary

Still closed: workspace read, arbitrary file read, repo search, workspace search, git status execution, shell execution, file edit/apply, provider tool calling, function calling, subagents, AgentLoop expansion, autonomous coding, memory mutation from UI, CORE_MEMORY write, persistent memory promotion, and production certification.

## Withdrawal Conditions

Withdraw if default start no longer works, Textual is not installed through packaging, UI remains a print loop, widgets call runtime side effects, palette executes before submit, help is transcript-dumped, CJK layout breaks, fallbacks break, regressions fail, high-risk capabilities open, or production certification is claimed.

## Known Limitations

The first Textual implementation is intentionally restrained. Fine-grained keyboard navigation, richer scroll anchoring, and visual spacing can still improve after manual acceptance.

## v0.44 Gate Recommendation

Do not open v0.44 workspace read until this full-screen TUI passes manual visual acceptance.

## Copy-Paste Restore Prompt

Restore v0.43.11 by preserving Textual as the default full-screen Schumpeter TUI, keeping classic/plain fallbacks, maintaining RuntimeAdapter boundaries, preserving help/palette/about overlays, testing responsive snapshots and Textual interaction, and keeping all high-risk capabilities closed before v0.44.
