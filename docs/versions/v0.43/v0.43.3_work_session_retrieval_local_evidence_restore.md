## 3. Current Version and Track Identity

Current version: `v0.43.3`.

Track: `v0.43 - Business Work Session Pilot & Process Intelligence Review Loop`.

This version opens deterministic retrieval over bounded ChantaCore-managed evidence only. It is not a Codex-like coding capability version and is not production certified.

## 4. v0.43.2 Baseline Summary

v0.43.2 defined the memory boundary and local note discipline. It opened bounded local work notes, memory candidates, `/note`, `/notes`, `/note last`, `/note from-artifact`, `/notes search`, `/memory-boundary`, and `/context`, while keeping persistent memory mutation closed.

## 5. v0.43.3 Goal

v0.43.3 lets a user find local operational evidence from prior business work sessions without broad filesystem search, repo search, shell execution, provider synthesis, or memory mutation.

## 6. Local Evidence Retrieval Concept

Local evidence retrieval is deterministic search over known ChantaCore evidence stores under the resolved profile home. It is evidence lookup, not persistent memory recall and not external search.

## 7. Allowed Evidence Sources

Allowed source kinds are:

* local work notes
* memory candidates
* business artifact cache
* feedback records
* trace summaries
* run reports
* session summaries
* current session summaries

Unavailable stores are disclosed and do not fail the whole retrieval.

## 8. Disallowed Search Sources

v0.43.3 does not search arbitrary user files, repository files, source trees, workspace files, external web, shell output, operating system search indexes, or CORE_MEMORY.

## 9. Evidence Boundary Policy

The default retrieval mode is deterministic local search. Provider invocation and prompt submission are disabled by default. Arbitrary file search, repo search, workspace search, broad filesystem scan, external search, shell execution, memory mutation, and CORE_MEMORY writes are all closed.

## 10. Evidence Source Policy

File-backed sources use fixed relative paths under the resolved ChantaCore home. Retrieval is read-only, append is disallowed, records are bounded, and secrets are treated as redacted evidence.

Known file-backed paths:

* `profiles/default-personal/state/work_notes/notes.jsonl`
* `profiles/default-personal/state/work_notes/memory_candidates.jsonl`
* `profiles/default-personal/state/feedback/feedback.jsonl`

## 11. Retrieval Commands

Slash commands:

* `/recall <query>` returns a user-friendly bounded evidence summary.
* `/evidence <query>` returns a source-oriented evidence list.
* `/evidence sources` shows available and unavailable evidence sources.
* `/evidence last` shows the last evidence pack in the current process.
* `/evidence explain` explains the retrieval boundary.

## 12. Recall Result Format

Recall output includes query, found evidence, source kind, snippet, relevance label, recommended next action, and what was not searched.

## 13. Evidence Source Disclosure

Each evidence pack discloses searched source kinds, unavailable source kinds, and explicitly not searched scopes: arbitrary files, repo, workspace, external web, shell output, and memory mutation.

## 14. Evidence Pack

An evidence pack contains matches, source disclosure, creation time, session id if available, and safety flags. It is bounded, non-provider, non-prompt, and not production certified.

## 15. Retrieval Trace Record

The retrieval trace record captures query text, result count, searched source kinds, session id if known, and high-risk flags. It is reconstructable as a Process Intelligence event.

## 16. PI Review Contract

The PI review record requires reconstructability, bounded sources only, explicit not-searched disclosure, and high-risk count zero.

## 17. Safety Boundary

Opened: bounded local evidence search.

Closed: arbitrary file search, repo search, workspace search, broad filesystem scan, external search, shell execution, provider tool calling, function calling, subagents, general AgentLoop, automatic memory mutation, CORE_MEMORY write, and production certification.

## 20. Withdrawal Conditions

Withdraw or roll back if retrieval searches outside the resolved ChantaCore home, reads arbitrary user-supplied paths, scans repo/source trees, calls provider by default, mutates memory, writes CORE_MEMORY, invokes shell/subagent/tool/function paths, or claims production certification.
