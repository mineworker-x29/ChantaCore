# v0.43.10.3 Help Command Information Architecture Restore

## Restore Purpose

Restore the Schumpeter TUI help-surface hotfix. v0.43.10.3 improves `/help` from a short command list into a usable, grouped, safe guidance surface.

## User-Observed Help Problem

The previous `/help` was too shallow. It did not explain what Schumpeter is, how to start, which commands support workflows, notes, evidence, grounded outputs, diagnostics, safety, or TUI usage.

## v0.43.10.3 Goal

Make `/help` clear, grouped, concise by default, detailed by topic, consistent with slash command names, and safe. Help rendering must not call providers, run shell/git, read repositories/workspaces, mutate memory, or claim production certification.

## Help Information Architecture

Default `/help` is an overview:

- Schumpeter identity
- quick start
- core commands
- links to detailed help topics

Detailed topics are opened explicitly with `/help <topic>`.

## Help Topics

Supported topics:

- `/help`
- `/help commands`
- `/help workflows`
- `/help evidence`
- `/help grounded`
- `/help notes`
- `/help status`
- `/help safety`
- `/help tui`
- `/help examples`

Unknown topics return a clean available-topic message.

## Command Registry Integration

Help uses a canonical command metadata wrapper and checks command names against the shared slash registry used by the palette. The wrapper adds human-readable Korean descriptions, usage, examples, provider policy labels, and side-effect labels.

## Provider / Side-Effect Labels

Detailed help uses human-facing labels:

- deterministic
- may use provider
- non-provider
- read-only
- local note write
- diagnostic
- debug-only
- not opened yet

Raw booleans are not shown.

## Safety Help Contract

`/help safety` explains closed capabilities in user-facing language:

- 저장소 직접 읽기
- repo search
- 셸/git 실행
- 파일 수정/apply
- provider 도구/functions
- subagent 실행
- 자동 메모리 승격
- production certification

It states that these can only open after v0.44 controlled design gates. It does not print raw safety footers.

## TUI Help Contract

`/help tui` explains normal text input, slash commands, command suggestions, `/help commands`, `/status`, `/exit`, and snapshot commands.

## Examples Help Contract

`/help examples` provides practical Korean examples for normal input, workflow commands, recall/evidence commands, and grounded output commands.

## Golden Help Cases

Golden cases cover:

- `/help`
- `/help commands`
- `/help safety`
- `/help examples`

Each golden case checks required content and forbidden noisy strings.
