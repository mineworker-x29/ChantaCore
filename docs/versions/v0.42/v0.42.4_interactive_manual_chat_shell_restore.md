## Interactive Manual Chat Shell Summary

v0.42.4 adds a manual CLI chat shell. It lets the user type repeated messages without retyping full `chanta-cli run` commands.

The chat shell is a UI loop. It is not a model-driven agent loop.

## Chat Command Contract

`chanta-cli chat` starts the manual shell.

Recommended options:

* `--profile default-personal`
* `--provider mock`
* `--session <session_id>`
* `--new-session`
* `--max-turns <n>`
* `--json`
* `--no-banner`

No `start` alias is required in v0.42.4.

## Manual UI Loop Contract

Allowed loop:

1. prompt user
2. receive explicit user input
3. classify input as internal command or user message
4. run safe internal read/report command or invoke one existing single-turn run
5. render output
6. wait for the next explicit user input

Forbidden loop:

* model-selected next action
* autonomous continuation
* hidden retry
* tool-call loop
* shell/action loop
* subagent delegation
* file mutation loop

## Internal Command Contract

Supported internal commands:

* `/help`
* `/exit`
* `/quit`
* `/status`
* `/provider`
* `/history`
* `/trace`
* `/run last`
* `/session`
* `/handoff`
* `/safety`

Internal commands do not call providers, submit prompts, execute shell, invoke subagents, or certify production.

## One-run-per-user-message Contract

Each explicit user message maps to at most one existing single-turn run path. Provider invocation and prompt submission may occur only through that existing run path.

The shell does not retry, self-continue, or launch follow-up actions.

## Chat Session Contract

Chat uses a resolved ChantaCore home and a session id. Existing run behavior appends user and assistant turns through bounded session stores and emits trace through existing run behavior.

Provider-generated assistant text remains untrusted for memory and execution.

## Chat Shell Safety Boundary

Closed in v0.42.4:

* autonomous AgentLoop
* retry loop
* provider doctor completion
* provider tool calling
* function calling
* read-only skill execution as actions
* shell execution
* file edit
* patch apply
* test execution through CLI
* subagent invocation
* child session creation
* Dominion runtime
* production certification

Raw dangerous command-like text is denied or safely warned. It is never executed.

## Scripted Test Helper Contract

`run_v042_chat_shell_scripted(inputs=[...])` simulates the same shell logic from in-memory inputs.

It does not require subprocess, pty, external scripts, or OS shell execution.

## Debug Handoff Contract

The chat debug handoff includes profile id, home path, session id, run ids, provider mode, turn count, and safety counts.

It does not include secret values and is suitable for GPT/Codex debugging.
