## Project Context for New Codex Session
ChantaCore is a personal OCEL/Process-Intelligence-native runtime. It must preserve structured evidence for prompts, provider calls, responses, sessions, traces, skills, denials, diagnostics, and feedback. v0.42.9 changes presentation defaults, not runtime authority.

## User UX Direction
The user direction is that ChantaCore should feel like a clean business/work agent, not a developer-only coding assistant. Debug details remain important but should live behind debug, verbose, JSON, trace, run-report, report bundle, status, handoff, and diagnostics surfaces.

## Business Agent UX Principles
Default output is for a business user. It should be calm, readable, and actionable. Operator/debug output remains available for process evidence review. The default surface must not lead with raw trace banners, internal version archaeology, or unstructured implementation details.

## Default vs Debug Output Policy
Default mode hides internal trace/runtime banners and the provider-untrusted warning. Debug and verbose modes show run id, session id, response parse status, error class, trace event count, and safety flags. JSON and existing report commands remain preserved.

## Run Output Polish
Default `chanta-cli run "..."` renders the assistant answer with a compact footer:

* run: `<run_id>`
* session: `<session_id>`

The old v0.41.5 traced banner remains available through debug/operator paths.

## Chat Output Polish
Chat starts with:

* ChantaCore Agent
* Profile
* Provider
* Session
* Type /help for commands. Type /exit to leave.

The default prompt is `ChantaCore>`. Help is grouped by conversation, status, review, and safety.

## Provider Status Polish
Provider status now presents readiness:

* Status: Ready / Not configured / Needs attention
* Mode
* Model
* Endpoint
* Mock provider
* Next action

Secrets are not shown.

## Empty Response User Guidance
Empty provider responses use plain language: ChantaCore could not find a final answer in the provider response. Guidance mentions reasoning-only local model output, LM Studio chat templates, max_tokens, provider connectivity, and run-report inspection.

## Command Guide
`chanta-cli commands` provides a business-user command map with Start, Talk, Provider, Review, Skills, Diagnostics, and Safety sections. It avoids internal class and artifact names.

## Runtime Identity Answer Policy
When asked who it is, ChantaCore should answer as a ChantaCore default-personal work/business assistant. Provider model identity is implementation detail, not primary runtime identity.

## Operation Readiness Report
The v0.42.9 readiness report checks quickstart, provider connectivity, configured run behavior, chat startup, grouped help, run output, debug details, trace timeline, report bundle, feedback note, and closed unsafe capabilities.
