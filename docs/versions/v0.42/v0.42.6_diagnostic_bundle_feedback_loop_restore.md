## Diagnostic Bundle Summary

v0.42.6 adds a local diagnostic bundle that can be pasted into GPT/Codex. The bundle includes runtime/version metadata, home/profile/provider status, redacted provider config, last run report, run history, trace summary, trace timeline summary, session show last, skill summary, denial summary, safety counts, feedback summary, latest feedback notes, known limitations, closed capability snapshot, v0.42 closure assessment, and next action.

The diagnostic bundle is read/report only by default.

## Diagnostic Bundle Contract

Supported commands:

* `chanta-cli report bundle`
* `chanta-cli report bundle --copy-paste`
* `chanta-cli report bundle --format text`
* `chanta-cli report bundle --format markdown`
* `chanta-cli report bundle --format json`

The bundle does not call providers, submit prompts, write files by default, execute shell, invoke subagents, read arbitrary paths, scan arbitrary directories, upload externally, or certify production.

## Diagnostic Redaction Contract

Diagnostic output redacts secret-like values, API key assignments, token-like values, and credential-looking values. Redaction preserves safe runtime context such as command names, profile id, and bounded home path context when safe.

The fake value `OPENAI_API_KEY=sk-test-secret-value` must not appear in persisted feedback records or report bundle output.

## Diagnostic PI Review Packet

The diagnostic PI review packet includes:

* run count summary
* provider count summary
* denial summary
* skill summary
* feedback summary
* safety summary
* process instance review readiness
* evidence handoff readiness
* high-risk counts zero status

This packet exists to make bundle evidence reviewable as process evidence.

## Feedback Loop Summary

v0.42.6 adds bounded local feedback notes so users can record real-use friction, bugs, trace concerns, provider issues, skill issues, safety concerns, Process Intelligence findings, docs gaps, performance issues, and ideas.

Feedback is not memory mutation, not automatic issue creation, and not automatic fixing.

## Feedback Store Contract

Feedback records are stored under the resolved profile home:

`profiles/default-personal/state/feedback/feedback.jsonl`

The store is bounded to home and append-only. It does not write `CORE_MEMORY.md`, profile config, provider config, session store, workspace files, or arbitrary paths.

## Feedback Redaction Contract

Feedback note text is redacted before persistence. Secret-like assignments, token-like values, and key-looking values are replaced with `<redacted>`. Raw secret-like values must not be persisted.

## Feedback Commands
