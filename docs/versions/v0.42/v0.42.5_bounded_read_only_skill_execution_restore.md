## Bounded Read-only Skill Execution Summary

v0.42.5 opens a skill execution gate only for explicitly registered safe metadata/report skills. A skill must be both read-only and bounded. Read-only means it does not mutate profile, provider config, memory, session, workspace, or external files. Bounded means it reads only known ChantaCore home/profile/session/trace/provider/config data with explicit input and output limits.

The only accepted side effect is bounded skill execution or denial trace evidence. The implementation exposes PI-reviewable trace record objects for execution and denial.

## Skill Execution Gate Contract

The registry gate status is `open_for_bounded_read_only`.

Executable skills must have:

* `execution_allowed=True`
* `risk_level=low`
* `requires_provider_call=False`
* `submits_prompt=False`
* `executes_shell=False`
* `mutates_workspace=False`
* `mutates_memory=False`
* `mutates_session=False`
* `allows_arbitrary_path=False`
* `allows_network=False`

Unknown, unsafe, or invalid skill requests are denied.

## Skill Input Schema Contract

Skill inputs are deterministic and bounded. Arbitrary path input, raw shell text, and provider prompts are not accepted.

Bounded options:

* `trace_timeline` accepts `--limit` with a maximum bound.
* `run_history` accepts `--limit` with a maximum bound.
* `session_show` accepts `--target`, `--session-id`, and `--max-turns` with bounds.

Unknown args, path-like args, and out-of-bound limits are denied.

## Skill Trace Evidence Contract

Successful bounded read-only skill execution produces a PI-reviewable trace record with event kind `read_only_skill_executed`. Denied skill requests produce `read_only_skill_denied`.

The trace evidence states:

* skill id
* profile id
* status
* object references
* provider invoked false
* prompt submitted false
* shell executed false
* subagent invoked false
* workspace mutated false
* memory mutated false
* session mutated false
* production certified false

## Skill Denial Contract

Unsafe, unknown, or invalid skill requests return a denial result. Denial evidence includes the skill id, denial reason, message, safe alternative, and non-execution safety flags.

Denial evidence does not execute command text, call providers, submit prompts, or mutate workspace/session/memory.

## Skill PI Review Contract

Each successful skill execution can be reviewed as a process event: skill request, gate decision, bounded input schema, read-only report output, and trace evidence. The PI review record marks input bounded, output bounded, high-risk counts zero, and reconstructable-as-process-event true.
