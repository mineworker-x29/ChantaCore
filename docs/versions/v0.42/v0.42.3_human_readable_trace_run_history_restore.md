## Human-readable Trace Summary

v0.42.3 adds a read-only human-readable layer over existing trace and session files. It does not replace structured trace commands.

Existing JSON-oriented commands remain available:

* `chanta-cli trace recent`
* `chanta-cli trace summary`
* `chanta-cli run-report last`

## Trace Timeline Contract

`chanta-cli trace timeline` reads existing trace events under the resolved home and renders a human-readable timeline.

It supports `--limit` and may support `--json` where CLI convention allows.

The command does not call providers, submit prompts, execute shell, invoke subagents, certify production, or append recursive trace events by default.

## Run History Contract

`chanta-cli run history` groups trace events by run id and links session id when present.

Each item shows provider/mock state, user input preview, assistant response preview, provider call transaction count, appended-session evidence, denial count, trace event count, and safety flags.

## Run Show Contract

`chanta-cli run show last` and `chanta-cli run show --run-id <run_id>` render a selected run as a process instance.

Expected successful mock lifecycle:

* run_started
* user_input_received
* prompt_assembled
* provider_text_call_started
* provider_text_call_completed
* session_turns_appended
* assistant_response_recorded
* run_completed

## Session Show Contract

`chanta-cli session show last` and `chanta-cli session show --session-id <session_id>` render session id, linked runs, turn previews, latest assistant preview, and related trace event count.

Provider-generated assistant text remains untrusted for memory and execution.

## Provider Call Count Semantics

Provider call event count counts `provider_text_call_started` and `provider_text_call_completed` events.

Provider call transaction count estimates actual provider run transactions by grouping started/completed events by `run_id`.

For a successful mock run, the normal interpretation is two provider call events and one provider call transaction.

## Process Instance View Contract

A run is reconstructable as a process instance when it has a run id and core lifecycle events. The view links the run to session evidence, provider event semantics, response recording, and safety counts.

## Denial Evidence Contract

Denial evidence shows denied command previews, matched patterns, denial count, and whether the trace proves non-execution. It proves non-execution only when denial events show `executed=False` or `shell_executed=False`.

## Safety Count Summary Contract

The safety count summary reports:

* shell_execution_count
* skill_execution_count
* subagent_invocation_count
* production_certification_count
* unsafe_denial_count
* provider_tool_calling_count
* function_calling_count

The v0.42.3 review is safe only when shell, subagent, and production certification counts remain zero.

## Debug Handoff Text Contract

Debug handoff text is compact copyable text for GPT/Codex review. It includes version, profile, home path, run id, session id, status, provider mode, provider count semantics, safety counts, and notes.

It must not include secret values.

## Trace History Safety Boundary

Trace/history/show commands are read-only. They do not call providers, submit prompts, mutate trace/session files by default, execute shell, invoke subagents, or certify production.

Provider text remains untrusted.
