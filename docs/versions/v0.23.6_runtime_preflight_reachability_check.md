# v0.23.6 Runtime Preflight / Reachability Check

Korean name: 런타임 사전점검·접근가능성 검사
Track: Internal Dominion Foundation

## Version Definition

v0.23.6 introduces foundation-level Dominion runtime preflight and reachability checks. It reads the Dominion static safety report, control plan, runtime inventory, capability digest, and Internal Dominion contract, then creates a `DominionRuntimePreflightReport`.

This is not live preflight. This release performs declared or simulated readiness checks only.

## Principles

- Preflight is not dispatch.
- Preflight is not authorization.
- Preflight is not provider-specific adapter execution.
- Preflight is not external runtime mutation.
- Foundation preflight must not call provider APIs.
- Foundation preflight must not ping external runtimes.
- Preflight pass only permits moving to Dominion Gate.
- Preflight pass does not make the plan safe to dispatch.

## Preflight Mode Policy

The default mode is `declared_only`.

Allowed v0.23.6 modes:

- `declared_only`
- `simulated`
- `adapter_contract_check`

Disallowed or future modes:

- `live_read_only_future`
- `blocked`

The mode policy always keeps:

- `live_provider_api_allowed=False`
- `external_runtime_touch_allowed=False`
- `network_allowed=False`
- `credential_materialization_allowed=False`
- `dispatch_allowed=False`
- `run_creation_allowed=False`
- `shell_allowed=False`
- `local_command_allowed=False`
- `local_runtime_provider_enabled=False`
- `general_agent_usability_enabled=False`
- `llm_judge_allowed=False`

## Status Rules

`passed` means the control plan exists, static safety passed, mode is allowed, declared reachability is sufficient, required bindings and policies are ready, and no forbidden effect occurred.

`warning` covers acceptable uncertainty such as provider adapter future requirement, stale or declared-only availability, partial operational readiness, or production environment requiring a future strong gate.

`failed` covers missing static safety, missing required bindings, missing input, missing output capture/redaction, or missing required operational policies.

`blocked` covers provider API call, runtime touch, dispatch attempt, live preflight attempted too early, premature Local Runtime Provider, premature General Agent UX, premature Schumpeter split, credential materialization/output, raw secret output, network/MCP/plugin/shell, vendor hardcoding, or other forbidden effects.

`eligible_for_dominion_gate` may be true only for `passed` or acceptable `warning`.

`safe_to_dispatch` is always false in v0.23.6.

## CLI Commands

Read-only/foundation-preflight CLI commands:

```powershell
python -m chantacore.cli dominion preflight check --plan-id dominion_control_plan:v0.23.4 --static-safety-report-id dominion_static_safety_report:v0.23.5
python -m chantacore.cli dominion preflight report --report-id dominion_runtime_preflight_report:v0.23.6
python -m chantacore.cli dominion preflight findings --report-id dominion_runtime_preflight_report:v0.23.6
python -m chantacore.cli dominion preflight summary --plan-id dominion_control_plan:v0.23.4
```

CLI output is sanitized and reports report id, plan id, preflight status, mode, gate eligibility, safety boundary flags, roadmap deferral flags, and next required step. It does not print credential values, raw secrets, or private full paths.

## Acceptance Criteria

- `DominionRuntimePreflightReport` can be created from a control plan and static safety report.
- Declared reachability and readiness checks run.
- Preflight may mark `eligible_for_dominion_gate`.
- `safe_to_dispatch=False` always.
- No live provider API call or external runtime touch occurs.
- No dispatch, gate, or authorization occurs.
- No local command execution or Local Runtime Provider implementation occurs.
- No General Agent Usability, ask, repl, or routing implementation occurs.
- No Schumpeter split or company wrapper is introduced.
- No credential value is stored or output.
- No LLM judge is used.
- OCEL/PIG/OCPX visibility exists.
- v0.23.5 through v0.20.x regression tests remain passable by the agreed test matrix.
