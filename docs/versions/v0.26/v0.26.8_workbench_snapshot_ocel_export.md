# v0.26.8 Workbench Snapshot / OCEL Export

## Version Definition

- Version: `v0.26.8`
- Name: Workbench Snapshot / OCEL Export
- Korean name: Workbench Snapshot.OCEL Export
- Track: Workspace Agent Workbench
- Previous: `v0.26.7` Workbench Command Surface
- Next: `v0.26.9` Workspace Agent Workbench Consolidation

`v0.26.8` creates refs-only snapshot and OCEL export artifacts. It is not memory
promotion, raw data export, external synchronization, runtime adapter behavior,
or process-mining execution.

## Objective

The objective is to preserve inspectable workbench state as OCEL-visible
snapshot/export records: selection, snapshot, manifest, ref bundle, sanitized
OCEL export package, event quality report, trace coverage report, redaction
report, reproducibility packet, export boundary descriptor, findings, and
report.

## Relationships

`v0.26.8` reads refs and summaries from `v0.26.7` command surface, `v0.26.6`
run dashboard/session monitor, `v0.26.5` approval console, `v0.26.4` evidence
inspector, `v0.26.3` provider browser, `v0.26.2` trace explorer, `v0.26.1`
view state/panel model, and `v0.26.0` workbench contract when available. It may
also preserve refs from `v0.25.8` trace telemetry and `v0.25.7` ask/REPL
reports. It does not execute or replay any source artifact.

## Grok/Codex Feedback Absorption Rule

ChantaCore may absorb external agent-usability feedback only as OCEL-visible
guidance, candidate, rationale, approval, inspection, dashboard, session-view,
telemetry, command-candidate, snapshot, and export surfaces.

Allowed absorption includes refs-only snapshots, refs-only OCEL export packages,
reproducibility packets, decision point refs, skill/action candidates,
route/provider/safety rationale refs, PIG guidance refs, human intervention
refs, approval/rejection/deferral refs, failure cause refs, event quality
reports, trace coverage reports, redaction reports, export manifests, and future
import/export boundary descriptions.

Forbidden absorption includes snapshot as memory promotion, snapshot as
persistent memory write, raw transcript/provider/secret export, external sync,
external adapter invocation, pm4py/ocpa runtime dependency, PIG as memory,
persona or policy mutation, PIG as execution authority, autonomous loops,
automatic rerun, automatic repair, command execution, provider invocation, and
file mutation.

## Core Principles

- Workbench Snapshot is not memory promotion.
- Snapshot selection is not memory candidate extraction.
- Snapshot manifest is not memory index.
- OCEL export package is refs-only by default.
- OCEL export package is not raw transcript export.
- OCEL export package is not raw provider output export.
- OCEL export package is not external sync.
- Reproducibility packet is not rerun permission.
- Event quality report is descriptive, not automatic optimization.
- Redaction report documents exclusion, not source deletion.
- Export boundary is not runtime adapter.
- PIG guidance export ref is not PIG memory.

## Service Design

The source service loads workbench view state, snapshot-export panel model,
command surface report, dashboard report, approval report, evidence report,
provider report, trace report, telemetry report, and PIG guidance refs when
available. It loads refs and summaries only.

Selection, snapshot, manifest, ref-bundle, export, quality, coverage, redaction,
reproducibility, boundary, finding, and report services create artifact records
without writing memory, syncing externally, invoking providers, executing
commands, or mutating files.

## Snapshot Source View

`WorkbenchSnapshotSourceView` contains workbench report refs, command candidate
refs, decision points, skill/action candidates, route/provider/safety
rationales, PIG guidance, human intervention, approval decision, failure cause,
trace, and OCEL projection refs. Raw provider output, raw transcripts, raw
secrets, and private full paths are excluded.

## Snapshot Selection

`WorkbenchSnapshotSelection` is refs-only. It records selected refs, selected
categories, omitted raw-data refs, omission reasons, and selection status. It
does not create memory candidates, promote memory, or select raw content.

## Snapshot, Manifest, Ref Bundle

`WorkbenchSnapshot` records the selected refs, manifest, ref bundle, redaction
report, reproducibility packet, event quality report, and trace coverage report.
It is refs-only and contains no raw transcript, raw provider output, raw secret,
credential, or private full path.

`WorkbenchSnapshotManifest` is not a memory index. It records included report,
trace, decision, candidate, approval, failure, and PIG refs plus excluded raw
data categories.

`WorkbenchSnapshotRefBundle` groups export refs for decision points, skill
candidates, action candidates, route/provider/safety rationales, PIG guidance,
human intervention, approval decisions, failure causes, and command candidates.

## Export Refs

Export refs preserve rationale and evidence refs while keeping execution flags
false. Skill/action/route/provider/command refs are not executed. Provider refs
do not include raw provider output. PIG refs are not memory, policy mutation, or
execution. Approval, failure, and intervention refs do not trigger execution,
retry, or repair.

## OCEL Export Package

The OCEL export package records object, event, relation, and effect type refs
plus counts. It is refs-only and does not mutate source artifacts, export raw
data, perform external sync, invoke adapters, or add pm4py/ocpa dependencies.

## Quality, Coverage, Redaction, Reproducibility

Event quality and trace coverage reports are descriptive only. They do not
optimize, rerun, retry, or repair. Redaction reports document excluded raw data
categories and preserved ref/summary counts; they do not delete source data.
Reproducibility packets list required report, boundary, command candidate,
approval decision, and export package refs but grant no rerun permission.

## Export Boundary Descriptor

The export boundary descriptor allows internal refs-only and sanitized OCEL
package targets. It forbids external sync, external adapter calls, raw
transcript/provider export, and memory promotion.

## Status Rules

The report passes when source view, selection, snapshot, manifest, ref bundle,
OCEL export package/manifest, event quality, trace coverage, redaction,
reproducibility, and export boundary records are created without forbidden
behavior.

It warns for partial refs or optional missing inputs, fails in strict mode when
mandatory source/panel/export artifacts are missing, and blocks on memory,
raw export, external sync, external adapters, pm4py/ocpa dependency, execution,
rerun/retry/repair, PIG mutation/execution, Schumpeter split, OpenCode/
OpenClaw/Hermes runtime dependency, or LLM judge findings.

## OCEL Mapping

Object, event, relation, allowed effect, and forbidden effect constants are
defined in the snapshot/export module. Allowed effects include read-only
observation, workbench snapshot creation, manifest creation, ref bundle
creation, OCEL export creation, event quality, trace coverage, redaction,
reproducibility, export boundary, and state candidate creation. Forbidden
effects include memory creation/promotion/write, persona mutation, raw export,
external sync/adapters, pm4py/ocpa runtime dependency, command/provider/local/
file/ask execution, final emission, rerun/retry/repair/autonomous loops, PIG
mutation/execution, Schumpeter split, and LLM judge use.

## CLI Commands

`v0.26.8` adds snapshot/export-only CLI commands:

- `python -m chantacore.cli workbench snapshot create`
- `python -m chantacore.cli workbench snapshot selection`
- `python -m chantacore.cli workbench snapshot manifest`
- `python -m chantacore.cli workbench snapshot refs`
- `python -m chantacore.cli workbench snapshot redaction`
- `python -m chantacore.cli workbench snapshot reproducibility`
- `python -m chantacore.cli workbench export ocel`
- `python -m chantacore.cli workbench export manifest`
- `python -m chantacore.cli workbench export quality`
- `python -m chantacore.cli workbench export trace-coverage`
- `python -m chantacore.cli workbench export boundary`
- `python -m chantacore.cli workbench snapshot report --report-id <id>`

The CLI creates or displays records only. It does not promote memory, export raw
material, sync externally, invoke pm4py/ocpa, execute commands/providers/ask,
or mutate files.
