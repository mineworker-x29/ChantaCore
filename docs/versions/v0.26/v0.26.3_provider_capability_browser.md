# v0.26.3 Provider / Capability Browser

Korean name: Provider·Capability Browser

Track: Workspace Agent Workbench

Previous: v0.26.2 Trace Explorer & Pipeline Timeline

Next: v0.26.4 Evidence / Report Inspector

## Objective

v0.26.3 creates provider and capability browser view artifacts for the
Workspace Agent Workbench. It reads existing v0.24 provider registry/capability
surfaces, v0.25 route/provider-selection/safety reports, v0.26.1 workbench view
state, and v0.26.2 trace explorer reports where available. It produces
refs-only provider cards, capability cards, boundary views, readiness views,
route compatibility rows, selection rationale views, boundary risk views, PIG
guidance views, failure mode views, human intervention point refs, inspection
summaries, and a provider browser report.

This release is view-only. It does not invoke providers, test-run providers,
register providers, implement adapters, render UI, execute ask/REPL, emit final
responses, execute local commands, promote memory, mutate persona, or persist
raw transcript/provider/secret material.

## Feedback Absorption Rule

Grok/Codex agent-usability feedback may be absorbed only as OCEL-visible
guidance, candidates, rationale, approval/deferral records, inspection surfaces,
and telemetry refs.

Allowed absorption:

- richer OCEL decision/event/relation coverage
- PIG guidance refs
- skill/action/provider candidates
- route/provider selection rationale
- provider capability readiness
- provider boundary risk view
- human intervention points
- approval/rejection/deferral records for later approval-console work
- failure cause classification refs
- refs-only session/context summaries
- evidence-bound action proposal metadata

Forbidden absorption:

- PIG as automatic memory
- PIG as persona mutation
- PIG as automatic behavior policy mutation
- PIG as execution authority
- autonomous multi-step execution loop
- automatic rerun or automatic repair
- provider boundary bypass
- external adapter runtime dependency
- process-mining runtime library dependency
- raw transcript persistence
- raw provider output dump

Directional external feedback is useful, but in the current roadmap it is
decomposed into OCEL-visible guidance, candidate, rationale, approval, and
inspection surfaces before any autonomy or memory interpretation.

## Relationships

v0.26.3 depends on v0.26.2 for Trace Explorer and Pipeline Timeline view
artifacts, but does not extend trace behavior. It depends on v0.26.1 for
WorkbenchViewState and the provider_browser panel model, but does not render
that panel. It depends on v0.26.0 for the Workbench contract. It reads v0.25.9
bounded agent foundation artifacts and v0.24.9 internal provider foundation
artifacts where available.

v0.26.x remains the Workspace Agent Workbench track. Memory Candidate &
Continuity remains deferred to v0.27. External provider adapter work remains
deferred to v0.29+. External Agent Dominion remains deferred to v0.30+.

OpenCode, OpenClaw, and Hermes remain reference architectures only. v0.26.3
does not add runtime dependencies, adapters, bridges, or copied runtime logic
from those projects.

## Core Principles

- Provider Browser is not provider invocation.
- Capability view is not permission.
- Provider boundary view is not boundary bypass.
- Provider selection rationale is explainability, not authority.
- PIG guidance is not memory.
- PIG guidance is not policy mutation.
- PIG guidance is not execution.
- Human intervention point is not approval by itself.
- Route compatibility is not route execution.
- Failure mode view is not provider test run.

## Included Scope

WorkbenchProviderBrowserRequest, WorkbenchProviderSourceView,
WorkbenchProviderBrowserView, WorkbenchProviderCard, WorkbenchCapabilityCard,
WorkbenchProviderBoundaryView, WorkbenchCapabilityReadinessView,

The service design loads prior artifacts through existing owners and converts
them into refs-only summaries:

- provider source view from registry, capability surface, routing, safety, and
  PIG refs
- provider cards and capability cards from provider registry/surface refs
- boundary and readiness views from declared provider policies
- route compatibility matrix from routing and capability refs
- provider selection rationale from v0.25.4 provider selection candidates
- boundary risk views from provider/capability boundary properties
- PIG guidance views from guidance refs only
- failure mode views from prior report refs only
- human intervention point refs from risk and boundary metadata
- provider inspection summary from the above view artifacts

watcher, autonomous loop, memory continuity, memory promotion, persistent memory

raw transcript/provider/secret persistence, or LLM judge.

## Status Rules

Passed when the provider browser panel, provider registry/capability surface,
provider/capability cards, boundary/readiness views, route compatibility matrix,
inspection summary, and safety boundary all pass.

Warning when optional route/provider selection, PIG guidance, failure mode, or
partial source refs are unavailable but no data is fabricated.

Failed when provider/capability cards, boundary views, readiness views, or route
compatibility matrix cannot be created.

Blocked when provider invocation, provider test run, boundary bypass, external
or vendor adapter, process-mining runtime dependency, PIG memory/policy/execution
use, ask/final/local command execution, direct file/subprocess access, command
rerun, automatic repair, memory/persona mutation, raw data exposure, Schumpeter
split, OpenCode/OpenClaw/Hermes runtime dependency, or LLM judge is detected.
