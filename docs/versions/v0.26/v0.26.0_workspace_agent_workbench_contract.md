## Objective

`v0.26.0` declares the contract-only Workspace Agent Workbench layer. It defines
surface modes, panel contracts, view permission policy, action boundary policy,
read-only inspection policy, approval policy, command boundary policy, snapshot
policy, trace privacy policy, OCEL visibility, roadmap boundaries, and a
contract report.

The release consumes `v0.25.9` Bounded General Agent Surface Foundation v1
artifacts as sanitized report references when available. It also relates back to
`v0.25.x` agent surface artifacts, `v0.24.9` Internal Provider / Local Runtime
Provider Foundation v1, and the `v0.23.9` OCEL-native Internal Dominion
Foundation.

## Core Principles

- Workbench contract is not UI implementation.
- Workbench surface is not autonomous execution.
- Workbench panel contract is not panel rendering.
- Workbench approval is not execution.
- Workbench command is not direct tool call.
- Workbench snapshot is not memory promotion.
- Workbench OCEL visibility is not raw transcript export.
- Workbench must increase human control, not create bypass paths.
- v0.27 memory continuity remains deferred.
- v0.29 external adapters remain deferred.
- v0.30 external agent dominion remains deferred.

OpenCode, OpenClaw, and Hermes remain reference architectures only. They may
inform future design patterns, but `v0.26.0` adds no runtime dependency,
adapter, bridge, fork-specific integration, or copied runtime logic.

## Surface Modes

The declared modes are `contract_view`, `trace_explorer_future`,
`pipeline_timeline_future`, `provider_browser_future`,
`evidence_inspector_future`, `safety_gate_view_future`,
`approval_console_future`, `run_dashboard_future`, `session_monitor_future`,
`command_surface_future`, `snapshot_export_future`, `blocked`, and `unknown`.

Future modes remain `contract_only`, `future_track`, `disabled`, or `blocked`.
They are not execution-capable, memory-capable, or external-adapter-capable in
this release.

## Panel Contracts

Required panel contracts are declared for trace explorer, pipeline timeline,
provider browser, evidence inspector, safety gate view, approval console, run
dashboard, session monitor, command surface, snapshot export, OCEL projection,
and telemetry metrics.

Each panel contract forbids raw data dumps, raw transcript exposure, raw
provider output exposure, raw secret output, provider invocation, direct
execution, memory promotion, and external adapter use.

## Policy Contracts

The view permission policy is deny-by-default while allowing read-only
inspection of sanitized trace, provider capability, evidence, safety, approval
candidate, run dashboard, and snapshot references.

The action boundary policy forbids direct provider invocation, direct file
access, direct subprocess use, direct local command execution, command rerun,
automatic repair, autonomous loops, and background execution. Future workbench
commands must use the existing `v0.25` agent surface, `v0.25.5` provider
invocation boundary, and `v0.24.7` local runtime gate.

The approval policy allows future approval candidates, manual approval, and
approval tokens, but approval is not execution and immediate or automatic
execution is forbidden.

The snapshot policy defers snapshot and OCEL export to `v0.26.8`. Snapshot is
not memory promotion, persistent memory write is forbidden, and raw transcript,
provider output, secrets, and private company material cannot be exported.

The trace privacy policy requires refs-only trace views by default and forbids
raw transcript, provider output, secret, and credential storage.

## Acceptance Criteria

- `WorkspaceAgentWorkbenchContract` can be created.
- Required `WorkbenchPanelContract` records can be created.
- All permission, boundary, approval, command, snapshot, privacy, OCEL, and
  roadmap policy contracts can be created.
- `WorkbenchContractReport` can be created.
- `ready_for_v0_26_1` can be true.
- `ready_for_v0_27` remains false.
- No actual UI or panel implementation is added.
- No ask/REPL execution, final response emission, provider invocation, local
  command execution, memory promotion, persistent memory write, persona
  mutation, external adapter, Schumpeter split, credential exposure, raw data
  persistence/output, or LLM judge is added.

## Withdrawal Conditions

Withdraw the `v0.26.0` contract-only readiness claim if this release renders
actual UI, creates actual panel state beyond contract declarations, executes
ask/REPL, emits responses, invokes providers, executes local commands, creates a
direct bypass, promotes memory, mutates persona, implements external adapters,
introduces Schumpeter split logic, persists raw transcripts/provider
outputs/secrets, exposes credentials, or uses an LLM judge.

## Validity Horizon

Valid until `v0.26.1` Workbench View State & Panel Model begins or the workbench
policy boundary changes.
