# v0.26.1 Workbench View State & Panel Model

Version: `v0.26.1`
Name: Workbench View State & Panel Model
Korean name: Workbench View State·Panel Model
Track: Workspace Agent Workbench
Previous: `v0.26.0` Workspace Agent Workbench Contract
Next: `v0.26.2` Trace Explorer & Pipeline Timeline

## Objective

`v0.26.1` creates the view-state and panel-model layer for the Workspace Agent
Workbench. It consumes the `v0.26.0` workbench contract and panel contracts as
sanitized references, then creates panel model records, panel registry, default
layout, selection state, filter state, focus state, navigation state, session
view, workbench view state, and a view-state report.

This release is still a model layer. It does not render UI, implement panel
behavior, execute agent turns, invoke providers, execute commands, promote
memory, or expose raw data.

## Relationship

`v0.26.1` follows the `v0.26.0` Workspace Agent Workbench Contract. It prepares
the substrate needed by `v0.26.2` Trace Explorer & Pipeline Timeline without
implementing trace-specific behavior.

It also depends on the `v0.25.9` Bounded General Agent Surface Foundation v1
handoff and trace/telemetry references from `v0.25.8` when available.

`v0.26.x` remains the Workspace Agent Workbench track. `v0.27.x` memory
continuity, `v0.28.x` public alpha / Schumpeter split preparation, `v0.29.x+`
external provider adapters, and `v0.30.x+` external agent dominion remain
deferred.

OpenCode, OpenClaw, and Hermes remain reference architectures only. No runtime dependency,
adapter, bridge, fork-specific integration, or copied runtime logic is added in `v0.26.1`.

## Principles

- Workbench view model is not rendered UI.
- Panel registry is not panel implementation.
- Panel slot is not visual component.
- Selection state is not approval.
- Filter state is not data deletion.
- Focus state is not provider invocation.
- Navigation state is not background routing.
- Session view is not durable memory.
- View state may reference artifacts, but must not inline raw transcript,
  provider output, or secrets.

## State Model

The view state is refs-only by default. Selection, filter, focus, navigation,
and session view records may point to prior workbench, trace, telemetry, or
session artifacts, but they do not inline raw transcript, provider output,
private full paths, credentials, or secrets.

Panel slots declare region and ordering only. A slot is not a visual component,
and a layout is not a dashboard implementation.

## Acceptance Criteria

- `WorkbenchViewState` can be created.
- Required `WorkbenchPanel` models can be created.
- `WorkbenchPanelRegistryView`, `WorkbenchPanelLayout`, selection, filter,
  focus, navigation, and session view records can be created.
- `WorkbenchViewStateReport` can be created.
- `ready_for_v0_26_2` can be true.
- `ready_for_v0_27` remains false.
- No actual UI or panel rendering is implemented.
- No panel-specific behavior is implemented.
- No ask/REPL execution, final response emission, provider invocation, local
  command execution, memory promotion, persistent memory write, persona
  mutation, external adapter, Schumpeter split, raw data persistence/output, or
  LLM judge is added.

## Withdrawal Conditions

Withdraw the `v0.26.1` view-state/panel-model readiness claim if this release
renders UI, implements panel behavior, implements trace/provider/evidence/
approval/dashboard/command/snapshot behavior early, executes ask/REPL, emits
responses, invokes providers, executes commands, promotes memory, mutates
persona, implements adapters, introduces Schumpeter split logic, persists raw
data, exposes credentials, or uses an LLM judge.

## Validity Horizon

Valid until `v0.26.2` Trace Explorer & Pipeline Timeline begins or workbench
view-state policy changes.
