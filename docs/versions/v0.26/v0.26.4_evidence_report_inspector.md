# v0.26.4 Evidence / Report Inspector

Korean name: Evidence·Report Inspector

Track: Workspace Agent Workbench

Previous: v0.26.3 Provider / Capability Browser

Next: v0.26.5 Safety Gate / Approval Console

## Objective

`v0.26.4` creates OCEL-visible Evidence / Report Inspector view artifacts. It
reads prior response assembly, evidence bundle, claim/support, safety gate,
routing, provider selection, provider browser, trace, PIG guidance, and failure
cause references when available. It creates refs-only inspector artifacts that
make support, uncertainty, limitation, rationale, and failure surfaces
inspectable without changing the underlying artifacts.

## Relationships

- `v0.26.3` supplies provider/capability browser refs, provider rationale,
  boundary risk, readiness, PIG guidance, failure mode, and human intervention
  point refs.
- `v0.26.2` supplies trace explorer and pipeline timeline refs.
- `v0.26.1` supplies WorkbenchViewState and the `evidence_inspector` panel
  model.
- `v0.25.6` supplies response assembly, evidence bundles, claims, claim
  supports, answer drafts, assembled response artifacts, uncertainty notes, and
  limitation notes.

## Grok / Codex Feedback Absorption

External agent-usability feedback may be absorbed only as OCEL-visible guidance,
candidate, rationale, approval-reference, inspection, and telemetry surfaces.
For this release that means decision evidence inspection, skill/action/route/
provider selection evidence inspection, safety rationale inspection, PIG
guidance inspection, failure cause inspection, unsupported claim inspection,
uncertainty and limitation inspection, and evidence-bound action proposal
metadata.

The feedback is not absorbed as autonomy or memory. PIG guidance is not long-term
memory, persona mutation, automatic behavior policy mutation, or execution
authority. Automatic rerun and automatic repair remain outside this release.

## Service Design

The inspector uses a prerequisite source service that loads prior owner-built
artifacts through their report services and keeps only references and sanitized
summaries. The downstream services build view artifacts, not executable work:

- source view service gathers refs and source status.
- report inspector service counts and classifies report refs.
- evidence bundle/item services expose evidence refs and counts.
- claim/support services expose support status and missing evidence reasons.
- decision, skill, action, route, provider, safety, PIG, failure, unsupported,
  uncertainty, and limitation services expose refs-only explanation surfaces.
- filter and selection services hide or select refs without deleting data,
  mutating access policy, approving, or executing.
- inspection summary service summarizes refs and boundary flags.
- finding/report services compute status and readiness.

## Status Rules

`passed` is possible when the inspector view, report inspector, evidence source,
claim/support/evidence views, rationale views, and inspection summary build
without any forbidden operation. `warning` is used for optional missing sources,
unsupported claims, uncertainty, limitations, or partial sources. `failed` is
used for missing inspectable sources under strict mode. `blocked` is used if raw
content is inlined, responses are rewritten, an LLM judge is used, providers are
invoked, routes or stages rerun, approval or commands execute, PIG is treated as
memory/policy/execution, memory/persona mutates, adapters are implemented, or
Schumpeter split is introduced.

## OCEL Mapping

The release adds object, event, relation, and effect mappings for evidence
inspector policy, request, source view, inspector view, report inspector view,
evidence bundle/item views, claim/support inspector views, decision/skill/action
/route/provider evidence views, safety rationale, PIG guidance inspector,
failure cause, unsupported claim, uncertainty, limitation, filter, selection,
inspection summary, finding, and report artifacts.

Allowed effect types are read-only observation, evidence/report/claim/decision/
skill/action/route/provider/safety/PIG/failure/unsupported inspection creation,
evidence inspection creation, and state candidate creation. Forbidden effects
include response rewrite, LLM judge, provider invocation, route/stage rerun,
approval execution, ask/REPL execution, final response emission, local command
execution, command rerun, automatic repair, memory/persona mutation, adapter
implementation, PIG memory/policy/execution, credential exposure, raw secret
output, raw provider output inline, and raw transcript persistence.

## Acceptance Criteria

- Evidence inspector, report inspector, evidence bundle/item, claim/support,
  decision, skill, action, route, provider, safety, PIG, failure, unsupported,
  uncertainty, limitation, filter, selection, inspection summary, finding, and
  report artifacts can be created.
- The artifacts are refs-only by default and OCEL-visible.
- `ready_for_v0_26_5` can be true and `ready_for_v0_27` remains false.
- No response rewrite/regeneration, LLM judge, provider invocation/test-run,
  route/stage rerun, approval execution, ask/REPL, final response emission,
  local command execution, command rerun, automatic repair, memory/persona
  mutation, adapter implementation, Schumpeter split, credential output, raw
  transcript persistence, raw provider output dump, or raw secret output occurs.

## Withdrawal Conditions

Withdraw this release if evidence/report inspection starts rewriting responses,
judging factuality with an LLM, invoking providers, rerunning routes or stages,
executing approvals or commands, mutating safety policy/PIG/memory/persona,
implementing adapters, persisting raw transcript/provider/secret material, or
introducing company-specific split behavior.

## Validity Horizon

Valid until v0.26.5 Safety Gate / Approval Console begins or evidence inspector
policy changes.
