# v0.26.2 Trace Explorer & Pipeline Timeline

Korean name: Trace Explorer·Pipeline Timeline

Track: Workspace Agent Workbench

Previous: v0.26.1 - Workbench View State & Panel Model

Next: v0.26.3 - Provider / Capability Browser

## Definition

v0.26.2 creates read-only Trace Explorer and Pipeline Timeline view artifacts
for the Workspace Agent Workbench track. It reads v0.26.1 view-state and panel
model records, v0.26.0 workbench contract boundaries, and v0.25.8
Agent Trace / Usability Telemetry artifacts when available.

The objective is explainability over prior trace refs. It does not add a
browser UI, server dashboard, visual rendering, provider browser behavior,
evidence inspector behavior, approval console behavior, run dashboard behavior,
command surface behavior, snapshot/export behavior, stage rerun, route rerun,
ask/REPL execution, final response emission, provider invocation, local command
execution, memory continuity, external adapters, Schumpeter split, raw
transcript/provider/secret persistence, or an LLM judge.

## Roadmap Policy

v0.26.x remains Workspace Agent Workbench. v0.27.x remains Memory Candidate &
Continuity. v0.28.x remains Public Alpha / Schumpeter Split Preparation.
v0.29.x+ remains External Skill / External Provider Adapter Development.
v0.30.x+ remains External Agent Dominion Bridge.

ChantaCore directly implements its own OCEL-native agent spine. OpenCode,
OpenClaw, and Hermes remain reference architectures only. They may inform
future architectural thinking, but v0.26.2 adds no runtime dependency, adapter,
bridge, fork-specific integration, or copied runtime logic for them.

## Principles

- Trace Explorer view is not UI rendering.
- Pipeline Timeline is not execution.
- Timeline node is not executable command.
- Stage node is a view of prior stage artifact.
- Decision node is a view of prior decision artifact.
- Provider node is trace view, not provider invocation.
- Response node is trace view, not response emission.
- Trace filter is not data deletion.
- Trace inspection is read-only.
- Trace inspection may summarize refs, but must not inline raw transcript,
  raw provider output, credentials, or secrets.

## Service Design

WorkbenchTraceExplorerPrerequisiteSourceService loads v0.26.1 view-state panel
models and v0.25.8 trace telemetry refs. The source service is read-only and
does not load raw transcripts, raw provider outputs, secrets, private full
paths, credentials, processes, plugins, or external providers.

WorkbenchTimelineNodeService creates stage, decision, route, provider, and
response node models. WorkbenchRelationEdgeService links those nodes as
model-only edges. WorkbenchTraceFilterStateService and
WorkbenchTraceSelectionViewService expose refs-only view state. Trace
inspection creates summaries and reports without mutating traces.

## Status Rules

Readiness can pass when view-state, trace explorer panel, pipeline timeline
panel, surface trace, stage nodes, decision nodes, relation edges, and
inspection report exist and privacy boundaries are respected.

Partial optional route/provider/response trace sources create warnings rather
than fabricated trace data. Missing required trace sources under strict
readiness fail or block. Any UI rendering, trace mutation, rerun, execution,
provider invocation, memory mutation, external adapter, raw data persistence,
Schumpeter split, OpenCode/OpenClaw/Hermes runtime dependency, or LLM judge
blocks the release.

Capability Browser.

## Acceptance Criteria

- WorkbenchTraceExplorerView can be created.
- WorkbenchPipelineTimeline can be created.
- WorkbenchStageNode, WorkbenchDecisionNode, and WorkbenchRelationEdge can be
  created.
- WorkbenchTraceFilterState and WorkbenchTraceInspectionReport can be created.
- ready_for_v0_26_3 can be true.
- ready_for_v0_27 remains false.
- No actual UI/panel rendering is implemented.
- No stage or route rerun occurs.
- No ask/REPL execution, final response emission, provider invocation, local
  command execution, memory promotion, persistent memory write, persona
  mutation, external adapter, OpenCode/OpenClaw/Hermes runtime dependency,
  Schumpeter split, credential output, raw transcript/provider/secret output,
  or LLM judge is introduced.

## Withdrawal Conditions

Withdraw v0.26.2 readiness if the implementation renders UI, mutates trace
artifacts, reruns stages or routes, executes ask/REPL, emits a response,
invokes providers, executes local commands, performs direct file or subprocess
bypass, starts autonomous/background work, promotes memory, mutates persona,
adds external adapters, introduces Schumpeter split, depends on
OpenCode/OpenClaw/Hermes runtimes, persists raw transcript/provider/secret
material, exposes credentials, or uses an LLM judge.

## Validity Horizon

Valid until v0.26.3 Provider / Capability Browser begins or trace explorer
policy changes.
