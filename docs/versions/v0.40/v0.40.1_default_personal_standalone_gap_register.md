# v0.40.1 Default Personal Standalone Gap Register

## 1. Current Status

v0.40.1 does not open the standalone Default Personal agent runtime.

It adds a gap register and acceleration assessment so the project can see what remains before a user-facing Default Personal smoke run is technically meaningful.

## 2. Why Default Personal Is Not Yet Standalone

Standalone Default Personal testing begins only when user input, profile loading, prompt assembly, AgentLoop, read-only SkillExecution, response generation, and event/trace emission are connected end-to-end.

v0.40.1 has not connected those surfaces end-to-end. It only records the missing surfaces and rehearses the existing sandbox apply/retest primitives inside a bounded sandbox context.

Default Personal is not a permission bypass mode. It does not override model, prompt, subagent, live apply, shell, network, Dominion, or production certification gates.

## 3. Required Components

The required standalone components are:

- ChatService.
- Orchestrator.
- AgentLoop.
- SkillRegistry.
- SkillExecutor.
- PromptAssembly.
- ProfileRuntime.
- SessionStore.
- EventTraceEmitter.
- UserFacingCLI.
- DefaultPersonalSmokeScenario.

## 5. Conservative v0.41.6 Smoke Run Target

The conservative first smoke run target remains v0.41.6.

This target assumes v0.41.0 through v0.41.5 must first establish profile runtime, CLI entry, prompt assembly, read-only skill registry, minimal AgentLoop, and event/trace emission.

## 6. Possible Acceleration Criteria

Acceleration is possible only if required gaps close earlier than expected:

- If ChatService, UserFacingCLI, AgentLoop, SkillExecutor, and ProfileRuntime are all missing, keep v0.41.6.
- If only UserFacingCLI and ProfileRuntime are missing, v0.41.4 may be possible.
- If only smoke scenario/docs remain, v0.41.3 may be possible.

Acceleration is not permission to open autonomous loops, model invocation, prompt submission, subagent invocation, live apply, Dominion runtime, or production certification.

## 7. Minimum User-Facing Commands

Minimum commands for the future runtime:

- chanta personal status.
- chanta personal ask.
- chanta personal inspect.
- chanta personal trace recent.
- chanta personal dry-run.

These commands are v0.41 targets, not v0.40.1 runtime features.

## 8. Minimum Read-Only Skills

Minimum read-only skills for the future runtime:

- status_summary.
- inspect_file.
- inspect_directory.
- summarize_recent_trace.
- dry_run_mission_loop.
- explain_boundary_report.

v0.40.1 does not open SkillLoader or SkillExecutor execution.

## 9. Minimum Event / Trace Emission

Minimum future events:

- user_goal_received.
- profile_loaded.
- prompt_assembled.
- skill_selected.
- skill_executed.
- observation_recorded.
- response_emitted.

v0.40.1 does not persist traces or write OCEL/OCPX artifacts.

## 10. v0.41 Recommended Sequence

Recommended v0.41 sequence:

1. v0.41.0 Default Personal Profile Runtime.
2. v0.41.1 User-facing CLI Entry.
3. v0.41.2 Prompt Assembly / Soul-Role-Domain Binding.

5. v0.41.4 Minimal AgentLoop.
6. v0.41.5 Event / Trace Emission.
7. v0.41.6 First Default Personal Smoke Run.

The sequence is a handoff draft. It is not standalone runtime activation.
