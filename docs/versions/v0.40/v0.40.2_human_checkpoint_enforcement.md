# v0.40.2 Human Checkpoint Enforcement

## 1. Why Human Checkpoint Is Mandatory

The second iteration is the first point where a bounded rehearsal could be mistaken for an autonomous loop. v0.40.2 makes the checkpoint mandatory so iteration 1 cannot proceed unless explicit human checkpoint decision metadata exists.

v0.40.2 does not open standalone Default Personal agent runtime. It only proves that a manual second iteration candidate can be gated by explicit human checkpoint metadata.

## 2. Checkpoint Request

ManualIterationCheckpointRequest is emitted after iteration 0. It is required, defaults to stop, and includes the decision options:

- approve_second_iteration_rehearsal,
- request_more_evidence,
- stop,
- do_nothing,
- reject.

The request does not grant runtime authority.

## 3. Checkpoint Decision

ManualIterationCheckpointDecision records the human decision as metadata. It may approve a second iteration rehearsal only when:

- decision is approve_second_iteration_rehearsal,
- approved_iteration_index is 1,
- approved_rehearsal_only is true,
- approval_grants_runtime_authority is false.

## 4. Approved Rehearsal Is Not Runtime Authority

Approval of a second iteration rehearsal does not grant runtime authority, live workspace mutation authority, model invocation authority, prompt submission authority, or subagent invocation authority.

The approval only makes a bounded manual sandbox rehearsal candidate eligible.

## 5. Missing Checkpoint Behavior

When checkpoint decision metadata is missing, SecondIterationEligibilityDecision returns:

- eligible false,
- reason human_checkpoint_missing,
- checkpoint_required true,
- checkpoint_present false,
- runtime_authority_granted false.

## 6. Stop / Do Nothing / Reject Behavior

The stop, do_nothing, and reject decisions all block iteration 1. They do not create rehearsal eligibility and do not grant runtime authority.

## 7. Request More Evidence Behavior

request_more_evidence blocks iteration 1 and indicates that more evidence is required before a second rehearsal can be considered.

## 8. Approved Second Iteration Rehearsal Behavior

approve_second_iteration_rehearsal allows only manual bounded second rehearsal eligibility. It does not allow autonomous continuation, retry loops, automatic repair, live workspace apply, model invocation, prompt submission, subagent invocation, or standalone runtime.

## 9. Audit Requirements

ManualTwoIterationAuditRecord confirms:

- max two iterations,
- checkpoint between iterations,
- no autonomous continuation,
- no runtime authority grant,
- no live workspace mutation,
- no model invocation,
- no prompt submission,
- no subagent invocation,
- standalone runtime not opened.

## 10. Safety Flags

ManualTwoIterationSafetyReport keeps unsafe surfaces false:

- safe_for_autonomous_loop false,
- safe_for_live_apply false,
- safe_for_model_invocation false,
- safe_for_prompt_submission false,
- safe_for_subagent_invocation false,
- safe_for_standalone_default_personal_runtime false.

ManualTwoIterationReadinessReport keeps ready_for_execution, ready_for_live_workspace_apply, ready_for_prompt_submission_to_model, ready_for_model_provider_invocation, ready_for_subagent_invocation, ready_for_autonomous_loop_runtime, ready_for_retry_loop, ready_for_multi_cycle_loop, ready_for_standalone_default_personal_runtime, ready_for_dominion_runtime, and production_certified false.
