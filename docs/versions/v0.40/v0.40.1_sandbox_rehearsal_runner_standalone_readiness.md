# v0.40.1 Sandbox Rehearsal Runner & Standalone Agent Readiness Clarification

Release name: v0.40.1 Sandbox Rehearsal Runner & Standalone Agent Readiness Clarification.

Track name: Standalone-Agent Preparation Track: Controlled MissionLoop Boundary + Sandbox Rehearsal + Manual Runtime Gap Accounting.

## 1. Purpose

v0.40.1 connects the v0.40.0 controlled mission-loop boundary to a bounded sandbox rehearsal surface. It can rehearse a single sandbox-only text replacement and a supplied-runner controlled retest using the existing v0.39 primitives, then record result, audit, safety, readiness, and standalone Default Personal gap metadata.

This release does not start the standalone Default Personal agent runtime.

## 2. Baseline from v0.40.0

v0.40.0 established metadata for:

- MissionLoopEnvelope.
- IterationState.
- StopConditionContract.
- LoopBudgetGate.
- HumanCheckpointGate.
- ProviderBoundaryGate.
- PromptSubmissionBoundary.
- VerifierSubagentBoundary.
- SimulatedMultiIterationLoopPacket.
- V040ReadinessReport.

v0.40.1 treats those as upstream boundary context. It does not replace or reinterpret v0.40.0.

## 3. Why v0.40.1 Adds Sandbox Rehearsal

v0.40.0 could describe, simulate, deny, and guide. It did not exercise the existing sandbox apply and controlled retest primitives together.

v0.40.1 adds a temp-sandbox rehearsal path so the project can verify whether the v0.39.3 sandbox-only apply primitive and the v0.39.4 supplied-runner retest primitive compose without opening live workspace mutation, raw shell, model invocation, prompt submission, subagent invocation, or autonomous continuation.

v0.40.1 reuses v0.39.3 through:

v0.40.1 reuses v0.39.4 through:

- create_repair_controlled_test_command_spec.

The v0.40.1 module does not define a new raw file mutation helper and does not define a subprocess or shell runner.

## 7. Sandbox Rehearsal Flow

1. Build SandboxRehearsalInput with an explicit sandbox_root_ref and relative target path.
2. Build SandboxRehearsalApplyPlan confirming v0.39.3 reuse and blocking live workspace, git apply, and apply_patch.
3. Build SandboxRehearsalRetestPlan confirming v0.39.4 reuse and blocking shell, network, and dependency install.
4. Apply exact text replacement inside the supplied sandbox root.
5. Invoke the v0.39.4 controlled retest boundary with a supplied or fake runner callable.
6. Return SandboxRehearsalResult, SandboxRehearsalAuditRecord, and SandboxRehearsalSafetyReport.

## 8. Safety Boundaries

The rehearsal remains sandbox-only. It requires exact original text match, explicit sandbox root, relative target path, and supplied-runner retest semantics.

The result must keep:

- sandbox_only true.
- live_workspace_mutated false.
- model_invoked false.
- prompt_submitted false.
- subagent_invoked false.
- external_agent_invoked false.
- runtime_authority_granted false.

## 9. Standalone Agent Runtime Clarification

v0.40 does not start the standalone Default Personal agent runtime.

v0.40 prepares the process-safe mission loop substrate required before the standalone agent runtime can be opened.

Standalone Default Personal testing begins only when user input, profile loading, prompt assembly, AgentLoop, read-only SkillExecution, response generation, and event/trace emission are connected end-to-end.

In the current roadmap, this begins in v0.41. The conservative first user-facing smoke run target is v0.41.6.

## 11. Acceleration Assessment toward v0.41 Smoke Run

The acceleration assessment can suggest an earlier smoke target only when core runtime gaps are smaller than expected:

- If ChatService, UserFacingCLI, AgentLoop, SkillExecutor, and ProfileRuntime are still missing, keep v0.41.6.
- If only UserFacingCLI and ProfileRuntime are missing, v0.41.4 may be possible.
- If only smoke scenario/docs remain, v0.41.3 may be possible.

The assessment does not start the standalone runtime.

## 13. Known Limitations

- v0.40.1 rehearses one bounded sandbox apply/retest path, not a manual two-iteration mission loop.
- v0.40.1 does not open SandboxRehearsalRunner as a user-facing command.
- v0.40.1 does not execute model calls, prompt submission, subagents, or external agents.
- v0.40.1 does not create persistent trace output.

## 14. Withdrawal Conditions

Withdraw v0.40.1 if any of the following appear:

- Live workspace mutation.
- Sandbox rehearsal mutation outside the temp sandbox.
- v0.39.3 primitive bypassed by a new unsafe apply helper.
- v0.39.4 primitive bypassed by a raw subprocess/shell runner.
- shell=True, os.system, eval, exec, or direct subprocess use.
- Model/provider invocation.
- Prompt submission.
- Subagent invocation.
- Standalone AgentLoop or standalone CLI/TUI.
- ready_for_live_workspace_apply true.
- ready_for_autonomous_loop_runtime true.
- ready_for_model_provider_invocation true.
- ready_for_subagent_invocation true.
- ready_for_dominion_runtime true.
- production_certified true.
