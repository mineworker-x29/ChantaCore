## 9. v0.40.6 Verifier Subagent Boundary Summary

v0.40.6 defines the verifier subagent boundary as a draft-only metadata surface. It can describe what a verifier would review, what evidence is required, what context would be isolated, and why dispatch is blocked.

It cannot invoke a subagent, create a child session, share parent raw transcript, grant subagent permissions, execute verifier tools, or trust verifier output.

## 10. Subagent Verification Request Draft Contract

`SubagentVerificationRequestDraft` is metadata-only and draft-only. `invocation_requested` may be true only as metadata. The draft keeps:

```text
metadata_only=True
draft_only=True
invocation_allowed=False
subagent_invoked=False
child_session_created=False
parent_raw_transcript_shared=False
runtime_authority_granted=False
```

## 11. Verifier Role Contract

Allowed verifier roles are review/evidence roles only:

```text
evidence_reviewer
boundary_reviewer
safety_invariant_reviewer
restore_document_reviewer
test_result_reviewer
readiness_flag_reviewer
```

Forbidden roles remain rejected:

```text
executor
patch_applier
provider_caller
prompt_submitter
live_workspace_operator
autonomous_repair_agent
dominion_operator
production_certifier
```

## 12. Verifier Evidence Requirement Contract

Evidence must be complete and matched before a verifier draft can be considered draft-eligible. Required evidence refs are:

```text
loop_decision_ref
checkpoint_decision_ref
safety_report_ref
negative_gate_report_ref
provider_prompt_boundary_report_ref
restore_document_ref
readiness_report_ref
```

Missing or mismatched evidence blocks draft eligibility. Complete evidence may set `eligible_for_verifier_draft=True`, but it still grants no invocation authority.

## 13. Verifier Context Isolation Contract

The context isolation contract keeps:

```text
parent_raw_transcript_shared=False
parent_hidden_context_shared=False
parent_credentials_shared=False
parent_runtime_authority_shared=False
summary_packet_required=True
structured_input_required=True
evidence_refs_required=True
child_context_isolated=True
```

Only a structured summary packet with evidence refs is allowed as future input metadata. Parent raw transcript and hidden context remain closed.

## 14. Verifier Dispatch Gate Contract

`VerifierSubagentDispatchGatePolicy` is deny-by-default:

```text
deny_by_default=True
invocation_allowed=False
child_session_creation_allowed=False
parent_context_sharing_allowed=False
permission_grant_allowed=False
metadata_only=True
```

`VerifierSubagentDispatchGateEvaluation` always blocks dispatch in v0.40.6:

```text
blocked=True
subagent_invoked=False
child_session_created=False
parent_raw_transcript_shared=False
executed=False
metadata_only=True
```

## 15. Verifier Result Quarantine Contract

A future verifier result must be quarantined before any trust, persistence, process-state use, memory use, or execution.

The quarantine contract keeps:

```text
raw_verifier_result_trusted=False
direct_persistence_allowed=False
direct_process_state_update_allowed=False
direct_memory_write_allowed=False
direct_execution_allowed=False
requires_schema_validation=True
requires_human_review=True
requires_provenance=True
```

The quarantine envelope keeps:

```text
quarantined=True
validated=False
human_reviewed=False
eligible_for_persistence=False
eligible_for_process_state=False
eligible_for_memory=False
eligible_for_execution=False
metadata_only=True
```

## 20. Why v0.40.6 Is Still Not Standalone

v0.40.6 adds verifier subagent boundary metadata only. A standalone runtime would require executable session orchestration, safe CLI/user surface, profile runtime, bounded tool execution, provider/prompt dispatch, subagent invocation rules, workspace mutation rules, and production-grade safety gates. Those are not opened in this release.

## 25. Known Limitations

v0.40.6 does not implement a verifier subagent runtime, child session, subagent dispatcher, subagent tool execution, prompt submission, provider invocation, or result trust pipeline.

v0.40.6 does not define the complete cross-version evidence matrix. That is deferred to v0.40.7.

v0.40.6 does not open standalone Default Personal runtime. That remains future-gated to v0.41+ and still depends on missing runtime surfaces such as ChatService, CLI, ProfileRuntime, AgentLoop, and SkillExecutor.
