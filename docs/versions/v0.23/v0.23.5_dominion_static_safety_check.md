views, and PIG/OCPX projection.

## Restore Boundary

- Static safety is not execution.
- Static safety check is not preflight.
- Static safety check is not provider API call.
- Static safety check is not external runtime touch.
- Static safety check is not authorization.
- Static safety check is not dispatch.
- `llm_judge_used: bool = False`
- Next version: v0.23.6 Runtime Preflight / Reachability Check.
- Static safety must not call providers or runtimes.
- Static safety must not use an LLM judge.
- Credential values and raw secrets must not be materialized or output.

## Verification

Run the primary tests and confirm all dangerous execution counts remain zero.

## Restore Detail

Restore v0.23.5 as deterministic static safety over Dominion control plans.
Static safety checks plan structure, policy, scope, and declared constraints
without executing the plan.

Required restored surfaces:

- static safety request/report model
- rule category model or equivalent rule representation
- finding model
- static safety service
- deterministic pass/warning/fail/block status rules
- OCEL mapping for static safety reports and findings
- PIG/OCPX projection entries for static safety visibility
- CLI static safety view

Static safety must make unsupported or dangerous conditions visible, but must
not validate them by calling a runtime, provider, shell, network, or LLM judge.

## Expected Model And Policy State

The restored static safety layer should preserve checks for:

- missing control plan
- missing target binding
- missing static constraints
- unsupported provider/runtime references
- credential materialization risk
- raw secret output risk
- provider-specific logic in core
- premature local runtime or agent UX behavior
- vendor hardcoding in core paths

The report should expose findings, severity, evidence refs, withdrawal
conditions, and status. Evidence refs must be sanitized.

## CLI And Trace Expectations

CLI output should show report id, status, finding count, severe finding types,
and non-secret evidence refs. OCEL should record static safety report creation,
not safety execution against live systems.

## Restore Procedure

1. Confirm `static_safety.py` exists.
2. Confirm static safety tests and boundary tests exist.
3. Build a static safety report from a declared control plan.
4. Confirm no provider/runtime/shell/network execution occurs.
5. Confirm findings are deterministic.
6. Run primary tests.
7. Run forbidden searches for shell, network, provider API calls, runtime touch,
   credentials, and LLM judge.

## Evidence Strength

Evidence strength is high for deterministic local tests and policy fields. It
is not evidence that a live provider action is safe; v0.23.5 does not test live
provider execution.

## Withdrawal Conditions

Withdraw this addendum if static safety starts executing checks against live
runtimes, provider APIs, shell commands, or LLM judgment.

## Validity Horizon

Valid while v0.23.x remains deterministic and non-executing.
