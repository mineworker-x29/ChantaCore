## Restore Detail

Restore v0.23.0 by rebuilding the Internal Dominion vocabulary before restoring
any later v0.23 subject. Later versions assume these objects, event names,
relation names, skill ids, and effect policies already exist.

Required restored surfaces:

- Internal Dominion contract model and report model
- seed subject definitions for contract, inventory, capability, request,
  candidate, plan, safety, preflight, gate, dispatch boundary, workbench, and
  consolidation
- skill taxonomy entries for Dominion contract view and future Dominion skills
- migration audit surface for reclassifying self-execution language out of
  v0.23.x
- OCEL object/event/relation/effect constants
- PIG and OCPX projection helpers for Internal Dominion visibility
- CLI contract view and migration/conformance visibility where present

The contract must describe Dominion as control grammar, not control execution.
If a future maintainer cannot determine this from the restored source alone,
the restore is incomplete.

## Expected Model And Policy State

The restored v0.23.0 model layer must preserve these semantic categories:

- internal dominion contract state
- internal dominion subject
- internal dominion skill
- internal dominion effect policy
- provider interface boundary
- gate contract
- migration/audit finding
- PIG report view
- OCPX projection view

Allowed effects remain non-executing, such as `read_only_observation` and
`state_candidate_created`. Forbidden effects include external runtime touch,
external control dispatch, shell execution, network access, credential exposure,
provider API call, authorization consumption, and raw secret output.

## CLI And Trace Expectations

The restored CLI should let a maintainer view the contract without mutating
state or executing any provider/runtime behavior. The expected trace posture is
OCEL-native: objects, events, relations, and effects must be representable in
OCEL terms and must not rely on JSONL as canonical state.

CLI output must avoid credential values, raw secrets, and private absolute
paths. Any report path or evidence reference should be relative, synthetic, or
sanitized.

## Restore Procedure

1. Confirm all primary files listed above exist.
2. Confirm the original version contract still exists in `docs/versions/`.
3. Import the Internal Dominion contract/report classes.
4. Confirm skill ids include `skill:dominion_contract_view`.

   seeded.
6. Confirm self-execution language is historical or future-track, not active
   v0.23 execution.
7. Run the primary tests.
8. Run the foundation forbidden-search checks.
