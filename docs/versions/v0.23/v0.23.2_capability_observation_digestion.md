## Restore Detail

Restore v0.23.2 as a capability observation and digestion layer. Observation
records possible capabilities; digestion normalizes them into Dominion-visible
candidate state. Neither step executes the capability.

Required restored surfaces:

- external capability candidate descriptors
- capability observation report
- capability digestion report
- risk and confidence fields based on deterministic rules
- source/evidence references
- OCEL object/event/relation mappings for observed and digested capabilities
- PIG/OCPX projection entries for capability observation
- CLI views for capability observation/digestion

Observed capabilities may be incomplete or unsupported. That must create
warnings, blocked status, or candidate-only state rather than an attempt to
execute or validate the capability against a live provider.

## Expected Model And Policy State

The restored capability layer should preserve:

- capability id and provider reference fields as descriptors
- input/output schema references as declared or candidate values
- risk notes and unsupported reasons
- effect policy that forbids actual capability execution
- provenance references that are sanitized and non-secret

Forbidden states include provider invocation, runtime touch, local command
execution, live capability probing, raw credential output, and LLM-judge-based
safety decisions.

## CLI And Trace Expectations

CLI output should show capability candidate ids, status, risk summaries, and
evidence refs. It must not print raw provider payloads, credentials, tokens, or
private absolute paths. OCEL traces should show observation/digestion events,
not execution events.

## Restore Procedure

1. Confirm `capability.py` exists.
2. Confirm capability observation tests and boundary tests exist.
3. Build an observation/digestion report from fixtures or declared input.
4. Confirm no provider invocation or runtime touch occurs.
5. Confirm all emitted effects are observation or candidate effects.
6. Run the primary tests.
7. Run forbidden searches for provider calls, command execution, credentials,
   and LLM judge references.
