## Restore Detail

Restore v0.23.7 as human review and gate-state creation. The gate can open,
reject, request more input, or produce no-action state according to declared
policy. Gate authorization may be created, but v0.23.7 must not consume it.

Required restored surfaces:

- human review request/report model
- Dominion gate report model
- Dominion gate authorization model
- gate condition policy
- decision/status rules
- finding model
- source/evidence refs
- OCEL mapping for gate reports and authorizations
- PIG/OCPX projection entries for review and gate visibility
- CLI gate and authorization views

The restored gate layer should preserve human review semantics without turning
approval into execution.

## Expected Model And Policy State

The restored gate layer should preserve:

- gate report id
- gate status and review decision
- authorization id if created
- single-use authorization semantics
- authorization consumed flag false
- expiration/scope fields if present
- source refs to static safety, preflight, plan, and action candidate
- report status and findings

Forbidden states include authorization consumption, provider API calls,
external runtime touch, actual dispatch, external run start, live status
tracking, live output fetch, and real external outcome recording.

## CLI And Trace Expectations

CLI output should show gate id, gate status, authorization id, single-use status,
consumed=false, and sanitized evidence refs. OCEL should record gate state and
authorization state, not dispatch.
