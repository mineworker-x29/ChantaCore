## Restore Detail

Restore v0.23.6 as declared runtime preflight and reachability boundary. The
preflight report describes whether declared prerequisites and interface
requirements are present. It does not perform a runtime ping.

Required restored surfaces:

- runtime preflight request/report model
- preflight mode policy
- provider/runtime interface descriptors
- reachability status fields
- preflight finding model
- source/evidence refs
- OCEL mapping for preflight reports and findings
- PIG/OCPX projection entries for preflight visibility
- CLI preflight view

The restored preflight layer should make future runtime requirements visible
while keeping v0.23.x non-executing.

## Expected Model And Policy State

The restored preflight layer should preserve:

- `live_preflight_performed=false`
- `external_runtime_touched=false`
- `provider_api_call_performed=false`
- `runtime_ping_performed=false` or equivalent absent/false behavior
- `dispatch_enabled=false`
- declared status for required provider/runtime interfaces
- missing/partial/blocked status if required declaration is absent

Forbidden states include network access, runtime ping, external API calls,
plugin loading, MCP connection, shell execution, local command execution, and
credential/token reading.

## CLI And Trace Expectations

CLI output should show preflight report id, plan id, declared interface status,
and safety flags. OCEL should record preflight boundary creation, not live
runtime observation.

## Restore Procedure

1. Confirm `runtime_preflight.py` exists.
2. Confirm runtime preflight tests and boundary tests exist.
3. Build a preflight report from a declared control plan.
4. Confirm all live preflight/runtime/provider flags are false.
5. Confirm missing declarations become findings rather than live probes.
6. Run primary tests.
7. Run forbidden searches for runtime touch, network, shell, provider calls,
   MCP, plugin loading, and credentials.

## Evidence Strength

Evidence strength is high for declared boundary behavior. Any claim about live
runtime reachability is unknown in v0.23.6 unless later provider/runtime tracks
provide verified live checks.

## Withdrawal Conditions

execution in v0.23.x.

## Validity Horizon
