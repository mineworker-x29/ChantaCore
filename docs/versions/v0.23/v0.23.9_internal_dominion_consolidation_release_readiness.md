## Expected Model And Policy State

The restored consolidation report should preserve:

- included versions v0.23.0 through v0.23.9
- release name `OCEL-native Internal Dominion Foundation v1`
- readiness status ready/warning/blocked
- release status releasable/releasable_with_warnings/blocked
- `ready_for_v0_24` calculated from coverage and safety
- `ready_for_v0_25=false`
- all provider/runtime/dispatch/authorization/live-output/credential/LLM flags
  false
- roadmap boundary pointing to v0.24.x Internal Provider / Local Runtime Provider
- future gaps for v0.24 through v0.29+ recorded as future-track, not blockers
  unless they contaminate v0.23.x

## CLI And Trace Expectations

CLI output should show release name, release status, readiness status,
ready_for_v0_24, ready_for_v0_25=false, safe_to_dispatch=false,
provider_api_call_performed=false, external_runtime_touched=false,
dispatch_performed=false, authorization_consumed=false, live status/output flags
false, real external outcome false, local runtime false, external adapter false,
Schumpeter split false, and sanitized evidence refs.

OCEL should record consolidation/report/workbench snapshot events, not
provider-invoked or external-control-dispatched events.

## Restore Procedure

1. Confirm `consolidation.py` exists.
2. Confirm consolidation tests and boundary tests exist.
3. Build the consolidation report.
4. Confirm all v0.23.0 through v0.23.8 subjects are present.
5. Confirm safety boundary dangerous counts are zero.
6. Confirm roadmap boundary is aligned to v0.24 next track.
7. Confirm future-track gaps are not treated as v0.23 blockers unless
   contamination exists.
8. Run primary tests and v0.23.8 dispatch boundary tests.
9. Run full regression when validating release readiness.
10. Run forbidden searches for provider calls, runtime touch, dispatch,
    authorization consumption, local runtime, external adapters, Schumpeter
    split, credentials, and LLM judge.

## Evidence Strength

Evidence strength is high for local consolidation models, tests, CLI surfaces,
and previously recorded full-suite outcomes. It is not evidence that v0.24
provider execution is implemented; v0.23.9 only decides readiness to begin that
next track.

## Withdrawal Conditions

Withdraw this addendum if consolidation masks missing v0.23 prerequisites,
starts execution, allows provider/runtime contact, consumes authorization,
exposes credentials, or introduces future-track work prematurely.

## Validity Horizon

Valid until v0.24 Internal Provider / Local Runtime Provider begins or v0.23.x
policy changes.
