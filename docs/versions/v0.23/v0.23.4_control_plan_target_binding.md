## Restore Detail

Restore v0.23.4 as the planning layer that binds action candidates to declared
targets and constraints. The control plan is a structured plan, not an execution
request.

Required restored surfaces:

- control plan model
- target binding model
- constraint model or equivalent structured constraints
- control plan service
- target binding service
- plan status calculation
- source/evidence refs
- OCEL mapping for control plans and target bindings
- PIG/OCPX projection entries for plan/binding visibility
- CLI views for plans and bindings

The plan must preserve enough detail for later safety, preflight, gate, and
dispatch-boundary stages to reason about it without executing anything.

## Expected Model And Policy State

The restored plan layer should preserve:

- plan id
- action candidate id
- provider/runtime/capability references as descriptors
- target binding references
- input binding policy, not credential values
- output policy, not fetched outputs
- required constraints such as scope, environment, authorization requirement,
  static safety requirement, preflight requirement, review gate requirement, and
  dispatch boundary requirement
- plan status and reasons

Forbidden states include actual dispatch, run creation, provider API call,
external runtime touch, local command execution, credential value output, and
provider-specific payload creation.

## CLI And Trace Expectations

CLI output should show plan id, target binding status, constraint status, and
sanitized references. OCEL should record plan/binding state creation, not
external control dispatch.

## Restore Procedure

1. Confirm `control_plan.py` exists.
2. Confirm control plan tests and boundary tests exist.
3. Build a plan from a descriptor-only action candidate.
4. Confirm constraints are present and non-executing.
5. Confirm target bindings are ids/descriptors, not live handles.
6. Run primary tests.
7. Run forbidden searches for dispatch, provider-specific payloads, credential
   values, and runtime touch.

## Evidence Strength

Evidence strength is high for source/test/policy surfaces. Exact restored plan
instances are unknown unless fixtures or OCEL records are provided.

## Withdrawal Conditions

Withdraw this addendum if a plan can start a run, call a provider, materialize
credentials, or bypass later safety/gate stages.

## Validity Horizon
