# v0.39.5 Before / After Repair Outcome Comparison

Release name: v0.39.5 Before / After Repair Outcome Comparison.

Parent track: Human-approved Sandbox Repair Apply & Re-test Loop with PI-native Self-Prompting Mission Loop Boundary.

## Relationships

v0.39.5 consumes v0.39.4 Post-Apply Controlled Re-test metadata: `RepairPostApplyRetestResult`, run record, bounded output capture, audit, and `V0394ReadinessReport`.

v0.39.5 consumes v0.39.3 Human-approved Patch Materialization & Sandbox Apply metadata: sandbox apply result, transaction, and audit.

v0.39.5 may consume v0.37 Controlled Sandbox Test Runner result envelopes, feedback reports, and failure diagnosis metadata as the before-test baseline. It does not invoke the v0.37 runner.

v0.39.5 may consume v0.38.1 repair evidence bundle metadata, v0.38.4 proposed patch envelope metadata, and v0.38.5 safety report metadata as evidence context. These sources are metadata inputs only.

## Definitions

Before/after repair outcome comparison is the metadata-only comparison between a supplied before-test result and a supplied after re-test result.

Before test outcome snapshot is bounded/redacted metadata representing baseline test outcome, selected tests, passed/failed/errored/skipped refs, timeout state, runner availability, and output digest/preview.

After re-test outcome snapshot is the corresponding post-apply controlled re-test metadata from v0.39.4 or equivalent supplied metadata.

Test outcome delta records transitions such as `fail_to_pass`, `pass_to_fail`, `fail_to_fail`, `pass_to_pass`, timeout transitions, missing-result states, incomparable states, and inconclusive states.

Failure delta assessment records resolved failures, residual failures, new failures, worsened failures, timeout/error changes, and diagnostic incomparability.

Regression signal detection records new failed tests, new errors, new timeouts, increased failure count, decreased pass count, output regression, flaky/inconclusive signals, scope changes, and insufficient data.

Residual failure assessment is represented through failure delta metadata and effectiveness assessment metadata. It is diagnostic metadata only.

Repair effectiveness assessment classifies the repair as an effective, partially effective, ineffective, regressive, or inconclusive candidate. It is not a repair correctness proof.

Do-nothing-after-apply comparison is mandatory. It records whether repair appears to outperform do-nothing, partially outperform do-nothing, or whether do-nothing remains preferred due to regression, inconclusive results, low coverage, or blocking issues.

Future process-state reconstruction input is metadata for v0.39.6. It is not process-state reconstruction.

## Concept Boundaries

v0.39.5 performs comparison only.

Comparison is not test execution. v0.39.5 does not run tests and does not invoke a controlled runner.

Effectiveness assessment is not correctness proof. Passing after-test is not production certification. Failing after-test is not automatic repair permission or retry permission.

Regression signal detection is not automatic rollback. Residual failure assessment is not automatic repair permission.

Future PI-native process-state reconstruction input is not process-state reconstruction.

Future PIG diagnostic input is not PIG recommendation execution.

Self-prompting remains draft/future-gated, not self-execution. Human handoff remains mandatory after comparison.

## Models

Comparison policy allows before/after pairing, delta analysis, failure delta assessment, regression signal detection, residual failure assessment, repair effectiveness assessment, do-nothing comparison, and future process-state reconstruction input. It blocks all runtime/action/persistence surfaces and is not correctness proof.

Comparison input is a comparison request, not a test execution request. It carries before result IDs, after re-test result IDs, sandbox apply IDs, evidence IDs, source refs, prohibited runtime actions, and task summary.

Outcome snapshot stores bounded/redacted supplied result metadata.

Outcome pair records before/after snapshots, comparable refs, before-only refs, after-only refs, scope change, completeness, and comparability.

Outcome delta records transition kind, count deltas, improvement, worsening, and inconclusive metadata.

Failure delta records resolved, residual, new, and worsened failure refs.

Regression signal records whether regression, new failure, timeout regression, scope change, or output regression was detected.

Effectiveness assessment records candidate effectiveness while keeping `correctness_proven` and `production_certified` false.

Do-nothing comparison is mandatory and remains available even when comparison is inconclusive.

Audit confirms no test execution, no controlled runner invocation, no shell, no subprocess, no patch application, no apply_patch, no git apply, no rollback execution, no repair execution, no process-state reconstruction, no OCEL write, no OCPX persistence, no PIG execution, no self-prompt execution, no subagent invocation, no model invocation, no external agent, no Dominion runtime, and no production certification.

Decision may mark future process-state reconstruction input ready. It never allows runtime/action execution now.

Comparison report aggregates pair, delta, failure delta, regression signal, effectiveness assessment, do-nothing comparison, audit, and decision.

Validation report confirms supplied before/after metadata comparison only and all no-execution boundaries.

No-execution guarantee keeps all required `no_*` fields true.

Readiness report summarizes comparison readiness and v0.39.6 design-stage handoff only.
