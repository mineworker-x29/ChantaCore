## Definitions

Self-Prompting Mission Loop is the PI-native single-iteration context used to draft a human-reviewable next-action packet. It requires human handoff, has max iteration count 1, and does not allow autonomous continuation.

Process-state-driven next-action draft is a bounded metadata recommendation derived from supplied process-state signals. It is not next-action execution.

Self-prompt draft is bounded prompt text metadata. It may be generated deterministically from supplied process-state metadata, but it is not submitted to any model and is not executed.

Agent-to-subagent prompt draft is a bounded prompt contract that a human may later review. It is not subagent invocation, model invocation, external-agent orchestration, or tool execution.

Subagent verification request draft is a bounded verification request artifact. It is not sent to a subagent.

Human handoff prompt is bounded handoff metadata that states the human decision points and blocked runtime actions. It is not approval capture.

Prompt safety assessment is metadata that marks draft artifacts safe as drafts only. It is not runtime permission.

Future CLI loop-state surface input is metadata for v0.39.8. It is not CLI execution, Codex CLI invocation, Claude Code invocation, OpenCode, Hermes, or OpenClaw execution.

## Models

Self-prompting policy allows loop context, next-action candidate generation, next-action decision, self-prompt draft generation, agent-to-subagent prompt draft generation, subagent verification request draft generation, prompt safety assessment, human handoff prompt, and future CLI loop-state surface input. It blocks prompt submission, model invocation, prompt execution, next-action execution, subagent invocation, external agent execution, autonomous loop, retry loop, multi-cycle loop, repair execution, test execution, patch application, rollback, OCEL/OCPX persistence, PIG execution, subprocess, shell, network, dependency install, and Dominion runtime.

Self-prompting input is a draft request, not an execution request. Its prohibited runtime actions include prompt submission, model provider invocation, prompt execution, next-action execution, subagent invocation, external agent, autonomous loop, retry loop, multi-cycle loop, repair execution, test execution, patch application, PIG execution, trace persistence, and Dominion.

Loop context uses the canonical name `Self-Prompting Mission Loop`, requires human handoff, uses max iteration count 1, current iteration index 0, and blocks autonomous continuation, retry loop, and multi-cycle loop.

Next-action candidates require human review, are draft-only, are not executable now, and grant no runtime authority.

Next-action decision considers do-nothing and keeps execution, auto-continue, subagent invocation, and model invocation false.

Next-action draft is bounded, redacted, draft-only, not executable, not sent to a model, not sent to a tool, and not sent to a subagent.

Self-prompt draft is bounded, redacted, draft-only, not submitted to a model, and not executed.

Agent-to-subagent prompt draft is bounded, redacted, draft-only, and keeps subagent, external agent, and model invocation false.

Subagent verification request draft is bounded, draft-only, not allowed to invoke a subagent, and not sent to a subagent.

Prompt safety assessment blocks prompt execution, model invocation, subagent invocation, and auto-continue. It may be safe as a draft, but safe to execute remains false.

Human handoff prompt is bounded, redacted, human-action-required metadata and keeps auto-continue false.

Draft packet contains loop context, candidates, decision, next-action draft, self-prompt draft, agent-to-subagent prompt draft, verification request draft, safety assessment, and human handoff prompt. It is draft-only and keeps submitted-to-model, prompt-executed, next-action-executed, subagent/model/external invocation, autonomous continuation, retry, multi-cycle, repair, test, patch, trace persistence, Dominion, production certification, and ready-for-execution false.

Decision may mark draft generation and future CLI loop-state input ready. It does not permit prompt execution, next-action execution, model provider invocation, subagent invocation, external agent execution, autonomous loop, retry, multi-cycle, repair, test, patch, trace persistence, Dominion runtime, or production certification.

Report may mark draft generation completed and future CLI loop-state input ready. It keeps all execution/action/runtime flags false.

Validation report confirms draft-only behavior, no model invocation, no prompt submission, no prompt execution, no next-action execution, no subagent invocation, no external agent, no autonomous loop, no retry loop, no multi-cycle loop, no repair execution, no test execution, no patch application, no trace persistence, no PIG execution, no Dominion runtime, and no production certification.

Draft-only guarantee keeps all required no_* guarantees true.

Readiness report may mark draft/context/handoff/future CLI readiness true and keeps execution/action/runtime flags false.
