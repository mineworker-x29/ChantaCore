## Loop Engineering PI-native Absorption

External Loop Engineering is absorbed as a PI-native operational pattern, not adopted as a top-level ChantaCore concept.

The PI-native terms are Self-Prompting Mission Loop, PI-native Mission Execution Loop, Process-State-Driven Self-Prompting, Agent-to-Subagent Prompting Cycle, Human-Handoff-Gated Next-Action Draft, CLI Loop-State Surface, and Process-State Repair Bundle.

Loop/system prompting is absorbed as draft-only process-state-driven prompt artifact generation. Worktree/parallel isolation is absorbed as sandbox workspace isolation contract. Maker/checker separation is absorbed as subagent verification request draft, not subagent invocation. External memory/state file is absorbed as process-state reconstruction metadata, not trace persistence. Loop control panel is absorbed as CLI preview surface, not CLI runtime execution. Human handoff remains mandatory.
