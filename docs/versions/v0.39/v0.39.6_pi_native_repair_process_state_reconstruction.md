## Definitions

PI-native repair process-state reconstruction is metadata-only reconstruction of the repair mission state from supplied approval, workspace, apply, re-test, and comparison artifacts.

OCEL-style repair event envelope is bounded event metadata. It is not an OCEL file write, JSONL write, log sink, or trace persistence.

Repair process object reference is metadata for process objects such as repair mission, approval gate, sandbox workspace, patch envelope, apply result, re-test result, comparison, effectiveness assessment, regression signal, do-nothing comparison, process-state projection, and human handoff.

Repair object relation snapshot is metadata describing relations such as approves, gates, targets, materializes, applies to, produces, verifies, compares, improves, regresses, leaves residual failure, requires human review, feeds future state, and blocks.

Repair trace segment is bounded trace metadata reconstructed from supplied object/event/relation metadata. It is not a persisted trace.

OCPX-style repair state projection is process-state projection metadata. It is not OCPX persistence and grants no runtime authority.

Repair mission state projection describes current mission state, effectiveness/regression/residual-failure signals, do-nothing posture, human handoff requirement, and future v0.39.7 input readiness. Mission state is not runtime authority.

PIG diagnostic input context is diagnostic input metadata. It is not PIG recommendation execution and grants no PIG runtime authority.

Future self-prompting next-action input is metadata for v0.39.7. It is not self-prompt generation, next-action draft generation, next-action execution, subagent prompt generation, or subagent invocation.

## Boundaries

v0.39.6 performs process-state reconstruction metadata only.

OCEL-style envelope is not OCEL file write. OCPX-style projection is not OCPX persistence. PIG diagnostic input is not PIG recommendation execution. Diagnostic context is not next-action execution.

No prompt generation is opened in v0.39.6. No subagent invocation is opened in v0.39.6. Human handoff remains mandatory.

Repair effectiveness signal is not correctness proof. Process-state projection is not production certification.

## Models

Process-state policy allows event envelope reconstruction, object references, object relations, trace segments, OCPX-style state projection, process-state transition, mission state projection, PIG diagnostic input context, and future self-prompting next-action input. It blocks persistence and runtime/action surfaces.

Process-state input is a reconstruction metadata request, not a persistence or execution request.

Object refs and object relations are metadata only.

Event envelopes are bounded and keep `persisted`, `written_to_ocel_file`, and `written_to_jsonl` false.

Trace segments may represent gaps. If gaps exist, the segment is incomplete.

State projection keeps `persisted`, `ocpx_state_written`, and `runtime_authority_granted` false.

State transition keeps `runtime_authority_granted` false.

Mission state requires human handoff and keeps next-action, self-prompt, subagent, and runtime authority flags false.

PIG diagnostic input context requires human review and keeps PIG execution, PIG runtime authority, and next-action draft generation false.

Decision may allow metadata reconstruction now and future v0.39.7 input, but it does not allow persistence, PIG execution, prompt generation/execution, next-action generation/execution, subagent invocation, model/external agent calls, patch apply, tests, repair execution, Dominion runtime, or production certification.

Reconstruction report may mark reconstruction completed and future self-prompting input ready, while keeping all persistence/execution/action flags false.

Validation report confirms supplied metadata reconstruction only.

No-persistence/no-execution guarantee confirms no OCEL file write, no JSONL write, no OCPX persistence, no trace persistence, no PIG execution, no PIG runtime authority, no self-prompt generation/execution, no next-action generation/execution, no subagent invocation, no model/external agent, no test execution, no patch application, no rollback, no repair execution, no autonomous/retry/multi-cycle loop, no Dominion runtime, and no production certification.

- Required models, helpers, docs, and tests exist.
- OCEL-style event envelopes are bounded and not persisted.
