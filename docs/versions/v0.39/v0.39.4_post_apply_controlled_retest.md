## Definitions

Post-apply controlled re-test is the bounded attempt to run approved tests after a sandbox patch was applied. It is limited to the controlled runner boundary and records output/result metadata for later human review and v0.39.5 comparison.

Controlled test runner invocation is a request to an existing or injected runner adapter. It is not shell execution, raw subprocess execution, command string execution, package installation, or network access.

Controlled test selection is metadata describing the approved bounded test subset. Unbounded scope is rejected. Full-suite intent is future-gated unless represented as bounded and approved.

Controlled test command spec is an argv-list metadata object with `shell=False`, timeout, sandbox cwd reference, and runner-adapter requirement. It is not an arbitrary command string.

Bounded output capture stores bounded/redacted stdout and stderr previews with digest metadata. It is not runtime process-state reconstruction.

Post-apply test result envelope stores outcome metadata such as passed, failed, error, timed out, blocked, or runner unavailable. Test result is not repair correctness proof.

Future before/after comparison input is metadata for v0.39.5. It is not before/after comparison.

## Concept Boundaries

Controlled re-test is not arbitrary test execution. Only a controlled runner adapter may be invoked.

Controlled runner invocation is not shell/subprocess execution by v0.39.4. v0.39.4 does not call raw subprocess, does not use `shell=True`, and does not pass command strings to a shell.

Passing re-test is not production certification. Failing re-test is not automatic repair permission. Human handoff remains mandatory after controlled re-test.

v0.39.4 does not run a repair loop, does not generate self-prompts, does not execute self-prompts, and does not invoke subagents.

It requires successful sandbox apply result metadata, sandbox apply audit metadata, workspace isolation metadata, controlled runner availability, shell false, timeout, bounded output capture, and sandbox cwd.

Test selection plan records selected bounded test refs, changed target refs, excluded refs, and whether scope was bounded and approved.

## Safe Defaults

If no controlled runner is supplied, v0.39.4 returns runner-unavailable metadata and does not execute tests.

If gates fail, v0.39.4 returns blocked metadata and does not call the runner.

If output exceeds bounds, stdout/stderr previews are truncated and digest metadata is retained.
