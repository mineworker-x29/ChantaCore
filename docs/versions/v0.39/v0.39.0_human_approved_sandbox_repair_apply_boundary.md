## PI-native Loop Terms

Self-Prompting Mission Loop is the canonical ChantaCore term. It means a PI-native mission execution structure where ChantaCore uses mission objective, process-state projection, trace history, verification contract, policy boundary, and PIG-style recommendation metadata to draft a future prompt/action request for itself or a delegated process/subagent. That draft is not automatically executed.

Process-State-Driven Self-Prompting is the narrower pattern where process-state projection and verification status shape a future prompt/action draft.

Agent-to-Subagent Prompt Draft is future metadata for a possible delegated verification or repair request. It is not subagent invocation and is not automatic dispatch.

Loop Engineering is absorbed as a PI-native operational pattern. It is not adopted as a top-level ChantaCore concept. Process Intelligence remains the higher-level substrate.

Self-prompting is not self-execution. Next-action draft is not next-action execution. Subagent prompt draft is not subagent invocation. Process-state recommendation is not runtime authority. Human-approved sandbox apply boundary is not actual sandbox apply. Sandbox apply is not live apply. Re-test boundary is not test execution. Process-state reconstruction boundary is not trace persistence. Human handoff remains mandatory.
