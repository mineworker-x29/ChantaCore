# v0.39.2 Sandbox Repair Workspace Isolation Contract

Release name: **v0.39.2 Sandbox Repair Workspace Isolation Contract**.

Parent track: **Human-approved Sandbox Repair Apply & Re-test Loop with PI-native Self-Prompting Mission Loop Boundary**.

v0.39.2 builds on v0.39.1 Approval Artifact Intake & Authenticity Gate, v0.39.0 Human-approved Sandbox Repair Apply Boundary, and v0.38.9 Bounded Repair Proposal Loop v1. v0.39.1 can represent an approval artifact as a future process-state gate. v0.39.2 consumes that gate as metadata and defines which declared workspace target may be treated as future v0.39.3 input.

## Boundary

v0.39.2 performs workspace isolation contract metadata only. It defines the sandbox repair workspace descriptor, workspace root reference metadata, workspace classification metadata, collision-prevention metadata, apply target isolation metadata, live workspace exclusion metadata, reference corpus exclusion metadata, secret path exclusion metadata, future patch materialization input metadata, and future sandbox apply input metadata.

It does not create the workspace. It does not inspect the workspace. It does not apply anything.

## Definitions

Sandbox repair workspace isolation is a PI-native metadata contract that identifies a declared sandbox target for a future human-approved repair apply stage.

Workspace descriptor is the record that states the candidate workspace kind, isolation strategy, trust classification, approval gate refs, patch envelope refs, safety report refs, and human review packet refs.

Workspace root reference metadata is a supplied reference string. It is not a filesystem read and is not a filesystem existence check.

Worktree-style isolation metadata records that a worktree-like pattern is a possible future sandbox isolation strategy. Worktree-style metadata is not git worktree execution.

Collision-prevention metadata records that parallel-agent overlap, shared live workspace writes, and branch collisions are blocked by contract. It is not parallel agent orchestration.

Apply target isolation metadata binds the declared sandbox target to approved patch envelope, safety report, and human review metadata. Binding is not patch application.

Live workspace exclusion metadata blocks live workspace read, write, and apply targets. Sandbox workspace is not live workspace.

Reference corpus exclusion metadata blocks using `references/OpenCode`, `references/Hermes`, or `references/OpenClaw` as future apply targets.

Secret path exclusion metadata blocks secret and credential areas from future apply targets.

## Distinctions
