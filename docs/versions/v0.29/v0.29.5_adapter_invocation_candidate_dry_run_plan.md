# v0.29.5 Adapter Invocation Candidate / Dry-Run Plan

## Version Definition

- Version: v0.29.5
- Name: Adapter Invocation Candidate / Dry-Run Plan
- Korean name: Adapter 호출 후보·Dry-Run 계획
- Track: External Skill / External Provider Adapter Development
- Previous: v0.29.4 Credential / Secret / Network Boundary
- Next: v0.29.6 Provider Invocation Approval / Audit / Rollback Boundary

v0.29.5 is invocation-candidate/dry-run-only. It creates candidate, preview, dry-run, no-op, failure, readiness, report, and audit artifacts for external adapter work. It does not invoke providers, call network, access credential values, retrieve secrets, execute commands, perform external side effects, mutate files, or implement live adapters.

## Objective

The objective is to consume v0.29.4 credential/secret/network boundary outputs and v0.29.3 permission/safety outputs, then produce adapter invocation candidates and dry-run plans that can feed v0.29.6 approval/audit/rollback boundary design. Candidate readiness is not live invocation readiness.

## Relationships

v0.29.4 created credential, secret, network, payload, redaction, provider SDK, and data exfiltration boundaries. v0.29.5 may reference those boundaries through metadata refs, but credential access candidate remains not credential access, network request candidate remains not network request, outbound domain policy remains not network call, and secret reference remains not secret value.

v0.29.6 must create Provider Invocation Approval / Audit / Rollback Boundary. v0.29.5 may prepare approval/audit/rollback inputs, but dry-run success is not approval, dry-run success is not provider invocation readiness, effect preview is not side effect, and risk preview is not safety approval.

## Boundary Behavior

Input envelopes are synthetic or metadata-only and contain no actual user data, company data, private data, credentials, raw traces, raw transcripts, or raw provider outputs. Payload previews are previews only and include no credential, private, or raw artifacts. Output previews describe expected fields and error fields without treating any value as a provider response.

Credential checks may confirm that a reference is present or required later, but they do not access, store, or log credential values and do not retrieve secrets. Network checks may confirm a boundary candidate is present or required later, but they do not send outbound request attempts. Permission/safety checks do not grant permission or approval.

Effect previews predict candidate effects without performing side effects or file mutation. Risk previews describe dimensions such as credential exposure, network access, private data exposure, provider side effect, command execution, data exfiltration, raw output persistence, audit gaps, rollback gaps, OCEL visibility gaps, RPA scope creep, and external dominion creep; they are not approvals.

Dry-run plans use metadata-only, mock-only, safe local no-network, not-run, or unknown modes. Dry-run steps validate the candidate path without provider calls, network calls, credential access, command execution, side effects, or file mutation. No-op plans remain available with deny, defer, mock-only, dry-run-only, v0.29.6 approval, and v0.29.8 preview fallback decisions.
