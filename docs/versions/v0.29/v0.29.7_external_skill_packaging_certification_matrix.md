# v0.29.7 External Skill Packaging / Certification Matrix

## Version Definition

- Version: v0.29.7
- Name: External Skill Packaging / Certification Matrix
- Korean name: External Skill packaging/certification matrix boundary
- Track: External Skill / External Provider Adapter Development
- Previous: v0.29.6 Provider Invocation Approval / Audit / Rollback Boundary
- Next: v0.29.8 Limited Provider Invocation Preview Gate

v0.29.7 is packaging/certification-matrix-only. It creates external skill manifests, adapter package manifests, dependency boundary profiles, exposure reports, certification policies, certification matrices, certification cases/results, boundary certification reports, RPA future-track notes, external dominion exclusions, readiness gates, audit trails, and reports. It does not publish packages, create release tags, certify live providers, invoke providers, call network, access credentials, retrieve secrets, execute commands, perform side effects, mutate files, execute rollback, automatically retry, or implement live adapters.

## Objective

Create packaging and certification evidence for external provider adapters while preserving no-provider-invocation, no-network, no-credential-access, and no-command-execution defaults.

## Relationship To v0.29.6

v0.29.6 produced approval/audit/rollback boundaries. v0.29.7 references those artifacts as metadata and certifies boundary completeness only. Approval/audit/rollback boundary is not provider invocation, approval candidate is not approval grant, rollback plan is not rollback execution, and certification evidence is not live invocation success.

## Relationship To v0.29.8

v0.29.8 may create Limited Provider Invocation Preview Gate. v0.29.7 may create readiness evidence for that gate, but certification matrix is not preview permission, certification pass is not provider invocation, certified mock is not certified live adapter, and limited preview still requires explicit v0.29.8 gating.

## Core Principles

- External skill manifest is not runtime enablement.
- Adapter package manifest is not package publish.
- Certification matrix is not production certification.
- Certified mock is not certified live adapter.
- No-network certification is not network access.
- No-credential certification is not credential access.
- Approval/audit certification is not approval grant.
- Rollback certification is not rollback execution.
- Certification pass is not limited invocation preview.
- Limited preview requires v0.29.8 gate.

## Service Design

AdapterPackagingCertificationPrerequisiteSourceService loads only metadata and refs from v0.29.6, v0.29.5, v0.29.4, v0.29.3, v0.29.2, v0.29.1, and v0.29.0. Package, certification, boundary certification, readiness gate, audit trail, finding, and report services construct static boundary evidence without raw secrets, raw traces, raw transcripts, raw provider outputs, or company-private material.

## Package Policy And Manifest

ExternalSkillPackagePolicy requires external skill manifests and package manifests while forbidding package publish, release tags, runtime binding, provider registration, and provider invocation. ExternalSkillManifest and AdapterPackageManifest are internal candidate records only; package_published and release_tag_created remain false.

## Dependency And Exposure

AdapterPackageDependencyProfile records that provider SDK runtime dependencies are not required now and are not added now. AdapterPackageExposureReport verifies no private data, credentials, secrets, raw provider outputs, raw payloads, or provider SDK runtime dependencies are exposed.

## Certification Matrix

AdapterCertificationMatrix contains cases for manifest completeness, dependency boundary, package exposure, mock mode, no network, no credential, no command, permission/safety, credential/network boundary, invocation dry-run, approval/audit/rollback, OCEL visibility, result boundary, failure/rollback/no-op, RPA future-track, and external dominion exclusion. All cases have live_provider_required, network_required, credential_value_required, and command_execution_required set to false.

## Boundary Certifications

Boundary certification reports cover mock mode, no-network, no-credential, no-command, permission/safety, credential/network, dry-run, approval/audit/rollback, OCEL visibility, result boundary, and failure/rollback/no-op. These reports certify boundary evidence only; they do not grant approval, access credentials, call network, emit provider invocation events, execute rollback, or perform retry.

## RPA And Dominion

RPAFutureTrackCertificationNote keeps RPA/A360/Brity/UiPath deferred and unimplemented. ExternalDominionExclusionCertification excludes external dominion from v0.29 and records v0.30+ as the future track.

## Gate And Audit Trail

AdapterCertificationReadinessGate computes manifest, package, dependency, exposure, certification matrix, case, boundary certification, and no-runtime-action completeness. It may set ready_for_v0_29_8 and ready_for_limited_invocation_preview_gate true while ready_for_provider_invocation, ready_for_network_access, ready_for_credential_access, and ready_for_command_execution remain false. AdapterCertificationAuditTrail excludes raw content, credential values, raw payloads, and raw provider outputs.
