# v0.29.8 Limited Provider Invocation Preview Gate

## Version Definition

- Version: v0.29.8
- Name: Limited Provider Invocation Preview Gate
- Korean name: 제한적 Provider 호출 Preview 게이트
- Track: External Skill / External Provider Adapter Development
- Previous: v0.29.7 External Skill Packaging / Certification Matrix
- Next: v0.29.9 External Provider Adapter Foundation Consolidation

v0.29.8 is limited-preview-gate-only. It creates preview eligibility matrices, bounded preview scopes, preview provider candidates, approval requirements, credential/network/payload/result bindings, audit/OCEL plans, rollback/no-op bindings, risk assessments, deny/defer reasons, decision records, preview gates, audit trails, handoff packets, and reports. It does not execute preview invocation, invoke providers, register providers, invoke provider SDKs, call network, access credentials, retrieve secrets, execute commands, perform side effects, mutate files, execute rollback, automatically retry, publish packages, create release tags, certify live providers, or implement live adapters.

## Objective

Create limited preview gate eligibility and v0.29.9 handoff material without executing provider invocation or enabling provider/network/credential/command runtime.

## Relationship To v0.29.7

v0.29.7 produced packaging and certification matrix artifacts. v0.29.8 consumes those reports as metadata. Certification pass is not limited invocation preview, certified mock is not certified live adapter, and preview authorization candidate is not provider invocation.

## Relationship To v0.29.9

v0.29.9 must consolidate v0.29.0 through v0.29.8. v0.29.8 produces handoff material for consolidation, but v0.29.9 consolidation is not unlimited provider runtime.

v0.30 External Agent Dominion Bridge remains a future-track item. RPA, A360, Brity, and UiPath adapter runtime also remain future-track and are not implemented by this version.

- Preview eligibility is not execution.

- Preview scope is not network access.
- Preview credential binding is not credential access.
- Preview network binding is not network call.

## Service Design

LimitedProviderInvocationPreviewPrerequisiteSourceService loads metadata and refs from v0.29.7 certification, v0.29.6 approval/audit/rollback, v0.29.5 invocation dry-run, v0.29.4 credential/network boundary, v0.29.3 permission/safety, v0.29.2 mock harness, v0.29.1 registry, and v0.29.0 contract/prohibition artifacts. It does not load raw secrets, raw traces, raw transcripts, raw provider outputs, or company-private material.

## Preview Policy

The policy enables preview gate artifact creation and disables preview execution now. Certification, explicit approval later, credential/network/payload/result boundaries, audit/OCEL plan, rollback/no-op boundary, and bounded preview scope are required.

## Eligibility Matrix

The eligibility matrix records candidates that are eligible for a future limited preview gate. Eligibility rows reference certification, approval, credential/network, dry-run, mock, and contract evidence while keeping eligible_for_live_invocation_now false.

## Scope And Candidates

Preview scopes are adapter-specific, capability-specific, payload-bounded, network-bounded, credential-reference-bounded, and time-bounded. Preview provider candidates are not provider invocation and keep provider_invoked_now, provider_registered_now, network_called_now, credential_accessed_now, and command_executed_now false.

## Bindings

Approval requirements require explicit user approval later and do not grant approval now. Credential bindings are secret-reference-only and do not access, store, or log credential values. Network bindings do not call network. Payload boundaries do not send payloads. Result boundaries do not receive provider responses or persist raw provider outputs.

## Audit, Rollback, Risk, And Decisions

Audit/OCEL plans record required preview events and do not emit provider invocation events now. Rollback/no-op bindings do not execute rollback. Risk assessments are not approval. Decision candidates and records may record preview_gate_candidate, deny, defer, or no-op decisions while keeping execution, provider, network, credential, and command permissions false.

## Gate, Audit Trail, And Handoff

LimitedPreviewGate computes readiness for v0.29.9 consolidation while ready_for_preview_execution_now and runtime readiness flags remain false. LimitedPreviewAuditTrail excludes raw content, credential values, raw payloads, and raw provider outputs. LimitedPreviewHandoffPacket targets v0.29.9, is refs-only, and records provider invocation, provider registration, SDK invocation, network call, credential access, command execution, live adapter runtime, RPA adapter runtime, and external agent dominion bridge as not implemented now.
