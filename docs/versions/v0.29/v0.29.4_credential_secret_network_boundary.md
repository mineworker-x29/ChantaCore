# v0.29.4 Credential / Secret / Network Boundary

Korean name: Credential·Secret·Network 경계

Track: External Skill / External Provider Adapter Development

## Version Definition

v0.29.4 defines the credential, secret, and network boundary for external adapter candidates. It follows v0.29.3 Permission / Safety / Scope Gate for External Adapters and prepares v0.29.5 Adapter Invocation Candidate / Dry-Run Plan.

This version is credential/secret/network-boundary-only. It may create credential access candidates and network request candidates, but it must not access credential values, retrieve secrets, call network, invoke provider SDKs, invoke providers, execute commands, or implement live adapters.

## Objective

The objective is to make credential sensitivity, secret reference requirements, network sensitivity, outbound domain constraints, provider SDK isolation, payload boundaries, redaction, audit, and data exfiltration boundaries explicit before v0.29.5 can define dry-run candidates.

## Relationship To v0.29.3

v0.29.3 produced permission/safety/scope decisions. v0.29.4 consumes the permission/safety report, gate, credential safety checks, network safety checks, decision records, safety classifications, action intents, and action scopes.

Permission/safety gate is not credential access. Approval requirement is not credential access. Scope declaration is not network access.

## Relationship To v0.29.5

v0.29.5 may create Adapter Invocation Candidate / Dry-Run Plan. v0.29.4 provides boundary inputs for that dry-run planning, while v0.29.5 must still not invoke providers, call network, access credential values, or send payloads.

## Credential Boundary

CredentialBoundaryPolicy forbids credential values, credential storage, credential logging, environment file creation, committed credentials, and credential materialization now. It permits only future credential access candidates, and those candidates are not credential access.

CredentialMaterialClassification records credential kind and sensitivity without credential value presence. SecretReferencePolicy requires opaque references that never include secret values. SecretReferenceDescriptor records a placeholder or future enterprise secret reference only.

ExternalSecretStoreContract records future store kinds and future rotation/audit/redaction expectations. It does not enable secret-store integration, retrieval, or writes.

CredentialRedactionPolicy and CredentialAuditPolicy require redaction and audit of references, not values. CredentialScopeBinding ties a future credential reference to minimal scope. CredentialAccessCandidate records a deferred candidate only.

## Network Boundary

NetworkBoundaryPolicy keeps no-network as the default, disables network access now, forbids outbound network and provider SDK network behavior now, and records that candidate creation is not a request.

OutboundDomainPolicy requires future allowlisting and forbids wildcard, unknown, private, and internal network domains now. OutboundDomainRule records future-review or deny decisions without allowing access.

NetworkRequestPolicy requires method, endpoint, payload, response, audit, timeout, and retry declarations. NetworkRequestCandidate records a deferred candidate only and never sends a request.

NetworkTimeoutRetryPolicy requires bounded timeout and retry behavior, forbids infinite retry, and forbids retry on authentication failure. The policy is not network execution.

## Provider SDK Boundary

ProviderSDKBoundaryPolicy forbids provider SDK runtime dependency, import, invocation, and network behavior now. ProviderSDKBoundaryReport confirms no SDK dependency, import, invocation, or SDK network call was added.

## Data Exfiltration And Payload Boundaries

DataExfiltrationBoundaryPolicy forbids private data, credential, and raw provider output exfiltration. PayloadBoundaryPolicy requires minimal payloads and excludes credentials, private data by default, raw traces, raw transcripts, and raw provider outputs. RequestResponseRedactionBoundary forbids raw request and response logging. NetworkAuditBoundary records audit requirements while excluding credential values and raw payloads.

## Gate And Audit Trail

CredentialNetworkBoundaryGate verifies credential boundary, secret reference boundary, absence of credential values, absence of credential storage/logging/environment files, no network calls, provider SDK isolation, outbound domain policy, payload boundary, data exfiltration boundary, and audit boundary.

CredentialNetworkBoundaryAuditTrail records request, source view, credential boundary report, network boundary references, and gate references. It includes no raw content, credential values, or raw payloads.

## Non-Goals

v0.29.4 does not store credential values, log credential values, create environment files, integrate with external secret stores, retrieve secrets, materialize secrets, invoke provider SDKs, invoke providers, register providers, call network, send outbound requests, call webhooks, open websocket connections, expand command execution, create shell execution surfaces, expand subprocess behavior, implement live adapters, implement RPA/A360/Brity/UiPath, implement external dominion, use Schumpeter private runtime, use actual user/company/private data, persist raw provider output, log raw payloads, add reference runtime dependencies, copy reference code, enable PIG execution authority, or use an LLM judge as sole boundary authority.

## Service Design

CredentialNetworkBoundaryPrerequisiteSourceService loads v0.29.3, v0.29.2, v0.29.1, and v0.29.0 metadata references only. It does not load raw secrets, raw traces, raw transcripts, raw provider outputs, or company-private material.

CredentialBoundaryService creates credential policy, material classifications, secret reference policy/descriptors, external secret store contract, redaction/audit policy, scope bindings, access candidates, and credential boundary report.

NetworkBoundaryService creates network policy, outbound domain policy/rules, network request policy/candidates, and timeout/retry policy.

ProviderSDKBoundaryService and DataExfiltrationBoundaryService create SDK, payload, redaction, audit, and exfiltration boundaries without runtime behavior.

Withdraw this version judgment if credential values are stored, logged, output, or detected; environment files are created; secret retrieval or materialization occurs; external secret store integration is implemented; provider SDK invocation or provider invocation occurs; network calls, outbound requests, webhook calls, or websocket connections occur; command execution expansion, shell true mode, or unbounded subprocess appears; live adapter or RPA/A360/Brity/UiPath implementation appears; private/company data or raw provider output is used or stored; raw payload logging occurs; provider/network/credential readiness becomes true; an LLM judge becomes the sole boundary authority; or GrowthKernel becomes an active dependency.
