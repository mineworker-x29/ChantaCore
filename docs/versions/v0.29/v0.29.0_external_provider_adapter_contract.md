Version: v0.29.0

## Objective

v0.29.0 opens the External Skill / External Provider Adapter Development track with contract-only metadata. It consumes the v0.28.9 refs-only handoff where available and defines the provider adapter contract, external skill contract, lifecycle, capability, schema, effect, permission, safety, credential, network, audit, rollback/no-op, OCEL visibility, mock/no-network, certification, provider-invocation prohibition, and command-execution prohibition surfaces.

This version does not implement a real adapter, provider registry, provider registration, provider invocation, provider SDK call, network call, credential storage, `.env` creation, command execution, shell surface, subprocess expansion, RPA/A360/Brity/UiPath adapter, external dominion, Schumpeter private runtime, or private/raw provider-output persistence.

## Relationship to v0.28.9

v0.28.9 may mark `ready_for_v0_29_contract=True`, but it keeps `ready_for_provider_invocation=False` and `ready_for_command_execution=False`. v0.29.0 trusts only the contract handoff. It does not reinterpret v0.29 readiness as provider invocation readiness.

## Core Principles

Adapter contract is not adapter implementation. Provider adapter contract is not provider invocation. External skill contract is not external skill execution. Capability declaration is not permission. Provider registration is not execution. Credential boundary is not credential storage. Secret reference is not secret value. Network boundary is not network access. Audit requirement is not completed audit. Rollback requirement is not rollback execution. OCEL visibility requirement is not provider action. v0.29 contract readiness is not provider invocation readiness. No-provider-invocation is the default outcome.

## Contract Surfaces

The provider adapter contract requires adapter interface, capability declaration, input and output schemas, effect boundary, permission, safety, credential, network, audit, rollback/no-op, OCEL visibility, mock/no-network mode, and certification. `adapter_implemented_now`, `provider_registered_now`, and `provider_invoked_now` remain false.

The external skill adapter contract requires manifest, capability declaration, effect boundary, permission scope, safety gate, OCEL visibility, mock mode, and certification. `external_skill_implemented_now` and `external_skill_executed_now` remain false.

Lifecycle states are declared, inventoried, registered, mocked, certified_mock, gated, dry_run_candidate, approval_required, preview_allowed, disabled, blocked, and deprecated. Registration is not execution. Certification is not live invocation. Preview is not unbounded runtime.

Capability declaration requires id, name, description, input schema, output schema, effect type declaration, risk level, and permission scope. Capability declaration is not permission or invocation.

Input schema contracts forbid raw secret input and raw private payloads by default. Output schema contracts forbid raw provider output persistence by default and require redaction, summary, and error boundaries.

Allowed effect types are read_only_observation, mock_provider_response, dry_run_plan_created, invocation_candidate_created, approval_record_created, audit_record_created, and ocel_trace_created. Forbidden effect types include provider_invoked, network_called, command_executed, credential_stored, external_side_effect_performed, file_mutated_by_adapter, and private_data_exfiltrated.

Permission, safety, credential, network, audit, rollback/no-op, OCEL, mock, and certification requirements are defined as future gates. They do not grant permission, open network access, store credentials, complete audit, execute rollback, or perform provider action in v0.29.0.

Provider invocation is forbidden now, including provider registration, SDK invocation, external API calls, and network calls. The future reopen version is v0.29.8 Limited Provider Invocation Preview Gate. Command execution expansion, shell execution, shell true, and unbounded subprocess are forbidden now.

## Withdrawal Conditions

Withdraw this contract-only judgment if v0.29.0 implements real adapters, registers or invokes providers, invokes provider SDKs, calls external APIs or networks, stores/logs credentials, creates `.env` files, expands command execution, adds shell true or unbounded subprocess behavior, implements RPA/A360/Brity/UiPath or external dominion, mixes Schumpeter private runtime into public core, uses actual user/company/private data, persists raw provider output, adds references runtime dependency/code copy, sets provider or command readiness true, or uses LLM judge as sole contract authority.

Validity horizon: Valid until v0.29.1 Provider Capability Inventory / Adapter Registry begins or External Provider Adapter Contract policy changes.
