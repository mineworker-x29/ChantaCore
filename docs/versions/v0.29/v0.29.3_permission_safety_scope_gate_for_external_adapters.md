# v0.29.3 Permission / Safety / Scope Gate for External Adapters

Korean name: External Adapter 권한·안전·범위 게이트

Track: External Skill / External Provider Adapter Development

## Version Definition

v0.29.3 defines a metadata-only permission, safety, and scope gate for external adapter candidates. It follows v0.29.2 Mock Adapter Harness / No-Network Default and prepares v0.29.4 Credential / Secret / Network Boundary.

This version is permission/safety/scope-gate-only. It creates policies, deny-first rules, action intent and scope records, scope matrix rows, permission decision candidates, safety classifications, approval requirements, decision records, safety checks, a gate, an audit trail, and a report. It does not grant real runtime permission.

## Objective

The objective is to evaluate external adapter candidates after mock/no-network validation without enabling live provider behavior. The output can say "deny", "defer", "mock only", "dry run only", or "requires future boundary"; it cannot authorize provider execution.

## Relationship To v0.29.2

v0.29.2 produced the Mock Adapter Harness / No-Network Default. v0.29.3 consumes the mock harness report, mock harness gate, mock run reports, effect boundary validation, no-network reports, SDK isolation reports, mock credential reports, and OCEL trace reports.

Mock success is not live adapter readiness. Mock success is not permission grant. Mock success is not safety pass for real provider invocation. Mock/no-network readiness allows only permission, safety, and scope evaluation.

## Relationship To v0.29.4

v0.29.4 must define Credential / Secret / Network Boundary. v0.29.3 may identify credential and network sensitivity, but credential need classification is not credential handling and network need classification is not network access.

## Permission, Safety, And Scope

AdapterPermissionPolicy establishes deny-first behavior, external-action permission requirements, minimal scopes, expiring scopes, approval records, and no ambient or wildcard permission.

AdapterSafetyPolicy requires safety classification and blocks unsafe action candidates. Private data, credential, network, external side effect, data exfiltration, command-like, and RPA paths are checked explicitly.

AdapterScopePolicy requires minimal, action-specific, expiring scopes that reference the adapter and capability. Private data, credential, and network scopes defer to future boundaries. Command scope remains forbidden now.

## Deny-First Rules

The deny-first rule set defaults to deny and allows only metadata decisions: deny, defer, require user approval, require future credential boundary, require future network boundary, mock only, dry run only, and no-op.

Deny rules cover provider invocation, network requirements, credential value requirements, command execution, private data, raw provider output persistence, RPA action, external dominion, missing scope, and missing safety classification.

## Action Intent And Scope Matrix

AdapterActionIntent records the intended adapter action and whether it would require provider invocation, network, credentials, command execution, private data, or raw provider output persistence. AdapterActionScope converts the intent into mock-only, metadata-only, future external read/write/side-effect, command-forbidden, RPA-deferred, or unknown scope.

AdapterScopeMatrix summarizes mock-only, approval-required, denied, deferred, and unknown rows. It is a decision matrix, not runtime authorization.

## Permission Evaluation

AdapterPermissionScopeEvaluation records required scope, minimality, expiry requirement, approval-record requirement, and whether a permission grant candidate is allowed. It always records real_permission_granted_now=False.

## Safety Classification

AdapterSafetyClassification supports safe mock-only, safe metadata-only, low risk read-only, medium risk external read, high risk external write, blocked private data, blocked credential, blocked network, blocked command-like, blocked RPA future-track, blocked external side effect, and unknown classes.

Credential and network classifications point to v0.29.4. External side effects point to v0.29.6. RPA defers to v0.29.7 or later certification and v0.29.8 preview gates.

## Approval And Decision Records

UserApprovalRequirement describes the approval surface required later. It is not approval and records approval_granted_now=False.

AdapterApprovalCandidate records a candidate only. The candidate is not permission grant and not execution.

AdapterPermissionDecisionRecord records the deny-first decision. It does not grant permission, approval, provider invocation, network calls, or command execution.

## Expiry Policy

AdapterScopeExpiryPolicy requires expiring scopes, forbids permanent permission, forbids silent restoration, and requires reapproval after expiry.

## Safety Checks

PrivateDataSafetyCheck blocks unsafe private data use and records no actual user/company data detection. CredentialReferenceSafetyCheck defers credential-sensitive paths to v0.29.4 and stores no credential values. NetworkNeedSafetyCheck defers network-sensitive paths to v0.29.4 and makes no network calls. CommandLikeActionSafetyCheck blocks command-like action now. ExternalSideEffectSafetyCheck defers side effects to v0.29.6. DataExfiltrationSafetyCheck blocks raw/private output exposure. RPAScopeDeferralCheck keeps RPA/A360/Brity/UiPath future-track.

## Gate And Audit Trail

AdapterPermissionSafetyGate checks deny-first status, scope completeness, safety completeness, approval requirements, safety checks, deferred credential/network/side-effect/RPA paths, and no forbidden runtime behavior.

AdapterPermissionSafetyAuditTrail records request, source view, gate, policy, decision, and safety check references. It includes no raw content.

## Non-Goals

v0.29.3 does not implement real provider adapters, grant live permission, grant approval as execution, register providers, invoke providers, call provider SDKs, call network, store credentials, create environment files, expand command execution, implement RPA/A360/Brity/UiPath, implement external dominion, use Schumpeter private runtime, use actual user/company/private data, persist raw provider output, add reference runtime dependencies, copy reference code, enable PIG execution authority, or use an LLM judge as sole authority.

## Service Design

AdapterPermissionSafetyPrerequisiteSourceService loads v0.29.2 and v0.29.1 metadata references only. It does not load raw secrets, raw traces, raw transcripts, raw provider outputs, or company-private material.

The report service composes source view, policies, deny-first rules, action intents, scopes, matrix rows, evaluations, classifications, findings, approval requirements, candidates, decision records, expiry policy, safety checks, gate, audit trail, and report.
