# v0.31.6 Dominion Target / Dominion Decision

## Version Name And Scope

Release name: `v0.31.6 Dominion Target / Dominion Decision`

Parent track: `v0.31.x Internal Triad Skill Foundation`

Immediate parent: `v0.31.5 Dominion Skill Foundation`

Inherited foundations:

- `v0.31.4 Digestion Candidate / Internalization Plan`

   run-preview, and readiness report models;

authority grant, permission, runtime registry, or execution readiness.

## Relationship To v0.31.5

`v0.31.5` produced signal artifacts:

- `DominionSkillOutput`
- `DominionControlBoundarySignal`
- `DominionGovernanceFinding`
- `DominionRouteDecision`

`v0.31.6` converts those signals into explicit governance records:

- `InternalDominionDecision`
- `DominionFutureGateItem`
- `DominionNoOpDecision`

The conversion is structural. It does not weaken the v0.31.5 no-execution,
no-control, and no-authority guarantees.

## Definition Of InternalDominionTarget

`InternalDominionTarget` is a governance record for a target-like surface that
requires dominion handling. It records target kind, status, source refs, control
surfaces, risk surfaces, boundary ids, future gate ids, evidence ids, a maximum
allowed descriptive/planning/simulation level, and prohibitions.

InternalDominionTarget is not external control.

## Target Versus External Runtime Control

External runtime control would invoke or operate a provider, network, command,
browser, RPA, gateway, delegation, or agent harness surface. `v0.31.6` does not
do that.

`InternalDominionTarget` always preserves:

- `ready_for_external_control=False`
- `ready_for_authority_grant=False`
- `ready_for_execution=False`

## InternalDominionDecision Versus Authority Grant

`InternalDominionDecision` records a governance decision type, reason, optional
descriptive/planning/simulation level, required boundaries, future gates,
reviews, evidence ids, blocked reasons, and withdrawal conditions.

InternalDominionDecision is not authority grant.

It always preserves:

- `approved_for_execution=False`
- `approved_for_external_control=False`
- `authority_granted=False`
- `ready_for_execution=False`

D4-D9 remain future-track and cannot be granted.

## DominionControlBoundary Versus Permission

`DominionControlBoundary` records governance metadata that blocks execution,
external control, and authority grant. It is not permission and does not enforce
runtime behavior by itself.

Default boundary behavior:

- `blocks_execution=True`
- `blocks_external_control=True`
- `blocks_authority_grant=True`

## DominionFutureGateItem Versus Readiness

`DominionFutureGateItem` records future conditions and prohibited surfaces. It is
not future readiness, not approval, and not implementation.

`ready_now=False` in v0.31.6.

## DominionNoOpDecision Versus Failure

`DominionNoOpDecision` is a valid governance outcome. It is not failure and
executes nothing.

- `requires_review`
- `blocked`
- `deferred`
- `future_track`

## Decision Type Taxonomy

`InternalDominionDecisionType` values:

- `deny`
- `defer`
- `no_op`
- `require_review`
- `require_future_gate`
- `record_target_only`
- `plan_only`
- `simulate_only`
- `block`
- `future_track`

No decision type grants execution. `plan_only` does not authorize action.
`simulate_only` does not execute simulation.

## Control Boundary Taxonomy

`DominionControlBoundaryKind` values include:

- no external control, provider invocation, network access, credential access,
  command execution, browser automation, RPA control, gateway control, packet
  send, raw output persistence, memory mutation, registry mutation, and policy
  activation;
- approval, audit, result boundary, rollback no-op, OCEL trace, and future gate
  requirements;
- `unknown`.

Boundary kind is governance metadata and not permission.

## Future Gate Taxonomy

`DominionFutureGateKind` values include external execution, provider invocation,
network access, credential access, command execution, browser runtime, RPA
runtime, gateway control, external agent delegation, registry mutation, memory
mutation, policy activation, tool execution, mission installation, production
certification, and `unknown`.

Future gate is not readiness, approval, or implementation.

## Reason Taxonomy

`DominionDecisionReasonKind` values include insufficient evidence, high risk,
missing boundary/audit/approval/result/rollback artifacts, OCEL incompatibility,
future gate requirement, policy block, no-op preference, descriptive record only,
conservative routing, and `unknown`.

Reason kind is explanatory and does not grant authority.

## Control Surface And Risk Surface Taxonomy

`DominionControlSurface` values include external runtime, provider, network,
credential, command, browser, RPA, gateway, packet, delegation, memory,
registry, policy, tool, mission, OCEL trace, and `unknown`.

`DominionRiskSurface` values include private data, credential exposure, network
side effect, command execution, provider invocation, browser automation, RPA
control, gateway send, external delegation, memory contamination, raw output
persistence, registry mutation, policy activation, mission installation, tool
execution, OCEL schema drift, and `unknown`.

## TargetDecisionSet

`DominionTargetDecisionSet` groups targets, boundaries, decisions, future gates,
no-op decisions, and evidence ids.

DominionTargetDecisionSet is not runtime registry.

It preserves:

- `ready_for_execution=False`
- `ready_for_external_control=False`
- `ready_for_authority_grant=False`

`ready_for_v0317_ocel_trace_integration` means next design stage only.

## Run Preview

`DominionDecisionRunPreview` is not run execution. It guarantees:

- no execution;
- no external control;
- no authority grant;
- no provider invocation;
- no network access;
- no credential access;
- no command execution;
- no registry mutation;
- no memory mutation.

## Readiness Report

`V0316ReadinessReport` records readiness for `v0.31.7 Triad Skill OCEL Trace
Integration` only.

It preserves:

- `ready_for_execution=False`
- `ready_for_external_control=False`
- `ready_for_authority_grant=False`
- `ready_for_dominion_runtime=False`

The readiness report is not runtime enablement.

## Safe Defaults

Safe defaults:

- target source refs do not fetch;
- target max allowed level defaults to `D3_SIMULATE`;
- targets cannot set external control, authority grant, or execution readiness;
- boundaries block execution, external control, and authority grant;
- decisions do not approve execution or grant authority;
- future gates are not ready now;
- no-op is valid;
- decision sets are not runtime registries;
- run previews guarantee no runtime operations.
