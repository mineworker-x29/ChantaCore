## Version Name And Scope

Release name: `v0.31.4 Digestion Candidate / Internalization Plan`

Parent track: `v0.31.x Internal Triad Skill Foundation`

Immediate parent: `v0.31.3 Digestion Skill Foundation`

## Definition Of Internal Candidate

An internal candidate is a reviewable design artifact. It describes a possible
future internal skill, tool contract, mission, policy, memory schema, prompt
pattern, trace event pattern, or result envelope.

Internal candidate is not active artifact.

Searchable invariants:

- `ready_for_activation=False`
- `ready_for_registry_mutation=False`
- `ready_for_execution=False`

## Internal Candidate Versus Active Artifact

Active artifacts would be registered, installed, invoked, persisted, or used by
runtime control paths. `v0.31.4` does none of those things.

Candidates can be inspected by tests and future design stages, but they do not
change active ChantaCore behavior.

## InternalSkillCandidate Versus Active Skill

`InternalSkillCandidate` records proposed skill name, kind, input/output contract
summaries, prohibited runtime actions, and required tests.

It is not an active skill and is not skill activation.

## InternalToolContractCandidate Versus Registered Tool

`InternalToolContractCandidate` records a proposed tool contract and schema
summary. It is not a registered tool. Its side-effect policy must not imply
execution.

## InternalMissionCandidate Versus Installed Mission

`InternalMissionCandidate` records a proposed mission name, schedule policy
summary, trigger policy summary, and fresh-session flag. It is not an installed
mission. Trigger and schedule policies are descriptive only.

## InternalPolicyCandidate Versus Active Policy

`InternalPolicyCandidate` records proposed policy name, scope, enforcement
summary, and default decision. It is not active policy and does not enforce at
runtime.

## InternalMemorySchemaCandidate Versus Memory Mutation

`InternalMemorySchemaCandidate` records proposed schema name, memory scope,
allowed/prohibited fields, and persistence policy summary. It is not a memory
writer, does not persist memory, and does not promote memory.

## InternalPromptPatternCandidate Versus Prompt Injection

`InternalPromptPatternCandidate` records a proposed prompt pattern and insertion
policy summary. It is not prompt injection and does not activate prompts.

## InternalTraceEventPatternCandidate Versus OCEL Event Emission

`InternalTraceEventPatternCandidate` records proposed event/object/relation type
patterns and OCEL visibility summary. It does not emit OCEL events and does not
activate trace handlers.

## InternalResultEnvelopeCandidate Versus Result Ingestion

`InternalResultEnvelopeCandidate` records required, optional, and forbidden
fields. `raw_output_allowed=False` and `memory_persistence_allowed=False`.

It is not result ingestion.

## InternalizationPlan Versus Implementation

`InternalizationPlan` is a plan artifact. It is not implementation, not generated
code, not runtime integration, and not execution.

All plan guarantees remain true:

- no execution;
- no registry mutation;
- no skill activation;
- no tool registration;
- no mission installation;
- no policy activation;
- no memory mutation.

## Internalization Plan

`InternalizationPlan` groups plan steps and review requirements. It records what
must be designed and reviewed later. It does not create files, activate runtime
paths, install anything, or mutate state.

## No-op Decision

`InternalizationNoOpDecision` is a valid outcome. It is not failure and executes
nothing.

## Run Preview

`InternalizationRunPreview` is not run execution. It lists planned modeling
steps and explicitly not-performed operations.

## Readiness Report

`V0314ReadinessReport` records readiness for later design stages only.

It preserves:

- `ready_for_activation=False`
- `ready_for_registry_mutation=False`
- `ready_for_execution=False`

The readiness report is not runtime enablement.

## Safe Defaults

Safe defaults:

- candidates default to inactive;
- result envelopes disallow raw output and memory persistence;
- candidate sets are not registries;
- plans are not implementations;
- review requirements block activation and registry mutation by default;
- run previews guarantee no runtime actions.

DigestionSkillOutput / DigestiblePatternSignal / DigestionRouteDecision
  -> InternalCandidateSourceRef

- active readiness flags;
