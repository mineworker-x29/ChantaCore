# v0.31.2 Observation Report / Capability Map

## Version Name And Scope

Release name: `v0.31.2 Observation Report / Capability Map`

Parent track: `v0.31.x Internal Triad Skill Foundation`

Immediate parent: `v0.31.1 Observation Skill Foundation`

Earlier inherited foundations:

- `v0.31.0 Internal Triad Skill Contract`
- `v0.30.9 External Dominion Control Plane Foundation Consolidation`

This release defines structured internal observation output artifacts:

- `InternalObservationReport`
- `InternalCapabilityMap`
- `ObservationGapRegister`
- `ObservationRiskMap`
- `ObservationEvidenceTable`
- `ObservationReportBundle`
- `ObservationReportRunPreview`
- `V0312ReadinessReport`

The scope is model and contract only. It is not runtime execution, not external
scanning, not source fetching, not read-only tool execution, not digestion
candidate creation, and not dominion target creation.

## Relationship To v0.31.1

`v0.31.1` introduced the Observation Skill foundation:

- source refs;
- artifact refs;
- evidence refs;
- findings;
- gaps;
- risk signals;
- observation input/output;
- no-op decisions;
- run previews;
- foundation and readiness reports.

`v0.31.2` consumes those already-available model artifacts as references and
structured objects. It can represent an `ObservationSkillOutput` as an
`InternalObservationReport`, and it can use observation findings, gaps, risk
signals, and evidence refs to populate report/map tables.

This is still not execution. `ObservationSkillOutput -> InternalObservationReport`
is a modeling transformation inside the contract layer, not a runtime run, not
artifact registration, and not external observation scanning.

## Relationship To v0.31.0 And v0.30.9

`v0.31.0` defines the Internal Triad contract grammar and guarantees that
internal triad contracts are not active skills. `v0.31.2` inherits that boundary:

- `ready_for_execution=False`
- `ready_for_skill_activation=False`
- `ready_for_external_scan=False`

`v0.30.9` provides an external dominion handoff vocabulary and consolidation
surface. `v0.31.2` may describe the existence of handoff-derived artifact refs,
but it must not re-open external dominion gates, must not claim external
execution readiness, and must not raise the max grantable dominion level beyond
the inherited `D3_SIMULATE` boundary.

## InternalObservationReport

`InternalObservationReport` is the structured report object for observation
outputs. It contains:

- `report_id`
- `observation_output_id`
- `source_input_id`
- `target_summaries`
- `findings`
- `gaps`
- `risk_signals`
- `evidence_ref_ids`
- `status`
- `summary`
- `report_gaps`
- `blocked_reasons`
- `ready_for_v0313_digestion_skill_foundation`
- `ready_for_v0315_dominion_skill_foundation`
- `ready_for_execution`
- `ready_for_skill_activation`
- `ready_for_external_scan`
- `metadata`

Validation requirements:

- `report_id` is non-blank.
- `summary` is non-blank.
- target summaries, findings, gaps, risk signals, and evidence refs are lists.
- `ready_for_execution` is always false.
- `ready_for_skill_activation` is always false.
- `ready_for_external_scan` is always false.
- a blocked report cannot be ready for `v0.31.3`.
- `v0.31.5` readiness requires an explicit dominion signal or conservative
  routing metadata.

Semantic boundary:

- Observation report is not external scan.
- Observation report is not a digestion candidate.
- Observation report is not a dominion target.
- Observation report is not active artifact registration.

## CapabilityMapEntry

`CapabilityMapEntry` is a single descriptive capability row. It contains:

- `entry_id`
- `target_id`
- `capability_name`
- `capability_kind`
- `classification`
- `source_finding_ids`
- `source_risk_signal_ids`
- `source_gap_ids`
- `evidence_ref_ids`
- `risk_posture`
- `evidence_quality`
- `boundary_surfaces`
- `effect_surfaces`
- `recommended_next_stage`
- `blocked_reasons`
- `metadata`

Validation requirements:

- `entry_id` is non-blank.
- `capability_name` is non-blank.
- source ids, evidence ids, boundary surfaces, and effect surfaces are lists.
- `blocked` classification requires `blocked_reasons`.
- metadata must not imply permission grant, runtime activation, digestion
  creation, or dominion target creation.

The entry exposes conservative boolean properties:

- `grants_permission=False`
- `activates_capability=False`
- `creates_digestion_candidate=False`
- `creates_dominion_target=False`

These properties are restore invariants. They are not runtime enforcement
systems by themselves; they record the model contract.

## CapabilityClassification

`CapabilityClassification` values:

- `unknown`
- `descriptive_only`
- `safe_descriptive`
- `digestion_signal`
- `dominion_signal`
- `blocked`
- `deferred`
- `future_track`
- `no_op`

Classification rules:

- `descriptive_only` means no internalization implication.
- `safe_descriptive` is not permission.
- `digestion_signal` is not an active digestion candidate.
- `dominion_signal` is not a dominion target.
- `blocked`, `deferred`, `future_track`, and `no_op` are valid outcomes.
- `unknown` remains conservative.

## Digestion Signal Versus Candidate

`digestion_signal` only means that a later design-stage digestion foundation may
review a capability entry. It does not instantiate a candidate, write a
candidate registry, create an internalization plan, install a tool, activate a
mission, or mutate memory.

If any restored implementation imports or creates active digestion objects, this
document's v0.31.2 boundary claim is invalid.

## Dominion Signal Versus Target

`dominion_signal` only means that a later dominion foundation may review a
boundary or governance signal. It does not create a dominion target, does not
create a dominion decision, does not grant authority, and does not control an
external runtime.

If any restored implementation creates active dominion target or decision
objects, this document's v0.31.2 boundary claim is invalid.

## Gap Register

`ObservationGapRegister` preserves diagnostic gap references:

- all gap ids;
- blocking gap ids;
- non-blocking gap ids;
- deferred gap ids;
- future-track gap ids;
- summary;
- evidence refs.

The gap register does not remediate. Blocking gaps are represented as state for
review; no repair, fetch, scan, retry, rollback, or task execution is performed.

## Risk Map

`ObservationRiskMap` preserves advisory risk signal references:

- all risk signal ids;
- high risk signal ids;
- critical risk signal ids;
- blocked risk signal ids;
- risk surfaces;
- recommended boundaries;
- evidence refs.

The risk map does not grant authority. It does not create dominion targets and
does not prove exploitability. It is a structured observation artifact for later
review.

## Evidence Table

`ObservationEvidenceTable` records evidence references and quality buckets:

- evidence refs;
- strong evidence refs;
- weak evidence refs;
- missing evidence items;
- conflicting evidence refs.

The evidence table is not runtime trust. It does not fetch evidence, does not
verify live systems, and does not approve execution.

## Observation Report Bundle

`ObservationReportBundle` groups the report, capability map, gap register, risk
map, and evidence table.

Nested references must match:

- `capability_map.observation_report_id == observation_report.report_id`
- `gap_register.observation_report_id == observation_report.report_id`
- `risk_map.observation_report_id == observation_report.report_id`
- `evidence_table.observation_report_id == observation_report.report_id`

The bundle is not active artifact registration. It is a model aggregate for
restore, review, and later design-stage handoff.

## Safe Defaults

The safe defaults are:

- readiness flags default false for execution, skill activation, and scan;
- helper functions construct contract artifacts only;

## Service Behavior

There is no service layer in this release.

The implemented behavior is pure object construction and validation:

- validate ids and list fields;
- reject true runtime readiness flags;
- reject metadata that would imply runtime, scan, permission, active
  registration, digestion creation, or dominion creation;
- classify observation findings into descriptive or next-stage signal buckets;
- bundle report/map/register/table artifacts without registering them.

No object stores, providers, adapters, runners, message channels, gateways, or
runtime schedulers are introduced.

`v0.31.2` introduces no persistence.

human-readable or testable contract artifacts, but they do not write memory,
