# v0.31.5 Dominion Skill Foundation

## Version Name And Scope

Release name: `v0.31.5 Dominion Skill Foundation`

Parent track: `v0.31.x Internal Triad Skill Foundation`

Immediate parent: `v0.31.4 Digestion Candidate / Internalization Plan`

Inherited foundations:

- `v0.31.3 Digestion Skill Foundation`
- `v0.31.2 Observation Report / Capability Map`
- `v0.31.1 Observation Skill Foundation`
- `v0.31.0 Internal Triad Skill Contract`
- `v0.30.9 External Dominion Control Plane Foundation Consolidation`

This release defines Dominion Skill foundation models and helpers. It does not
control external systems, does not grant authority, and does not create dominion
target or dominion decision artifacts.

## Restore Goal

A future maintainer should be able to restore:

1. dominion source, focus, signal, route, posture, and blocker taxonomies;
2. dominion source reference and input models;
3. control boundary signal, governance finding, blocker, and route decision models;
4. dominion output, no-op, run-preview, foundation report, and readiness models;
5. pure helper functions that build foundation artifacts only;
6. tests proving all execution, control, authority, target, and decision flags stay false.

Restore is invalid if a route signal becomes an active target, decision,
authority grant, external control path, or runtime readiness.

## Definition Of Dominion Skill

Dominion Skill is the internal triad skill that classifies governance,
authority, external-control, and high-risk runtime boundary questions before a
later stage defines target and decision models.

Dominion Skill Foundation is not external control.

## Foundation Versus External Control

External control would invoke, authorize, command, register, send, execute, or
delegate through runtime surfaces. `v0.31.5` does none of those things.

The release is a contract/control-plane model layer only.

## DominionControlBoundarySignal Versus Target

`DominionControlBoundarySignal` records an observed or inferred boundary need.
It is not a target, does not create target artifacts, and does not mark anything
live-runtime-ready.

`dominion_target_signal` is not a target.

## DominionGovernanceFinding Versus Decision

`DominionGovernanceFinding` records governance interpretation, risk posture,
boundary posture, feasibility posture, evidence ids, and blocked reasons.

It is not a decision and does not grant authority.

`dominion_decision_signal` is not a decision.

## DominionRouteDecision Versus Authority Grant

`DominionRouteDecision` records route classification. It is not authority grant,
not implementation, and not external control.

`ready_for_v0316_dominion_target_decision` means next design stage only.

## Feasibility, Boundary, And Risk Posture

`DominionFeasibilityPosture` values:

- `unknown`, `low`, `medium`, `high`, `blocked`, `future_track`

`DominionBoundaryPosture` values:

- `unknown`, `descriptive_only`, `observation_only`, `plan_only`,
  `simulate_only`, `blocked`, `future_track`

`DominionRiskPosture` values:

- `unknown`, `low`, `medium`, `high`, `critical`, `blocked`, `future_track`

High feasibility is not control readiness. `simulate_only` does not execute
simulation. Risk posture is advisory and does not grant authority.

## Blocker Taxonomy

`DominionBlockerKind` values include insufficient evidence, missing
internalization context, missing boundary/risk/approval/audit/result/rollback
classification, unsafe network/credential/command/provider/browser/RPA/gateway
and delegation surfaces, memory contamination, raw output persistence, registry
mutation, policy activation, future gate, OCEL incompatibility, and `unknown`.

A blocker is diagnostic. It does not remediate.

## DominionSkillInput

`DominionSkillInput` contains source refs, candidate set refs, internalization
plan refs, digestion refs, route decision refs, risk/gap refs, requested focus,
task summary, source version, evidence refs, and prohibited runtime actions.

It is not an execution request.

## DominionControlBoundarySignal

`DominionControlBoundarySignal` records signal id, optional source/target/candidate
ids, focus, signal kind, route, title, summary, boundary summary, risk posture,
boundary posture, feasibility posture, evidence ids, blocker ids, assumptions,
and limitations.

High or critical risk requires conservative routing evidence in the form of
blockers, limitations, or future/review/blocked/no-op route.

## DominionGovernanceFinding

`DominionGovernanceFinding` summarizes governance classification. It does not
create `v0.31.6` artifacts and does not grant authority.

## DominionBlocker

`DominionBlocker` records diagnostic blockers and routing hints. It does not
execute remediation.

## DominionRouteDecision

`DominionRouteDecision` records route, signal ids, blocker ids, reason, evidence
ids, and design-stage readiness for `v0.31.6`.

All runtime/control/authority creation flags remain false.

## DominionSkillOutput

`DominionSkillOutput` groups boundary signals, findings, blockers, route
decisions, signal id lists, evidence refs, blocked reasons, and no-op reason.

Dominion output is not active artifact registration. It is not authority grant,
not external control, not target creation, and not decision creation.

## DominionNoOpDecision

`DominionSkillNoOpDecision` is a valid outcome. No-op is not failure and executes
nothing.

## DominionRunPreview

`DominionSkillRunPreview` is not run execution. It guarantees:

- no execution;
- no external control;
- no authority grant;
- no target creation;
- no decision creation;
- no registry mutation;
- no memory mutation.

## DominionSkillFoundationReport

`DominionSkillFoundationReport` records supported taxonomies, supported output
artifact kinds, prohibited runtime actions, next-stage design readiness, evidence
refs, and withdrawal conditions.

It is not runtime enablement.

## Readiness Report

`V0315ReadinessReport` records readiness for `v0.31.6` design models only.

It preserves:

- `ready_for_execution=False`
- `ready_for_skill_activation=False`
- `ready_for_external_control=False`
- `ready_for_authority_grant=False`
- `ready_for_dominion_runtime=False`

The readiness report is not runtime enablement.

## Safe Defaults

Safe defaults:

- source refs do not fetch;
- input is not an execution request;
- boundary signals do not create target or decision artifacts;
- findings do not grant authority;
- route decisions keep runtime/control/authority flags false;
- output is not active artifact registration;
- run previews guarantee no runtime operations;
- no-op remains valid.

## Service Behavior

There is no service layer. Behavior is pure validation and model construction.

Validation rejects:

- blank required ids;
- invalid enum values;
- non-list list fields;
- source fetch metadata;
- execution request metadata;
- target/decision creation metadata;
- authority grant metadata;
- external control metadata;
- true runtime/control/authority readiness flags.

## Persistence And Canonical State Policy

No persistence is introduced.

Dominion Skill foundation artifacts are not canonical runtime state. They do not
write memory, promote memory, persist raw external output, append OCEL events,
or register active artifacts.

## Runtime Integration

There is no runtime integration.

Absent:

- external control;
- authority grant;
- target creation;
- decision creation;
- provider invocation;
- network or credential access;
- command execution;
- browser automation;
- RPA control;
- gateway send;
- packet send;
- rollback;
- retry.

`v0.31.5` does not enable D4-D9. It does not enable provider, network,
credential, command, RPA, browser, gateway, or runtime control.
