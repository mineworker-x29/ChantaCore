# v0.31.1 Observation Skill Foundation

Release name: v0.31.1 Observation Skill Foundation

Parent track: Internal Triad Skill Foundation

## Relationship To v0.31.0

v0.31.0 defined the Internal Triad Skill Contract. It introduced the common contract grammar for Observation, Digestion, and Dominion skills:

- `TriadSkillKind`
- `TriadSkillStatus`
- `TriadSkillExecutionMode`
- `TriadSkillBoundaryPolicy`
- `TriadSkillContract`
- `ObservationSkillContract`
- `DigestionSkillContract`
- `DominionSkillContract`
- input and result envelopes
- no-execution guarantees
- contract set and readiness report

v0.31.1 builds on that contract layer by defining the first concrete foundation vocabulary for the Observation side of the triad. It does not weaken v0.31.0 no-execution guarantees. It does not activate the Observation skill as runtime behavior.

## Relationship To v0.30.9 V031 Handoff

v0.30.9 consolidated External Dominion Control Plane Foundation v1 and produced `ExternalDominionV031HandoffPacket` as a handoff artifact for the v0.31 Internal Triad Skill Foundation.

v0.31.1 may represent v0.30.9 handoff items as:

- `ObservationTargetRef`
- `ObservationArtifactRef`
- `ObservationEvidenceRef`

Those references are descriptive. They do not fetch files, URLs, source refs, external targets, provider data, browser state, RPA state, gateways, packets, credentials, or runtime outputs.

v0.31.1 does not reopen v0.30 runtime gates. External execution readiness remains false. D4-D9 remain future-track.

## Definition Of Observation Skill

Observation Skill means a contract/control-plane layer that reads already-available contract artifacts and structures them into observation inputs and outputs.

Observation Skill may describe:

- external target refs;
- artifact refs;
- evidence refs;
- requested observation focus;
- findings;
- gaps;
- risk signals;
- no-op decisions;
- run previews;
- foundation and readiness reports.

Observation Skill is not external scanning.

Observation Skill input is not an execution request.

Observation Skill output is not a digestion candidate.

Observation Skill output is not a dominion target.

Observation Skill run preview is not run execution.

Source refs are not fetched.

ready_for_v0312 means next design stage only.

ready_for_execution remains false.

ready_for_skill_activation remains false.

ready_for_external_scan remains false.

v0.31.1 does not enable read-only tools.

## Observation Versus External Scanning

Observation is the structuring of already-available references. External scanning would touch external systems, inspect live targets, fetch source refs, fetch URLs, call providers, use browser/RPA/gateway runtimes, execute read-only tools, or access credentials.

v0.31.1 implements observation vocabulary only. It does not implement external scanning.

## Source Ref Versus Fetching Source

`source_ref` is a pointer-like string. It may name or identify a source artifact, but it is not dereferenced in v0.31.1.

The following are forbidden in v0.31.1:

- source_ref fetch;
- URL fetch;
- provider fetch;
- filesystem fetch beyond already supplied artifact refs;
- live target read;
- read-only tool execution.

`ObservationTargetRef.fetches_source_ref` returns false.

`ObservationArtifactRef.fetches_artifact` returns false.

`ObservationEvidenceRef.runtime_trust` returns false.

## Safe Defaults

Default builders:

- create references without fetching them;
- create inputs without execution-request semantics;
- create findings without certification, permission, digestion, or dominion creation;
- create gaps without remediation;
- create risk signals without authority grants;
- create outputs with execution, activation, and external-scan readiness false;
- create no-op decisions that execute nothing;
- create run previews that guarantee no execution, no scan, no tool execution, no registry mutation, and no memory mutation;
- create reports that are readiness for v0.31.2 design only.

