# v0.33.5 Safe Workspace Inspection Tool Pack

Parent track: Internal General Agent Runtime MVP.

v0.33.5 builds on v0.33.4 Safe Read-only Tool Registry and v0.33.3 Session Runtime / Turn State Machine. It inherits the v0.33.0-v0.33.4 gates: boundary/profile/prompt/session/tool-registry artifacts remain non-provider, non-agent-step, non-write, and non-OCEL-runtime surfaces.

## Definition

Safe workspace inspection is a bounded, policy-gated, Python-internal read-only inspection surface. It may inspect path metadata, bounded project trees, safe text files, bounded text search results, and reference inventory metadata after root/path/secret/binary/size/depth checks pass.

Safe workspace inspection is not general tool execution. It does not open shell access, command execution, provider invocation, agent step execution, workspace mutation, or OCEL runtime emission. `ready_for_safe_workspace_inspection_execution` may be true only for this bounded v0.33.5 inspection pack. `ready_for_execution` and `ready_for_general_tool_execution` remain false.

## Boundaries

Safe text read is not arbitrary file read. A read is allowed only after path normalization, allowed-root validation, secret-pattern checks, extension checks, size checks, and binary checks. Text output is capped by `max_read_chars`.

Reference inventory summary is not reference code execution. `references/OpenCode` and `references/Hermes` may be inspected only through the same bounded safe read-only path policy. Reference code is never executed, imported, installed, or tested. Secret-like, large, binary, outside-root, and symlink-escape paths are skipped or denied.

Pathlib-based inspection is not shell execution. v0.33.5 uses standard-library path operations such as path resolution, stat metadata, directory iteration, and approved text reads. It does not use shell commands, subprocesses, package managers, grep/find/cat/ls, or external harnesses.

Session state, prompt assembly output, and read-only tool registry metadata may point to this pack later. v0.33.5 itself does not run an agent step. v0.33.7 may trace this later, but v0.33.5 does not emit runtime OCEL events.

## Policy

Root sandbox policy:
- Allowed roots are explicit.
- Blocked roots are explicit.
- Symlink escape is always false.
- Reference roots do not imply execution/import/install permission.

Path normalization policy:
- Normalize before metadata or content inspection.
- Deny outside-root and traversal paths.
- Return denied/skipped records for expected policy blocks.

Secret and binary policy:
- Secret-like names include `.env`, `secret`, `key`, `token`, `credential`, `pem`, and `id_rsa` patterns.
- Secret-like files are skipped or denied before content access.
- Binary-like files are skipped.
- Oversized files are skipped.

Limit policy:
- Max depth, entries, file size, read chars, search matches, search files, and line length prevent unbounded traversal and output.
- Search output is bounded and excerpted.

## Flow

`ReadOnlyToolRegistry / WorkspaceInspectionRequest / PathPolicy -> WorkspaceInspectionDecision -> safe bounded inspection -> WorkspaceInspectionToolResult -> WorkspaceInspectionReport -> V0335ReadinessReport -> v0.33.6 Agent Step Runner`

v0.33.6 and v0.33.7 are design-stage handoffs only. v0.33.5 does not execute agent steps and does not emit runtime traces.

## Hard Prohibitions

v0.33.5 does not execute shell commands, does not use subprocess, does not write/edit/patch files, does not create/delete/rename/chmod/chown files, does not install dependencies, does not execute OpenCode/Hermes/OpenClaw, does not import reference code, does not run reference tests, does not read secret-like files, does not read arbitrary binary files, does not call providers, does not execute agent steps, and does not emit runtime OCEL events.

Model/provider/network/credential/command/write/edit/patch/registry/memory/OCEL/UI/external-control/authority readiness remains false.

## Acceptance Criteria

- Bounded safe read-only workspace inspection exists.
- Path root sandbox, secret-pattern block, binary/oversized skip, bounded tree inspection, bounded safe text read, bounded search, and metadata-only reference inventory summary exist.
- Denied/skipped results are safe outcomes.
- No shell/subprocess/command execution, write/edit/patch, reference execution/import/install, provider invocation, agent step execution, or OCEL emission is introduced.

## Withdrawal Conditions

Withdraw v0.33.5 readiness if any implementation path shells out, writes/mutates files, reads secret-like content, reads binary content as text, escapes allowed roots, follows unsafe symlink escapes, executes/imports reference code, calls providers, accesses network/credentials, mutates registry/memory, or emits/persists runtime traces.
