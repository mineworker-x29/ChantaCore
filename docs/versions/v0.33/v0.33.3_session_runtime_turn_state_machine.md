# v0.33.3 Session Runtime / Turn State Machine

Release name: v0.33.3 Session Runtime / Turn State Machine

Parent track: Internal General Agent Runtime MVP

v0.33.3 builds on v0.33.2 Prompt Assembly Pipeline, v0.33.1 Agent Profile Runtime, and v0.33.0 Internal Runtime Boundary & Permission Gate.

## Definition

Session Runtime / Turn State Machine is the deterministic in-memory control contract for session, turn, step, transition, failure, snapshot, and report artifacts. v0.33.3 may construct session/turn/step state artifacts and may validate deterministic transitions.

Session state machine is not agent execution. State transition is not side-effect execution. Transition decision is not model response. Step record is not step execution. Session snapshot is not persistence.

PromptAssemblyOutput can be referenced by ID or metadata, but it is not sent to a model in v0.33.3. AgentRuntimeProfile can be referenced, but it is not executed as a session authority.

## Reference Context

references/OpenCode is read-only design/reference context only.

references/Hermes is read-only design/reference context only.

Reference context can be carried as IDs, labels, path-ref notes, or source refs, but it is not read. Reference files are not read. Reference code is never executed. Reference code is never imported. Dependency install is prohibited.

## Outcomes

Safe fail, no-op, block, cancel, and ask-user-required outcomes are valid non-executing results. A terminal outcome is not persistence. response_ready is not a provider response by itself.

## Gate Inheritance

v0.33.3 inherits v0.33.0-v0.33.2 gates and does not bypass v0.30, v0.31, or v0.32 restrictions.

## Acceptance Criteria

- In-memory session/turn/step artifacts can be constructed.
- Deterministic transition validation exists.
- Invalid transitions are denied.
- Safe-fail/no-op/block outcomes exist.
- Session snapshot is not persistence.
- State transition is not side-effect execution.
- references/OpenCode and references/Hermes remain read-only design/reference context only.
- Runtime/model/provider/tool/workspace/OCEL readiness flags remain false.
- Focused tests pass.

## Withdrawal Conditions

This contract should be withdrawn or treated as invalid if implementation code introduces model/provider calls, agent or model step execution, tool execution, workspace inspection, reference file access, reference code execution/import/install, network or credential access, command execution, workspace write/edit/patch behavior, registry or memory mutation, OCEL emission, runtime trace persistence, UI runtime, external control, authority grant, or any runtime/model/provider/tool/workspace/OCEL readiness flag set true.

## Handoff

v0.33.3 is ready only for v0.33.4 Safe Read-only Tool Registry, v0.33.6 Agent Step Runner, and v0.33.7 Runtime OCEL Trace Emitter design-stage handoff. It does not authorize execution.
