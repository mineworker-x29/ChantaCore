# v0.33.7 Runtime OCEL Trace Emitter

Parent track: Internal General Agent Runtime MVP.

v0.33.7 builds on v0.33.6 Agent Step Runner MVP, v0.33.5 Safe Workspace Inspection Tool Pack, and v0.33.3 Session Runtime / Turn State Machine. It introduces bounded internal runtime OCEL trace artifact emission only.

## Scope

v0.33.7 may create bounded in-memory OCEL-style trace artifacts from supplied runtime artifacts:

- `AgentRuntimeSession` / `AgentRuntimeTurn` / `AgentRuntimeStep` identifiers and summaries.
- `PromptAssemblyOutput` identifiers and summaries when supplied as prior artifact refs.
- `AgentActionProposal`, `AgentActionDecision`, `AgentSafeToolExecutionRequest`, `AgentSafeToolExecutionResult`.
- `AgentStepOutput` and `AgentStepExecutionRecord`.
- `WorkspaceInspectionToolResult` summaries or references already produced by v0.33.5.

Trace artifact creation is not persistent trace write. A `RuntimeOCELTracePacket` is a returned data artifact, not a database record, runtime log, file, or external sink payload.

OCEL event creation is not an external side effect. OCEL object and relation creation is not runtime registry mutation or domain object mutation.

## Policy

The trace policy blocks raw model output, raw tool output, full file content, secret content, credential content, persistent writes, and external sinks. Attributes are bounded and sanitized before they enter events, objects, relations, or packet-level attribute lists.

Secret-like patterns include `.env`, `secret`, `key`, `token`, `credential`, `pem`, and `id_rsa`. Suspicious values are redacted. Long values are truncated. Raw model/tool output is summarized by status, counts, identifiers, and bounded summaries rather than copied.

`references/OpenCode` and `references/Hermes` may appear only as safe reference-context refs or path labels inherited from earlier bounded artifacts. v0.33.7 does not read reference files and does not execute, import, install, or test reference code.

## Taxonomies

Event taxonomy covers session, turn, step, supplied output, action proposal/decision, safe tool request/result, final response, no-op, safe-fail, and completion trace events.

Object taxonomy covers session, turn, step, runtime boundary, runtime profile, prompt output, supplied model output, action proposal, action decision, safe tool request/result, workspace inspection result, reference context, no-op record, and safe-fail record.

Relation taxonomy covers session-turn, turn-step, step-profile, step-prompt, step-output, proposal-decision, safe-tool request/result, response, and terminal outcome relationships.

Attribute taxonomy covers summary, status, decision kind, result kind, risk kind, redaction status, bounded output, source/evidence refs, path refs, safe tool names, token estimates, and timestamps.

Sink taxonomy allows returned trace packets and in-memory test sinks. Future internal stores are future-stage only. External trace sinks remain blocked.

## Flow

`AgentStepOutput / AgentStepExecutionRecord / SessionSnapshot / WorkspaceInspectionResult -> RuntimeOCELTraceEmissionInput -> TraceDecision -> TracePacket -> ValidationReport -> TraceEmissionReport -> V0337ReadinessReport -> v0.33.8 CLI Agent Run Surface`

The emitter operates only on supplied in-memory artifacts. It does not execute agent steps, execute tools, inspect workspace files, read files, call providers, or persist traces.

## Hard Prohibitions

v0.33.7 does not call model providers. It does not invoke provider SDKs. It does not execute agent steps, model steps, tools, or workspace inspection. It does not inspect workspace files and does not read files.

v0.33.7 does not write trace files, database records, runtime logs, or arbitrary files. It does not persist runtime traces and does not use external trace sinks.

v0.33.7 does not execute shell commands, use subprocess, access network, access credentials, write/edit/patch files, execute OpenCode/Hermes/OpenClaw, import reference code, install dependencies, mutate registries, mutate memory, implement UI runtime, perform external control, or grant authority.

v0.33.7 does not store raw model output, raw tool output, secret content, credential content, or full file contents.

## Readiness

`ready_for_bounded_internal_ocel_trace_emission` may be true for returned bounded trace artifacts.

`ready_for_general_ocel_emission`, `ready_for_persistent_trace_write`, `ready_for_external_trace_sink`, and `ready_for_execution` remain false. Provider, tool, command, network, credential, workspace write, code edit, patch, registry, memory, UI, external control, and authority readiness remain false.

## Acceptance Criteria

- Bounded runtime OCEL trace packets can be created from supplied fake/in-memory step outputs and session snapshots.
- Events, objects, relations, and attributes are bounded and sanitized.
- Raw output and secret/credential content are redacted or omitted.
- Validation reports block unsafe packets.
- Trace packets are returned artifacts only, not persistence.
- v0.33.0-v0.33.6 gates remain inherited.

## Withdrawal Conditions

Withdraw v0.33.7 readiness if implementation writes trace files, database records, runtime logs, or external sink payloads; records raw model/tool output; traces secret/credential content; reads files; inspects workspaces; executes tools or agent steps; calls providers; uses shell/subprocess/network; or sets any general execution, general OCEL, persistent trace, or external sink readiness flag true.

## Handoff

v0.33.7 is a design and artifact handoff for v0.33.8 CLI Agent Run Surface. v0.33.8 may display or surface returned trace packets later, but v0.33.7 itself does not implement CLI runtime execution or persistent trace storage.
