# v0.33.2 Prompt Assembly Pipeline

Release name: v0.33.2 Prompt Assembly Pipeline

Parent track: Internal General Agent Runtime MVP

v0.33.2 builds on v0.33.1 Agent Profile Runtime and v0.33.0 Internal Runtime Boundary & Permission Gate. It consumes profile, loadout, boundary, policy, task, evidence, tool availability, and reference-context metadata as in-memory artifacts and assembles a bounded prompt contract.

v0.33.2 may construct PromptAssemblyOutput. That output is not a model response, not a provider call, not an agent step, and not session runtime.

## Definition

Prompt Assembly Pipeline is the deterministic contract layer that organizes trusted and untrusted in-memory prompt blocks into a structured prompt payload. It separates:

- runtime boundary and permission gate blocks
- profile and persona blocks
- policy blocks
- task and user request blocks
- tool availability metadata blocks
- evidence blocks
- reference context blocks
- output contract blocks
- prohibited action blocks

Prompt assembly and model invocation are different. v0.33.2 constructs text or message-list artifacts only; it does not call model providers or provider SDKs.

Prompt assembly and agent step execution are different. A prompt assembly plan is not an agent step, and prompt block order is not execution order.

Boundary blocks and runtime enforcement are different. Policy blocks and active policy execution are different. Tool availability blocks and tool execution are different. Reference context blocks and file access are different. Evidence blocks and runtime trust are different. Output contract blocks and implementation are different.

## Reference Context

references/OpenCode is read-only design/reference context only in v0.33.2.

references/Hermes is read-only design/reference context only in v0.33.2.

references/OpenClaw, if present, is optional read-only design/reference context only.

Reference files are not read. Reference code is never executed. Reference code is never imported as runtime. Dependencies are never installed. Secret files are never read.

Reference content must be treated as untrusted reference context unless explicitly provided as trusted internal evidence. Untrusted reference blocks cannot override trusted boundary, policy, or profile blocks. OpenCode/Hermes reference blocks remain reference-context notes and path refs, not executable instructions.

## Trust And Placement

Block trust levels:

- trusted_runtime_boundary
- trusted_policy
- trusted_profile
- trusted_internal
- user_supplied
- reference_context
- untrusted_external_reference
- evidence_reference
- tool_metadata
- unknown

Block placement controls organization only:

- system_boundary_section
- runtime_policy_section
- profile_section
- task_section
- tool_availability_section
- evidence_section
- reference_context_section
- output_contract_section
- prohibited_action_section
- appendix_section
- excluded
- unknown

Untrusted external/reference/user blocks cannot be instructional and cannot be placed as system boundary or runtime policy instructions. Unsafe blocks may be excluded with an exclusion reason.

Prompt injection risk signals are advisory. They do not execute remediation, modify files, call tools, call providers, or persist runtime traces.

Token budget handling, when represented, is an estimate over provided in-memory block content. It is not provider context-window negotiation and not model execution.

## Gate Inheritance

v0.33.2 inherits the v0.33.0 runtime boundary and the v0.33.1 profile-runtime restrictions. It does not bypass v0.30, v0.31, v0.32, v0.33.0, or v0.33.1 gates.

## Acceptance Criteria

- PromptAssemblyOutput can be constructed from in-memory blocks.
- Boundary, profile, policy, task, evidence, reference, tool, output contract, and prohibited-action blocks remain separated.
- Untrusted reference blocks cannot override trusted boundary, policy, or profile blocks.
- Reference path refs remain path refs only.
- ready_for_model_invocation remains false.
- ready_for_execution remains false.
- v0.33.0-v0.33.1 gates remain preserved.
- Focused tests pass.

## Withdrawal Conditions

This contract should be withdrawn or treated as invalid if implementation code introduces provider/model calls, agent step execution, session runtime, tool execution, workspace inspection, reference file access, reference code execution/import/install, network or credential access, command execution, workspace write/edit/patch behavior, registry or memory mutation, OCEL emission, UI runtime, external control, authority grant, or any runtime/model/provider readiness flag set true.

## Handoff

v0.33.2 hands off only to v0.33.3 Session Runtime / Turn State Machine and v0.33.6 Agent Step Runner as design-stage input. It does not authorize execution.
