# v0.33.0 Internal Runtime Boundary & Permission Gate

Release name: v0.33.0 Internal Runtime Boundary & Permission Gate.

Parent track: Internal General Agent Runtime MVP.

v0.33.0 begins the internal runtime track after v0.32.9 External Harness
Observation & Digestion Pipeline v1. The v0.32.9 handoff provides observed
external harness context, reference corpus path refs, and design-stage candidate
metadata. v0.33.0 turns that handoff into an internal runtime boundary and
permission gate contract only.

v0.33.0 defines the gate that later stages must obey. It does not run the agent
runtime, execute read-only tools, inspect workspace files, call providers, emit
runtime OCEL events, or implement CLI runtime execution.

## Boundary Meaning

An internal runtime boundary records which internal surfaces may be discussed as
design-stage surfaces and which surfaces remain prohibited runtime surfaces.
Runtime boundary is not runtime execution.

A permission gate evaluates a requested surface and produces a decision such as
allowed design-stage only, allowed boundary definition only, denied, blocked,
deferred, ask-user-required, review-required, future-gate-required, or no-op.
Permission gate is not permission grant.

Allowed design-stage surface is not an executable surface. In v0.33.0 every
`executable_in_v0330` flag remains false.

## Reference Corpus

Reference corpus path refs may include `references/OpenCode`,
`references/Hermes`, and optionally `references/OpenClaw`. These are read-only
design/reference corpus only in v0.33.0.

`references/OpenCode` role in v0.33: read-only design/reference corpus only.
`references/Hermes` role in v0.33: read-only design/reference corpus only.
`references/OpenClaw` role in v0.33: optional read-only design/reference corpus
only.

A reference corpus path is not execution permission, import permission,
dependency installation permission, test execution permission, secret-file read
permission, or read-only inspection permission. Reference code is never executed.
Reference code is never imported as runtime. Dependencies are never installed.
Secret files are never read.

## Future Stages

Safe read-only tool registry is future-stage v0.33.4 and remains non-executable
in v0.33.0. Safe workspace inspection is future-stage v0.33.5 and remains false.
Agent step runner is future-stage v0.33.6 and remains false. Runtime OCEL trace
emission is future-stage v0.33.7 and remains false. CLI runtime is future-stage
v0.33.8 and remains false.

`ready_for_v0331_agent_profile_runtime` and `ready_for_v0332_prompt_assembly`
mean next design-stage handoff only. They do not imply execution readiness.

## Taxonomies

`InternalRuntimeTrackKind` records the v0.33 roadmap stages. Track kind is
roadmap metadata and does not enable execution.

`InternalRuntimeSurfaceKind` records planned internal surfaces such as profile
resolution, prompt assembly, session state machine, read-only tool registry,
safe workspace inspection, model step boundary, agent action proposal,
permission gate, internal OCEL trace, CLI surface, reference corpus context, and
external harness context. It also records prohibited surfaces such as provider
boundary, workspace write, code edit, patch application, shell command, network,
credential, browser, RPA, gateway, external control, and authority grant.

`InternalRuntimeCapabilityKind` records design capabilities and dangerous
capabilities. Read-only capabilities are future-stage and not executable in
v0.33.0. Dangerous capabilities remain prohibited.

`InternalRuntimeActionKind` records permission decision actions. It is not
execution.

`InternalRuntimeRiskSurfaceKind` is advisory/governance metadata and does not
prove exploitability.

`InternalRuntimePermissionDecisionKind`, `InternalRuntimeBoundaryStatus`,
`InternalRuntimeReadinessLevel`, and `InternalRuntimeReferenceCorpusPosture`
preserve non-runtime semantics. Boundary readiness is not runtime readiness.

## Models

`InternalRuntimeCapabilityFlagSet` keeps runtime readiness false while allowing
v0.33.1/v0.33.2 design-stage handoff flags.

`InternalRuntimeAllowedSurface` records design-stage allowed surfaces and keeps
`executable_in_v0330` false.

`InternalRuntimeProhibitedSurface` records prohibited surfaces, blocked execution
and blocked runtime readiness. It is represented in the gate, but it is not
runtime enforcement by itself.

`InternalRuntimeReferenceCorpusBoundary` records reference path refs and
prohibited file patterns such as `.env`, secret, key, token, credential, pem,
and id_rsa patterns. It is not file access.

`InternalRuntimeBoundary` combines allowed surfaces, prohibited surfaces,
reference corpus boundary, capability flags, gaps, blocked reasons, evidence,
and withdrawal conditions.

`InternalRuntimePermissionRequest` is not permission grant and not execution.
`InternalRuntimePermissionDecision` is not runtime permission grant.
`InternalRuntimeDeniedAction` is a safe outcome. `InternalRuntimeGateEvaluation`
is not execution.

`InternalRuntimeNoExternalSideEffectGuarantee` confirms no runtime execution or
external side effect. `V033RoadmapOverview` records the roadmap only and is not
implementation. `V0330ReadinessReport` is not runtime enablement.

## Hard Prohibitions

v0.33.0 does not execute external harnesses, OpenCode, Hermes, OpenClaw, or
references code. It does not import reference code as runtime, install
dependencies, run tests inside references, live-scan external systems, fetch
source refs or URLs, read secret files, load plugins, register or invoke tools,
execute read-only tools, inspect workspace files, run agent steps, invoke
providers, access network, access credentials, execute commands, write
workspace files, edit code, apply patches, control browser/RPA/gateway, send
packets, mutate registry, mutate memory, emit OCEL runtime events, persist OCEL
objects or relations, write runtime traces, implement UI runtime, control
external runtimes, grant authority, or grant D4-D9.

## Gate Inheritance

v0.33.0 preserves v0.30, v0.31, and v0.32 gates. It does not bypass the
External Harness Observation & Digestion Pipeline v1 consolidation. v0.32
external candidates and dominion candidates remain design-stage artifacts.

## Acceptance Criteria

- Boundary and permission-gate contract models exist.
- Reference corpus boundaries represent OpenCode and Hermes as read-only
  design/reference corpus path refs only.
- Runtime, provider, command, workspace, OCEL, UI, external control, authority,
  and D4-D9 readiness flags remain false.
- Focused tests pass.
- Handoff to v0.33.1 and v0.33.2 is clear as design-stage only.

## Withdrawal Conditions

Withdraw v0.33.0 readiness if any implementation executes runtime behavior,
executes read-only tools, inspects workspace files, calls providers, emits
runtime OCEL events, executes or imports reference code, installs dependencies,
reads secret files, grants authority, grants D4-D9, mutates registry or memory,
or sets any runtime readiness flag true.
