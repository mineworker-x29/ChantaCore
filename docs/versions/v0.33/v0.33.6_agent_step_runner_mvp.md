# v0.33.6 Agent Step Runner MVP

Parent track: Internal General Agent Runtime MVP.

v0.33.6 builds on v0.33.5 Safe Workspace Inspection Tool Pack, v0.33.4 Safe Read-only Tool Registry, and v0.33.3 Session Runtime / Turn State Machine. It inherits the v0.33.0-v0.33.5 gates: real provider calls, autonomous runtime loops, general tool execution, shell commands, workspace writes, reference execution/import/install, and OCEL runtime emission remain prohibited.

## Definition

v0.33.6 may execute one bounded internal agent step. A bounded step means:

- Accept supplied/mock model output as an in-memory artifact.
- Parse a deterministic, bounded action proposal.
- Evaluate the proposal against runner, boundary, registry metadata, and safe tool policy.
- Optionally call v0.33.5 safe workspace inspection functions if the action decision explicitly allows it.
- Return `AgentStepOutput`, blocked/no-op/ask-user output, report, and readiness metadata.

This is not autonomous agent runtime. It does not run multi-step planning, retry loops, shell commands, arbitrary tools, provider calls, or external harnesses.

## Model Output Policy

Supplied/mock model output is not real provider invocation. v0.33.6 does not call OpenAI, Claude, local providers, or provider SDKs. Supplied model output is untrusted and cannot override boundary, policy, or tool safety. Structured actions are preferred. Raw text parsing is minimal and bounded; it does not evaluate or execute code.

A final response result may be constructed from supplied/mock output. It is not a provider response.

## Proposal And Decision Policy

An `AgentActionProposal` is not execution. An `AgentActionDecision` is permission metadata until it routes to an explicitly allowed safe path.

Unsafe proposals, unknown proposals, prohibited command/write/provider/network/credential/external-control proposals, and unsupported tool calls become blocked/no-op/ask-user outputs. They do not trigger remediation by execution.

`allow_safe_workspace_inspection` permits only v0.33.5 bounded safe workspace inspection tools. It does not permit general tool execution.

## Safe Tool Bridge

The only executable bridge in v0.33.6 is to v0.33.5 safe workspace inspection functions:

- `inspect_project_tree_readonly`
- `inspect_file_metadata_readonly`
- `read_text_file_safe`
- `search_text_in_workspace_readonly`
- `summarize_reference_inventory_readonly`
- `inspect_workspace_path_readonly`

The bridge requires an allowed action decision and a v0.33.5 path policy. v0.33.5 still enforces root, symlink, size, secret, binary, and output bounds.

`references/OpenCode` and `references/Hermes` remain safe read-only only through the v0.33.5 policy. Reference code is never executed, imported, installed, or tested. Secret-like reference files are skipped or denied.

## Future Stages

v0.33.7 may trace this later, but v0.33.6 does not emit runtime OCEL events and does not persist runtime traces. v0.33.8 may expose a CLI run surface later, but v0.33.6 does not implement CLI runtime execution.

## Flow

`AgentRuntimeSession / PromptAssemblyOutput / supplied model output / ReadOnlyToolRegistry / WorkspaceInspectionPolicy -> AgentStepInput -> parse proposal -> evaluate decision -> optional v0.33.5 safe tool execution -> AgentStepOutput -> AgentStepReport -> V0336ReadinessReport -> v0.33.7 Runtime OCEL Trace Emitter`

## Hard Prohibitions

v0.33.6 does not call real model providers, does not invoke provider SDKs, does not execute general tools, does not shell out, does not use subprocess, does not execute commands, does not write/edit/patch files, does not create/delete/rename/chmod/chown files, does not execute OpenCode/Hermes/OpenClaw, does not import reference code, does not install dependencies, does not read secret-like files, does not access network or credentials, does not mutate registry/memory, does not emit runtime OCEL events, does not persist runtime traces, does not implement UI runtime, does not enable external control, and does not grant authority.

`ready_for_safe_workspace_inspection_execution` may be true only through v0.33.5 policy. `ready_for_execution`, `ready_for_general_agent_execution`, `ready_for_real_model_invocation`, `ready_for_provider_invocation`, and `ready_for_general_tool_execution` remain false.

## Acceptance Criteria

- One bounded internal step can process supplied/mock output.
- Safe action parsing and decision evaluation exist.
- Approved safe workspace inspection proposals can call v0.33.5 functions under policy.
- Unsafe proposals are blocked/no-op/ask-user.
- No provider, autonomous loop, general tool, shell, write, reference execution/import/install, secret read, OCEL emission, or trace persistence is introduced.

## Withdrawal Conditions

Withdraw v0.33.6 readiness if any implementation path calls a real provider, invokes provider SDKs, shells out, runs commands, writes/edits/patches files, executes arbitrary tools, executes/imports/installs reference code, reads secret-like content, accesses network/credentials, mutates registry/memory, emits OCEL events, persists runtime traces, enables UI runtime, enables external control, grants authority, or sets general execution readiness true.
