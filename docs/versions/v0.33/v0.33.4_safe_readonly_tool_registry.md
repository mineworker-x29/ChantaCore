# v0.33.4 Safe Read-only Tool Registry

Release name: v0.33.4 Safe Read-only Tool Registry

Parent track: Internal General Agent Runtime MVP

v0.33.4 builds on v0.33.3 Session Runtime / Turn State Machine and inherits the v0.33.0 Internal Runtime Boundary & Permission Gate.

## Definition

Safe Read-only Tool Registry is the contract layer for read-only tool descriptor metadata, safety policies, call proposals, permission decisions, blocked actions, validation reports, registry reports, and readiness reports.

v0.33.4 may define safe read-only tool descriptors. v0.33.4 may validate tool descriptors. v0.33.4 may evaluate tool call proposals.

Tool registry is not tool execution. A tool descriptor is not an executable tool. Tool availability is not tool invocation. A tool call proposal is not invocation. A permission decision is not execution. A safety policy is metadata and is not runtime enforcement by itself. Registry metadata is not active runtime registry mutation.

Allowed future read-only execution decisions are not execution in v0.33.4. They only identify candidates for later v0.33.5/v0.33.6 design-stage gates.

## Reference Context

references/OpenCode is read-only design/reference context only.

references/Hermes is read-only design/reference context only.

Reference corpus remains path-ref/design context only. Reference files are not read. Reference code is never executed. Reference code is never imported. Dependency install is prohibited. Secret files are never read.

The registry may describe future tools that could inspect reference corpus safely in v0.33.5, but v0.33.4 does not access reference files.

## Acceptance Criteria

- Safe read-only tool descriptors can be defined.
- Tool descriptors are non-executable.
- Tool registry is not executable runtime registry.
- Tool proposals are not invocations.
- Permission decisions are not executions.
- Dangerous surfaces are blocked.
- Secret-like patterns are prohibited.
- references/OpenCode and references/Hermes remain read-only design/reference context only.
- Runtime/tool/file/model/provider/workspace/OCEL readiness flags remain false.
- Focused tests pass.

## Withdrawal Conditions

This contract should be withdrawn or treated as invalid if implementation code introduces tool execution, read-only tool execution, workspace inspection, file read, reference file access, reference code execution/import/install, model/provider invocation, agent/model step execution, network or credential access, command execution, workspace write/edit/patch behavior, registry or memory mutation, OCEL emission, runtime trace persistence, UI runtime, external control, authority grant, or any runtime/tool/file/model/provider/workspace/OCEL readiness flag set true.

## Handoff

v0.33.4 is ready only for v0.33.5 Safe Workspace Inspection Tool Pack and v0.33.6 Agent Step Runner design-stage handoff. It does not authorize execution.
