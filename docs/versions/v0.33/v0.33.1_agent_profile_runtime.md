# v0.33.1 Agent Profile Runtime

Release name: v0.33.1 Agent Profile Runtime.

Parent track: Internal General Agent Runtime MVP.

v0.33.1 builds on v0.33.0 Internal Runtime Boundary & Permission Gate and the
v0.32.9 External Harness Observation & Digestion Pipeline v1 handoff. It
resolves provided in-memory profile metadata into an `AgentRuntimeProfile`
contract. It does not run the agent runtime.

## Definition

Agent Profile Runtime is the profile-resolution layer for persona, mode,
loadout, policy overlay, tool availability view, and reference context. It may
represent Vera, ChantaCore mode, persona projection, runtime loadout, policy
overlay, and read-only reference corpus roles as contract metadata.

Profile runtime is not agent step execution. Profile resolution is not
permission grant. Persona projection is not authority. Mode binding is not tool
activation. Policy overlay is not active policy enforcement. Tool availability
view is not tool execution. Reference context is not reference file access.

## Reference Context

`references/OpenCode` role in Agent Profile Runtime: read-only design/reference
corpus only. `references/Hermes` role in Agent Profile Runtime: read-only
design/reference corpus only. `references/OpenClaw` may be represented as an
optional read-only design/reference corpus path ref.

OpenCode and Hermes remain read-only design/reference corpus only. Reference
code is never executed. Reference code is never imported as runtime.
Dependencies are never installed. Secret files are never read.

## Future Stages

v0.33.2 Prompt Assembly Pipeline is future-stage. v0.33.1 does not assemble a
final prompt payload. v0.33.3 Session Runtime / Turn State Machine is
future-stage. v0.33.1 does not create session runtime.

v0.33.4 Safe Read-only Tool Registry is future-stage. v0.33.1 does not execute
read-only tools or access a tool registry. v0.33.5 Workspace Inspection is
future-stage. v0.33.1 does not inspect workspace files.

v0.33.1 does not call providers and does not emit runtime OCEL events.

`ready_for_v0332_prompt_assembly` and `ready_for_v0333_session_runtime` mean
next design-stage handoff only. `ready_for_execution` remains false.

## Taxonomies

`AgentProfileKind` covers Vera, ChantaCore, Chanta Research Group, Schumpeter,
generic internal agent, test profile, and unknown. Profile kind is descriptive
and does not grant authority.

`AgentProfileModeKind` covers default, research, coding assistant, process
intelligence, external harness analysis, reference corpus review, safe readonly,
boundary review, test, and unknown. Mode kind does not activate tools.

`AgentProfileSourceKind`, `AgentProfileProjectionKind`,
`AgentProfilePolicyOverlayKind`, `AgentProfileLoadoutKind`,
`AgentProfileContextRoleKind`, `AgentProfileResolutionStatus`,
`AgentProfileReadinessLevel`, and `AgentProfileEvidenceQuality` preserve
non-runtime semantics.

## Models

`AgentProfileRuntimeFlagSet` records v0.33.2/v0.33.3 design-stage handoff flags
and keeps all runtime readiness flags false.

`AgentProfileSourceRef` is not fetch, memory read, or execution.
`AgentProfilePersonaProjection` is not authority grant. `AgentProfileModeBinding`
is not tool activation. `AgentProfilePolicyOverlay` is not active policy
enforcement. `AgentProfileReferenceContext` is not file access.
`AgentProfileToolAvailabilityView` is not tool registry access.

`AgentRuntimeLoadout` is not runtime action. `AgentRuntimeProfile` is not an
active session. `AgentProfileResolutionDecision` is not permission grant.
`AgentProfileResolutionReport` is not runtime execution.

`AgentProfileRuntimeRunPreview` and
`AgentProfileRuntimeNoExecutionGuarantee` explicitly preserve no execution.
`V0331ReadinessReport` is not runtime enablement.

## Flow

InternalRuntimeBoundary / V033HandoffPacket / profile metadata / references
path refs -> AgentProfileResolutionInput -> AgentProfileResolutionDecision ->
AgentRuntimeLoadout -> AgentRuntimeProfile -> AgentProfileResolutionReport ->
V0331ReadinessReport -> v0.33.2 Prompt Assembly Pipeline.

v0.33.3 Session Runtime / Turn State Machine may receive design-stage handoff
metadata from v0.33.1. That handoff is not execution readiness.

final prompt payloads, create session runtime, execute read-only tools, inspect
workspace files, execute external harnesses, execute OpenCode/Hermes/OpenClaw,
execute references code, import reference code as runtime, install dependencies,
run reference tests, live-scan external systems, fetch source refs or URLs, read
secret files, load plugins, register or invoke tools, invoke providers, access
network, access credentials, execute commands, write workspace files, edit code,
apply patches, control browser/RPA/gateway, send packets, mutate registries,
mutate memory, emit runtime OCEL events, persist OCEL objects or relations,
write runtime logs, write database records, implement UI runtime, control
external runtimes, grant authority, or grant D4-D9.

## Gate Inheritance

v0.33.1 inherits the v0.33.0 boundary and permission gate. It preserves v0.32.9
external harness observation/digestion handoff semantics. It does not bypass
v0.30, v0.31, v0.32, or v0.33.0 gates.

## Acceptance Criteria

- Profile runtime contract models exist.
- In-memory fake Vera/OpenCode/Hermes profile metadata can resolve an
  `AgentRuntimeProfile`.
- References remain path-ref/read-only design context only.
- Runtime readiness flags remain false.
- Focused tests pass.
- Handoff to v0.33.2 and v0.33.3 is clear as design-stage only.

## Withdrawal Conditions

Withdraw v0.33.1 readiness if any implementation runs the agent runtime,
executes an agent step, executes prompt assembly, creates session runtime,
executes read-only tools, inspects workspace files, calls providers, executes or
imports reference code, installs dependencies, reads secret files, mutates
registry or memory, emits OCEL events, grants authority, controls external
runtimes, or sets runtime readiness flags true.
