# v0.25.6 Response Assembly & Evidence Binder

Korean name: 응답 조립·근거 Binder

Track: Bounded General Agent Surface & Internal Tool Routing

Previous: v0.25.5 Internal Provider Invocation Orchestrator

Next: v0.25.7 Ask / REPL Surface

## Version Definition

v0.25.6 implements response assembly and evidence binding for the public
ChantaCore agent surface. It consumes v0.25.5 provider invocation evidence
seeds or v0.25.3 non-route gate outcomes, creates evidence bundles, binds
claims to evidence, creates answer drafts, and assembles a response artifact
for v0.25.7.

v0.25.x remains Bounded General Agent Surface & Internal Tool Routing.

## Relationships

v0.25.5 provides provider result refs, result bundles, and evidence seeds.
v0.25.6 may bind those refs into an evidence bundle, but it does not invoke
providers.

v0.25.4 creates route plans and provider selections. v0.25.6 does not route or
execute route steps.

v0.25.3 provides safety gate outcomes for no-action, clarification,
needs-more-input, blocked, and deferred response drafts.

v0.25.2, v0.25.1, and v0.25.0 remain the intent/task, turn envelope, and agent
surface contract prerequisites. v0.24.9 remains the internal provider
foundation prerequisite.

## Direction

ChantaCore directly implements its OCEL-native agent spine. OpenCode,
OpenClaw, and Hermes remain reference architectures only. They do not become
runtime dependencies, adapters, bridges, fork-specific integrations, or copied
runtime logic in v0.25.6.

Workspace workbench remains deferred to v0.26. Memory candidate and continuity
remain deferred to v0.27. External provider and agent adapters remain deferred
to v0.29+. External agent dominion remains deferred to v0.30+.

## Principles

- Response assembly is not ask/repl emission.
- Evidence binding is not raw provider output dump.
- Provider result is not automatically a fact.
- Observation, inference, uncertainty, and limitation must be separated.
- No-action response must include rationale.
- Clarification response must ask specific missing-input questions.
- Blocked response must include policy-grounded reason and safe alternative
  when possible.
- Conclusion should appear once.
- Ask/REPL surface is deferred to v0.25.7.
- Agent trace and usability telemetry are deferred to v0.25.8.

## Service Design

AgentResponseAssemblyPrerequisiteSourceService loads the agent surface
contract, provider invocation report, provider evidence seed, provider result
bundle, safety gate report, gate outcome envelope, intent report, task frame,
and turn envelope when available.

AgentEvidenceSourceRefService creates sanitized source references. It does not
include raw content, raw secrets, credentials, or private full paths.

AgentEvidenceItemService creates evidence items from provider result refs,
provider evidence seeds, and safety gate decisions.

AgentEvidenceBundleService computes provider, policy, and uncertainty evidence
counts.

AgentClaimService creates provider observation, inference, no-action rationale,
clarification request, blocked rationale, deferred scope, uncertainty, and
confirmed fact claims as appropriate.

AgentClaimSupportService binds claims to evidence or marks them unsupported.

AgentResponseSectionService creates response sections and ensures there is one
final conclusion section.

AgentAnswerDraftService creates provider-backed and non-provider response
drafts. AgentAssembledResponseService assembles a response artifact without
emitting it.

## Evidence Binding

Provider evidence is represented by sanitized result refs. Raw provider output
is not inlined. A provider result ref supports a provider observation, not an
automatic fact. Inferences are labeled separately from observations.

## Status Rules

passed means a response source exists, evidence was bound, claims/supports and
sections were created, the response artifact was assembled, and final emission,
provider invocation, local command execution, memory mutation, external adapter
use, raw provider output, raw secrets, and LLM judging did not occur.

warning means evidence is partial, uncertainty or limitations are present, a
non-route outcome was assembled, or unsupported non-critical claims were marked.

failed means no response source exists under strict mode or a draft/assembled
response cannot be created without a dangerous bypass.

blocked means raw provider output inline, raw secret output, credential
exposure, private full path output, provider invocation, local command
execution, direct file access, ask/REPL execution, memory/persona mutation,
external adapter use, Schumpeter split, or LLM judge use is detected.

## Acceptance Criteria

- AgentResponseAssemblyReport can be created.
- AgentEvidenceBundle can be created from provider evidence seed or gate
  outcome.
- AgentClaim and AgentClaimSupport can be created.
- Provider-backed and non-provider response drafts can be created.
- AgentAssembledResponse can be created.
- Response is evidence-bound.
- Fact, provider observation, inference, uncertainty, and limitation are
  separated.
- No-action, clarification, blocked, and deferred outcomes can produce drafts.
- Final response is not emitted.
- No provider invocation occurs.
- No local command execution occurs.
- No ask/REPL execution occurs.
- No memory promotion or persistent memory write occurs.
- No persona mutation occurs.
- No external provider or agent adapter is implemented.
- No OpenCode/OpenClaw/Hermes runtime dependency is added.
- No Schumpeter split is introduced.
- No credential value is stored or output.
- No raw provider output or raw secret is inlined.
- No LLM judge is used.

## Withdrawal Conditions

Withdraw v0.25.6 readiness if the response assembler emits through ask/REPL,
invokes providers, executes commands, directly reads/searches/inspects,
inlines raw provider output, exposes raw secrets/credentials/private full
paths, promotes memory, mutates persona, introduces external adapters,
introduces Schumpeter split logic, or uses an LLM judge.

## Validity Horizon

Valid until v0.25.7 ask/repl emission begins or response/evidence policy
changes.
