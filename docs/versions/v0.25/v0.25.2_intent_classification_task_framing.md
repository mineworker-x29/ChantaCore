# v0.25.2 Intent Classification & Task Framing

Korean name: 의도 분류·과업 프레이밍

Track: Bounded General Agent Surface & Internal Tool Routing

Previous: v0.25.1 Turn Envelope & Interaction Context

Next: v0.25.3 Safety / No-Action / Clarification Gate

## Version Definition

v0.25.2 implements deterministic intent classification and task framing after
v0.25.1 turn envelope creation. It may classify an `AgentTurnEnvelope` into an
`AgentIntentDescriptor`, create task goals, constraints, input requirements,
risk previews, and a task frame candidate for v0.25.3.

v0.25.2 is intent/task-only. It does not run a safety gate, finalize no-action,
blocked, or clarification decisions, route tools, select or invoke providers,
execute local commands, run ask/REPL, promote memory, write persistent memory,
mutate persona, or use an LLM judge.

## Objective

The objective is to convert sanitized turn-envelope input into a deterministic
classification report:

- `AgentIntentClassificationPolicy`
- `AgentIntentClassificationRequest`
- `AgentIntentTaxonomy`
- `AgentIntentRule` and `AgentIntentRuleResult`
- `AgentIntentDescriptor`
- `AgentTaskGoal`
- `AgentTaskConstraint`
- `AgentTaskInputRequirement`
- `AgentTaskRiskPreview`
- `AgentTaskFrame`
- `AgentTaskFrameCandidate`
- `AgentIntentFinding`
- `AgentIntentClassificationReport`

## Relationship To Earlier Versions

v0.25.2 reads v0.25.1 `AgentTurnEnvelope` and `AgentTurnReport` outputs.
It preserves v0.25.0 Agent Surface Contract boundaries and v0.24.9 Internal
Provider / Local Runtime Provider Foundation v1. v0.23.9 remains the
OCEL-native Internal Dominion Foundation v1 prerequisite.

v0.25.x remains the Bounded General Agent Surface & Internal Tool Routing
track.

## Long-Term Direction

ChantaCore directly implements its own OCEL-native agent spine. OpenCode,
OpenClaw, and Hermes remain reference architectures only:

- OpenCode patterns may inform future session, tool registry, permission, and
  provider abstraction.
- Hermes patterns may inform future isolation, delegation, and bounded memory.
- OpenClaw patterns may inform future assistant UX and external-agent risk
  handling.

No runtime dependency, adapter, bridge, fork-specific integration, or copied
runtime logic for these projects is introduced in v0.25.2.

## Core Principles

- Task framing is not tool routing.
- Intent category is not provider selection.
- Risk preview is not safety gate.
- Needs-more-input candidate is not clarification gate.
- Blocked candidate is not final blocked decision.
- No-action candidate is not final no-action decision.
- Safety/no-action/clarification gate is deferred to v0.25.3.
- Tool routing is deferred to v0.25.4.
- Provider invocation is deferred to v0.25.5.
- Ask/REPL surface is deferred to v0.25.7.

## Service Design

- `AgentIntentPrerequisiteSourceService` loads read-only prerequisite refs.
- `AgentIntentClassificationPolicyService` builds the intent/task-only policy.
- `AgentIntentTaxonomyService` builds the taxonomy.
- `AgentIntentRuleRegistry` lists deterministic rules.
- `AgentIntentRuleEngine` evaluates rules without model or provider calls.
- `AgentIntentDescriptorService` builds bounded confidence descriptors.
- `AgentTaskGoalService` extracts task goals.
- `AgentTaskConstraintService` extracts policy/context constraints.
- `AgentTaskInputRequirementService` infers missing inputs.
- `AgentTaskRiskPreviewService` creates a risk preview only.
- `AgentTaskFrameService` creates the task frame.
- `AgentTaskFrameCandidateService` creates a candidate for v0.25.3.
- `AgentIntentFindingService` records warnings and blockers.
- `AgentIntentClassificationReportService` assembles the report.

## Taxonomy

The taxonomy includes general answers, workspace overview, repository search,
file read, process state inspection, local runtime candidate, local runtime
execution request, diagnostics, explanation, planning, architecture design,
implementation prompt, verification prompt, checklist, consolidation,
no-action candidate, needs-more-input candidate, blocked candidate, and
unknown.

## Deterministic Rule Classification

Rules are keyword, phrase, structural, context-ref, negative, or fallback
rules. The default implementation is deterministic/rule-based. External LLM/API
classification and LLM safety judging are disabled.

## Task Framing

Task framing creates goals, constraints, input requirements, and a risk preview.
It may identify likely no-action, blocked, or needs-more-input candidates, but
those are not final decisions. v0.25.3 must perform the safety/no-action/
clarification gate.

## Status Rules

Passed means sanitized turn input, taxonomy, rule results, intent descriptor,
task frame, risk preview, and candidate were created without execution.
Warning covers low confidence, ambiguity, multiple intents, missing input,
future-track risks, and sensitive-like input already sanitized by v0.25.1.
Failed covers missing turn envelope, missing sanitized request, missing
taxonomy, no classifiable intent under strict mode, task frame failure, or
unbounded confidence. Blocked covers premature safety gate, routing, provider
invocation, local command execution, ask/REPL execution, memory promotion,
persona mutation, external adapter, runtime dependency, credential exposure,
raw secret output, or LLM judge.

## Acceptance Criteria

- `AgentIntentClassificationReport` can be created.
- `AgentIntentDescriptor` can be created from `AgentTurnEnvelope`.
- `AgentTaskFrame` can be created.
- `AgentTaskRiskPreview` can be created.
- Task frame candidate points to v0.25.3.
- Classification is deterministic/rule-based by default.
- No LLM judge is used.
- No safety gate is executed.
- No final no-action, blocked, or clarification decision is made.
- No tool routing occurs.
- No provider invocation occurs.
- No local command execution occurs.
- No ask/REPL execution occurs.
- No memory promotion or persistent memory write occurs.
- No persona mutation occurs.
- No external provider or external agent adapter is implemented.
- No OpenCode/OpenClaw/Hermes runtime dependency is added.
- No Schumpeter split is introduced.
- No credential value is stored or output.
