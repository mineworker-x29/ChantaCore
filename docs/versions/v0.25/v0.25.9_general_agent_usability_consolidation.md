# v0.25.9 General Agent Usability Consolidation

Korean name: 일반 Agent 사용성 통합·릴리즈 준비성

Release name: Bounded General Agent Surface Foundation v1

Track: Bounded General Agent Surface & Internal Tool Routing

Next: v0.26.0 Workspace Agent Workbench Contract

## Objective

v0.25.9 consolidates v0.25.0 through v0.25.8 into the first bounded general
agent surface foundation release. It creates release-readiness artifacts:
subject components, capability map, coverage matrix, pipeline boundary report,
safety boundary report, trace/telemetry coverage report, roadmap boundary
report, gap register, release manifest, v0.26 readiness report, workbench
handoff packet, and consolidation report.

Consolidation is not new agent execution. Release readiness is not workspace
workbench. `ready_for_v0_26` means the v0.26 workbench contract can begin next;
it does not mean the v0.26 workbench is implemented.

## Relationships

- v0.25.0 provides the Agent Surface Contract.
- v0.25.1 provides Turn Envelope & Interaction Context.
- v0.25.2 provides Intent Classification & Task Framing.
- v0.25.3 provides Safety / No-Action / Clarification Gate.
- v0.25.4 provides Tool Routing Plan & Provider Selection.
- v0.25.5 provides Internal Provider Invocation Orchestrator.
- v0.25.6 provides Response Assembly & Evidence Binder.
- v0.25.7 provides Ask / REPL Surface.
- v0.25.8 provides Agent Trace / Usability Telemetry.
- v0.24.9 provides the previous Internal Provider / Local Runtime Provider
  Foundation v1 release reference.

v0.25.9 consumes those existing release artifacts as read-only evidence. It does
not create a new user turn and does not rerun the pipeline.

## Direct Development Strategy

ChantaCore directly implements its own OCEL-native agent spine. OpenCode /
OpenClaw / Hermes remain reference architectures only. Do not add runtime
dependency, adapter, bridge, fork-specific integration, or copied runtime logic
for OpenCode/OpenClaw/Hermes in v0.25.9.

## Core Principles

- Consolidation is not new agent execution.
- Release readiness is not workspace workbench.
- Bounded agent surface closes user-facing substrate, not product UI.
- Ask/REPL history may be summarized, but no new ask/REPL turn may run.
- Provider history may be summarized, but no new provider may be invoked.
- Trace/telemetry may be summarized, but no background collector may start.
- `ready_for_v0_26` means workbench can be built next, not that workbench is
  implemented.
- Memory continuity is deferred to v0.27.
- External provider adapters are deferred to v0.29+.
- External agent dominion is deferred to v0.30+.

2. v0.25.2 intent/task

only. Direct bypass remains forbidden.

## Safety Boundary Report

The safety boundary report may summarize nonzero historical ask, REPL turn,
final response emission, provider invocation, bounded local command execution,
response assembly, trace record, and telemetry report counts when they are
observed through their owning release boundaries.

Dangerous counts must remain zero: direct provider invocation, direct file
access, direct subprocess, direct local command execution, command rerun,
automatic repair, autonomous loop, background execution, self-prompt loop,
workspace workbench, memory promotion, persistent memory write, persona
mutation, external provider adapter, external agent adapter, credential
exposure, raw secret output, raw provider output inline, raw transcript
persistence, Schumpeter split, and LLM judge. Any dangerous nonzero count blocks
release.

response, invokes providers, executes commands, starts background collection,
implements workbench UI, promotes memory, mutates persona, persists raw
transcripts, exposes credentials or raw secrets, inlines raw provider output,

dependency, introduces Schumpeter split, or uses an LLM judge.
