# v0.25.8 Agent Trace / Usability Telemetry

Korean name: Agent Trace·사용성 Telemetry

Track: Bounded General Agent Surface & Internal Tool Routing

Previous: v0.25.7 Ask / REPL Surface

Next: v0.25.9 General Agent Usability Consolidation

## Version Definition

v0.25.8 implements report-derived agent trace and descriptive usability
telemetry for the public ChantaCore agent surface. It reads existing v0.25.7
ask/repl and pipeline artifacts, projects them into OCEL-style trace views, and
computes bounded usability metrics from those reports.

## Relationships

v0.25.7 remains the ask/repl and surface emission owner. v0.25.8 does not run
ask/repl and does not emit final responses.

v0.25.6 remains the response assembly and evidence binding owner. v0.25.8 only
observes assembled response references.

v0.25.5 remains the internal provider invocation owner. v0.25.8 does not invoke
providers.

v0.25.4 remains the route plan and provider selection owner. v0.25.3 remains
the safety/no-action/clarification gate owner. v0.25.2 remains intent/task
framing. v0.25.1 remains turn envelope/context. v0.25.0 remains the agent
surface contract. v0.24.9 remains the internal provider foundation.

## Roadmap Policy

v0.25.x is Bounded General Agent Surface & Internal Tool Routing. v0.26.x is
Workspace Agent Workbench. v0.27.x is Memory Candidate & Continuity. v0.28.x is
Public Alpha / Schumpeter Split Preparation. v0.29.x+ is External Skill /
External Provider Adapter Development. v0.30.x+ is External Agent Dominion
Bridge.

ChantaCore directly implements its OCEL-native agent spine. OpenCode,
OpenClaw, and Hermes remain reference architectures only. v0.25.8 does not add
runtime dependencies, adapters, bridges, fork-specific integrations, or copied
runtime logic for them.

## Principles

- Trace recording is not agent execution.
- Telemetry is not workspace workbench.
- Telemetry is not memory promotion.
- Trace is not raw transcript storage.
- Metric is not optimization by itself.
- OCEL projection is not mutation of original artifacts.
- Usability telemetry is descriptive, not autonomous control.
- Ask/repl execution remains v0.25.7.
- Workspace workbench is deferred to v0.26.
- Memory continuity is deferred to v0.27.

## Service Design

AgentTracePrerequisiteSourceService loads references to already existing
ask/repl artifacts. It does not run the ask/repl service or the v0.25 pipeline.
AgentTraceSourceBundleService creates a reference-only source bundle.
AgentSurfaceTraceService builds stage, decision, route, provider, and response
emission trace views. AgentTurnOCELProjectionService projects those views into
OCEL type/event/relation lists without mutating original artifacts.
AgentUsabilityMetricComputationService computes descriptive metrics from the
source report and trace. AgentTraceTelemetryReportService assembles the final
trace telemetry report.

## Status Rules

Passed means an ask/repl report source exists, source refs are bundled, a
surface trace is created, OCEL projection is created, usability metrics are
computed, a telemetry report is created, and no new execution or raw output
persistence occurs.

Warning means a source bundle is partial, a stage trace is missing, provider
trace is partial, latency data is unavailable, or response emission trace is
missing while the trace remains usable.

Failed means a strict request lacks a required source or trace/projection/
metric computation cannot be created.

Blocked means ask/repl execution, provider invocation, local command execution,
direct file/subprocess access, background daemon, continuous watcher,
autonomous optimization, workspace workbench, memory/persona mutation, external
adapter use, raw transcript persistence, credential exposure, raw secret output,
raw provider output inline, Schumpeter split, or LLM judge use is detected.

## Acceptance Criteria

- AgentTraceTelemetryReport can be created.
- AgentSurfaceTrace can be created from a v0.25.7 ask/repl report.
- AgentTurnOCELProjection can be created.
- AgentUsabilityMetricSet can be computed.
- AgentUsabilityTelemetryReport can be created.
- Trace is report-derived.
- Telemetry is descriptive only.
- No ask/repl execution, final response emission, provider invocation, local
  command execution, background daemon, continuous watcher, autonomous
  optimization, workspace workbench UI, memory promotion, persistent memory
  write, persona mutation, external adapter, OpenCode/OpenClaw/Hermes runtime
  dependency, Schumpeter split, credential output, raw transcript persistence,
  raw provider output, raw secret output, or LLM judge is introduced.

## Withdrawal Conditions

Withdraw v0.25.8 readiness if trace/telemetry runs ask/repl, emits responses,
invokes providers, executes commands, starts background collection, performs
autonomous optimization, implements workspace workbench UI, promotes memory,
mutates persona, persists raw transcripts, exposes credentials or raw secrets,
inlines raw provider output, introduces external adapter logic, introduces
Schumpeter split logic, or uses an LLM judge.

## Validity Horizon

Valid until v0.25.9 consolidation begins or trace/telemetry policy changes.
