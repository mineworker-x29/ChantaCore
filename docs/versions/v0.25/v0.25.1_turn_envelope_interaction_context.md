# v0.25.1 Turn Envelope & Interaction Context

Korean name: Turn Envelope·상호작용 Context

Track: Bounded General Agent Surface & Internal Tool Routing

Previous: v0.25.0 Agent Surface Contract

Next: v0.25.2 Intent Classification & Task Framing

## Version Definition

v0.25.1 implements the turn envelope and conversation-local interaction context
layer for the bounded agent surface. It creates request, context, envelope, and
trace records only. It does not execute ask/REPL, classify intent, frame tasks,
evaluate a safety gate, route tools, invoke providers, execute local commands,
or promote memory.

## Objective

The objective is to turn a user request into a sanitized
`AgentUserRequestView`, a conversation-local `AgentRequestContextView`, an
`AgentTurnContext`, an `AgentTurnEnvelope`, and an OCEL-visible
`AgentTurnTrace`. The resulting `AgentTurnReport` can be used by v0.25.2 as an
input for future intent classification.

## Relationship To Earlier Versions

v0.25.1 builds on the v0.25.0 Agent Surface Contract. It preserves the
v0.24.9 Internal Provider / Local Runtime Provider Foundation v1 and the
v0.23.9 OCEL-native Internal Dominion Foundation v1 as prerequisites.

v0.25.x remains the Bounded General Agent Surface & Internal Tool Routing
track. v0.25.1 is envelope/context-only.

## Long-Term Direction

ChantaCore directly implements its own OCEL-native agent spine.
OpenCode, OpenClaw, and Hermes remain reference architectures only:

- OpenCode patterns may inform future session, tool registry, permission, and
  provider abstraction design.
- Hermes patterns may inform future isolation, delegation, and bounded memory
  design.
- OpenClaw patterns may inform future assistant UX and external-agent risk
  handling.

No runtime dependency, adapter, bridge, fork-specific integration, or copied
runtime logic for these projects is introduced in v0.25.1.

## Service Design

The v0.25.1 services are deterministic:

- `AgentTurnEnvelopePolicyService` builds an envelope-only policy.
- `AgentInteractionSessionService` creates conversation-local sessions only.
- `AgentTurnSourceDescriptorService` describes sanitized request sources.
- `AgentUserRequestSanitizationService` detects and redacts secret-like,
  credential-like, and private-path-like input before durable views are built.
- `AgentContextBoundaryPolicyService` defines conversation-local context limits.
- `AgentContextRefService` creates, filters, and sanitizes context references.
- `AgentRequestContextViewService` builds bounded context views.
- `AgentTurnContextService` builds a turn context without intent, routing, or
  provider invocation.
- `AgentTurnEnvelopeService` creates the turn envelope.
- `AgentTurnTraceService` creates OCEL-visible trace references.
- `AgentTurnFindingService` reports warnings and blocked premature behavior.
- `AgentTurnReportService` assembles the report.

## Request Sanitization

Raw user request text is not stored. The request view stores only sanitized
request text and a bounded excerpt. Secret-like, credential-like, and
private-path-like input produces redaction findings. Raw secrets and raw
credentials are not persisted.

## Context Boundary

Context scope is `conversation_local`. Context references may point to
workspace, repository, process, local runtime, provider registry, prior turn,
agent surface contract, or user-supplied context records, but raw provider
output, raw secrets, and private full paths are not inlined. Persistent memory
read/write and memory promotion remain disabled.

## Status Rules

Passed means the envelope, context view, trace, and report are created without
premature execution or persistence hazards. Warning is used for sanitized or
truncated views. Failed is used for missing required request data or missing
strict prerequisites. Blocked is used for premature ask/REPL, intent
classification, safety gate, routing, provider invocation, command execution,
memory promotion, persistent memory write, persona mutation, external adapter,
Schumpeter split, raw secret persistence, credential exposure, or LLM judge.

## Acceptance Criteria

- `AgentTurnEnvelope` can be created.
- `AgentInteractionSession` can be created as conversation-local or ephemeral.
- `AgentUserRequestView` sanitizes request text.
- `AgentRequestContextView` can be created without persistent memory.
- `AgentTurnContext` can be created.
- `AgentTurnTrace` is OCEL-visible.
- `AgentTurnReport` can be created.
- `ready_for_v0_25_2` can be true.
- `ready_for_v0_26` remains false.
- No ask/REPL execution occurs.
- No intent classification occurs.
- No safety gate occurs.
- No tool routing occurs.
- No provider invocation occurs.
- No local command execution occurs.
- No memory promotion or persistent memory write occurs.
- No persona mutation occurs.
- No external provider or external agent adapter is implemented.
- No OpenCode/OpenClaw/Hermes runtime dependency is added.
- No Schumpeter split is introduced.
- No credential value is stored or output.
- No LLM judge is used.

## Withdrawal Conditions

This record must be withdrawn if v0.25.1 starts ask/REPL, classifies intent,
frames tasks, evaluates a safety gate, routes tools, invokes providers,
executes commands, writes persistent memory, promotes memory, mutates persona,
implements external adapters, adds OpenCode/OpenClaw/Hermes runtime
dependencies, introduces Schumpeter split logic, stores raw secrets, exposes
credentials, or uses an LLM judge.

## Validity Horizon

Valid until v0.25.2 intent classification begins or turn/context policy
changes.
