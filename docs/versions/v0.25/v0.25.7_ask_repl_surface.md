# v0.25.7 Ask / REPL Surface

Korean name: Ask·REPL 사용자 표면

Track: Bounded General Agent Surface & Internal Tool Routing

Previous: v0.25.6 Response Assembly & Evidence Binder

Next: v0.25.8 Agent Trace / Usability Telemetry

## Version Definition

v0.25.7 implements the bounded, synchronous user-facing ask and REPL surface
for the public ChantaCore agent surface. It may run one explicit user turn
through the v0.25 pipeline and emit an assembled response to the user surface.

## Relationships

v0.25.6 creates AgentAssembledResponse and AgentResponseAssemblyReport.
v0.25.7 may emit only those assembled response artifacts.

v0.25.5 remains the only stage that can invoke registered internal providers.
v0.25.7 does not directly invoke providers.

v0.25.4 remains the route plan and provider selection owner. v0.25.7 does not
route outside that stage.

v0.25.3 remains the safety, no-action, clarification, blocked, and deferred
gate owner. v0.25.7 does not replace safety decisions.

v0.25.2 remains the intent classification and task framing owner. v0.25.1
remains the turn envelope and interaction context owner. v0.25.0 remains the
agent surface contract. v0.24.9 remains the internal provider foundation.

## Roadmap Policy

v0.25.x is Bounded General Agent Surface & Internal Tool Routing. v0.26.x is
Workspace Agent Workbench. v0.27.x is Memory Candidate & Continuity. v0.29.x+
is External Skill / External Provider Adapter Development. v0.30.x+ is
External Agent Dominion Bridge.

ChantaCore directly implements its OCEL-native agent spine. OpenCode,
OpenClaw, and Hermes remain reference architectures only. v0.25.7 does not add
runtime dependencies, adapters, bridges, fork-specific integrations, or copied
runtime logic for them.

## Principles

- Ask surface is not autonomous execution.
- REPL is user-driven, not self-driven.
- One user turn creates at most one bounded pipeline run.
- Surface emission is not response assembly.
- Surface emission must use an assembled response from v0.25.6.
- Ask/REPL must not bypass stage boundaries.
- Provider invocation must occur only through v0.25.5.
- Local runtime execution must remain inside the v0.24 gated provider boundary.
- No-action, clarification, blocked, and deferred are valid emitted responses.
- Agent trace and usability telemetry are deferred to v0.25.8.

## Non-Goals

v0.25.7 does not implement autonomous loops, background execution,
self-prompting, automatic rerun, automatic repair, direct provider bypass,
direct file read, direct repository search, direct process inspection, command
string execution, local command execution, file mutation, memory continuity,
memory promotion, persistent memory write, persona mutation, Workspace Agent
Workbench, external provider adapters, external agent adapters, Schumpeter split
logic, company-specific wrappers, network fetch, plugin loading, credential
storage/output, raw provider output dump, raw secret output, or LLM judge logic.

## Service Design

AgentAskPipelineRunnerService calls existing v0.25 stage services in order. It
records skipped route/provider stages for non-route outcomes and keeps one
pipeline run per user turn. AgentSurfaceEmissionService emits only a v0.25.6
AgentAssembledResponse. AgentReplSessionService starts sessions and processes
explicit user turns without self-triggering follow-up turns. Command history is
sanitized and non-persistent in this version.

## Status Rules

Passed means an ask request or REPL turn creates one bounded pipeline run,
obtains or uses a v0.25.6 assembled response, emits it, and avoids direct
provider/file/process/subprocess/local command bypasses.

Warning means the emitted response is no-action, clarification, blocked,
deferred, truncated, or includes safe stage warnings.

Failed means the assembled response or emission cannot be created without a
dangerous bypass.

Blocked means autonomous execution, background execution, self-prompting,
direct provider or command bypass, memory/persona mutation, external adapter
use, credential exposure, raw secret output, raw provider output inline,
Schumpeter split, or LLM judge use is detected.

## Acceptance Criteria

- AgentAskReplReport can be created.
- Agent ask can emit an existing v0.25.6 assembled response.
- Agent ask can run the full v0.25 bounded pipeline from raw text.
- REPL session can process explicit user turns.
- Final response emission is allowed only from v0.25.6 assembled response.
- No autonomous loop, background execution, or self-prompting occurs.
- No direct provider invocation, direct file access, direct subprocess/local
  command execution, command rerun, or automatic repair occurs.
- No memory promotion, persistent memory write, persona mutation, external
  provider/agent adapter, OpenCode/OpenClaw/Hermes runtime dependency,
  Schumpeter split, credential output, raw provider output, raw secret output,
  or LLM judge is introduced.

## Withdrawal Conditions

Withdraw v0.25.7 readiness if ask/repl self-prompts, starts background work,
creates autonomous follow-up turns, bypasses v0.25 stages, directly invokes
providers, directly reads files, executes commands, promotes memory, mutates
persona, emits raw secrets, emits raw provider output, introduces external
adapter logic, introduces Schumpeter split logic, or uses an LLM judge.

## Validity Horizon

Valid until v0.25.8 trace/telemetry begins or ask/repl surface policy changes.
