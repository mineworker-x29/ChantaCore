# v0.25.3 Safety / No-Action / Clarification Gate

Korean name: 안전·무행동·명확화 Gate

Track: Bounded General Agent Surface & Internal Tool Routing

## Version Definition

v0.25.3 implements the deterministic safety, no-action, clarification,
needs-more-input, blocked, deferred, and allow-route gate for the bounded agent
surface. It consumes v0.25.2 intent classification and task framing outputs and
produces an AgentSafetyGateReport.

## Objective

The objective is to decide whether a framed request may proceed to v0.25.4 Tool
Routing Plan & Provider Selection, or whether v0.25.6 Response Assembly &
Evidence Binder should explain a no-action, clarification, needs-more-input,
blocked, deferred, or failed outcome.

## Relationships

v0.25.2 provides AgentIntentClassificationReport, AgentTaskFrameCandidate, and
AgentTaskRiskPreview. v0.25.1 provides AgentTurnEnvelope and AgentTurnReport.
v0.25.0 provides the Agent Surface Contract. v0.24.9 remains the Internal
Provider / Local Runtime Provider Foundation reference source.

The v0.25.x policy remains Bounded General Agent Surface & Internal Tool
Routing. v0.26.x is Workspace Agent Workbench. v0.27.x is Memory Candidate &
Continuity. v0.29.x+ is External Skill / External Provider Adapter Development.
v0.30.x+ is External Agent Dominion Bridge.

## Long-Term Direction

ChantaCore directly implements its own OCEL-native agent spine. OpenCode,
OpenClaw, and Hermes remain reference architectures only. v0.25.3 adds no
runtime dependency, adapter, bridge, fork-specific integration, or copied
runtime logic for those projects.

## Core Principles

- Safety gate is not tool routing.
- No-action is a valid agent decision.
- Clarification is a valid agent decision.
- Blocked decision is a policy outcome, not an error.
- Allow-route is not provider invocation.
- Allow-route is not execution authorization.
- User request is not permission.
- LLM proposal is not safety judgment.
- Tool routing is deferred to v0.25.4.
- Provider invocation is deferred to v0.25.5.
- Response assembly for non-route outcomes is deferred to v0.25.6.
- Ask/REPL is deferred to v0.25.7.
- Memory continuity is deferred to v0.27.

## Non-Goals

v0.25.3 does not create tool routing plans, select providers, invoke providers,
execute local commands, execute ask/REPL, perform command reruns, mutate files,
promote memory, write persistent memory, mutate persona, implement workspace
workbench, implement external provider or external agent adapters, introduce
Schumpeter split logic, expose credentials or raw secrets, or use an LLM judge.

## Service Design

The service layer is deterministic and rule-based by default:

- AgentSafetyGatePrerequisiteSourceService loads prior read models when
  available.
- AgentSafetyGatePolicyService builds the gate policy.
- AgentSafetyRuleRegistry lists required safety rules.
- AgentSafetyRuleEngine evaluates rule outcomes from the task frame candidate
  and risk preview.
- AgentSafetyGateDecisionService produces the primary outcome.
- AgentNoActionDecisionService, AgentClarificationDecisionService,
  AgentNeedsMoreInputDecisionService, AgentBlockedDecisionService, and
  AgentDeferredDecisionService create optional outcome records.
- AgentGateOutcomeEnvelopeService wraps the outcome without routing or
  provider invocation.
- AgentSafetyGateReportService assembles the report.

## Acceptance Criteria

- AgentSafetyGateReport can be created.
- AgentSafetyGateDecision can be created.
- no_action, clarification, needs_more_input, blocked, deferred, and allow_route
  outcomes can be produced.
- allow_route leads to v0.25.4, not provider invocation.
- non-route outcomes lead to v0.25.6 response assembly.
- Safety gate is deterministic and rule-based by default.
- No LLM judge is used.
- No tool routing plan is created.
- No provider selection or invocation occurs.
- No local command execution, ask/REPL execution, memory promotion, persistent
  memory write, persona mutation, external adapter, OpenCode/OpenClaw/Hermes
  runtime dependency, Schumpeter split, or credential output is added.

## Withdrawal Conditions

Withdraw v0.25.3 readiness if the implementation creates a tool route plan,
selects or invokes a provider, executes a local command, executes ask/REPL,
promotes or writes memory, mutates persona, adds an external adapter, adds an
OpenCode/OpenClaw/Hermes runtime dependency, introduces Schumpeter split logic,
stores or outputs credentials/raw secrets, or uses an LLM judge.

## Validity Horizon

Valid until v0.25.4 routing begins or safety/no-action/clarification policy
changes.
