# v0.25.4 Tool Routing Plan & Provider Selection

Korean name: Tool Routing 계획·Provider 선택

Track: Bounded General Agent Surface & Internal Tool Routing

## Version Definition

v0.25.4 implements deterministic tool route planning and provider selection
only. It consumes v0.25.3 AgentSafetyGateReport and AgentGateOutcomeEnvelope,
then creates an AgentToolRoutingReport and AgentToolRoutePlan when the gate
outcome is allow_route.

## Objective

The objective is to map an allowed agent task to a route kind, view internal
provider capabilities, select candidate providers, define route steps and
dependencies, and mark the route as ready for v0.25.5 provider invocation or
v0.25.6 response assembly.

## Relationships

v0.25.3 provides the safety/no-action/clarification gate. v0.25.2 provides
intent classification and task framing. v0.25.1 provides turn envelope and
conversation-local interaction context. v0.25.0 provides the Agent Surface
Contract. v0.24.9 provides the Internal Provider / Local Runtime Provider
Foundation v1. v0.24.1 provider registry and capability surface remain
read-model references when available.

The v0.25.x policy remains Bounded General Agent Surface & Internal Tool
Routing. v0.26.x is Workspace Agent Workbench. v0.27.x is Memory Candidate &
Continuity. v0.28.x is Public Alpha / Schumpeter Split Preparation. v0.29.x+
is External Skill / External Provider Adapter Development. v0.30.x+ is
External Agent Dominion Bridge.

## Long-Term Direction

ChantaCore directly implements its own OCEL-native agent spine. OpenCode,
OpenClaw, and Hermes remain reference architectures only. v0.25.4 adds no
runtime dependency, adapter, bridge, fork-specific integration, or copied
runtime logic for those projects.

## Core Principles

- Tool route plan is not provider invocation.
- Provider selection is not provider execution.
- Route step is not action execution.
- Route dependency is not execution ordering by itself.
- Allow-route is not provider permission.
- Provider invocation is deferred to v0.25.5.
- Response assembly is deferred to v0.25.6.
- Ask/REPL surface is deferred to v0.25.7.
- Local runtime execution must go through the v0.24 gate sequence.
- External provider adapters are deferred to v0.29+.
- External agent dominion is deferred to v0.30+.

## Included Scope

v0.25.4 includes AgentToolRoutingPolicy, AgentToolRoutingRequest,
AgentProviderCapabilityCatalogView, AgentProviderCapabilityRef,
AgentRouteIntentMapping, AgentProviderSelectionCandidate,
AgentProviderSelection, AgentRoutePrecondition, AgentToolRouteStep,
AgentToolRouteDependency, AgentRouteRiskReview, AgentToolRoutePlan,
AgentToolRoutingFinding, and AgentToolRoutingReport.

## Non-Goals

v0.25.4 does not invoke providers, execute route steps, execute local commands,
perform bounded local command execution, execute ask/REPL, assemble responses,
mutate files, promote memory, write persistent memory, mutate persona,
implement workspace workbench, implement external provider or external agent
adapters, introduce Schumpeter split logic, expose credentials or raw secrets,
or use an LLM judge.

## Service Design

The service layer is deterministic and rule-based by default:

- AgentToolRoutingPrerequisiteSourceService loads prior read models.
- AgentToolRoutingPolicyService builds the routing policy.
- AgentProviderCapabilityCatalogService builds a v0.24 provider capability
  catalog view.
- AgentRouteIntentMappingService maps intent categories to route kinds.
- AgentProviderSelectionCandidateService creates provider candidates.
- AgentProviderSelectionService selects required providers without invoking
  them.
- AgentRoutePreconditionService builds route preconditions.
- AgentToolRouteStepService builds planned steps.
- AgentToolRouteDependencyService builds dependency records.
- AgentRouteRiskReviewService reviews route risk without execution.
- AgentToolRoutePlanService creates the route plan.
- AgentToolRoutingReportService assembles the report.

## Provider Capability Catalog

The catalog view includes workspace_read_provider, repository_search_provider,
file_read_provider, ocel_inspection_provider, pig_inspection_provider,
ocpx_projection_provider, local_runtime_provider, diagnostic_provider, and
candidate_generation_provider references. All v0.25.4 capability refs have
invocation_enabled_in_v025_4=false and executable_in_v025_4=false. External
provider adapter count is zero.

## Route Mappings

Route mappings cover general_answer, workspace_overview, repository_search,
file_read, process_state_inspection, local_runtime_candidate,
local_runtime_execution_request, diagnostic_request, explanation_request,
planning_request, architecture_design, implementation_prompt_request,
verification_prompt_request, checklist_request, consolidation_request,
no_action_candidate, needs_more_input_candidate, blocked_candidate, and unknown.

## Status Rules

Provider-backed planned routes are ready for v0.25.5 Internal Provider
Invocation Orchestrator. Response-only and no_route_due_to_gate outcomes are
ready for v0.25.6 Response Assembly & Evidence Binder. ready_for_v0_26 remains
false.

## OCEL Mapping

v0.25.4 adds object, event, relation, and effect constants for tool routing
policy, provider capability catalog view, provider selection, route steps,
route dependencies, route risk review, route plan, and route report. OCEL
remains the canonical substrate; JSONL is not introduced as a canonical store.

## CLI Commands

- `python -m chanta_core.cli.main agent route plan --text "<request>"`
- `python -m chanta_core.cli.main agent route plan --safety-report-id <id>`
- `python -m chanta_core.cli.main agent route providers`
- `python -m chanta_core.cli.main agent route mappings`
- `python -m chanta_core.cli.main agent route report --report-id <id>`
- `python -m chanta_core.cli.main agent route findings --report-id <id>`

CLI output is sanitized and reports no route execution, provider invocation,
local command execution, ask/REPL execution, response assembly, memory
promotion, persona mutation, credential exposure, raw secret output, or LLM
judge use.

## PIG/OCPX

The PIG subject is tool_routing_plan_provider_selection. The OCPX state is
agent_tool_route_plan_created. Effect types are read_only_observation,
agent_tool_route_plan_created, agent_provider_selection_created,
agent_route_risk_review_created, and state_candidate_created.

## Acceptance Criteria

- AgentToolRoutingReport can be created.
- AgentToolRoutePlan can be created from an allow_route safety gate outcome.
- AgentProviderSelection can be created.
- Provider capability catalog can view v0.24 providers.
- Route mappings exist for major v0.25 intent categories.
- Route steps and dependencies are planned.
- Local runtime execution flow preserves v0.24 candidate/safety/gate/output
  sequence.
- Provider invocation is not performed.
- Route execution is not performed.
- Local command execution is not performed.
- Ask/REPL execution is not performed.
- Response assembly is not performed.
- No memory promotion, persistent memory write, persona mutation, external
  adapter, OpenCode/OpenClaw/Hermes runtime dependency, Schumpeter split,
  credential output, or LLM judge is added.

## Withdrawal Conditions

Withdraw v0.25.4 readiness if the implementation invokes providers, executes
route steps, executes local commands, executes ask/REPL, assembles responses,
promotes or writes memory, mutates persona, adds an external adapter, adds an
OpenCode/OpenClaw/Hermes runtime dependency, introduces Schumpeter split logic,
stores or outputs credentials/raw secrets, or uses an LLM judge.

## Validity Horizon

Valid until v0.25.5 provider invocation begins or routing/provider policy
changes.
