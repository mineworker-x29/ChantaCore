# v0.22.2 Patch Draft / Diff Preview

Korean name: 패치 초안·Diff 미리보기

Track: Self-Modification Safety

Release status: implemented as candidate-only/read-only preview artifacts.

## Version Definition

v0.22.2 introduces the OCEL-native patch draft and diff preview step for the self-modification safety layer.

Patch draft is not patch apply.
Diff preview is not file mutation.
Diff preview is not dry-run.
Preview-created does not mean safety-checked.
Safety-checked does not mean dry-run-checked.
Dry-run-checked does not mean approved.
Approved does not mean applied.

This version creates only bounded, sanitized preview artifacts from an existing patch candidate. It does not write files, call apply_patch, apply patches, run dry-run, approve static safety, approve human review, open an apply gate, execute tests or lint, use an LLM judge, or mutate memory/persona/overlay state.

## Relationship To Earlier Versions

v0.22.0 defines the OCEL-native Self-Modification Safety Contract, lifecycle policy, patch policy, gate contract, and non-executable seed skills.

v0.22.1 creates modification requests and patch candidates. Those candidates remain candidate_only and do not contain patch drafts or diff hunks.

v0.22.2 consumes a patch candidate and creates PatchDraft plus DiffPreviewArtifact read models. It does not change the candidate into an applied state.

v0.21.9 remains the Deep Self-Introspection Foundation v1 consolidation source. v0.20.9 remains the Self-Awareness Foundation v1 consolidation source.

## Data Models

PatchDraftCreateRequest selects a patch_candidate_id, preview format, optional sanitized operation hints, and bounded limits for hunks and line previews.

PatchTargetContextRef records target metadata and whether bounded/provided context is available. It records redacted/truncated flags and does not expose raw full file content.

PatchAnchorRef records a sanitized anchor reference. Ambiguous anchors are findings and can produce needs_more_input.

PatchOperationDraft records an operation type, sanitized old/new text previews, line counts, and disabled safety flags. applies_cleanly remains unknown.

DiffHunkPreview records a sanitized hunk preview. It is a preview-only read model and not proof that the operation applies cleanly.

PatchDraftFinding records policy, context, anchor, operation, and safety-boundary findings.

PatchDraft has lifecycle_state=preview_created and draft_status=draft_only. It keeps static_safety_checked, dry_run_checked, review_approved, apply_gate_opened, applied, file_write_enabled, apply_patch_enabled, execution_enabled, materialized, and promoted set to false.

DiffPreviewArtifact has preview_status=preview_only and applies_cleanly=null. It keeps static_safety_checked, dry_run_checked, applied, file_write_enabled, and apply_patch_enabled set to false.

PatchDraftNoActionCandidate and PatchDraftNeedsMoreInputCandidate are valid candidate-only outcomes and do not apply patches.

## Service Design

PatchDraftPolicyService decides whether the request can create a preview, needs more input, or should create no_action.

PatchDraftSourceService loads a patch candidate from the candidate owner and collects only sanitized/provided target context. CLI rendering may use a synthetic candidate reference for read-only display when no persistence exists.

PatchAnchorCollector creates anchors from sanitized hints and marks ambiguity.

PatchOperationDraftBuilder creates operation drafts only for allowed patch policy types.

DiffPreviewBuilder creates DiffPreviewArtifact and DiffHunkPreview records from PatchOperationDraft records. It does not run apply checks.

PatchDraftFindingService creates findings for missing candidates, non-candidate-only candidates, applied candidates, unavailable context, ambiguous anchors, forbidden operations, scope/line limits, and deferred safety stages.

SelfModificationDiffPreviewService coordinates the read-only flow and exposes PIG/OCPX/CLI renderers.

## Generation Rules

The patch candidate must exist, must have candidate_status=candidate_only, and must not already be applied.

Target context must be provided as sanitized context or collected by a bounded owner. Raw full file content is not an output.

Allowed operation types follow the v0.22.0 patch policy: text_replace, insert_after, append_block, comment_only_change.

Forbidden operation types create findings or no_action. max_hunks, max_added_lines, and max_removed_lines are enforced.

Private or secret target risks block preview creation or produce no_action.

v0.22.2 never claims applies_cleanly=true. It never sets static_safety_checked=true, dry_run_checked=true, review_approved=true, apply_gate_opened=true, or applied=true.

## CLI Commands

- python -m chanta_core.cli self-modification diff preview --candidate-id patch_candidate:test
- python -m chanta_core.cli self-modification diff preview --candidate-id patch_candidate:test --format structured
- python -m chanta_core.cli self-modification patch-draft create --candidate-id patch_candidate:test
- python -m chanta_core.cli self-modification patch-draft view --draft-id patch_draft:test
- python -m chanta_core.cli self-modification diff view --preview-id diff_preview:test
- python -m chanta_core.cli self-modification diff findings --preview-id diff_preview:test

CLI output shows draft_id, preview_id, candidate_id, lifecycle_state=preview_created, static_safety_checked=false, dry_run_checked=false, applies_cleanly=unknown, applied=false, file_write_enabled=false, apply_patch_enabled=false, and next required step=v0.22.3 Patch Static Safety Check.

CLI output must not print private full paths, raw full file content, or raw secrets.
