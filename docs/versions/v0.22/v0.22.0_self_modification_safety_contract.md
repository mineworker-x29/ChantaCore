# v0.22.0 OCEL-native Self-Modification Safety Contract

Korean name: OCEL-native 자기수정 안전 계약

Track: Self-Modification Safety

## Objective

v0.22.0 introduces the contract foundation for the OCEL-native Self-Modification Safety layer. It registers the layer, seed subjects, seed skill contracts, risk profile, gate contract, observability contract, allowed patch policy, lifecycle policy, OCEL mapping skeleton, PIG report skeleton, OCPX projection skeleton, CLI read-only views, and conformance tests.

This version does not enable file write, patch application, patch draft generation, diff generation, dry-run execution, rollback execution, test/lint/shell execution, network access, MCP connection, plugin loading, external harness execution, memory/persona/overlay mutation, LLM patch generation, LLM judge, or autonomous self-editing.

## Relationship To Prior Foundations

- v0.20.9 consolidated OCEL-native Self-Awareness Foundation v1.
- v0.21.9 consolidated OCEL-native Deep Self-Introspection Foundation v1.
- v0.22.0 depends on those foundations as read-only source truth and starts a separate Self-Modification Safety track.

## Core Principles

- Self-modification safety is not self-modification.
- Patch candidate is not patch apply.
- Diff preview is not file mutation.
- Review approval is not execution.
- Apply requires explicit apply gate.
- Rollback plan is required before mutation.
- Post-apply verification is mandatory after mutation.

## Included Scope

v0.22.0 defines contract-only read models for modification request, patch candidate, diff preview, static safety, dry-run record, review gate, apply gate, rollback plan, post-apply verification, and modification outcome.

All seed skills are contract-only, stub, and non-executable.

## Non-Goals

v0.22.0 does not implement actual file write, actual patch application, patch draft generation, diff generation, dry-run execution, rollback execution, test/lint/shell execution, network/web fetch, MCP connection, plugin loading, external harness execution, memory/persona/overlay mutation, capability registry mutation, policy/gate mutation, LLM patch generator activation, LLM judge, or autonomous self-edit loop.

- `SelfModificationReportService` builds contract report, PIG report, OCPX projection, and sanitized CLI views.

## Lifecycle Policy

States are defined as requested, candidate_created, preview_created, safety_checked, dry_run_checked, pending_review, approved_for_apply, apply_gate_opened, applied, post_apply_verified, outcome_recorded, blocked, rejected, no_action, and needs_more_input.

The `apply_gate_opened -> applied` transition is defined as a future transition and is not executable in v0.22.0.

## Patch Policy

Allowed patch types are bounded text operations: text_replace, insert_after, append_block, and comment_only_change.

Forbidden patch types include binary_patch, broad_rewrite, delete_file, rename_file, chmod_change, dependency_install, secret_file_change, and generated_file_mass_update.

Patch scope limits are one file, three hunks, eighty added lines, and eighty removed lines. Anchor text and workspace path policy are required. Binary files, secret files, and private paths are not allowed.

## OCEL Mapping

The OCEL mapping skeleton includes self-modification safety layer, contract, subject, skill contract, risk profile, gate contract, observability contract, patch policy, lifecycle policy, contract report, seed subject object types, deep-self consolidation report, capability/runtime/policy/trace/candidate-memory reports, execution envelope, PIG report, and OCPX projection.

The effect types are read_only_observation and state_candidate_created. `workspace_file_changed` is future/deferred and not active in v0.22.0.
