# v0.23.2 Capability Observation & Digestion for Dominion

Version: v0.23.2
Version name: Capability Observation & Digestion for Dominion
Korean name: 지배 대상 Capability 관찰·소화
Track: Internal Dominion Foundation
Layer: internal_dominion

## Objective

v0.23.2 implements declarative capability observation and digestion for Internal Dominion. It reads v0.23.1 runtime inventory plus declared/static/operator-provided capability descriptors, then normalizes them into provider-neutral `ExternalCapabilityCandidate` objects.

## Relationship

v0.23.0 defines the Internal Dominion contract. v0.23.1 declares runtime, agent, tool, system, and control-surface inventory. v0.23.2 observes and digests declared capability descriptors over that inventory.

## Policy Reminder

v0.23.x is Internal Dominion Foundation. v0.23.x is not Self-Execution Safety. Self-Execution Safety is reclassified to v0.24.x Local Runtime Provider. Vendor systems are future external provider skills/provider adapters. GrowthKernel is a future consumer / future optimizer, not an active runtime dependency.

## Principles

- Capability observation is not provider API discovery.
- Capability digestion is not execution.
- Capability candidate is not control request.
- Capability candidate is not dispatch.
- Every capability must be risk-classified before control planning.

## Included Scope

Data models include `DominionCapabilityObservationRequest`, `CapabilityObservationSource`, `ExternalCapabilityDescriptor`, `DominionActionVerbDescriptor`, `CapabilitySchemaDescriptor`, `CapabilityBoundaryDescriptor`, `CapabilityRiskProfile`, `CapabilityDigestionRule`, `CapabilityDigestionResult`, `ExternalCapabilityCandidate`, `CapabilityObservationFinding`, `CapabilityObservationDigestSnapshot`, and `CapabilityObservationDigestReport`.

Services include source loading, descriptor parsing, capability normalization, action verb normalization, schema metadata creation, boundary creation, risk classification, digestion rules, candidate creation, finding generation, and report rendering.

## Non-Goals

This release does not perform provider API capability discovery, external runtime touch, external control dispatch, external run start, capability execution, RPA or agent runtime API calls, local command execution, network fetch, MCP connection, plugin loading, credential/token reading, credential/token storage/output, control request creation, action candidate creation, control plan creation, preflight execution, human gate creation, GrowthKernel runtime dependency, or vendor-specific adapter implementation.

## Risk Rules

Read-only verbs such as observe, list, describe, inspect, status, get, and read classify as `read_only`. Trigger/start/run/update/create/assign/approve/reject classify as future-gated mutating or high-risk actions. Stop/cancel classify as high risk. Delete/remove/destroy classify as destructive. Unknown verbs create warnings. Production-impacting, destructive, and credential-sensitive capabilities require future gate/preflight before planning.

Schema descriptors store metadata only. Sensitive fields may be listed as metadata, but credential values are not stored or output.

## PIG/OCPX

PIG uses `version=v0.23.2`, `layer=internal_dominion`, and `subject=capability_observation_digestion`. It records no dispatch, no control request, no control plan, no provider API call, no external runtime touch, no credential exposure, and no raw secret output.

## Acceptance Criteria

Capability observation/digestion report builds. External capability descriptors are normalized. External capability candidates are created. Input/output schema and risk/boundary descriptors are created. Production/destructive/credential-sensitive risks are reported. No provider API call, external runtime touch, control request, control plan, dispatch, or credential output occurs.

## Withdrawal Conditions

Withdraw if provider API discovery, runtime touch, dispatch, control request creation, control plan creation, preflight, credential value output, GrowthKernel active dependency, or vendor-specific core adapter logic is introduced.

## Validity Horizon

Valid until declared capability descriptors, inventory, provider registry, or company system landscape changes.
