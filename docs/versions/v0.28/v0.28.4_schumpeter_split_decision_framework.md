# v0.28.4 Schumpeter Split Decision Framework

Korean name: Schumpeter Split 의사결정 프레임워크  
Track: Public Alpha / Schumpeter Split Preparation  
Previous: v0.28.3 Public-Private Boundary / Redaction / Reference Policy  
Next: v0.28.5 Schumpeter Split Preparation Profile

## Objective

v0.28.4 creates the Schumpeter split decision framework for `references/schumpeter`. It inventories metadata, reviews license/private risk, compares architecture and capabilities, scores options, records reuse dispositions, recommends a safe default, and creates an audit trail. It does not implement a split.

## Relationships

v0.28.3 provides the public/private boundary, reference governance, license boundary, package exposure boundary, and release gate inputs. v0.28.4 consumes those signals. Unknown Schumpeter license or private-risk status is not safe.

v0.28.5 may consume this decision framework to prepare a Schumpeter private overlay/profile boundary. v0.28.5 still must not implement company wrapper runtime unless a later explicit gate authorizes it.

## Principles

- Schumpeter split decision framework is not Schumpeter split.
- Reference inventory is not adoption.
- Reference evaluation is not code migration.
- Reuse disposition is not runtime dependency.
- Concept reuse is not code copy.
- Test reuse is not production integration.
- Documentation reuse is not private-data exposure.
- Private overlay preparation is not company wrapper runtime.
- Unknown license status is not safe.
- Unknown private-data risk is not safe.
- No split, defer split, and keep reference-only are valid outcomes.
- Public ChantaCore must not depend on private Schumpeter artifacts.

## Decision Policy

The runtime policy enables decision framework, reference inventory, capability comparison, license review, private-risk review, public/private boundary review, reuse disposition, decision records, and audit. It disables actual split, company wrapper, private distribution runtime, private repo creation, merge into public core, reference runtime dependency, reference code copy, file movement, destructive redaction, package publish, release tag creation, external adapters, provider invocation, command execution, and runtime continuity injection.

The default posture is `keep_reference_only_and_prepare_private_overlay`.

## Source View And Inventory

The source view inspects `references/schumpeter` metadata only. It separates reference root, docs, models/code-like files, tests, config, and runtime-like metadata refs. It does not expose raw contents.

The inventory creates SchumpeterReferenceArtifact records with artifact kind, origin status, license status, private/company risk, reuse values, compatibility placeholders, and recommended initial disposition. Runtime dependency and code copy are false for every artifact.

## Reviews And Comparisons

License review treats unknown and incompatible licenses as blockers for reuse/public-core adoption. Private risk review treats unknown private data risk as unsafe, keeps public-core adoption blocked, and routes the subject to private-overlay-only or quarantine review.

Architecture comparison covers OCEL trace, reflective substrate, memory, workbench, provider boundary, skill boundary, runtime loop, safety boundary, and governance boundary. Capability comparison rows capture ChantaCore status, Schumpeter reference status, overlap, reuse value, risk, and disposition.

OCEL/PIG/OCPX and Workbench/Memory compatibility reviews support concept reuse and future private overlay discussion, not direct code adoption.

## Reuse, Risk, Options, And Criteria

Reuse value assessments can recommend concept-only, docs-only, tests-later, model-later, private-overlay-later, reference-only, discard, quarantine, or unknown modes. They never add runtime dependency or copy code.

Risk assessments include IP/license risk, private data risk, public-core contamination risk, runtime dependency risk, credential exposure risk, and maintainability risk. Split implementation is blocked now.

The option catalog includes no_split, reference_only, public_core_private_overlay, private_distribution_profile, separate_private_repo, merge_schumpeter_into_core, and deprecate_legacy_schumpeter. High-risk implementation options remain blocked.

Decision criteria cover IP/license risk, company private data contamination risk, public-core contamination risk, architecture overlap, code/concept/test/doc reuse value, OCEL/PIG/OCPX compatibility, memory/workbench compatibility, runtime dependency risk, maintainability, testability, deployment boundary clarity, credential exposure risk, future external adapter compatibility, and organizational usefulness.

## Disposition And Recommendation

Reuse disposition policy allows adopt_concept, port_model_later, port_test_later, port_document_later, keep_reference_only, quarantine_due_to_license_or_private_risk, and discard. Disposition decisions are not migration.

The safe default recommendation is `keep_reference_only_and_prepare_private_overlay`. The decision record records keep-reference-only and prepare-private-overlay decisions without split implementation, company wrapper, runtime dependency, code copy, or file movement.

## Audit

The audit trail records request, source view, inventory, license reviews, private-risk reviews, capability matrix, option assessments, disposition decisions, and decision records. Raw content is not included.

## OCEL / PIG / OCPX

Allowed effects are read-only observation, Schumpeter split decision framework creation, reference inventory, license review, private-risk review, capability comparison, reuse disposition decision, recommendation, decision record, audit, and state candidate creation.

Forbidden effects include Schumpeter split implementation, company wrapper, private repo creation, merge into public core, reference runtime dependency, reference code copy, file movement, destructive redaction, package publish, release tag creation, external adapters, provider invocation, command execution, runtime continuity injection, private exposure, credential/secret exposure, raw trace/transcript/provider output exposure, PIG execution authority, and LLM judge use.

The PIG subject is `schumpeter_split_decision_framework`. The OCPX state is `schumpeter_split_decision_framework_created`.

## Non-goals

v0.28.4 does not split Schumpeter, implement company wrapper, create a private repo, merge Schumpeter into public core, copy code, add runtime dependency, move files, destructively redact files, publish packages, create release tags, implement adapters, invoke providers, execute runtime commands, expose private/company/raw/credential material, enable PIG execution authority, or use an LLM judge as sole authority.

## Acceptance Criteria

The policy, request, source view, inventory, artifacts, license/private risk reviews, architecture and capability comparisons, compatibility reviews, reuse/risk assessments, option catalog, option assessments, criteria, scores, disposition candidates/decisions, recommendation, decision records, audit trail, findings, and report can be created. Unknown license/private risk is not safe. The recommendation remains `keep_reference_only_and_prepare_private_overlay` unless stronger reviewed evidence appears.
