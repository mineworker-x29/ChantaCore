# v0.28.2 Packaging / Distribution / Type Boundary

- Version: `v0.28.2`
- Name: Packaging / Distribution / Type Boundary
- Korean name: Packaging·Distribution·Type Boundary
- Track: Public Alpha / Schumpeter Split Preparation
- Previous: `v0.28.1` Release Hygiene / Repository Governance Blocking Gate
- Next: `v0.28.3` Public-Private Boundary / Redaction / Reference Policy

## Objective

`v0.28.2` defines packaging, distribution, dependency, type-marker,
package-data, build-smoke, import-smoke, and CLI-smoke boundaries. It may report
package distribution readiness, but it must not publish a package, create a
release tag, or claim public alpha readiness by itself.

## Principles

- Packaging boundary is not package publishing.
- Distribution readiness is not release readiness.
- Build smoke is not release artifact publication.
- Import smoke is not runtime execution expansion.
- CLI smoke is not command execution expansion.
- `py.typed` marker is type distribution metadata, not runtime behavior.
- Runtime dependencies and dev/test dependencies must be separated.
- `pytest` must not be runtime dependency.
- Runtime data and `references/` must not be included in package by default.
- Company/private config, secrets, credentials, raw traces, raw transcripts, and
  raw provider outputs must not be package data.
- No-publish and packaging-blocked are valid outcomes.

## Relationship To v0.28.1

`v0.28.1` remains the required input for public alpha release claims.
`v0.28.2` may proceed when the hygiene gate is warning or blocked, but
`package_distribution_ready` does not imply `public_alpha_ready`.

## Data Models

The version defines `PackagingDistributionTypeBoundaryPolicy`,
`PackagingReadinessRequest`, `PackagingSourceView`,
`PyprojectPackageMetadataReport`, dependency boundary and dependency reports,
pytest runtime dependency violation report, build backend report, include/exclude
policy, package data boundary, `PyTypedMarkerReport`,
`TypeDistributionBoundaryReport`, wheel/sdist/import/CLI smoke plans and
reports, distribution artifact policy, package publish blocker, remediation
plan, readiness gate, findings, and `PackagingReadinessReport`.

## Service Design

Services read metadata and refs only. Smoke reports may be planned or represented
as not-run metadata reports. Package publishing, upload, release tag creation,
and official release artifact creation remain blocked.

## OCEL / PIG / OCPX

OCEL mappings are declared for packaging boundary creation, metadata reports,
dependency reports, package data/type reports, smoke reports, package publish
blocker, and readiness gate evaluation. Forbidden effects include package
publishing/upload, release tags, official release artifacts, public alpha
release, split/adapter implementation, provider invocation, command execution,
private exposure, and LLM judge use.

The PIG subject is `packaging_distribution_type_boundary`.
The OCPX state is `packaging_distribution_type_boundary_created`.

## CLI

Commands are available under `python -m chantacore.cli alpha packaging ...`:
`policy`, `source-view`, `pyproject`, `dependencies`, `runtime-deps`,
`dev-deps`, `optional-deps`, `pytest-boundary`, `build-backend`,
`include-exclude`, `package-data`, `py-typed`, `type-boundary`, `wheel-smoke`,
`sdist-smoke`, `import-smoke`, `cli-smoke`, `publish-blocker`, `remediation`,
`readiness`, and `report`.

## Acceptance Criteria

All packaging boundary reports can be created. Package data excludes runtime
data, references, raw traces, raw transcripts, raw provider outputs, secrets,
credentials, and company/private material. `pytest` runtime dependency is
surfaced. Package publish and release tag creation remain blocked. No public
alpha release, Schumpeter split, external adapter, provider invocation, command
execution, private exposure, or LLM sole authority is introduced.

## Withdrawal Conditions

Withdraw this boundary if it publishes or uploads a package, creates a release
tag or official release artifact, includes runtime/private/raw/reference data as
package data without policy, leaves pytest runtime dependency while declaring
package-ready, expands import/CLI smoke into provider or command execution,
implements split/adapters, adds references runtime dependency, or uses an LLM
judge as sole authority.

## Validity Horizon

Valid until `v0.28.3 Public-Private Boundary / Redaction / Reference Policy`
begins or packaging policy changes.
