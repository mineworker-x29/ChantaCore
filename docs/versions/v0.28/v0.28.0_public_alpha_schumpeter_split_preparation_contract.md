# v0.28.0 Public Alpha / Schumpeter Split Preparation Contract

## Version Definition

- Version: `v0.28.0`
- Name: Public Alpha / Schumpeter Split Preparation Contract
- Korean name: Public Alpha·Schumpeter Split Preparation 계약
- Track: Public Alpha / Schumpeter Split Preparation
- Previous: `v0.27.9` Memory Candidate & Continuity Consolidation
- Next: `v0.28.1` Release Hygiene / Repository Governance Blocking Gate

## Objective

`v0.28.0` is contract-only. It prepares ChantaCore for public alpha candidacy
and Schumpeter split decision-making without implementing public release,
package publishing, release tagging, company distribution, external adapters,
runtime expansion, provider invocation, command execution, or Schumpeter split.

The contract absorbs the Grok review as roadmap control logic: the v0.26 to
v0.27 to v0.28 handoff is structurally consistent, Schumpeter and external
adapter surfaces remain out of v0.27, release hygiene debt remains a public
alpha blocker, and v0.29/v0.30 high-risk surfaces require contract-first
preflight criteria.

## Relationship To v0.27.9

`v0.27.9` produced Memory Candidate & Continuity Foundation v1. `v0.28.0` reads
that consolidation report, release manifest, public alpha handoff packet, and
readiness report as refs-only prerequisites. It does not alter memory,
continuity, lifecycle, runtime, Default Agent, DecisionService, SkillRouter,
SafetyGate, PermissionPolicy, persona, or behavior policy.

## Relationship To v0.26.10 Hygiene Debt

`v0.26.10` Release Hygiene / Governance Hardening or an equivalent gate is
required before public alpha release claims. Hygiene unknown is not hygiene
passed. `v0.28.0` models the debt and establishes `v0.28.1` as the blocking
gate, but it does not claim public alpha readiness.

## Core Principles

- Public Alpha readiness is not production readiness.
- Public Alpha readiness is not automatically true just because architecture is
  mature.
- Schumpeter Split Preparation is not Schumpeter implementation.
- Schumpeter split decision is not split execution.
- Reference inventory is not code adoption.
- Reference reuse disposition is not runtime dependency.
- Release hygiene unknown is not release hygiene passed.
- External adapter preflight is not adapter implementation.
- `v0.29` readiness is not `v0.29` implementation.
- No-release / defer-alpha is a valid outcome.

## Public Alpha Scope And Stage

The allowed public alpha surfaces are public docs, sanitized examples,
synthetic demo data, OCEL store validation, OCPX projection demo, PIG report
demo, Workbench report surfaces, Memory foundation report surfaces, and CLI
smoke surfaces. Forbidden surfaces include company private material,
credentials, raw runtime trace, raw transcript, raw provider output, external
provider invocation, command execution expansion, runtime continuity injection,
Schumpeter company wrapper, and RPA adapter.

Readiness stages are separated as alpha architecture candidate, alpha
repository candidate, alpha package candidate, and alpha release candidate.
Architecture readiness is not release readiness; repository readiness is not
package readiness; package readiness is not production readiness.

## Release Hygiene Debt

The release hygiene debt policy requires v0.26.10 or equivalent debt to be
resolved or explicitly dispositioned. Clean worktree, release tag, license,
changelog, third-party notice when references exist, runtime data hygiene, and
reference license policy are represented as required release-claim inputs.

`v0.28.1` is the blocking gate. Repository release readiness must remain false
when hygiene is unknown, and public alpha readiness must remain false when
hygiene has failed.

## Packaging Readiness Boundary

Packaging readiness is deferred to `v0.28.2`. Pyproject validation, runtime/dev
dependency separation, `py.typed`, include/exclude policy, runtime data
exclusion, references policy, wheel smoke, sdist smoke, import smoke, and CLI
smoke are required by policy. Package publishing is disabled in `v0.28.0`.

## Public-Private Boundary

The public-private boundary is deferred to `v0.28.3`. Public core/private
overlay is required. Public core must not contain company material, credentials,
internal endpoints, raw traces, raw transcripts, or raw provider outputs.
Sanitized examples and synthetic demo data may be allowed. Schumpeter requires a
private overlay.

## Schumpeter Split Decision Framework

Schumpeter split preparation is not split implementation. The decision
framework is deferred to `v0.28.4`, and split preparation is deferred to
`v0.28.5`. The default option is
`keep_reference_only_and_prepare_private_overlay`. Actual decision and split
implementation are false in `v0.28.0`.

Decision outputs include keep reference only, prepare private overlay, prepare
distribution profile, defer split, block split, deprecate legacy Schumpeter,
extract concepts only, extract tests only, extract docs only, and migrate a
specific module later.

Decision options include no split, reference only, public core/private overlay,
private distribution profile, separate private repo, merge Schumpeter into core,
and deprecate legacy Schumpeter. These options are evaluation artifacts, not
implementation actions.

Evaluation criteria include IP/license risk, company-private data contamination
risk, public core contamination risk, architecture overlap, code/concept/test/doc
reuse value, OCEL/PIG/memory/workbench compatibility, runtime dependency risk,
maintainability, testability, deployment boundary clarity, credential exposure
risk, future external adapter compatibility, and organizational usefulness.

## Schumpeter Reference And Reuse Policy

`references/schumpeter` is reference-only by default. It is not a runtime
dependency and is not copied into core by default. Origin, license status,
private data risk, company material risk, concept value, code reuse value, test
reuse value, doc reuse value, OCEL compatibility, and recommended disposition
must be reviewed.

Allowed reuse dispositions include adopt concept, port model later, port test
later, port document later, keep reference only, quarantine due to license or
private risk, and discard. Reuse requires decision record, license review, and
public-private boundary review. Runtime dependency and code copy are forbidden
now.

## External Adapter Preflight Boundary

External adapter preflight is deferred to `v0.28.8`, and implementation is
deferred to `v0.29.x`. Provider invocation and adapter registration are
forbidden now. Capability declaration is not permission, provider adapter
contract is not provider invocation, and safety, permission, audit, rollback,
credential, and network boundaries are required before invocation.

The v0.29 risk reopen criteria require contract-first work, preflight before
provider or command reopening, credential and network boundaries, adapter
certification, OCEL visibility, permission gate, safety gate, and no background
execution without a gate.

## Safety Boundary

Provider invocation, command execution, file mutation expansion, runtime
continuity injection, autonomous memory-driven execution, external adapter,
Schumpeter split, and company deployment are disabled now. Secret, credential,
raw trace, raw transcript, and raw provider output exposure are forbidden. PIG
execution authority and LLM judge as sole readiness authority are forbidden.

## Non-Goals

`v0.28.0` does not implement public alpha release, package publishing, release
tag creation, Schumpeter split, company wrapper/profile runtime, company/private
config, endpoint or credential handling, references/schumpeter code adoption,
references/schumpeter runtime dependency, external provider adapters, external
agent dominion bridge, RPA adapters, provider invocation, command execution,
runtime continuity injection, autonomous memory-driven execution, PIG execution
authority, or LLM judge as sole readiness authority.
