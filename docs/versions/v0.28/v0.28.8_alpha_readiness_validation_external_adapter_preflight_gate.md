# v0.28.8 Alpha Readiness Validation / External Adapter Preflight Gate

Korean name: Alpha 준비성 검증·External Adapter 사전 게이트

Track: Public Alpha / Schumpeter Split Preparation

Previous: v0.28.7 Alpha Documentation / Onboarding / Example Pack

Next: v0.28.9 Public Alpha / Schumpeter Split Preparation Consolidation

## Objective

v0.28.8 validates v0.28.0 through v0.28.7 and creates the external adapter preflight gate required before v0.29. It is validation/preflight-gate only.

Alpha readiness validation is not Public Alpha release. Validation pass is not package publish. Package build validation is not package upload. Docs validation is not production support. External adapter preflight is not external adapter implementation. Provider invocation readiness is not provider invocation. Command execution reopen criteria is not command execution expansion. Credential boundary is not credential storage. Network boundary is not network access. v0.29 readiness is not v0.29 implementation.

## Relationship

v0.28.7 created public-safe docs, onboarding, and example-pack artifacts. v0.28.8 validates those docs against feature flags, safety boundaries, package boundaries, public/private policy, smoke scenarios, and forbidden overclaims.

v0.29 remains future-track External Provider Adapter Contract work. v0.28.8 may prepare preflight criteria for v0.29 but does not implement adapters, register providers, invoke providers, open network access, or expand command execution.

## Validation Scope

The validation scope includes v0.28 coverage validation, regression and boundary validation, forbidden pattern scan, release hygiene validation, packaging validation, package build validation, import and CLI smoke validation, public-private validation, documentation and example validation, smoke demo validation, safety validation, and Schumpeter preparation validation.

Unknown validation result is not passed for release claims or external adapter readiness claims. No-release and no-adapter remain valid outcomes.

## External Adapter Preflight

The external adapter preflight defines provider invocation reopen criteria, command execution reopen criteria, credential boundary preflight, network boundary preflight, permission boundary preflight, safety gate preflight, audit/rollback/OCEL visibility preflight, and adapter certification preflight.

Provider invocation requires a v0.29 contract, provider inventory, capability registry, permission gate, safety gate, credential boundary, network boundary, audit trace, rollback boundary, OCEL visibility, and user approval surface. Command execution reopen criteria require v0.29-or-later contract, allowlist, sandbox boundary, permission gate, safety gate, audit trace, rollback/no-op boundary, no shell true, and no unbounded subprocess.

## Service Design

The service layer loads prerequisite metadata and refs only. It must not load raw secrets, raw traces, raw transcripts, raw provider outputs, company-private material, private configs, or actual user/company data. Safe local tests may be run by the operator, but the validation model itself does not add command execution capability.
