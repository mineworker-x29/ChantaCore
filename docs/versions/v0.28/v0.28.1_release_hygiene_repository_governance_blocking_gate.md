# v0.28.1 Release Hygiene / Repository Governance Blocking Gate

- Version: `v0.28.1`
- Name: Release Hygiene / Repository Governance Blocking Gate
- Korean name: Release Hygiene·Repository Governance 차단 게이트
- Track: Public Alpha / Schumpeter Split Preparation
- Previous: `v0.28.0` Public Alpha / Schumpeter Split Preparation Contract
- Next: `v0.28.2` Packaging / Distribution / Type Boundary

## Objective

`v0.28.1` creates the first blocking gate for public alpha release claims. It
models the `v0.26.10` release hygiene debt explicitly and evaluates repository
governance readiness using read-only metadata. Unknown hygiene is not passed.

## Principles

- Release hygiene gate is not release.
- Repository governance gate is not auto-remediation.
- Clean worktree unknown is not clean.
- Missing LICENSE blocks public alpha release claims.
- Missing CHANGELOG blocks public alpha release claims.
- Missing third-party notices with references present blocks public alpha release claims.
- Tracked runtime DB / SQLite / bak files block release readiness.
- References policy missing blocks public alpha release claims when `references/` exists.
- No-release and defer-alpha are valid outcomes.
- Architecture readiness is not repository release readiness.

## Scope

Allowed artifacts include release hygiene policy, repository governance policy,
repository state snapshot, version/worktree/tag/governance-file reports,
license/changelog/third-party reports, pyproject/py.typed/CI reports,
gitignore/data/artifact reports, references inventory/license/governance
reports, forbidden artifact scan, remediation plan, no-release/defer-alpha
decisions, public alpha release claim gate, and the gate report.

Non-goals are auto-fix, LICENSE or CHANGELOG creation, release tag creation,
package publishing, public alpha implementation, Schumpeter split, company
wrapper, external adapters, provider invocation, runtime command expansion,
private material exposure, and LLM judge sole authority.

## Data Models

The version defines `ReleaseHygieneBlockingGatePolicy`,
`RepositoryGovernancePolicy`, `RepositoryStateSnapshot`,
`VersionConsistencyReport`, `CleanWorktreeReport`,
`ReleaseTagReadinessReport`, governance-file and packaging metadata reports,
runtime data and references reports, remediation plan, `NoReleaseDecision`,
`DeferAlphaDecision`, `PublicAlphaReleaseClaimGate`, findings, and
`ReleaseHygieneGateReport`.

## Service Design

Services load metadata and refs only. If a check cannot be verified without a
bounded repository service, it is marked `unknown` rather than passed. The
remediation plan records recommendations but performs no auto-fix.

## OCEL / PIG / OCPX

OCEL object/event/effect mappings are declared for release hygiene gate
evaluation, repository governance reports, remediation plans, no-release and
defer-alpha decisions, and public alpha release claim gate evaluation. Forbidden
effects include auto-fix, tag creation, package publishing, provider or command
execution, split/adapter implementation, private exposure, and LLM judge use.

The PIG subject is `release_hygiene_repository_governance_blocking_gate`.
The OCPX state is `release_hygiene_repository_governance_blocking_gate_evaluated`.

## CLI

Commands are available under `python -m chantacore.cli alpha hygiene ...`:
`gate`, `snapshot`, `version`, `worktree`, `tag`, `governance-files`,
`license`, `changelog`, `third-party`, `pyproject`, `py-typed`, `ci`,
`gitignore`, `data`, `artifacts`, `references`, `forbidden-scan`,
`remediation`, `release-claim`, and `report`.

## Acceptance Criteria

All gate reports can be created. Unknown hygiene does not pass. Missing
required governance files, references governance gaps, tracked runtime artifacts,
and forbidden exposure artifacts block release claims. No auto-fix, tag
creation, package publishing, split, adapter, provider invocation, command
execution, private exposure, or LLM sole authority is introduced.

## Withdrawal Conditions

Withdraw this gate if it auto-fixes the repository, creates governance files or
tags, publishes packages, treats unknown hygiene as passed, treats dirty
worktree or tracked runtime artifacts as release-ready, allows references
without governance policy, invokes providers or runtime commands, implements
split/adapters, exposes private material, or uses an LLM judge as sole gate
authority.

## Validity Horizon

Valid until `v0.28.2 Packaging / Distribution / Type Boundary` begins or release
hygiene policy changes.
