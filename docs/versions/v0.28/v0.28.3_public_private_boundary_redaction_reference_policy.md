# v0.28.3 Public-Private Boundary / Redaction / Reference Policy

Korean name: Public-Private Boundary·Redaction·Reference Policy  
Track: Public Alpha / Schumpeter Split Preparation  
Previous: v0.28.2 Packaging / Distribution / Type Boundary  
Next: v0.28.4 Schumpeter Split Decision Framework

## Objective

v0.28.3 defines the public/private boundary, redaction-preview policy, and reference governance policy for Public ChantaCore candidacy. It is boundary, classification, preview, and report only.

## Relationships

v0.28.0 created the Public Alpha / Schumpeter Split Preparation contract. v0.28.1 created the release hygiene blocking gate. v0.28.2 created the packaging, distribution, and type boundary. v0.28.3 refines those outputs by deciding which artifacts are public-eligible, private-only, reference-only, quarantine candidates, or blocked for release claims.

v0.28.4 must consume the PublicPrivateBoundaryReport, ReferenceGovernanceReport, ReferenceLicenseBoundaryReport, and package exposure findings before any Schumpeter split decision framework can proceed. v0.28.3 does not make the split decision.

## Principles

- Public-private boundary is not repo split.
- Redaction policy is not destructive redaction.
- Redaction preview is not file mutation.
- Quarantine recommendation is not deletion.
- Reference policy is not reference adoption.
- Reference inventory is not runtime dependency.
- Public example must be sanitized or synthetic.
- Public docs must not expose company/private material.
- Schumpeter references are reference-only until v0.28.4 decision framework.
- No-public-release and no-redaction-apply are valid outcomes.

## Policies

PublicArtifactPolicy allows public docs, architecture docs, sanitized examples, synthetic demo data, public-safe tests, generic OCEL/PIG/Workbench/Memory reports, package source code, and type metadata.

PrivateArtifactPolicy keeps company config, credentials, endpoints, RPA adapter config, A360/Brity/UiPath details, actual process logs, user session traces, raw transcripts, raw provider outputs, private overlay profiles, and deployment secrets outside the public core.

Classification rules require credentials, secrets, company endpoints, raw traces, raw transcripts, raw provider outputs, runtime DBs, backup files, references, examples, docs, package data, Schumpeter references, and RPA details to be classified before public release claims.

RedactionPolicy enables preview-only redaction for credentials, secrets, internal endpoints, private paths, raw traces, raw transcripts, and raw provider outputs. It explicitly disables destructive redaction, source mutation, and source deletion.

ReferenceCodePolicy keeps references reference-only by default, forbids runtime dependency adoption, forbids code copy into public core now, and defers Schumpeter reference disposition to v0.28.4.

PublicDatasetPolicy, ExampleDataPolicy, SanitizedExamplePolicy, SyntheticDataPolicy, and DocumentationExposurePolicy require public examples to be sanitized or synthetic and forbid actual user/company data, raw DBs, raw event logs, credentials, company material, and raw provider output.

## OCEL / PIG / OCPX

OCEL object, event, and effect mappings are extended for public/private boundary policies, artifact classifications, redaction previews, reference governance, package exposure, remediation, and release gate evaluation.

Allowed effects are read-only observation, boundary creation, artifact classification, redaction preview creation, reference governance report creation, package exposure report creation, public/private release gate evaluation, remediation plan creation, and state candidate creation.

Forbidden effects include destructive redaction, source deletion, file movement, repo split, Schumpeter split, company wrapper, package publishing, release tag creation, external adapters, provider invocation, command execution, runtime continuity injection, reference runtime dependency, reference code copy, private exposure, credential exposure, secret exposure, raw trace exposure, raw transcript exposure, raw provider output exposure, PIG execution authority, and LLM judge use.

The PIG subject is `public_private_boundary_redaction_reference_policy`. The OCPX state is `public_private_boundary_redaction_reference_policy_created`.

## Non-goals

v0.28.3 does not apply redaction, delete files, move files, split repositories, implement Schumpeter split, implement company wrapper, publish packages, create release tags, implement external adapters, invoke providers, execute commands as a runtime capability, add references as runtime dependencies, copy references into core, expose private/company material, expose credentials or secrets, expose raw traces/transcripts/provider outputs, or use an LLM judge as sole authority.

## Acceptance Criteria

- Public/private runtime policy can be created.
- Public/private artifact policies can be created.
- Classification rules and artifact classifications can be created.
- Redaction policy, previews, and decisions can be created without mutation.
- Secret, credential, company, and raw exclusion policies can be created.
- Reference governance and license reports can be created.
- Public dataset, example, sanitized, synthetic, and docs policies can be created.
- Package exposure boundary report can be created.
- Remediation and quarantine recommendations can be created without moving or deleting files.
- PublicPrivateReleaseGate can be evaluated.
- PublicPrivateBoundaryReport can be created.
- `ready_for_v0_28_4` can be true as decision-framework-start readiness, not split implementation readiness.

## Withdrawal Conditions

Withdraw this version judgment if destructive redaction, source deletion, file movement, repo split, Schumpeter split, company wrapper, package publishing, release tagging, external adapter implementation, provider invocation, command execution capability, reference runtime dependency, reference code copy, private exposure, credential/secret exposure, raw trace/transcript/provider output exposure, PIG execution authority, or LLM sole authority appears.
