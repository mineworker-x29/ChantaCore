# v0.28.5 Schumpeter Split Preparation Profile

Version: v0.28.5  
Name: Schumpeter Split Preparation Profile  
Korean name: Schumpeter Split 준비 프로파일  
Track: Public Alpha / Schumpeter Split Preparation  
Previous: v0.28.4 Schumpeter Split Decision Framework  
Next: v0.28.6 Public Alpha Runtime Profile / Smoke Demo Flow

## Objective

v0.28.5 creates a preparation profile for a possible future Schumpeter private overlay or company distribution. It consumes the v0.28.4 decision framework and prepares boundaries, contracts, previews, risk registers, decisions, and a handoff packet. It does not implement a Schumpeter runtime.

## Relationship to v0.28.4

v0.28.4 provides the Schumpeter split recommendation, split decision records, reuse disposition decisions, and decision audit trail. v0.28.5 may prepare a future private overlay/profile only when those records allow or recommend it. Missing, unknown, or blocked v0.28.4 evidence must not be treated as split permission.

Decision recommendation is not implementation authorization. Prepare private overlay is not company wrapper runtime. The default posture remains `keep_reference_only + prepare_private_overlay`; `no_split`, `keep_reference_only`, and `defer_private_overlay_preparation` remain valid outcomes.

## Relationship to v0.28.6

v0.28.6 must consume only public-safe ChantaCore surfaces for the runtime profile and smoke demo flow. It must not use Schumpeter private/company artifacts. The v0.28.5 handoff packet marks public-safe refs separately from private-overlay future refs and blocked or deferred refs.

## Core Principles

- Schumpeter preparation profile is not Schumpeter implementation.
- Private overlay contract is not private overlay runtime.
- Config overlay contract is not config file creation.
- Company distribution boundary is not company deployment.
- Skill boundary is not skill implementation.
- Provider boundary is not provider adapter.
- RPA adapter deferral is not RPA integration.
- Reference-only Schumpeter remains reference-only unless later explicitly dispositioned.
- Public ChantaCore must not depend on private Schumpeter artifacts.
- Private overlay must not contaminate public core.
- No private config is generated now.
- No company credential is generated or stored now.
- No provider invocation occurs now.
- No command execution occurs now.
- No split is a valid outcome.
- Defer preparation is a valid outcome.

## Profile Boundary

`SchumpeterProfileBoundary` names Schumpeter as a profile candidate and separates public core responsibilities from private overlay responsibilities. Public core remains generic. The private overlay is a future candidate only. Forbidden cross-boundary dependencies include public-core imports of private overlay, private config in public core, company credentials in public core, and references/schumpeter runtime dependency.

## Naming Policy

`SchumpeterNamingPolicy` keeps the public name as `ChantaCore` and the private overlay candidate name as `Schumpeter`. ChantaCore is not renamed. The Schumpeter name is a profile candidate, not a runtime implementation.

## Public Core / Private Overlay Policy

`PublicCorePrivateOverlayPolicy` keeps public core generic, forbids committing private overlay material into public core, forbids public core imports of private overlay, and states that the private overlay contract is not runtime.

## Private Overlay Contract

`SchumpeterPrivateOverlayContract` is a draft contract for possible future contents: private config, company provider configs, company RPA adapter configs, company-specific docs, deployment profiles, private skill profiles, and private provider profiles. It forbids committed credentials, committed secrets, raw company logs, raw user traces, raw provider outputs, uncontrolled provider invocation, and uncontrolled command execution.

Required future gates include public/private boundary, credential boundary, provider adapter gate, RPA adapter gate, deployment gate, and audit gate.

## Config Overlay Contract

`SchumpeterConfigOverlayContract` allows a config overlay later but creates no config file now. Company endpoints are forbidden now. Credentials and secrets require future policy and external secret-store handling.

## Data Boundary

`SchumpeterDataBoundaryPolicy` keeps company data, raw process logs, raw user session traces, raw transcripts, and raw provider outputs private-only. Synthetic demo data may be public. Sanitized summaries may be public only with policy. Public core must not store company data.

## Skill Boundary

`SchumpeterSkillBoundaryPolicy` keeps public core skills generic. Company-specific skills, RPA skills, A360, Brity, and UiPath are forbidden now. Private overlay skill registration is deferred and requires v0.29 or later adapter gates.

## Provider Boundary

`SchumpeterProviderBoundaryPolicy` keeps the public provider boundary generic. Company provider configs, provider adapter implementation, and provider invocation are forbidden now. Provider work requires v0.29 or later plus credential boundary gates.

## RPA Adapter Deferral

`SchumpeterRPAAdapterDeferralPolicy` defers RPA adapters, A360, Brity, and UiPath to v0.29 or later. RPA adapter deferral is not RPA integration and is not a public alpha feature.

## Runtime Boundary

`SchumpeterRuntimeBoundaryPolicy` forbids runtime wrapper implementation, runtime continuity injection, autonomous memory execution, provider invocation, and command execution now. Future private runtime requires safety and permission gates.

## Memory / Workbench Boundary

`SchumpeterMemoryWorkbenchBoundaryPolicy` keeps public memory foundation generic. Private company memory records are forbidden now. Company workbench reports are private-only. Continuity injection for private overlay is deferred.

## Deployment Boundary

`SchumpeterDeploymentBoundaryPolicy` forbids company deployment, deployment automation, private repo creation, and company release pipeline creation now. A future deployment gate is required.

## Manifest Preview

`SchumpeterPrivateOverlayManifestPreview` proposes profile metadata, private config boundary, provider boundary, RPA adapter boundary, data boundary, skill boundary, memory/workbench boundary, deployment boundary, and audit boundary. The preview is not file creation. It creates no manifest file and no private config.

## Capability Map

`SchumpeterProfileCapabilityMap` classifies capabilities as public core, private overlay candidate, company distribution candidate, future external adapter, future RPA adapter, future deployment, forbidden now, or unknown. Future/private capability entries have `implementation_allowed_now=false`.

## Risk Register

`SchumpeterProfileRiskRegister` records risks for public/private contamination, credential exposure, company data exposure, provider invocation without a gate, command execution without a gate, RPA scope creep, reference code copy, runtime dependency leak, deployment boundary violation, unknown license, and unknown private risk.

## Preparation Decision

`SchumpeterPreparationDecision` may choose prepare private overlay profile, prepare profile boundary only, prepare handoff only, defer preparation, block preparation, keep reference-only, no-op, or unknown. In v0.28.5 every decision is non-implementation: no config, provider adapter, or RPA adapter is created.

## Handoff Packet

`SchumpeterPreparationHandoffPacket` targets v0.28.6. It separates public core safe refs, private overlay future refs, blocked/deferred refs, future v0.29 adapter refs, and future v0.30 dominion refs. It is refs-only and implementation is not performed now.

Not implemented now includes actual Schumpeter split, company wrapper runtime, private repo creation, private config generation, provider adapter, RPA adapter, A360 adapter, Brity adapter, UiPath adapter, company deployment, and runtime continuity injection.

## Audit Trail

`SchumpeterPreparationAuditTrail` links request, source view, profile boundary, overlay contract, config/data/skill/provider/RPA/runtime boundaries, capability map, risk register, decisions, and handoff packet. Raw content is not included.

## Non-Goals

v0.28.5 does not implement actual Schumpeter split, company wrapper runtime, private distribution runtime, private repo creation, private config generation, credentials, endpoints, provider adapters, RPA adapters, A360, Brity, UiPath, references runtime dependency, references code copy, file movement, package publishing, release tag creation, provider invocation, command execution, runtime continuity injection, private/company/raw/credential exposure, PIG execution authority, or LLM judge sole authority.
