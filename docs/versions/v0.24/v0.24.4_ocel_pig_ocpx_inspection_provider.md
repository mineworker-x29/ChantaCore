# v0.24.4 OCEL / PIG / OCPX Inspection Provider

Korean name: OCEL·PIG·OCPX 검사 Provider

Track: Internal Provider / Local Runtime Provider

Previous: v0.24.3 Repository Search / File Read Provider

Next: v0.24.5 Local Runtime Command Candidate Provider

## Objective

v0.24.4 introduces read-only inspection providers for existing process-intelligence state:

- OCEL type catalog, bounded recent event view, and bounded object-centric trace view.
- PIG report index and bounded sanitized report views.
- OCPX projection index and bounded sanitized projection views.

This release depends on the v0.24.0 Internal Provider Contract, the v0.24.1 Provider Registry and Capability Surface, the v0.24.2 read-only workspace provider, the v0.24.3 repository search/file read provider, and the v0.23.9 OCEL-native Internal Dominion consolidation.

## Policy Boundary

v0.24.x remains Internal Provider / Local Runtime Provider work.

Process inspection is read-only. OCEL inspection is not event log mutation. PIG inspection is not graph recomputation. OCPX inspection is not projection mutation.

Recent event and object trace views are bounded. Payload, report, and projection excerpts must be sanitized before output. Raw payload output, raw secret output, credential exposure, local command execution, external provider calls, and provider adapter execution are forbidden.

v0.25 General Agent Usability, v0.26 Workspace Agent Workbench, v0.27 Memory Candidate and Continuity, v0.28 Schumpeter split preparation, and v0.29+ external adapters remain deferred.

## Non-Goals

v0.24.4 does not append, write, delete, migrate, or mutate OCEL state. It does not mutate or recompute PIG graphs. It does not mutate or recompute OCPX projections. It does not replace the canonical OCEL substrate. It does not execute local commands, create local runtime command candidates, install packages, fetch network resources, load plugins, connect MCP, implement ask/repl/tool routing, add external adapters, introduce Schumpeter split logic, store credentials, output raw secrets, or use an LLM judge.

## Service Design

The process inspection provider is a deterministic read-only facade. It uses existing provider registry surfaces and static OCEL mapping constants, plus explicitly supplied in-memory OCEL/PIG/OCPX artifacts when available. Missing process-intelligence sources are represented as warnings in standard mode and failures in strict mode where applicable.

The implementation intentionally avoids store initialization, event append APIs, graph recomputation APIs, projection recomputation APIs, process spawning, shell execution, network access, and plugin loading.

## Inspection Policy

Defaults:

- `read_only=True`
- `ocel_mutation_enabled=False`
- `pig_mutation_enabled=False`
- `ocpx_mutation_enabled=False`
- `event_append_enabled=False`
- `projection_recompute_enabled=False`
- `graph_recompute_enabled=False`
- `raw_payload_output_enabled=False`
- `secret_redaction_required=True`
- `private_path_sanitization_required=True`
- `local_command_execution_enabled=False`

Bounds:

- `max_recent_events_default=200`
- `max_trace_events_default=500`
- `max_report_chars_default=40000`
- `max_projection_chars_default=40000`

## Status Rules

`passed` means declared provider surfaces exist, available process state can be inspected in bounded sanitized form, and no mutation/recompute/raw output/local command/credential exposure occurs.

`warning` means provider surfaces exist but optional OCEL events, PIG reports, or OCPX projections are missing, truncated, or sanitized.

`failed` means a required surface or required strict source is unavailable, bounds are missing, or a disallowed raw payload request cannot be represented safely.

`blocked` means mutation, recomputation, raw payload output, raw secret output, credential exposure, local command execution, external runtime touch, external provider call, Schumpeter split, vendor-specific runtime logic, or LLM judge behavior is detected.

Event types include `process_state_inspection_requested`, `process_inspection_policy_created`, `ocel_type_catalog_created`, `ocel_recent_events_inspected`, `ocel_object_trace_inspected`, `pig_report_index_created`, `pig_report_view_created`, `ocpx_projection_index_created`, `ocpx_projection_view_created`, and `process_state_inspection_report_created`.

Relation types include `uses_ocel_inspection_provider`, `uses_pig_inspection_provider`, `uses_ocpx_projection_provider`, `not_ocel_mutated`, `not_pig_mutated`, `not_ocpx_mutated`, `not_projection_recomputed`, `not_graph_recomputed`, `not_local_command_executed`, and `defers_local_runtime_candidate_to_v0_24_5`.

Forbidden effects include OCEL event append/write, OCEL object/relation mutation, PIG graph mutation/recomputation, OCPX projection mutation/recomputation, event log migration, raw payload output, raw secret output, local command execution, shell/network/package/destructive command, external runtime touch, credential exposure, and external provider calls.

## PIG/OCPX

The PIG report subject is `ocel_pig_ocpx_inspection_provider`. It records the read-only process inspection principles and safety boundary flags.

The OCPX projection state is `process_intelligence_state_inspected`. Target read models include OCEL inspection, OCEL type catalog, OCEL recent event view, OCEL object trace view, PIG inspection, OCPX inspection, process state inspection, and v0.24 readiness.

## Acceptance Criteria

- `ProcessStateInspectionReport` can be created.
- OCEL type catalog can be inspected when mapping exists.
- Recent event, object trace, PIG report, and OCPX projection views are bounded and sanitized when sources exist.
- Missing process state sources are represented honestly as warnings or failures.
- No OCEL/PIG/OCPX mutation occurs.
- No graph or projection recomputation occurs.
- No raw payload or raw secret output occurs.
- No local command execution occurs.
- No external provider adapter or Schumpeter split is introduced.
- No credential value is stored or output.
- No LLM judge is used.
- v0.24.3, v0.24.2, v0.24.1, v0.24.0, v0.23.9, v0.23.x, v0.22.x, v0.21.x, and v0.20.x behavior remains preserved.

## Withdrawal Conditions

Withdraw this release claim if later evidence shows event log mutation, PIG/OCPX mutation or recomputation, raw payload output, raw secret output, credential exposure, local command execution, external runtime touch, external provider adapter behavior, vendor-specific runtime logic in core, Schumpeter split logic, or LLM judge behavior was introduced.

## Validity Horizon

Valid until OCEL/PIG/OCPX state, inspection policy, provider registry surfaces, or v0.24 roadmap boundaries change.
