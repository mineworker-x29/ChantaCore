# v0.24.6 Local Runtime Static Safety / Preflight

Korean name: 로컬 런타임 정적 안전성·사전점검

Track: Internal Provider / Local Runtime Provider

## Definition

v0.24.6 implements deterministic static safety and declared preflight over v0.24.5 `LocalRuntimeCommandCandidate` objects. It may produce v0.24.7 execution eligibility, but it does not execute commands.

## Principles

- Static safety check is not command execution.
- Declared preflight is not live command execution.
- Preflight is not process spawn.
- Allowlist compatibility is not authorization to execute.
- Execution eligibility is not execution.
- Execution eligibility only permits moving to v0.24.7 execution gate.

## Scope

Included scope: safety/preflight request, command allowlist policy, forbidden pattern policy, static safety rules/results/findings/report, declared preflight policy/check/result/report, execution eligibility, combined safety/preflight report, OCEL mapping, PIG/OCPX projection, and CLI views.

Non-goals: local command execution, subprocess, shell, live tool availability check, output capture, execution gate opening, bounded command run, external provider adapter, General Agent UX, Schumpeter split, credential output, and LLM judge.

## Policies

The allowlist is deny-by-default and contains static descriptors for `python --version`, `git status --short`, `git diff --stat`, `python -m compileall`, `python -m pytest`, `python -m ruff check`, and `python -m mypy`. These are static descriptors, not execution authorization.

Forbidden patterns include shell tools, network tools, package managers, destructive command words, credential env patterns, and path escape patterns.

## CLI

- `python -m chantacore.cli provider local-runtime allowlist`
- `python -m chantacore.cli provider local-runtime safety --candidate-id <id>`
- `python -m chantacore.cli provider local-runtime preflight --candidate-id <id>`
- `python -m chantacore.cli provider local-runtime safety-preflight --candidate-id <id>`
- `python -m chantacore.cli provider local-runtime eligibility --candidate-id <id>`
- `python -m chantacore.cli provider local-runtime safety-report`
- `python -m chantacore.cli provider local-runtime preflight-report`
- `python -m chantacore.cli provider local-runtime safety-findings`

The CLI prints `version=v0.24.6`, `declared_preflight_only=true`, `live_preflight_performed=false`, `execution_allowed_now=false`, and all command execution/output capture flags as false.

## OCEL/PIG/OCPX

OCEL mappings are declared for safety/preflight request, allowlist, forbidden policy, static safety rules/results/findings/report, declared preflight checks/results/report, execution eligibility, and combined report. Effects are limited to read-only observation, static safety checked, preflight checked, execution eligibility created, and state candidate creation.

The PIG report subject is `local_runtime_static_safety_preflight`. The OCPX state is `local_runtime_command_candidate_safety_preflight_checked`.

## Acceptance Criteria

`LocalRuntimeStaticSafetyReport`, `LocalRuntimePreflightReport`, `LocalRuntimeSafetyPreflightReport`, and `LocalRuntimeExecutionEligibility` can be created without executing commands or performing live checks.

## Withdrawal Conditions

Withdraw v0.24.6 readiness if live command execution, subprocess, shell, live tool availability check, output capture, execution gate opening, credential materialization, external adapter logic, or LLM judge is introduced.

## Validity Horizon

Valid until command candidate, allowlist policy, workspace context, repository context, or local runtime policy changes.
