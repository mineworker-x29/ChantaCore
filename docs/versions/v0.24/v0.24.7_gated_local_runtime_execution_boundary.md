# v0.24.7 Gated Local Runtime Execution Boundary

Korean name: 게이트 기반 로컬 런타임 실행 경계

Track: Internal Provider / Local Runtime Provider

## Definition

v0.24.7 introduces the first gated, bounded local runtime execution boundary. It may execute only eligible, allowlisted argv commands after v0.24.6 safety/preflight and v0.24.7 gate authorization.

## Principles

- Gated execution is not unrestricted shell.
- Bounded local command run is not arbitrary subprocess.
- Execution authorization is single-use and scoped.
- Execution must consume authorization exactly once.
- Execution must be OCEL-visible.
- Execution must be bounded by timeout and output caps.
- Output must be redacted before display.
- Side effects must be scanned and reported.

## Scope

Included scope: gate request/policy/conditions/state, single-use authorization, runner policy, process spec, bounded run, process result, output capture policy/capture, side-effect policy/scan, execution findings/report, OCEL mapping, PIG/OCPX projection, and CLI views.

Non-goals: unrestricted command execution, arbitrary subprocess, command string execution, shell execution, package install, network command, destructive command, file write/edit/delete/move/rename, external provider adapter, General Agent UX, Workspace Agent Workbench, Memory Candidate continuity, Schumpeter split, credential output, LLM judge, output summarization, failure explanation, automatic repair loop, and automatic rerun loop.

## Implementation Rule

The only allowed process execution path is `BoundedLocalCommandRunner`. It uses argv list, explicit workspace CWD, explicit timeout, bounded capture, and `shell=False`. `compileall` remains disabled by default because it can create cache artifacts unless a future explicit side-effect policy permits it.

## CLI

- `python -m chantacore.cli provider local-runtime gate --eligibility-id <id>`
- `python -m chantacore.cli provider local-runtime run --authorization-id <id>`
- `python -m chantacore.cli provider local-runtime execution-report --report-id <id>`
- `python -m chantacore.cli provider local-runtime run-result --run-id <id>`
- `python -m chantacore.cli provider local-runtime side-effects --run-id <id>`
- `python -m chantacore.cli provider local-runtime execution-findings --report-id <id>`

The CLI prints `version=v0.24.7`, gate and authorization status, bounded run status when applicable, `shell_used=false`, output redaction status, side-effect scan status, and forbidden effect flags as false. It does not print credential values, raw secrets, private full paths, or unbounded output.

## OCEL/PIG/OCPX

OCEL mappings are declared for gate, authorization, process spec, bounded run, process result, output capture, side-effect scan, execution finding, and execution boundary report. Effects are limited to read-only observation, gate opened, authorization created/consumed, bounded local command executed, local output captured, side-effect scan created, and state candidate creation.

The PIG report subject is `gated_local_runtime_execution_boundary`. The OCPX state is `gated_local_runtime_execution_boundary_created`.

## Acceptance Criteria

`LocalRuntimeExecutionBoundaryReport` can be created, gate opens only with valid v0.24.6 eligibility, authorization is single-use and scoped, bounded runner uses `shell=False`, timeout/output caps/redaction/side-effect scan are enforced, and forbidden network/package/destructive/external-provider paths remain blocked.

## Withdrawal Conditions

Withdraw v0.24.7 readiness if subprocess appears outside the bounded runner, shell strings are executed, arbitrary executable names are allowed, network/package/destructive commands pass, credential values are stored or output, private full paths are printed, or LLM judge is introduced.

## Validity Horizon

Valid until execution policy, allowlist policy, command candidate, workspace context, or local runtime policy changes.
