## Version Definition

v0.24.3 activates the repository search provider and file read provider as bounded, read-only internal providers. It follows the v0.24.0 Internal Provider Contract, the v0.24.1 Provider Registry & Capability Surface, and the v0.24.2 Read-only Workspace Provider.

v0.24.x remains Internal Provider / Local Runtime Provider. v0.25.x remains General Agent Usability & Tool Routing. v0.26.x remains Workspace Agent Workbench. v0.27.x remains Memory Candidate & Continuity. v0.28.x remains Public Alpha / Schumpeter Split Preparation. v0.29.x+ remains External Skill / External Provider Adapter Development.

## Objective

v0.24.3 introduces bounded repository file-name/text/symbol-ish search and bounded sanitized file excerpts. It creates the semantic surface needed by later OCEL/PIG/OCPX inspection work without adding local command execution, external provider adapters, file mutation, or agent UX.

## Core Principles

- Repository search is bounded observation, not full repository dump.
- File read is bounded excerpt, not unrestricted content extraction.
- Secret-like content must be redacted before output.
- Binary files must not be emitted as raw text.
- Private paths must be sanitized.
- Ignore policy must be applied before search/read output.
- No write/edit/delete operation is allowed.
- No local command execution is allowed.

## Service Design

`RepositoryFileProviderContractSourceService` reads the prior provider contract, registry, provider surfaces, and workspace snapshot when available. `RepositoryProviderPolicyService` and `FileReadPolicyService` build deny-by-default policy objects. `RepositorySearchScopeService` limits search to declared workspace roots and enforces max file and match limits. `RepositorySearchService` performs bounded file-name/text/symbol-ish search with ignored/hidden/symlink/binary handling and redacted match context. `FileReadService` reads only bounded windows and emits sanitized excerpts. `RepositoryFileProviderFindingService` maps policy violations to findings. Report services assemble CLI-safe reports, PIG reports, and OCPX projections.

## Search Policy

Search is read-only. File-name search does not read file content. Text and symbol-ish search may scan text files internally, but output is limited to sanitized bounded match context. Hidden and ignored files are excluded by default. Symlinks are not followed. Binary-like files are skipped. `max_files_scanned`, `max_matches`, match context, and output size limits are enforced.

## File Read Policy

File read is read-only and bounded. `excerpt` and `bounded_file` modes enforce line and byte windows. `metadata_only` does not include content. Full file dump and unrestricted read are forbidden. Binary raw output is blocked. Secret-like content is redacted before output.

## Redaction And Sanitization

Private full paths are not output. Relative/sanitized paths are used. Secret-like content such as password, token, secret, credential, API key, bearer-style values, and common token prefixes is redacted before any CLI/report output. Raw secret output is a withdrawal condition.

## Status Rules

Passed means required provider surfaces exist, scope stays inside workspace roots, ignore and sanitization policies are applied, bounded search/read completes under limits, and no dangerous boundary is crossed.

Warning means results were truncated, some files were skipped, or secret-like content was redacted.

Failed means a required provider surface, usable root, sanitization policy, ignore policy, max limit, or supported file type is missing.

Blocked means credential exposure, raw secret output, private full path output, unrestricted file read, full repository dump, file mutation, local command execution, external runtime touch, network/shell/package/destructive command, vendor-specific runtime logic, Schumpeter split, or LLM judge appears.
