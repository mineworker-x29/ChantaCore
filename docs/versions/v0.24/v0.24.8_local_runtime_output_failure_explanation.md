# v0.24.8 Local Runtime Output / Failure Explanation

Korean name: 로컬 런타임 출력·실패 설명 Provider

Track: Internal Provider / Local Runtime Provider

Previous: v0.24.7 Gated Local Runtime Execution Boundary

Next: v0.24.9 Internal Provider Consolidation

## Definition

v0.24.8 adds read-only interpretation over existing v0.24.7 bounded local runtime run artifacts. It may read an execution boundary report, bounded run metadata, process result metadata, bounded/redacted output captures, and side-effect scan metadata. It creates output summaries, deterministic failure classifications, failure explanations, specialized summaries, side-effect interpretations, diagnostic findings, and non-executing next action candidates.

This version depends on the v0.24.0 Internal Provider Contract, v0.24.1 registry/capability surface, v0.24.5 command candidates, v0.24.6 static safety/preflight, and v0.24.7 execution boundary. It preserves v0.23.9 Internal Dominion Foundation v1 as the previous foundation release.

## Core Principles

- Output interpretation is not command execution.
- Failure explanation is not automatic repair.
- Next action candidate is not action execution.
- Output summary is not raw output dump.
- Failure classification is deterministic, not LLM judging.
- No command may be rerun in v0.24.8.
- No file may be modified in v0.24.8.

## Non-goals

v0.24.8 does not execute commands, rerun commands, spawn processes, use shell, call subprocess, perform automatic repair, apply patches, mutate files, implement agent UX, introduce external provider adapters, introduce Schumpeter split logic, expose credentials, dump raw output, or use an LLM judge.

## Service Design

`LocalRuntimeOutputSourceService` provides read-only references to existing v0.24.7 artifacts. `LocalRuntimeOutputBundleService` converts already-bounded captures into an interpretation bundle. `LocalRuntimeOutputSummaryService` produces bounded key lines and high-level status. `LocalRuntimeFailureClassificationRuleRegistry` supplies deterministic rules, and `LocalRuntimeFailureClassificationService` selects classifications from exit status, timeout, known output patterns, and side-effect metadata. Explanation and next-action services remain candidate-only and non-mutating.

## Classification Rules

Required rules include success, nonzero exit, timeout, missing executable, pytest failure/no-tests, ruff issue, mypy error, compileall error, git-not-repository, git-dirty, permission denied, dependency missing, unexpected side effect, output truncated, and output redacted. Confidence is rule-based and limited to low/medium/high.

## Status Rules

Passed status requires a bounded output bundle, sanitized summary, deterministic classification, explanation, non-executing next actions, and no rerun or mutation. Warnings cover truncated/redacted output, classifiable failures, timeouts, missing tools, partial summaries, and unknown side-effect scans. Blocked status is reserved for rerun attempts, subprocess/shell use, repair, patch application, mutation, credential exposure, raw secret output, external runtime touch, external adapter implementation, Schumpeter split, or LLM judge use.

## OCEL / PIG / OCPX

OCEL object, event, relation, and effect mappings are extended for local runtime output interpretation, failure classification, failure explanation, diagnostic findings, specialized summaries, side-effect interpretation, next action candidates, and output failure reports.

PIG subject: `local_runtime_output_failure_explanation`.

OCPX state: `local_runtime_output_failure_explained`.

## Acceptance Criteria

- `LocalRuntimeOutputFailureReport` can be created.
- Existing bounded output can be summarized without rerun.
- Failure classification is deterministic.
- Failure explanation is generated without LLM judge.
- Specialized summaries degrade to partial/unknown without hallucinating counts.
- Next action candidates are non-executing and non-mutating.
- No command rerun, subprocess, shell, repair, patch, file mutation, external adapter, Schumpeter split, credential exposure, raw secret output, or LLM judge is introduced.
- OCEL/PIG/OCPX visibility exists.

## Withdrawal Conditions

Withdraw readiness if v0.24.7 bounded output artifacts are not available, output is not bounded/redacted, raw output or private paths are emitted, deterministic rules are replaced by an LLM judge, command rerun appears, or any mutation/repair path is added.

## Validity Horizon

Valid until bounded run output, side-effect scan shape, output policy, or failure classification rules change.
