# v0.24.5 Local Runtime Command Candidate Provider

Korean name: 로컬 런타임 명령 후보 Provider

Track: Internal Provider / Local Runtime Provider

## Definition

v0.24.5 introduces deterministic local runtime command candidate creation. It reads the v0.24.0 provider contract, v0.24.1 registry/capability surface, and available v0.24.2 through v0.24.4 provider contexts, then creates inert argv/cwd/env/timeout/output/risk candidates.

## Principles

- Command candidate is not command execution.
- Argv candidate is not process spawn.
- Command template is not allowlist approval.
- Candidate creation is not static safety pass.
- Candidate creation is not runtime preflight.
- Candidate creation is not execution authorization.
- Local runtime provider is not unrestricted shell.

## Scope

Included scope: `LocalRuntimeCommandCandidatePolicy`, intent descriptor, inert command template catalog, argv candidate, workspace-bound CWD candidate, non-materialized env policy candidate, timeout policy candidate, output policy candidate, side-effect risk preview, candidate set, findings, report, OCEL mapping, PIG/OCPX projection, and CLI views.

Non-goals: local command execution, subprocess, shell, static safety, runtime preflight, execution gate, bounded command run, output capture from a real process, external provider adapter, General Agent UX, Workspace Agent Workbench, Memory Candidate continuity, Schumpeter split, credential output, and LLM judge.

## Template Catalog

The catalog includes inert descriptors for:

- `python_version_check_candidate`
- `git_status_short_candidate`
- `git_diff_stat_candidate`
- `python_compileall_candidate`
- `pytest_candidate`
- `ruff_check_candidate`
- `mypy_candidate`

These strings are descriptors only. They are not invoked by v0.24.5.

## CLI

- `python -m chantacore.cli provider local-runtime templates`
- `python -m chantacore.cli provider local-runtime candidate --goal "<goal>"`
- `python -m chantacore.cli provider local-runtime candidate --category test --target tests`
- `python -m chantacore.cli provider local-runtime report`
- `python -m chantacore.cli provider local-runtime findings`

The CLI prints `version=v0.24.5`, `provider=local_runtime_provider`, `candidate_only=true`, and all execution flags as false. Private full paths and credential values are not printed.

## OCEL/PIG/OCPX

OCEL object, event, relation, and effect mappings are declared for command candidate policy, intent, templates, argv/cwd/env/timeout/output candidates, side-effect risk preview, candidate set, findings, and report. Effects are limited to read-only observation, command candidate creation, local runtime intent classification, and state candidate creation.

The PIG report subject is `local_runtime_command_candidate_provider`. The OCPX state is `local_runtime_command_candidate_created`.

## Acceptance Criteria

`LocalRuntimeCommandCandidateReport` can be created, local runtime intent can be classified, inert templates can be selected, candidate status can be `ready_for_static_safety`, `needs_more_input`, `no_action`, or `blocked`, and no local command execution path is introduced.

## Withdrawal Conditions

Withdraw v0.24.5 readiness if command execution, subprocess, shell string execution, static safety, preflight, execution gate, credential materialization, external adapter logic, or private full path output is introduced.

## Validity Horizon

Valid until command template catalog, workspace context, repository context, or local runtime policy changes.

Next version: v0.24.6 Local Runtime Static Safety / Preflight.
