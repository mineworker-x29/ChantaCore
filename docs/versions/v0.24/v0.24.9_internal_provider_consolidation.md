## Definition

v0.24.9 closes the v0.24.x Internal Provider / Local Runtime Provider foundation. It is consolidation and release-readiness only. It may read prior v0.24.0 through v0.24.8 reports and artifacts as already-existing evidence and create a foundation snapshot, subject components, capability map, coverage matrix, safety boundary report, runtime boundary report, roadmap boundary report, gap register, release manifest, v0.25 readiness report, consolidation report, and read-only workbench snapshot.

The release consolidates:

- v0.24.0 Internal Provider Contract
- v0.24.1 Provider Registry / Capability Surface
- v0.24.2 Read-only Workspace Provider
- v0.24.3 Repository Search / File Read Provider
- v0.24.4 OCEL / PIG / OCPX Inspection Provider
- v0.24.5 Local Runtime Command Candidate Provider
- v0.24.6 Local Runtime Static Safety / Preflight
- v0.24.7 Gated Local Runtime Execution Boundary
- v0.24.8 Local Runtime Output / Failure Explanation

It preserves v0.23.9 Internal Dominion Foundation v1 as the previous foundation release.

## Core Principles

- Consolidation is not new provider execution.
- Release readiness is not Agent UX.
- Provider foundation closes substrate, not product surface.
- Bounded local execution history may be summarized, but no new command may run.
- `ready_for_v0_25` means the substrate can support v0.25, not that v0.25 is implemented.
- v0.24.9 preserves public-core safety and OCEL visibility.

## Consolidation Artifacts

Subject components identify each v0.24.x provider subject, introduced version, skills, status, read-only/execution capability, visibility in OCEL/PIG/OCPX, and limitations. The capability map lists implemented capabilities and forbidden effects. The coverage matrix records model, service, CLI, tests, boundary tests, docs, OCEL, PIG, OCPX, and safety visibility.

The safety boundary report allows nonzero counts only for bounded observations and prior gated bounded execution history. It requires zero counts for command rerun, uncontrolled execution, unrestricted shell, arbitrary subprocess, network, package install, destructive command, unexpected file mutation, credential exposure, raw secret output, raw output dump, external provider adapter, external runtime touch, Schumpeter split, and LLM judge.

The runtime boundary report checks candidate provider, static safety, declared preflight, gated execution boundary, output/failure explanation, bounded runner isolation, shell false, argv only, single-use authorization, timeout, output cap, redaction, side-effect scan, and no unrestricted shell/network/package/destructive execution.

The roadmap report keeps v0.25 General Agent Usability, v0.26 Workspace Agent Workbench, v0.27 Memory Candidate / Continuity, v0.28 Schumpeter split preparation, v0.29+ external provider adapters, and GrowthKernel bridge deferred.

## Acceptance Criteria

- `InternalProviderConsolidationReport` can be created.
- Release manifest, v0.25 readiness, safety boundary, runtime boundary, and workbench snapshot can be created.
- Release status and readiness status are computed.
- `ready_for_v0_25` can be true when criteria pass.
- `ready_for_v0_26=false`.
- No new provider invocation, command execution, rerun, subprocess, shell, repair, patch, file mutation, external adapter, General Agent UX, Schumpeter split, credential exposure, raw secret output, raw output dump, or LLM judge is introduced.
- OCEL/PIG/OCPX visibility exists.

## Withdrawal Conditions

Withdraw readiness if any required v0.24.x subject is missing, dangerous safety counts are nonzero, uncontrolled execution appears, unrestricted shell or arbitrary subprocess appears, command rerun or repair appears, unexpected mutation occurs, credential/raw secret exposure occurs, external adapter or Schumpeter split logic appears, GrowthKernel becomes a runtime dependency, General Agent UX is implemented prematurely, or an LLM judge is used.

## Validity Horizon

Valid until v0.25 Agent Surface work begins or internal provider policy changes.
