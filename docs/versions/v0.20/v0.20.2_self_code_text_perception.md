# ChantaCore v0.20.2 - Self-Code/Text Perception

## Version Definition

- Version: `0.20.2`
- Version name: `Self-Code/Text Perception`
- Korean name: `자기 코드·텍스트 지각`
- Track: `OCEL-native Self-Awareness`
- Effect type: `read_only_observation`

## Objective

v0.20.2 implements bounded, policy-gated, redacted text perception for files inside the public ChantaCore workspace.

Self-awareness is not self-modification.

This is perception, not search or summary. The runtime may observe an explicitly requested text slice from one approved file. It must not scan the workspace for keywords, summarize content, parse symbols, parse ASTs, edit files, execute commands, or promote content into memory/persona/overlay state.

## Relationship To Earlier v0.20 Versions

v0.20.0 introduced the `self_awareness` contract layer.

v0.20.1 introduced metadata-only self-workspace awareness:

- `skill:self_awareness_workspace_inventory`
- `skill:self_awareness_path_verify`
- workspace path policy
- metadata-only inventory

v0.20.2 preserves those capabilities and implements only:

- `skill:self_awareness_text_read`

Later skills such as workspace search, Python symbols, markdown structure, project structure, config surface, test surface, capability registry, runtime boundary, plan candidates, and todo candidates remain contract-only.

## Included Scope

- Single-file text perception inside the workspace.
- Path policy reuse from v0.20.1.
- Bounded read budgets by bytes and lines.
- Preview, line range, head, and tail modes.
- Secret-like file blocking.
- Binary file blocking.
- Private-boundary blocking.
- Redaction for secret-like text and private path literals.
- CLI visibility for blocked and allowed reads.
- PIG/OCPX coverage for limited text perception.

## Non-Goals

v0.20.2 does not implement workspace search, semantic search, code symbol parsing, AST parsing, markdown summary, deterministic summary, LLM summary, file write/edit behavior, shell/test/lint execution, network fetch, MCP connection, plugin loading, external harness execution, sidecar/event bus behavior, memory/persona/overlay mutation, or plan/todo materialization.

File mutation and execution are not part of v0.20.2.

## Data Models

- `SelfTextReadRequest`
- `SelfTextReadPolicyDecision`
- `SelfTextSlice`
- `SelfTextRedactionFinding`
- `SelfTextReadResult`

The result carries either a bounded redacted `SelfTextSlice` or a structured blocked decision. Blocked reads do not include content.

## Service Design

- `SelfTextReadPolicyService`
  - `classify_file_for_text_read(path_resolution)`
  - `decide(request)`
- `SelfTextReaderService`
  - `read_text_slice(request, decision)`
- `SelfTextRedactionService`
  - `redact(text)`
- `SelfCodeTextPerceptionSkillService`
  - `read_text(request)`

User-facing skill invocation goes through the existing explicit read-only gate and explicit skill invocation surface.

## Text Read Policy

The policy reuses v0.20.1 workspace path resolution. It blocks traversal, outside-workspace paths, private boundaries, symlink traversal, missing files, directories, binary files, unsupported extensions, `.env` and `.env.*`, and obvious key/certificate-like files.

Decode errors return structured blocked results.

## Read Budget

Default read budget:

- `max_bytes=16384`
- `max_lines=300`

Hard caps:

- `max_bytes=65536`
- `max_lines=1000`

Preview and head modes read from the beginning. Tail mode returns the bounded tail slice. Line range mode returns only the requested line interval, subject to the same byte and line caps. Oversized reads return truncated bounded slices rather than unbounded content.

## Redaction Policy

Initial redaction covers:

- AWS-style access keys
- private key headers
- password/token/api_key/secret assignment-like lines
- Bearer token-like values
- GitHub token-like prefixes
- private workspace/member path literals
- high-entropy token-like strings

When redaction applies, returned content is redacted and `SelfTextRedactionFinding` entries are recorded.

## CLI Commands

- `python -m chanta_core.cli self-awareness text read README.md --max-lines 40`
- `python -m chanta_core.cli self-awareness text read README.md --start-line 1 --end-line 20 --mode line_range`
- `python -m chanta_core.cli self-awareness text read README.md --mode head --max-lines 80`
- `python -m chanta_core.cli self-awareness text read README.md --mode tail --max-lines 80`
- `python -m chanta_core.cli self-awareness text read .env`

CLI output shows workspace-relative path, mode, line range, bytes read, lines read, truncation status, redaction status, and `effect=read_only_observation`. Blocked reads show the blocked finding and do not print content.

## Tests

Tests cover allowed reads, bounded previews, line ranges, head/tail modes, traversal/outside/private-boundary blocking, directory/missing/binary/secret file blocking, oversized truncation, redaction, CLI behavior, safety flags, PIG/OCPX coverage, v0.20.0 contract preservation, and v0.20.1 workspace awareness preservation.

## Acceptance Criteria

- Text read works only for allowed text-like files inside workspace.
- Reads are bounded by `max_bytes`, `max_lines`, and line range.
- Binary, secret, private, outside, and traversal paths are blocked.
- Secret-like content is redacted before return.
- No search, summary, or symbol parsing is implemented.
- No write/shell/network/MCP/plugin/external harness execution occurs.
- No memory/persona/overlay mutation occurs.
- OCEL/PIG/OCPX visibility exists.
- v0.20.0 and v0.20.1 tests still pass.

## Withdrawal Conditions

Withdraw v0.20.2 acceptance if any path can bypass workspace policy, if any read is unbounded, if blocked reads expose content, if secret-like content is returned unredacted, if search/summary/symbol parsing is introduced, if workspace/canonical state is mutated, if external execution is enabled, or if private material is added to public code/docs/tests.

## Next Version

`v0.20.3 Self-Code Search Awareness` may add explicit bounded search awareness only if v0.20.2 bounded text perception remains policy-gated, redacted, and non-mutating.
