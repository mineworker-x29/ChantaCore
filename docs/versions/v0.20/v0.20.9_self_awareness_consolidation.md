# v0.20.9 Self-Awareness Consolidation

Korean name: 자각 계층 통합 정리

Track: OCEL-native Self-Awareness

## Version Definition

v0.20.9 consolidates v0.20.0 through v0.20.8 into the release-ready
Self-Awareness Foundation v1 unit.

In the release-line orientation, `v0.20.x` is OCEL-native Self-Awareness: the
system observes its own workspace, code, project surfaces, candidates,
verification reports, and intentions as OCEL object-centric processes.

Self-awareness is not self-modification.

Self-awareness consolidation is release readiness, safety closure, and gap
registration.

In plain terms, this release is release readiness, safety closure, and gap registration.

## Objective

This version produces ecosystem, capability, coverage, safety, candidate,
verification, gap, release manifest, and consolidation report read models. It
does not add new perception, search, summary, verification, intention, planning,
execution, approval, promotion, materialization, or mutation capabilities.

## Relationship To v0.20.0 Through v0.20.8

- v0.20.0 introduced the self-awareness layer contract.
- v0.20.1 introduced workspace path verification and metadata-only inventory.
- v0.20.2 introduced bounded text/code perception.
- v0.20.3 introduced bounded literal workspace search.
- v0.20.4 introduced deterministic structure summarization.
- v0.20.5 introduced project surface structure awareness.
- v0.20.6 introduced self-surface verification.
- v0.20.7 introduced self-directed intention candidates.
- v0.20.8 introduced the read-only self-awareness workbench.
- v0.20.9 closes the foundation track as a release-readiness unit.

## Non-Goals

v0.20.9 does not implement candidate approval, candidate promotion, task queue
creation, plan execution, todo materialization, file write/edit/apply_patch,
shell/test/lint execution, network/web fetch, MCP connection, plugin loading,
external harness execution, memory/persona/overlay mutation, LLM judge, LLM
summary, or autonomous mission loop.

## Service Design

The consolidation service reuses the self-awareness registry, PIG/OCPX report
service, and v0.20.8 workbench snapshot. It generates read-only count, status,
and reference summaries only.

## Readiness Rules

Readiness is blocked if dangerous, mutation, promotion, materialization,
private-boundary, raw-secret, or executable-candidate counts are nonzero, or if
required coverage is missing for an implemented capability.

Readiness is warning when only future-track or warning gaps remain. It is ready
only when implemented coverage is complete and all dangerous counts are zero.

## Safety Boundary Report

The safety report tracks write, shell, network, MCP, plugin, external harness,
memory, persona, overlay, canonical promotion, materialization, dangerous
capability, private boundary, and raw secret exposure counts.

## Candidate Inventory

The candidate inventory reports summary, project structure, verification, plan,
todo, no-action, and needs-more-input candidate counts. Promoted, materialized,
or executable candidates become violations.

## Gap Register

Required future gaps are recorded for config surface, test surface, capability
registry, runtime boundary, write/edit safety, shell execution safety,
network/MCP/plugin safety, external adapter implementation, mission loop, and
GrowthKernel bridge. These are future-track gaps unless the current release
claims them as implemented.

## Release Manifest

The release manifest defines Self-Awareness Foundation v1, includes v0.20.0
through v0.20.9, excludes write/shell/network/MCP/plugin/external harness and
self-execution capabilities, and lists future work without treating those
excluded capabilities as part of v0.20.x.

## Consolidation Report

The consolidation report ties together the ecosystem snapshot, capability map,
coverage matrix, safety report, candidate inventory, verification summary,
workbench snapshot, gap register, and release manifest.

## OCEL Mapping

v0.20.9 extends the self-awareness mapping with consolidation object, event,
and relation types. OCEL remains the canonical substrate. JSONL is not
introduced as a canonical store.

## Tests

Tests cover ecosystem snapshot, capability map, coverage matrix, safety report,
candidate inventory, verification summary, gap register, release manifest,
consolidation report, CLI commands, PIG/OCPX, non-goal drift, and regression
preservation.

## Acceptance Criteria

- Consolidation report builds.
- Release manifest builds.
- Gap register builds.
- Safety report confirms dangerous/mutation/promotion/materialization counts
  are zero.
- Capability map and coverage matrix include v0.20.0 through v0.20.8.
- Remaining gaps are future-track or warning, not blockers.
- No new execution, promotion, materialization, or mutation is introduced.
- No write/shell/network/MCP/plugin/external harness execution occurs.
- OCEL/PIG/OCPX visibility exists.
- v0.20.0 through v0.20.8 tests still pass.

## Withdrawal Conditions

Withdraw release readiness if consolidation adds new capability expansion,
executes, approves, promotes, materializes, mutates workspace/memory/persona or
overlay state, exposes private material, prints raw file content, prints raw
secrets, exposes full private paths, uses LLM judge/summary, connects to MCP,
loads plugins, or performs network/web fetches.

## Validity Horizon

Valid until the next self-awareness track expands capability scope or safety
assumptions change.
