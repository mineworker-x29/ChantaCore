# v0.20.6 Self-Surface Verification

Korean name: 자기 표면 검증 자각

Track: OCEL-native Self-Awareness

## Version Definition

v0.20.6 introduces deterministic self-surface verification for the public ChantaCore self-awareness outputs created from v0.20.1 through v0.20.5.

Self-awareness is not self-modification.

Verification reports are not canonical memory.

## Objective

ChantaCore can verify the structure of self-awareness outputs: path resolution, workspace inventory, text read, workspace search, markdown structure, python symbols, project structure, project surface candidates, redaction findings, truncation findings, and candidate promotion boundaries.

This is evidence checking, not test execution or LLM judging. It does not claim semantic truth. It checks whether a surface carries enough evidence, remains within read-only boundaries, and preserves candidate-only semantics.

## Relationship To Earlier Versions

- v0.20.0 introduced the self_awareness layer contract.
- v0.20.1 introduced workspace path verification and metadata-only inventory.
- v0.20.2 introduced bounded text/code perception.
- v0.20.3 introduced bounded literal workspace search.
- v0.20.4 introduced deterministic markdown/python structure summarization.
- v0.20.5 introduced deterministic project surface structure awareness.

Reports default to `review_status=candidate_only`, `canonical_promotion_enabled=false`, and `promoted=false`.

## Service Design

- `SelfSurfaceVerificationPolicyService` blocks unsupported target types.
- `EvidenceConsistencyVerifier` checks evidence refs, line refs, search matches, markdown headings, python symbols, surface candidate evidence, and truncation limitations.
- `BoundaryConsistencyVerifier` detects unexpected mutation/execution flags and secret-like returned strings.
- `CandidateSemanticsVerifier` detects promotion and memory/persona/overlay mutation violations.
- `SelfSurfaceVerificationService` composes the checks and produces the report.

## Evidence Verification Policy

Evidence checks are structural. v0.20.6 does not re-read full files. Missing `evidence_refs`, invalid line refs, search matches without source path/line number, headings or symbols without line numbers, weak surface candidates, and truncated outputs without limitations become findings.

## Boundary Verification Policy

Verification detects `mutates_workspace`, `mutates_memory`, `mutates_persona`, `mutates_overlay`, `uses_shell`, `uses_network`, `uses_mcp`, `loads_plugin`, `executes_external_harness`, and `dangerous_capability`. It also checks for secret-like returned content and missing `read_only_observation` effect metadata.

## Candidate Semantics Verification Policy

Candidate outputs must remain `candidate_only`. `canonical_promotion_enabled=true`, `promoted=true`, or memory/persona/overlay mutation flags are violations. The verification report itself is also candidate-only and never promoted.

## Status Rules

- Critical finding: `failed`
- Error finding: `failed`
- Warning finding only: `warning`
- Info/ok findings only: `passed`
- Unsupported target type: `blocked`
- `strictness=strict` upgrades warning findings to errors.

## CLI Commands

- `python -m chanta_core.cli self-awareness verify surface --target-type self_awareness_layer --strictness strict`
- `python -m chanta_core.cli self-awareness verify surface --target-type project_structure_candidate --target-id <id>`
- `python -m chanta_core.cli self-awareness verify surface --target-payload <json-file>`

CLI output shows target type, target id, status, finding counts by severity, evidence coverage, boundary status, candidate status, promotion flags, and `content_printed=false`.

## PIG/OCPX Report Update

- `llm_judge_enabled=false`
- `test_execution_enabled=false`
- `shell_execution_enabled=false`
- `canonical_promotion_enabled=false`
- `memory_mutation_enabled=false`
- `persona_mutation_enabled=false`
- `overlay_mutation_enabled=false`

OCPX state is `self_surface_verification_awareness`. Candidate types include `surface_verification_report`.

## Tests

Tests cover valid and invalid payloads, evidence refs, line refs, search match source fields, heading/symbol line numbers, weak surface candidates, truncation limitations, secret-like snippets, mutation/execution flags, candidate promotion violations, strictness behavior, CLI output, PIG/OCPX coverage, and earlier self-awareness regressions.

## Acceptance Criteria

- Surface verification works for self-awareness outputs.
- Missing or invalid evidence is reported.
- Boundary violations are detected.
- Candidate promotion violations are detected.
- Verification report is candidate-only and not promoted.
- No test/lint/shell/network/MCP/plugin/external harness execution occurs.
- No LLM judge is used.
- No memory/persona/overlay mutation occurs.
- OCEL/PIG/OCPX visibility exists.
- v0.20.0 through v0.20.5 tests still pass.

## Withdrawal Conditions

Withdraw v0.20.6 if it executes tests, lint, shell commands, network calls, MCP, plugins, external harnesses, LLM judging, semantic truth verification, file writes, candidate promotion, or memory/persona/overlay mutation, or if it exposes raw private/secret material.
