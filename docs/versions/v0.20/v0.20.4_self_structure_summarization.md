# v0.20.4 Self-Structure Summarization

Korean name: 자기 구조 요약 자각

Track: OCEL-native Self-Awareness

## Version Definition

v0.20.4 introduces deterministic structure summary candidates for allowed workspace text and code files. Self-awareness is not self-modification.

This version follows v0.20.0 self-awareness contracts, v0.20.1 workspace path and inventory policy, v0.20.2 bounded text perception and redaction, and v0.20.3 bounded literal search.

## Objective

The objective is deterministic structure extraction, not LLM summary. Summary candidates are not canonical memory. They are read-only evidence candidates that remain `candidate_only` unless a future reviewed promotion path is designed separately.

## Included Scope

- `skill:self_awareness_markdown_structure`
- `skill:self_awareness_python_symbols`
- Markdown headings, frontmatter keys, and section line ranges
- Python imports, from-imports, top-level functions, async functions, classes, and simple assignments
- Helper summaries for JSON, YAML, TOML shallow keys
- Explicit or fallback plain text preview
- Candidate semantics with `canonical_promotion_enabled=false` and `promoted=false`

## Non-Goals

- LLM summary
- Semantic summary
- Embedding or vector summary
- Cross-file dependency graph
- Import resolution
- Full project architecture inference
- Runtime introspection
- Config/test surface public skill activation
- File write/edit/delete
- Shell, network, MCP, plugin, or external harness execution
- Memory, persona, or overlay mutation
- Plan or todo materialization

## Service Design

- `SelfStructureSummaryPolicyService` reuses path and text read policy.
- `MarkdownStructureExtractor` extracts heading structure and frontmatter keys.
- `PythonSymbolExtractor` uses bounded approved text and safe AST parsing without execution.
- `ShallowKeyExtractor` extracts key names only.
- `SelfStructureSummaryService` creates the candidate.
- `SelfStructureSummarizationSkillService` exposes markdown and Python skill routes.

## Markdown Extraction Policy

Markdown extraction is deterministic. It records heading level, title, line number, slug, heading count, maximum heading depth, frontmatter key names, and section line ranges. It does not include section body content and does not produce LLM prose.

## Python Symbol Extraction Policy

Python extraction parses only bounded, policy-approved, redacted text. It never executes code, never resolves imports, and never builds dependency graphs. Syntax errors become `parse_error` values instead of crashes.

## Shallow Key Extraction Policy

JSON and TOML use safe parsers within read budget. YAML uses conservative line-based key extraction. Values are never exposed.

## Summary Candidate Semantics

Candidates are evidence objects, not canonical memory. Every candidate has:

- `review_status=candidate_only`
- `canonical_promotion_enabled=false`
- `promoted=false`
- `evidence_refs`
- `limitations`
- `confidence`

## OCEL Mapping

Object coverage adds structure summary request, structure summary candidate, markdown heading, Python symbol, shallow key summary, plain text preview summary, and summary policy decision.

Event coverage adds structure summary requested, policy checked, blocked, markdown extracted, Python symbols extracted, shallow keys extracted, plain text preview created, candidate created, truncated, and redaction applied.

Relation coverage adds summarizes file, extracts heading, extracts symbol, extracts key, and produces summary candidate.

OCEL remains the canonical process substrate. JSONL is not introduced as the canonical store.

## CLI Commands

- `python -m chanta_core.cli self-awareness summarize markdown README.md`
- `python -m chanta_core.cli self-awareness summarize python src/chanta_core/cli/main.py`
- `python -m chanta_core.cli self-awareness summarize auto pyproject.toml`
- `python -m chanta_core.cli self-awareness summarize auto config.yaml --max-lines 300`

CLI output shows workspace-relative path, summary kind, candidate id, review status, promotion flags, and structure metadata only. It does not show private full paths, raw secret values, full body content, or LLM summary text.

## PIG/OCPX Report Update

PIG report includes:

- `markdown_structure=implemented_deterministic`
- `python_symbols=implemented_deterministic`
- `shallow_key_summary=helper_deterministic`
- `project_structure=not_implemented`
- `config_surface=not_implemented`
- `test_surface=not_implemented`
- `llm_summary_enabled=false`
- `canonical_promotion_enabled=false`

OCPX state is `deterministic_structure_summary_awareness`, with candidate type `summary_candidate` and effect types `read_only_observation` and `state_candidate_created`.

## Tests

Tests cover markdown headings, frontmatter keys, Python imports/functions/classes/assignments, syntax errors, JSON/TOML/YAML shallow keys, plain text preview, read budgets, blocked paths/files, redaction, candidate-only semantics, CLI, reports, and regression across v0.20.0 through v0.20.3.

## Acceptance Criteria

- Markdown and Python deterministic structure summaries work for allowed files.
- Summary candidates are `candidate_only` and not promoted.
- No LLM summary is implemented or invoked.
- No secret, private, or raw body content is exposed.
- No write/shell/network/MCP/plugin/external harness execution occurs.
- No memory/persona/overlay mutation occurs.
- OCEL/PIG/OCPX visibility exists.
- v0.20.0, v0.20.1, v0.20.2, and v0.20.3 tests still pass.

## Withdrawal Conditions

Withdraw this version if summaries bypass workspace policy, expose raw secret/private content, create canonical memory, use model-generated summary text, mutate state, or enable external execution.
