# v0.21.8 - Deep Self-Introspection Workbench

Korean name: 심층 자기 관조 워크벤치

Track: Deep Self-Introspection

## Version Definition

v0.21.8 implements the read-only OCEL-native workbench for the Deep
Self-Introspection layer. It provides a sanitized operator surface over
capability truth, runtime boundary, policy/gate truth, trace integrity, context
projection, candidate/memory boundary, claim consistency, contradiction
register, findings, safety boundary, and OCEL/PIG/OCPX coverage.

## Core Principles

- Deep self-introspection is not self-modification.
- Workbench is not correction.
- Workbench is not approval.
- Workbench is not promotion.
- Workbench is not execution.
- Workbench is an OCEL-native read model.

## Relationship To Earlier Versions

- v0.20.9 provides the Self-Awareness Foundation v1.
- v0.21.0 provides the OCEL-native Deep Self-Introspection Contract.
- v0.21.1 provides self-capability registry awareness.
- v0.21.2 provides self-runtime boundary awareness.
- v0.21.3 provides self-policy/gate awareness.
- v0.21.4 provides self-trace integrity awareness.
- v0.21.5 provides self-context projection awareness.
- v0.21.6 provides self-candidate/memory boundary awareness.
- v0.21.7 provides self-claim consistency and contradiction register.

v0.21.8 reads these subject reports and summarizes them. It does not mutate
them.

## Included Scope

- Workbench snapshot construction.
- Version and subject coverage rows.
- Subject status rows.
- Findings and contradiction counts.
- Safety boundary counts.
- OCEL object/event/relation coverage.
- PIG/OCPX coverage status.
- Read-only CLI output.

## Non-Goals

v0.21.8 does not implement correction, approval, promotion, execution, claim
rewrite, contradiction resolution, trace repair/backfill/rewrite, context
injection, prompt/system prompt mutation, memory/persona/overlay mutation,
candidate materialization, task queue creation, network/web fetch, MCP
connection, plugin loading, external harness execution, LLM judge, or LLM
planner.

## Service Design

- `DeepSelfCoverageService` builds subject coverage rows.
- `DeepSelfSubjectStatusService` reads subject reports.
- `DeepSelfSafetyBoundaryService` inspects safety counters.
- `DeepSelfFindingsService` aggregates sanitized findings.
- `DeepSelfContradictionViewService` renders unresolved contradiction counts.
- `DeepSelfOCELCoverageService` checks mapping coverage.
- `DeepSelfPigOCPXStatusService` checks PIG/OCPX visibility.
- `DeepSelfWorkbenchService` builds the snapshot and CLI surface.

## Workbench Status Rules

OK:
- implemented subjects have report visibility;
- no critical/error workbench findings exist;
- safety boundary counters are zero;
- OCEL/PIG/OCPX coverage is complete.

Warning:
- warning findings exist;
- optional projection is partial;
- low-risk contradictions are open;
- coverage is partial.

Violation:
- safety boundary count is nonzero;
- claim consistency failed;
- trace integrity failed;
- candidate/memory boundary failed;
- context projection private payload risk exists;
- critical/error findings exist.
