# v0.21.7 - Self-Claim Consistency & Contradiction Register

Korean name: 자기 주장 정합성·모순 등록부

Track: Deep Self-Introspection

## Version Definition

v0.21.7 implements the seventh concrete OCEL-native deep self-introspection subject:
read-only self-claim consistency and contradiction registration.

It checks whether ChantaCore self-claims are supported by OCEL evidence and
consistent with capability truth, runtime boundary, policy/gate truth, trace
integrity, context projection, and candidate/memory boundary state.

## Core Principles

- Deep self-introspection is not self-modification.
- LLM statement is not evidence.
- Claim without OCEL evidence is not truth.
- Contradiction register is not auto-correction.
- Claim consistency check is not claim rewriting.

## Relationship To Earlier Versions

- v0.20.9 provides the OCEL-native Self-Awareness Foundation v1.
- v0.21.0 defines the Deep Self-Introspection Contract.
- v0.21.1 provides self-capability registry awareness.
- v0.21.2 provides self-runtime boundary awareness.
- v0.21.3 provides self-policy/gate awareness.
- v0.21.4 provides self-trace integrity awareness.
- v0.21.5 provides self-context projection awareness.
- v0.21.6 provides self-candidate/memory boundary awareness.

v0.21.7 reads those states as evidence sources. It does not mutate them.

## Included Scope

- Structured claim payload intake.
- Deterministic claim extraction from known structured sources.
- Evidence requirement mapping.
- Evidence matching against OCEL/deep-self truth read models.
- Unsupported, contradicted, and unverifiable claim classification.
- Read-only contradiction register entries with status `open`.
- CLI, PIG, OCPX, and OCEL mapping visibility.

## Non-Goals

v0.21.7 does not implement claim auto-fix, assistant response rewrite, prompt
modification, system prompt modification, memory/persona/overlay mutation,
capability registry mutation, policy/gate mutation, trace repair/backfill,
candidate promotion, materialization, external execution, or LLM judging.

## Service Design

- `SelfClaimSourceService` loads sanitized report, workbench, projection,
  candidate, assistant-message, or payload source refs.
- `SelfClaimExtractor` extracts structured claims deterministically.
- `ClaimEvidenceRequirementMapper` maps claim types to required evidence.
- `ClaimEvidenceMatcher` reads capability/runtime/policy/trace/context and
  candidate-memory truth state as evidence.
- `SelfClaimConsistencyFindingService` creates findings.
- `SelfContradictionRegisterService` creates read-only open register entries.
- `SelfClaimConsistencyService` composes the report.

## Claim Evidence Rules

- `capability_claim` requires `capability_record` and `capability_truth_report`.
- `execution_claim` requires `execution_envelope` and an OCEL event.
- `read_claim` requires `self_text_read_completed`, `text_slice`, and
  `workspace_file`.
- `search_claim` requires `workspace_search_result` and
  `workspace_search_match`.
- `verification_claim` requires a verification report.
- `promotion_claim` requires explicit promotion evidence, but current boundary
  state disables promotion.
- `memory_claim` requires explicit memory entry/write evidence, but current
  boundary state disables memory mutation.
- `canonical_truth_claim` cannot be satisfied by candidate-only, report-only,
  or projection-only objects.

## Truth Check Rules

Passed:
- all checked claims are supported by OCEL/deep-self evidence;
- no contradicted claims exist;
- candidate/report/projection-only objects are not claimed as canonical truth.

Warning:
- low-risk claims are unverifiable;
- informational claims lack enough structure;
- evidence is partial or stale.

Failed:
- capability, runtime, policy, execution, memory, promotion,
  materialization, external-contact, or canonical-truth claims contradict
  current deep-self evidence.

Blocked:
- the source would expose private/raw material;
- the source type is unsupported;
- evidence source access would violate the public boundary.

## Acceptance Criteria

- Claim consistency report builds.
- Contradiction register builds.
- Claims are checked against OCEL evidence and deep-self truth states.
- Unsupported, contradicted, and unverifiable claims are classified.
- Capability/runtime/policy/trace/context/candidate-memory contradictions are
  detected.
- No claim rewrite, auto-correction, memory mutation, candidate promotion,
  trace repair, LLM judge, or external execution occurs.
- OCEL/PIG/OCPX visibility exists.

## Withdrawal Conditions

Withdraw v0.21.7 readiness if claim truth is decided by LLM judgment, if
unsupported claims become supported without OCEL evidence, if contradiction
entries resolve automatically, if memory/persona/overlay or trace state is
mutated, or if OCEL mapping is removed.

reporting assumptions for deep self-introspection state.
