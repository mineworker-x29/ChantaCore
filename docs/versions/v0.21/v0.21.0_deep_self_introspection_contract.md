# v0.21.0 OCEL-native Deep Self-Introspection Contract

Korean name: OCEL-native 심층 자기 관조 계약

Track: Deep Self-Introspection

## Version Definition

v0.21.0 opens the Deep Self-Introspection track as a contract-only layer.
It follows v0.20.x OCEL-native Self-Awareness Foundation and depends on the
v0.20.9 Self-Awareness Foundation v1 consolidation.

Deep self-introspection is not self-modification.
Deep self-introspection is not execution.
Deep self-introspection is not permission escalation.
Deep self-introspection is not memory promotion.
Deep self-introspection must be OCEL-native.

## Objective

This version registers the `deep_self_introspection` layer, seed subjects, seed
skill IDs, risk contract, gate contract, observability contract, OCEL mapping
skeleton, PIG report skeleton, OCPX projection skeleton, CLI views, tests, and
restore documentation.

v0.21.0 is contract-only. It does not implement actual deep introspection
analysis.

## Relationship To v0.20.x

v0.20.x records workspace, code, project surfaces, verification reports,
candidate queues, intention candidates, workbench state, and release closure as
OCEL-native self-awareness read models. v0.21.0 does not replace those read
models. It declares the next layer that may later inspect runtime, capability,
policy, context, trace, candidate-memory, and self-claim consistency from those
OCEL-visible sources.

## Why OCEL-native

Deep self-introspection must be OCEL-native because it reasons over ChantaCore's
own process objects, events, relations, reports, envelopes, and read models. A
plain CLI dump or ad hoc report is insufficient unless it is backed by explicit
OCEL object/event/relation contracts and source references.

## Non-Goals

v0.21.0 does not implement actual capability registry analysis, runtime
boundary analysis, policy/gate mapping, trace integrity scan, context projection
analysis, candidate-memory boundary checking, self-claim consistency checking,
file write/edit/apply_patch, shell/test/lint execution, network/web fetch, MCP
connection, plugin loading, external harness execution, runtime credential
probing, provider API probing, memory/persona/overlay mutation, candidate
auto-promotion, permission escalation, task queue creation, autonomous mission
loop, LLM judge, or LLM planner.

## Service Design

`DeepSelfIntrospectionRegistryService` lists subjects, resolves a subject,
lists seed skill IDs, and builds the contract.

`DeepSelfIntrospectionConformanceService` checks that subjects and seed skills
exist, that OCEL mappings/source objects/read models are present, and that the
risk/gate contracts remain read-only and non-mutating.

`DeepSelfIntrospectionReportService` builds the PIG report skeleton, OCPX
projection skeleton, and contract report.

## OCEL Mapping

Object types include the deep self-introspection layer, contract, subject, skill
contract, risk profile, gate contract, observability contract, report, all seed
subject object types, self-awareness consolidation objects, execution envelope,
PIG report, and OCPX projection.

Event types include layer registered, contract registered, subject registered,
contract checked, conformance report created, PIG report created, and OCPX
projection created.

Relation types include subject declaration, mapping to OCEL object/event/relation,
source object requirements, read model requirements, report requirements,
contract checks, workbench visibility, self-awareness consolidation derivation,
and envelope recording.

Effect types are `read_only_observation` and `state_candidate_created`. OCEL
remains the canonical substrate. JSONL is not introduced as canonical store.

## CLI Commands

The repository package name is `chanta_core`.

- `python -m chanta_core.cli deep-self contract`
- `python -m chanta_core.cli deep-self subjects`
- `python -m chanta_core.cli deep-self show subject:capability_registry`
- `python -m chanta_core.cli deep-self conformance`
- `python -m chanta_core.cli deep-self pig-report`
- `python -m chanta_core.cli deep-self ocpx-projection`

CLI output must show `layer=deep_self_introspection`, `status=contract_only`,
`read_only=True`, `ocel_native=True`, seed subjects, seed skill IDs, and disabled
mutation/execution/permission escalation. It must not print raw file content,
raw secrets, private material, or full private paths.

## PIG/OCPX Report

The PIG report skeleton uses version `v0.21.0`, layer
`deep_self_introspection`, source layer `self_awareness`, and requirement
`self_awareness_consolidation=v0.20.9`. It lists all seed subjects and shows
mutation, permission escalation, external execution, and LLM judge disabled.

The OCPX projection skeleton uses state
`deep_self_introspection_contract_registered`, references v0.20.x
self-awareness source read models, declares v0.21.x target read models, and
uses effect types `read_only_observation` and `state_candidate_created`.

## Tests

Tests cover contract construction, layer identity, seed subject and skill
registration, risk flags, gate flags, observability flags, subject OCEL mapping,
PIG/OCPX skeletons, CLI views, no JSONL canonical store, non-goal drift, and
v0.20.x/v0.20.9 regression preservation.

## Acceptance Criteria

- Deep self-introspection contract exists.
- All seed subjects and seed skills are registered.
- All seed skills are contract-only and non-executable.
- OCEL object/event/relation skeleton exists.
- PIG/OCPX skeletons exist.
- CLI read-only views work.
- No mutation, execution, permission escalation, or LLM judge is introduced.
- v0.20.9 and v0.20.x tests still pass.

## Withdrawal Conditions

Withdraw v0.21.0 if v0.20.9 consolidation is absent, any seed skill becomes
executable, actual analysis is implemented under the contract release, OCEL
mapping is removed, JSONL becomes canonical, LLM judge/planner behavior appears,
shell/env/provider probing appears, permission escalation is added, or
memory/persona/overlay/capability-registry/policy mutation is introduced.

## Validity Horizon

Valid until v0.21.1 begins subject-specific Self-Capability Registry Awareness
or until the deep self-introspection source graph assumptions change.
