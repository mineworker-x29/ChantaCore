# v0.21.2 Self-Runtime Boundary Awareness

Korean name: 자기 런타임 경계 관조

Track: Deep Self-Introspection

Previous: v0.21.1 Self-Capability Registry Awareness

Next: v0.21.3 Self-Policy/Gate Awareness

## Definition

v0.21.2 implements read-only OCEL-native awareness of ChantaCore's known
runtime boundary truth. It views runtime profile, runtime mode, track state,
workspace boundary, private boundary, canonical store policy, execution
boundary, and optional runtime claim contradictions.

Runtime boundary awareness is not runtime probing.

Runtime boundary truth > persona claim.

Runtime boundary view is read-only.

Deep self-introspection is not self-modification.

## Relationship

v0.20.9 consolidated the OCEL-native Self-Awareness Foundation v1. v0.21.0
registered the Deep Self-Introspection contract. v0.21.1 added capability
registry truth. v0.21.2 uses that capability truth as one source read model for
runtime boundary awareness.

## Included Scope

- Runtime profile and mode descriptor view
- Active and disabled track state view
- Workspace boundary summary using aliases
- Private boundary summary without private material
- Canonical store policy view
- Execution boundary view
- Runtime boundary truth report
- Claim-safe optional contradiction checks
- OCEL/PIG/OCPX visibility
- Read-only CLI views

## Non-Goals

v0.21.2 does not implement shell/env probing, broad environment dumps,
credential probing, provider/API probing, network fetch, MCP connection,
plugin loading, external harness execution, file write/edit/apply_patch,
shell/test/lint execution, runtime mode change, profile change, workspace root
change, private boundary change, permission grant creation, capability
enablement, task queue creation, scheduler registration, memory/persona/overlay
mutation, LLM judge, or LLM planner.

## Service Design

- RuntimeBoundarySourceService loads known public descriptors from existing
  registry and boundary contracts without probes.
- RuntimeBoundaryFindingService evaluates descriptor consistency.
- RuntimeBoundaryTruthCheckService creates report-only truth reports.
- SelfRuntimeBoundaryAwarenessService builds snapshots, truth reports, PIG
  reports, OCPX projections, and sanitized CLI output.

Missing information is represented as unknown, limitation, or finding. It is
not resolved through runtime probing.

## Truth Check Rules

Passed:

- canonical_store=ocel
- canonical_jsonl_allowed=false
- canonical_markdown_allowed=false
- private material and private full path exposure are disabled
- write/shell/network/MCP/plugin/external harness are disabled
- memory/persona/overlay mutation are disabled
- candidate promotion/materialization are disabled
- no unsafe runtime claim exists

Warning:

- track state is unknown
- runtime profile is partially unknown
- workspace scope is ambiguous
- optional boundary source is missing
- future tracks are present and correctly disabled

Failed:

- canonical store is not OCEL
- private material or private full path exposure is allowed
- write/shell/network/MCP/plugin/external harness is enabled
- memory/persona/overlay mutation is enabled
- candidate promotion/materialization is enabled
- ambient filesystem access claim is allowed
- autonomous runtime claim is allowed
- permission escalation claim is allowed

CLI output shows runtime profile/mode summary, active/disabled tracks,
workspace/private boundary summary, canonical store policy, execution boundary

secrets, environment variable dumps, credentials, or shell probe results.

## Tests

Tests cover snapshot construction, read-only flags, profile/mode descriptors,
track states, future safety tracks, workspace/private boundary descriptors,
canonical store policy, execution boundary disabled flags, truth check pass and
failure cases, no shell/env/provider probing, CLI commands, PIG report, OCPX
projection, and regression preservation for v0.21.1, v0.21.0, v0.20.x,
observation/digestion, and gate/envelope suites.

## Acceptance Criteria

- Runtime boundary snapshot builds.
- Runtime boundary truth report builds.
- Store policy confirms OCEL canonical.
- Disabled safety tracks are visible.
- Private boundary is visible and protected.
- Execution boundary confirms write/shell/network/MCP/plugin/external harness
  disabled.
- No runtime probing, credential probing, provider probing, permission
  escalation, or mutation occurs.
- OCEL/PIG/OCPX visibility exists.
- v0.21.1, v0.21.0, and v0.20.x tests still pass.

## Withdrawal Conditions

Withdraw this version's readiness if runtime boundary awareness performs
runtime probing, dumps environment variables, probes credentials or providers,
changes profile/mode/workspace/private boundary state, creates permission
grants, enables capabilities, mutates memory/persona/overlay, uses an LLM
judge/planner, or exposes private material, raw secrets, raw file content, or
full private paths.

## Validity Horizon

This judgment is valid until v0.21.3 Self-Policy/Gate Awareness changes policy
or gate assumptions.
