# v0.21.3 Self-Policy/Gate Awareness

Korean name: 자기 정책·게이트 관조

Track: Deep Self-Introspection

Previous: v0.21.2 Self-Runtime Boundary Awareness

Next: v0.21.4 Self-Trace Integrity Awareness

## Definition

v0.21.3 implements read-only OCEL-native awareness of ChantaCore's policy and
gate truth. It maps proposal policy, review policy, permission boundary,
execution gates, execution envelope policy, hard-block rules, candidate-only
policy, promotion gate, materialization gate, no_action policy,
needs_more_input policy, and optional policy claim contradictions.

Policy awareness is not policy mutation.

Gate awareness is not permission grant.

Review approval is not execution permission.

No-action is a valid policy outcome.

Deep self-introspection is not self-modification.

## Relationship

v0.20.9 consolidated the OCEL-native Self-Awareness Foundation v1. v0.21.0
registered the Deep Self-Introspection contract. v0.21.1 added capability
truth. v0.21.2 added runtime boundary truth. v0.21.3 uses those read models as
source state for policy/gate awareness.

## Included Scope

- Proposal and review policy descriptors
- Permission boundary descriptor
- Read-only, write, shell, network, MCP, plugin, external harness, promotion,
  and materialization gate descriptors
- Execution envelope policy descriptor
- Promotion and materialization gate descriptors
- Hard-block and candidate-only policy rules
- Claim-safe optional contradiction checks
- PIG/OCPX/OCEL visibility
- Read-only CLI views

## Non-Goals

v0.21.3 does not implement policy change, gate change, permission grant
creation, review decision creation, proposal accept/reject execution, skill
invocation execution, execution envelope execution, candidate promotion,
candidate materialization, file write/edit/apply_patch, shell/test/lint
execution, network/web fetch, MCP connection, plugin loading, external harness
execution, memory/persona/overlay mutation, LLM judge, or LLM planner.

## Service Design

- PolicyGateSourceService builds read-only descriptors from existing policy,
  review, gate, envelope, promotion, materialization, capability truth, and
  runtime boundary owners.
- PolicyGateFindingService evaluates unsafe or missing policy boundaries.
- PolicyGateTruthCheckService creates report-only truth reports.
- SelfPolicyGateAwarenessService builds snapshots, truth reports, PIG reports,
  OCPX projections, and sanitized CLI output.

Missing policy data is represented as unknown, limitation, or finding. It is
not converted into an implicit allow.

## Truth Check Rules

Passed:

- proposal/review separation holds
- review approval does not imply execution
- no_action is valid
- needs_more_input is valid
- deny_by_default=true
- permission grant creation is disabled
- permission escalation is disabled
- read-only invocation requires gate/envelope
- blocked attempts are envelope-visible
- write/shell/network/MCP/plugin/external harness gates are disabled or
  hard-blocked
- promotion/materialization gates are disabled
- raw private payload is not allowed

Warning:

- optional workbench visibility is partial
- some policy source is unknown
- reviewer note support is missing
- optional policy claim is unknown

Failed:

- review approval implies execution
- no_action is missing as valid outcome
- needs_more_input is missing as valid outcome
- permission grants are allowed unexpectedly
- permission escalation is allowed unexpectedly
- execution envelope is not required
- blocked attempts are not auditable
- write/shell/network/MCP/plugin/external harness gate is enabled
- promotion/materialization gate is enabled
- raw private payload is allowed

## Acceptance Criteria

- Policy/gate map snapshot builds.
- Policy/gate truth report builds.
- Review approval does not imply execution.
- No-action and needs-more-input are valid outcomes.
- Permission grant creation is disabled.
- Read-only gate/envelope requirement is visible.
- Write/shell/network/MCP/plugin/external harness gates are disabled or
  hard-blocked.
- Promotion/materialization gates are disabled.
- No policy/gate mutation, review decision, permission grant, skill invocation,
  or execution occurs.
- OCEL/PIG/OCPX visibility exists.
- v0.21.2, v0.21.1, v0.21.0, and v0.20.x tests still pass.

## Withdrawal Conditions

Withdraw this version's readiness if policy/gate awareness mutates policy or
gate state, creates review decisions, creates permission grants, invokes
skills, executes envelopes, enables candidate promotion/materialization,
performs file/shell/network/MCP/plugin/external execution, uses an LLM
judge/planner, mutates memory/persona/overlay, or exposes private material,
raw secrets, raw file content, or full private paths.
