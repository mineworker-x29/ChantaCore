# v0.21.4 Self-Trace Integrity Awareness

Korean name: 자기 트레이스 정합성 관조
Track: Deep Self-Introspection

Previous: v0.21.3 Self-Policy/Gate Awareness

Next: v0.21.5 Self-Context Projection Awareness

## Definition

v0.21.4 implements read-only OCEL-native awareness of ChantaCore's own trace
integrity. It checks whether OCEL events, objects, relations, execution
envelopes, candidates, evidence refs, and process chains are internally
consistent.

Trace integrity awareness is not trace repair.

Missing trace is a finding, not implicit backfill.

OCEL remains canonical.

Deep self-introspection is not self-modification.

## Relationship

v0.20.9 consolidated the OCEL-native Self-Awareness Foundation v1. v0.21.0
registered the Deep Self-Introspection contract. v0.21.1 added capability
truth. v0.21.2 added runtime boundary truth. v0.21.3 added policy/gate truth.
v0.21.4 uses those read models as source state for trace integrity awareness.

## Included Scope

- Envelope to OCEL event consistency checks
- Event-object coverage checks
- Object-relation integrity checks
- Candidate source/evidence lineage checks
- Process chain integrity checks
- Missing, dangling, orphan, and duplicate trace findings
- PIG/OCPX visibility for trace integrity state
- Read-only CLI views

## Non-Goals

v0.21.4 does not modify OCEL events, objects, relations, execution envelopes,
or candidates. It does not generate missing relations, replay events, re-emit
events, rewrite events, migrate databases, promote candidates, materialize
candidates, mutate memory/persona/overlay, grant permissions, mutate policy, or
use an LLM judge/planner.

## Integrity Rules

Passed when requested trace elements are linked, required event-object
relations exist, candidates have source/evidence lineage, blocked attempts are
envelope-visible, effect types match contracts, and JSONL is not canonical.

Warning when optional relations are missing, verification refs are absent for
low-risk candidates, process chain information is partial, or scope is
truncated.

Failed when an envelope lacks an OCEL event, an event requiring an envelope has
none, a required relation is missing, a relation is dangling, a candidate lacks
source/evidence lineage, JSONL is treated as canonical, or an effect type
mismatch involves mutation/execution.

Blocked when the requested scope is unsupported, OCEL source is unavailable, or
the check would expose private boundary material.

## Acceptance Criteria

- Trace integrity report builds.
- Envelope to OCEL consistency is checked.
- Event-object relation coverage is checked.
- Candidate lineage is checked.
- Process chain integrity is checked.
- Missing/dangling/orphan traces become findings.
- No trace repair, implicit backfill, replay, re-emission, or rewrite occurs.
- No mutation, execution, LLM judge, or external execution is introduced.
- OCEL/PIG/OCPX visibility exists.
- v0.21.3, v0.21.2, v0.21.1, v0.21.0, and v0.20.x tests remain green.

## Withdrawal Conditions

Withdraw this version if trace awareness modifies OCEL records, modifies
execution envelopes, modifies candidates, silently creates missing relations,
replays or re-emits events, treats JSONL as canonical, exposes private material,
or introduces mutation/execution/LLM judge behavior.

## Validity Horizon

about how process-aware context projections reference trace evidence.
