# v0.21.5 Self-Context Projection Awareness

Korean name: 자기 컨텍스트 투영 관조

Track: Deep Self-Introspection

Previous: v0.21.4 Self-Trace Integrity Awareness

Next: v0.21.6 Self-Candidate/Memory Boundary Awareness

## Definition

v0.21.5 implements read-only OCEL-native awareness of ChantaCore's own context
projection. It inspects which structured process-state sources are represented
in the current projection, which sources are omitted, compacted, stale, or
truncated, and whether candidate-only or raw transcript material is being
misrepresented as canonical truth.

Context projection awareness is not context injection.

Projection view is not canonical truth.

Raw transcript is not process-state.

Candidate-only must not appear as canonical memory.

Deep self-introspection is not self-modification.

## Relationship

v0.20.9 consolidated the OCEL-native Self-Awareness Foundation v1. v0.21.0
registered the Deep Self-Introspection contract. v0.21.1 added capability
truth. v0.21.2 added runtime boundary truth. v0.21.3 added policy/gate truth.
v0.21.4 added trace integrity truth. v0.21.5 uses those read models as source
state for context projection awareness.

## Included Scope

- Context projection source visibility
- Projected item metadata
- Omitted, compacted, stale, and truncated source accounting
- Candidate-only versus canonical truth separation
- Raw transcript as process-state detection
- Private payload projection risk detection
- Budget, freshness, gap, and risk findings
- PIG/OCPX visibility for context projection state
- Read-only CLI views

## Non-Goals

v0.21.5 does not inject context, modify prompts, modify system prompts, promote
memory, promote candidates, mutate projection sources, run compaction, create
automatic summaries, call an LLM summary/judge, canonicalize raw transcript,
write files, execute shell/test/lint, fetch network resources, connect MCP,
load plugins, execute external harnesses, grant permissions, mutate policy, or
mutate memory/persona/overlay state.

## Service Design

`ContextProjectionSourceService` reads existing context/projection-compatible
source state through sanitized references. It represents OCPX, PIG, workbench,
capability truth, runtime boundary, policy/gate, trace integrity, and candidate
memory boundary sources without exposing raw prompt bodies or raw transcripts.

Service components:

- `ContextProjectionSourceService`
- `ContextProjectionItemService`
- `ContextProjectionBudgetService`
- `ContextProjectionFreshnessService`
- `ContextProjectionGapService`
- `ContextProjectionTruthCheckService`
- `SelfContextProjectionAwarenessService`

Missing sources become gaps, findings, or limitations. No source is implicitly
allowed or silently canonicalized.

## Truth Rules

Passed when projection sources are OCEL/OCPX/PIG/workbench-derived,
candidate-only items are marked candidate-only and not canonical truth, stale
or truncated items are marked, capability/runtime/policy/trace truth sources
are present, raw transcript is not process-state, and private payloads are not
projected.

Warning when optional sources are omitted, projection is partially stale,
budget is near limit, non-critical sources are compacted, or old reports lack
freshness metadata.

Failed when a candidate is projected as canonical truth, raw transcript is used
as process-state, projection exceeds runtime boundary, projection claims
capabilities beyond capability truth, projection ignores policy/gate hard
blocks, projection ignores trace integrity failure, private payload is
projected, or truncation is unmarked.

Blocked when projection source is unavailable, checking would expose a private
payload, or projection scope is unsupported.

## Tests

Tests cover snapshot construction, read-only and non-mutating flags, OCPX/PIG
and workbench-derived source refs, raw transcript exclusion, projection item
flags, candidate-only truth separation, gap detection, freshness, budget,
private payload risk, runtime/capability/policy/trace contradictions, CLI
views, PIG/OCPX projection, and regression against v0.21.4 through v0.20.x.

## Acceptance Criteria

- Context projection snapshot builds.
- Projection sources, items, gaps, freshness, and budget are visible.
- Candidate-only items are not treated as canonical truth.
- Raw transcript is not treated as process-state.
- Projection risks and gaps become findings.
- No context injection, prompt mutation, compaction execution, LLM
  summary/judge, or memory promotion occurs.
- OCEL/PIG/OCPX visibility exists.
- v0.21.4, v0.21.3, v0.21.2, v0.21.1, v0.21.0, and v0.20.x tests remain green.

## Withdrawal Conditions

Withdraw this version if context projection awareness injects context, mutates
prompt content, runs compaction, promotes memory/candidates, treats raw
transcript as canonical process-state, projects private payloads, exposes raw
private material, or introduces mutation/execution/LLM judge behavior.

## Validity Horizon

Valid until v0.21.6 Self-Candidate/Memory Boundary Awareness changes the
candidate-memory boundary assumptions used by the projection view.
