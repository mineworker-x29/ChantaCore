# v0.21.6 Self-Candidate/Memory Boundary Awareness

Korean name: 자기 후보·기억 경계 관조

Previous: v0.21.5 Self-Context Projection Awareness

## Definition

v0.21.6 implements read-only OCEL-native awareness of ChantaCore's own
candidate/memory boundary. It checks whether candidates, reports, summaries,
verification outputs, intention bundles, projections, and workbench artifacts
remain separated from canonical memory, persona, overlay, materialized task
state, and executable state.

Candidate is not memory.

Report is not canonical truth.

Projection is not promotion.

Review status is not promotion status.

Candidate creation is not materialization.

## Relationship

v0.20.9 consolidated the OCEL-native Self-Awareness Foundation v1. v0.21.0
registered the Deep Self-Introspection contract. v0.21.1 added capability
truth. v0.21.2 added runtime boundary truth. v0.21.3 added policy/gate truth.
v0.21.4 added trace integrity truth. v0.21.5 added context projection truth.
v0.21.6 uses those read models as source state for candidate/memory boundary
awareness.

## Included Scope

- Candidate and report source refs
- Candidate-only and report-only state descriptors
- Promotion boundary
- Materialization boundary
- Memory write and auto-promotion boundary
- Persona and overlay mutation boundary
- Projection risk where candidates or reports are confused with canonical state
- PIG/OCPX visibility for candidate-memory boundary state
- Read-only CLI views

## Non-Goals

v0.21.6 does not promote candidates, create memory entries, update memory,
update persona, update overlays, materialize candidates, create todo files,
create task queues, register schedulers, mutate policy, mutate the capability
registry, mutate projection state, inject context, mutate prompts, write files,
execute shell/test/lint, fetch network resources, connect MCP, load plugins,
execute external harnesses, or use an LLM judge/planner.

## Service Design

`CandidateMemoryBoundarySourceService` reads candidate/report-compatible source
state through sanitized refs. It reads context projection and policy/gate state
only as source state. It does not promote candidates, mutate memory, mutate
persona, mutate overlay, or materialize anything.

Service components:

- `CandidateMemoryBoundarySourceService`
- `CandidateStateInspector`
- `CandidateMemoryBoundaryFindingService`
- `CandidateMemoryBoundaryTruthCheckService`
- `SelfCandidateMemoryBoundaryAwarenessService`

Missing or ambiguous source state becomes a finding or limitation, not an
implicit permission to promote or materialize.

## Truth Rules

Passed when all candidate/report artifacts remain candidate-only or
report-only, canonical promotion is disabled, promoted/materialized/executable
state is false, memory write and memory auto-promotion are disabled,
persona/overlay mutation is disabled, candidates are not projected as canonical
memory, and reports are not projected as canonical truth.

Warning when candidate metadata lacks optional verification or evidence refs
but remains candidate-only/report-only, or when older artifacts lack explicit
boundary metadata without evidence of promotion or mutation.

Failed when promoted, materialized, execution-enabled, canonical promotion,
memory auto-promotion, memory write, persona/overlay mutation, candidate as
canonical memory, report as canonical truth, or private persona material
exposure risk is detected.

Blocked when source state is unavailable, the requested scope is unsupported,
or checking would expose private material.

## Acceptance Criteria

- Candidate/memory boundary snapshot builds.
- Candidate/report states are visible.
- Promotion/materialization/memory/persona/overlay boundaries are visible.
- Candidate-only outputs are not treated as memory.
- Reports are not treated as canonical truth.
- Violations become findings.
- No promotion, memory mutation, persona/overlay mutation, materialization, LLM
  judge, or external execution occurs.
- OCEL/PIG/OCPX visibility exists.
- v0.21.5, v0.21.4, v0.21.3, v0.21.2, v0.21.1, v0.21.0, and v0.20.x tests remain green.

## Withdrawal Conditions

Withdraw this version if candidate/memory boundary awareness promotes or
materializes candidates, mutates memory/persona/overlay, treats candidates as
canonical memory, treats reports as canonical truth, exposes private persona
material, or lacks OCEL/PIG/OCPX mapping.
