# v0.35.4 Diff Proposal Envelope

Parent track: Controlled Patch Proposal Layer.

Release name: v0.35.4 Diff Proposal Envelope.

## Relationships

v0.35.4 builds on v0.35.3 Reference-informed Patch Plan & Change Set Graph. It consumes `PatchPlan`, `PatchChangeSetGraph`, and `PatchTargetFilePlan` as planning metadata.

It may also consume v0.35.2 `PatchContextSnapshot` and `PatchContextEvidenceBundle`, v0.35.1 `PatchIntentScopeBundle`, and the v0.35.0 `ReferencePatternDigest` as metadata only. These inputs are not executed, imported, installed, tested, or treated as runtime authority.

## Definition

`DiffProposalEnvelope` is the container for bounded diff proposal artifacts. It may hold:

- `UnifiedDiffProposal`
- `StructuredPatchProposal`
- `PatchFileProposal`
- `PatchHunkProposal`
- source refs, gaps, risks, validation reports, and readiness metadata

Diff proposal is not patch apply. Unified diff artifact text is not `git apply`. Structured patch proposal is not file write. Patch hunk proposal is not applied hunk. Patch file proposal is not file overwrite. Before/after preview is not filesystem mutation. Diff proposal validation is not approval.

## Reference-Informed Diff Policy

ReferencePatternDigest, context snapshots, evidence bundles, and patch plans may inform proposal metadata. Rejected reference patterns must stay rejected. Future-track reference patterns remain future-track metadata. No OpenCode, Hermes, or OpenClaw code is executed or imported.

## Taxonomies

Modes include plan-only no diff, unified diff artifact, structured patch artifact, combined unified and structured artifact, blocked, review required, no-op, and unknown.

Formats include unified diff, structured patch, file-level summary, hunk-level summary, before/after preview, proposal metadata only, no diff, and unknown.

Sources include v0.35.3 patch plan, change-set graph, target file plan, v0.35.2 context snapshot and evidence bundle, v0.35.1 intent/scope bundle, v0.35.0 ReferencePatternDigest, reference-informed pattern use, test fixture, and unknown.

Status and readiness values distinguish draft, created, created with gaps, validated, validated with warnings, blocked, review required, future gated, artifact readiness, design handoff to v0.35.5/v0.35.6, and not ready.

Decision and risk taxonomies cover artifact allowance, metadata-only allowance, design-stage handoff, denial/block/review, insufficient context, ungrounded or overbroad diff risk, scope escape, secret or credential exposure, copied code, licensing, unsafe readiness flags, provider/network, shell, dependency, test, reference execution, patch apply, workspace write, code edit, raw source dump, and unknown.

Hunk kinds include add, remove, replace, insert before, insert after, metadata-only change, documentation change, test change, safety boundary change, no-op hunk, and unknown.

Validation kinds include syntax shape, scope alignment, source context alignment, no-apply, no-write, no-secret, no-command, no-dependency-install, no-reference-execution, review-required, and unknown checks.

## Generation Policy

`DiffProposalGenerationPolicy` may allow bounded unified diff text, structured patch artifacts, and before/after previews. It must keep false:

- `allow_patch_apply`
- `allow_workspace_write`
- `allow_code_edit`
- `allow_test_execution`
- `allow_shell`
- `allow_dependency_install`

The policy requires no-apply and no-write checks. It is not apply permission.

## Flow

`PatchPlan + ChangeSetGraph + PatchContextSnapshot + EvidenceBundle + ReferencePatternDigest -> DiffProposalInput -> StructuredPatchProposal -> UnifiedDiffProposal -> DiffProposalEnvelope -> V0354ReadinessReport -> v0.35.5 Patch Risk & Conformance Scanner`

v0.35.6 Human Review Packet is also a design-stage handoff. v0.35.4 does not perform risk conformance scanning or human approval.

## Hard Prohibitions

v0.35.4 may create bounded diff proposal artifacts and unified diff artifact text. It may create structured patch proposal artifacts, file proposals, hunk proposals, and before/after preview metadata.

v0.35.4 does not apply patches. It does not call `apply_patch`. It does not call `git apply`. It does not write or edit files. It does not execute tests. It does not execute shell, subprocess, commands, package scripts, dependency installs, OpenCode, Hermes, OpenClaw, reference imports, provider SDKs, network calls, credential reads, secret reads, persistent trace writes, UI runtime, external control, or authority grants.

## Acceptance Criteria

- `DiffProposalEnvelope`, `UnifiedDiffProposal`, `StructuredPatchProposal`, `PatchFileProposal`, and `PatchHunkProposal` exist.
- Diff proposal generation policy permits bounded artifacts but blocks apply/write/edit/test/shell/install.
- Validation reports do not certify apply, write, edit, or execution.
- Bounded artifact text can be generated from supplied fake plan/context metadata.
- Missing plan/context metadata produces gaps, review-required, blocked, or future-gated metadata instead of unsafe inference.
- All unsafe readiness flags remain false.

## Withdrawal Conditions

Withdraw this release boundary if any runtime path is introduced for patch application, workspace write, code edit, `apply_patch`, `git apply`, shell/subprocess/command execution, test execution, dependency install, reference execution/import/install, provider/network/credential/secret access, persistent trace writes, UI runtime, external control, or authority grant.

## Handoff

v0.35.4 is ready only for v0.35.5 Patch Risk & Conformance Scanner and v0.35.6 Human Review Packet design-stage handoff. `ready_for_execution` remains false.
