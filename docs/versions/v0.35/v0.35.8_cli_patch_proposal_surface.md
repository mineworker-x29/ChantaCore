# v0.35.8 CLI Patch Proposal Surface

Release name: v0.35.8 CLI Patch Proposal Surface.

Parent track: Controlled Patch Proposal Layer.

v0.35.8 consumes v0.35.7 `PatchProposalTracePacket` metadata, v0.35.6 human review packet metadata, v0.35.5 risk report metadata, v0.35.4 diff proposal metadata, v0.35.3 patch plan metadata, v0.35.2 patch context metadata, and v0.35.1 intent/scope metadata. It also follows the bounded CLI lessons from v0.33.8 CLI Agent Run Surface and v0.34.8 CLI Model-backed Agent Step Surface.

## Definition

The CLI Patch Proposal Surface is a pure Python, bounded, in-memory parser and handler layer for patch proposal preview commands. It parses CLI-like argv lists into `CLIPatchProposalInvocation` artifacts, evaluates them against `CLIPatchProposalSurfacePolicy`, and returns `CLIPatchProposalRunOutput` metadata.

CLI surface is not shell execution. CLI argv parsing is not OS command execution. `patch-diff-preview` is not patch apply. `patch-review` is not approval-for-apply. `patch-trace-preview` is not persistent trace write. A denied command is not remediation execution.

v0.35.8 may expose bounded CLI-facing patch proposal handlers. v0.35.8 does not execute shell commands. It does not use subprocess. It does not apply patches. It does not call `apply_patch`. It does not call `git apply`. It does not write or edit files. It does not run tests. It does not install dependencies. It does not execute or import OpenCode, Hermes, or OpenClaw. It does not invoke Claude Code, Codex CLI, or external coding agents. It does not implement Dominion runtime. It does not implement infinite agent loops. It does not persist trace files or database records.

## Digestion And Dominion Policy

The CLI follows Digestion-first / Dominion-fallback policy. It may report external-agent-control-pattern awareness as metadata only. Commands that request `run-claude-code`, `run-codex`, `run-opencode`, `run-hermes`, `run-openclaw`, `dominion`, `external-agent-loop`, `infinite-loop`, `recursive-agent`, `agent-chain`, or `harness-execute` are denied or blocked. They are never executed.

OpenCode, Hermes, and OpenClaw references remain static metadata sources only. The CLI denies reference execution, import, install, test, and harness orchestration commands.

## Policy And Artifacts

`CLIPatchProposalSurfacePolicy` allows bounded preview booleans and blocks shell, subprocess, command execution, patch application, workspace write, code edit, `apply_patch`, `git apply`, test execution, dependency install, reference execution/import, external agent execution, Claude Code invocation, Codex CLI invocation, Dominion runtime, infinite agent loop, provider invocation, network access, credential access, secret read, persistent trace write, and UI runtime.

`CLIPatchProposalInvocation` stores argv as a bounded list of strings. The argv list is never passed to shell or subprocess.

`CLIPatchProposalInvocationDecision` may allow only bounded preview dispatch. All unsafe `*_allowed` booleans remain false.

`CLIPatchProposalRuntimeContext` is in-memory metadata only and cannot allow patch application, workspace write, code edit, shell, external agent execution, or Dominion runtime.

`CLIPatchProposalCommandResult`, `CLIPatchProposalRunOutput`, and `CLIPatchProposalRunReport` are bounded returned artifacts. Rendering output is not file write.

`CLIPatchProposalNoExternalSideEffectGuarantee` records no shell execution, no subprocess execution, no command execution, no patch application, no workspace write, no code edit, no test execution, no dependency install, no reference execution/import, no external agent execution, no Claude Code invocation, no Codex CLI invocation, no Dominion runtime, no infinite loop, no provider/network/credential/secret access, no persistent trace write, no UI runtime, and no authority grant.

Trace preview uses v0.35.7 returned packet creation only and does not persist traces.

## Safe Defaults

Safe defaults enable argument parsing, bounded preview dispatch, denied unsafe command handling, and in-memory rendered output only.

`ready_for_cli_patch_proposal_surface` may be true. `ready_for_bounded_cli_patch_proposal_preview` may be true. Intent/context/plan/diff/risk/review/trace preview readiness may be true. `ready_for_v0359_consolidation` may be true only as design-stage handoff.

`ready_for_patch_application` remains false. `ready_for_workspace_write` remains false. `ready_for_code_edit` remains false. `ready_for_shell_execution` remains false. `ready_for_external_agent_execution` remains false. `ready_for_dominion_runtime` remains false. `ready_for_persistent_trace_write` remains false. `ready_for_execution` remains false.

## Hard Prohibitions

v0.35.8 must not pass argv to subprocess or shell. It must not execute commands, apply patches, write files, edit code, call `apply_patch`, call `git apply`, run tests, install dependencies, execute/import references, invoke external agents, invoke Claude Code, invoke Codex CLI, implement Dominion runtime, implement infinite loops, call providers, use network, read credentials/secrets, persist traces, write OCEL/JSONL/log/database records, implement UI runtime, mutate registries or memory, or grant authority.

## Acceptance Criteria

- Safe CLI commands return bounded in-memory preview artifacts only.
- Unknown and unsafe commands are denied.
- Patch apply/write/edit/test/install/external-agent/Dominion commands are denied, blocked, no-op, or future-gated.
- CLI output rendering never writes to disk.
- Trace preview uses v0.35.7 returned trace packet only.
- All unsafe readiness flags remain false.

## Withdrawal Conditions

Withdraw this boundary if any runtime path passes argv to shell/subprocess, applies patches, writes or edits files, calls `apply_patch`, calls `git apply`, runs tests, installs dependencies, executes/imports references, invokes external agents, invokes Claude Code/Codex CLI, implements Dominion runtime, implements infinite loops, uses providers/network/credentials/secrets, persists trace files/logs/databases, implements UI runtime, or grants authority.

## Handoff

v0.35.8 is ready only for v0.35.9 Controlled Patch Proposal Layer Consolidation design-stage handoff. `ready_for_execution` remains false.
