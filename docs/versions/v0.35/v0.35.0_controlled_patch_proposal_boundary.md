# v0.35.0 Controlled Patch Proposal Boundary Foundation

Parent track: Controlled Patch Proposal Layer.

v0.35.0 builds on v0.34.9 Controlled Model Invocation Boundary v1. It defines the boundary for future controlled patch proposal artifacts, but it does not generate patch proposals, apply patches, write files, edit code, run tests, execute shell commands, install dependencies, or execute reference harnesses.

## Boundary Definition

Controlled patch proposal boundary means a contract layer that can describe future patch intent, scope, read-only context, change planning, diff proposal envelopes, review packets, and trace packets. In v0.35.0 it is boundary metadata only.

Patch proposal is not patch apply. Diff proposal is not file write. Patch plan is not code edit. Review metadata is not approval authority. Approval metadata is not apply permission.

Patch application is not part of v0.35. Patch apply belongs to a later human-approved sandbox track, likely v0.36.

## Reference Corpus Policy

OpenCode and Hermes are read-only static references. v0.35.0 may inspect bounded file names, docs, configs, and small excerpts to create `ReferencePatternDigest` metadata. OpenCode, Hermes, and OpenClaw are not executed, imported, installed, or tested. Their code is not used as a live adapter and implementation bodies are not copied.

ReferencePatternDigest is evidence/design metadata, not executable source. ReferenceCorpusInventory is not an import path. ReferenceHarnessPattern is observed design, not copied code. ReferencePatternAdaptationMap is metadata, not implementation execution.

## Taxonomies

The foundation defines track, surface, capability, risk, decision, status, readiness, write posture, apply posture, reference corpus, inspection status, reference pattern, and pattern disposition taxonomies.

Proposal/design surfaces include patch intent, patch scope policy, read-only context snapshot, reference pattern digest, patch plan, change-set graph, diff proposal envelope, structured patch proposal, patch risk report, human review packet, and approval metadata. Apply/write/edit/test/install/provider/credential/UI surfaces are prohibited in v0.35.0.

## Policies And Models

The boundary includes:

- `ReferenceHarnessPattern`
- `ReferencePatternDigest`
- `ReferencePatternAdaptationMap`

- allowed/prohibited surface records
- permission request/decision, denied action, and gate evaluation records
- risk register
- no-apply guarantee
- roadmap overview
- readiness report

Safe defaults keep `PatchWritePosture` at no write and `PatchApplyPosture` at no apply.

## Flow

v0.34.9 handoff plus read-only references/OpenCode and references/Hermes digest feed into `ControlledPatchProposalBoundary`, then `V0350ReadinessReport`, then v0.35.1 Patch Intent & Scope Policy and v0.35.2 Read-only Patch Context & Reference Corpus Collector.

## Hard Prohibitions

v0.35.0 does not implement runtime patch apply, runtime file write, runtime code edit, runtime patch proposal generation, generated diff execution, apply_patch runtime calls, git apply runtime calls, shell/subprocess/command execution, test execution, dependency installation, reference execution/import/install/test runs, provider invocation, network access, credential/secret reads, autonomous general agent runtime, general tool execution, persistent trace write, UI runtime, or authority grants.

## Readiness

`ready_for_v0351_patch_intent_scope_policy` and `ready_for_v0352_readonly_patch_context_collector` may be true only as design-stage handoff. `ready_for_reference_pattern_digest` may be true after the digest document is created.

`ready_for_patch_proposal`, `ready_for_patch_application`, `ready_for_workspace_write`, `ready_for_code_edit`, `ready_for_apply_patch`, `ready_for_git_apply`, `ready_for_test_execution`, `ready_for_shell_execution`, `ready_for_subprocess_execution`, `ready_for_command_execution`, `ready_for_dependency_install`, `ready_for_reference_execution`, `ready_for_reference_import`, `ready_for_execution`, and `production_certified` remain false.

## Acceptance Criteria

- Controlled patch proposal boundary models exist.
- Reference corpus inspection policy blocks execution/import/install/test/shell/secret/raw dumps.
- ReferencePatternDigest document exists.
- Surface policy blocks apply/write/edit/shell/test/install.
- Decisions never allow apply/write/edit/shell/test/install.
- No-apply guarantee exists and all no_* fields are true.
- Focused v0.35.0 tests pass.

## Withdrawal Conditions

Withdraw v0.35.0 if any runtime patch apply, workspace write, code edit, apply_patch, git apply, shell/subprocess/command, test execution, dependency install, reference execution/import/install, provider/network/credential/secret path, persistent trace write, UI runtime, authority grant, unsafe readiness flag, or production certification is introduced.
