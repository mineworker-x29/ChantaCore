# v0.35.2 Read-only Patch Context & Reference Corpus Collector

Parent track: Controlled Patch Proposal Layer.

Release name: v0.35.2 Read-only Patch Context & Reference Corpus Collector.

## Relationships

v0.35.2 builds on v0.35.1 Patch Intent & Scope Policy. It consumes `PatchIntentEnvelope`, `PatchScopePolicy`, `PatchTargetSelector`, and `PatchIntentScopeBundle` as metadata for bounded read-only context collection.

v0.35.2 also consumes v0.35.0 ReferencePatternDigest metadata, preferably from `docs/versions/v0.35/v0.35.0_reference_pattern_digest.md`. References under `references/OpenCode`, `references/Hermes`, and optional `references/OpenClaw` are static read-only material only.

v0.35.2 inherits the v0.33.5 Safe Workspace Inspection Tool Pack posture: bounded inspection may read safe text metadata, but it does not open shell, command execution, workspace mutation, provider invocation, autonomous runtime, or general tool execution.

## Definition

Read-only patch context collection creates bounded evidence artifacts for future planning:

- `PatchContextRequest`
- `PatchContextTarget`
- `PatchContextFileSummary`
- `PatchContextSearchFinding`
- `PatchContextReferenceSummary`
- `PatchContextEvidenceItem`
- `PatchContextEvidenceBundle`
- `PatchContextSnapshot`
- validation/report/preview/no-write/readiness artifacts

Patch context collection is not patch planning. A context snapshot is not a patch plan or patch proposal. An evidence bundle is not a diff proposal. A read-only file summary is not file edit. A search finding is not shell grep output. A reference summary is not reference execution or import.

## Taxonomies

Collection modes include intent/scope bundle only, read-only workspace context, read-only reference corpus context, combined workspace and reference context, digest metadata only, blocked, no-op, and unknown.

Source kinds include v0.35.1 intent/scope bundle, v0.35.1 scope policy, v0.35.1 target selector, v0.35.0 ReferencePatternDigest, v0.33.5 safe workspace inspection, workspace source/test/doc files, OpenCode/Hermes/OpenClaw reference corpus, test fixture, and unknown.

Target statuses include selected, allowed read-only, collected, collected with warnings, skipped, denied, blocked, too large, binary skipped, secret skipped, outside scope, no-op, and unknown.

Evidence kinds include file metadata, bounded text excerpt, bounded search finding, symbol or heading summary, test file summary, documentation summary, reference digest summary, reference static pattern summary, skipped target record, denied target record, gap record, and unknown.

File roles and reference roles are descriptive only. They do not grant write, apply, execution, import, or install authority.

## Collection Policy

`PatchContextCollectionPolicy` may allow:

- file metadata,
- bounded text excerpts,
- bounded search,
- reference digest metadata,
- reference corpus read-only summaries.

It must block:

- secret file read,
- credential file read,
- binary file read,
- outside-scope read,
- shell,
- subprocess,
- workspace write,
- code edit,
- patch application,
- raw source dump.

Limits such as max target count, max files per target, max file size, max excerpt chars, max total excerpt chars, and max search findings keep output bounded.

## Safe Flow

`PatchIntentScopeBundle + PatchScopePolicy + TargetSelectors + ReferencePatternDigest -> PatchContextRequest -> Read-only Target Collection -> FileSummaries/SearchFindings/ReferenceSummaries -> EvidenceBundle -> PatchContextSnapshot -> V0352ReadinessReport -> v0.35.3 Reference-informed Patch Plan`

The v0.35.4 Diff Proposal Envelope handoff is design-stage only. v0.35.2 does not generate a diff proposal.

## Reference Corpus Policy

OpenCode, Hermes, and OpenClaw may only be summarized as static read-only sources under strict limits. v0.35.2 does not execute, import, install, test, or run reference code. It does not copy implementation bodies. It skips or denies unsafe files, secret-like files, binary files, oversized files, outside-root paths, symlink escapes, and blocked reference paths.

If a reference directory is absent, inaccessible, too large, or unsafe, the collector records that honestly as a gap or skipped target.

## Hard Prohibitions

v0.35.2 does not create patch plans. It does not create change-set graphs. It does not generate diff proposals. It does not generate patch proposals. It does not apply patches. It does not write or edit files. It does not call apply_patch. It does not run git apply. It does not execute shell, subprocess, commands, tests, package scripts, dependency installs, provider SDKs, network calls, credential reads, secret reads, reference imports, reference execution, persistent trace writes, UI runtime, external control, or authority grants.

## Acceptance Criteria

- Read-only context collection models and helpers exist.
- Context policy blocks write/apply/shell/secret/raw dump surfaces.
- Safe text summaries and search findings are bounded.
- Reference summaries confirm no execution/import/install/test.
- Evidence bundles and snapshots keep patch plan/diff/proposal/execution readiness false.
- Unsafe files are skipped, denied, or blocked as safe outcomes.

## Withdrawal Conditions

Withdraw v0.35.2 readiness if any runtime path is introduced for patch planning, change-set graph generation, diff proposal generation, patch proposal generation, patch application, workspace write, code edit, shell, subprocess, command execution, test execution, dependency installation, reference execution/import/install, provider/network/credential/secret access, raw source dumping, persistent trace writing, UI runtime, external control, or authority grant.

## Handoff

v0.35.2 is ready only for v0.35.3 Reference-informed Patch Plan & Change Set Graph and v0.35.4 Diff Proposal Envelope design-stage handoff. `ready_for_execution` remains false.
