# v0.35.6 Human Review Packet & Approval Gate Metadata

Parent track: Controlled Patch Proposal Layer.

Release name: v0.35.6 Human Review Packet & Approval Gate Metadata.

## Relationships

v0.35.6 consumes v0.35.5 `PatchProposalRiskReport` and `PatchRiskScanDecision` as risk metadata. It may also consume v0.35.4 `DiffProposalEnvelope`, `UnifiedDiffProposal`, and `StructuredPatchProposal`, v0.35.3 `PatchPlan` and `PatchChangeSetGraph`, v0.35.2 `PatchContextSnapshot` and evidence metadata, v0.35.1 intent/scope metadata, and the v0.35.0 `ReferencePatternDigest`.

All inputs are metadata/static read-only sources. They are not executed, imported, installed, tested, applied, or treated as runtime authority.

## Definition

`PatchReviewPacket` is a bounded human review artifact for patch proposal metadata. It may contain artifact summaries, risk summaries, checklist items, approval gate metadata, reviewer decision placeholders, and review decision records.

Human review packet is not patch application. Approval metadata is not apply permission. Reviewer decision placeholder is not runtime authority. `approved_for_review` is not `approved_for_apply`. Review checklist is not enforcement execution. Review readiness is not write readiness.

## Taxonomies

Review modes include risk-report-only, diff-and-risk review, full patch proposal review, metadata-only review, blocked, needs revision, no-op, and unknown.

Source kinds include v0.35.5 risk report and risk decision, v0.35.4 diff artifacts, v0.35.3 patch plan, v0.35.2 context snapshot, v0.35.1 intent/scope bundle, v0.35.0 ReferencePatternDigest, reviewer metadata, test fixture, and unknown.

Statuses include draft, review packet created, checklist created, approval gate metadata created, pending human review, approved for review, rejected, needs revision, blocked, future gated, no-op, and unknown.

Readiness levels include review contract ready, review packet ready, checklist ready, approval gate metadata ready, design handoff for v0.35.7/v0.35.8, blocked, future track, and not ready.

Decision and outcome values may represent ready for human review, approved for review, rejected, needs revision, blocked by risk/scope/missing artifact, future gate required, pending, acceptable for human review, blocked, no-op, and unknown. None of these values grants apply permission.

Review risks include missing risk report, blocking risk present, unresolved review item, missing diff proposal, missing patch plan, missing context snapshot, scope violation, safety regression, secret or credential risk, reference execution risk, patch apply risk, workspace write risk, test execution risk, shell execution risk, dependency install risk, authority grant risk, and unknown.

Checklist items cover scope, intent, context evidence, diff shape, risk report, no secret exposure, no provider/network opening, no shell command, no write/apply runtime, no reference execution, tests planned but not run, docs planned but not written, human decision needed, future apply sandbox required, and unknown.

Approval gates include no-apply gate, human review required, approval metadata only, future human-approved apply sandbox, blocked gate, no-op gate, and unknown.

Reviewer roles include human reviewer, owner, maintainer, safety reviewer, domain reviewer, automated placeholder, and unknown.

## Review Policy

`PatchReviewPolicy` may allow `approved_for_review`. It must keep false:

- `allow_approved_for_apply`
- `allow_patch_application`
- `allow_workspace_write`
- `allow_code_edit`
- `allow_test_execution`
- `allow_shell`
- `allow_dependency_install`

Review policy is not apply permission.

## Hard Prohibitions

v0.35.6 may create human review packets, approval gate metadata, reviewer decision placeholders, and approved-for-review metadata.

v0.35.6 does not approve patch application. It does not grant apply permission. It does not apply patches. It does not call `apply_patch`. It does not call `git apply`. It does not write or edit files. It does not execute tests. It does not execute shell, subprocess, commands, dependency installs, OpenCode, Hermes, OpenClaw, reference imports, provider SDKs, network calls, credential reads, secret reads, persistent trace writes, UI runtime, external control, or authority grants.

## Acceptance Criteria

- Human review packet, checklist, artifact summary, risk summary, approval gate metadata, reviewer placeholder, decision record, validation, readiness, report, preview, guarantee, and readiness report models exist.
- Acceptable-for-review risk metadata can produce a pending/ready human review packet.
- Blocking risk metadata produces a blocked packet.
- Missing risk metadata produces blocked or review-required packet metadata.
- Approval gate metadata never allows apply permission, workspace write, code edit, or patch application.
- Decision records may approve for review but never approve for apply.
- All unsafe readiness flags remain false.

## Withdrawal Conditions

Withdraw this release boundary if any runtime path is introduced for apply approval, patch application, workspace write, code edit, `apply_patch`, `git apply`, shell/subprocess/command execution, test execution, dependency install, reference execution/import/install, provider/network/credential/secret access, persistent trace writes, UI runtime, external control, or authority grant.

## Handoff

v0.35.6 is ready only for v0.35.7 Patch Proposal OCEL Trace Packet and v0.35.8 CLI Patch Proposal Surface design-stage handoff. `ready_for_execution` remains false.
