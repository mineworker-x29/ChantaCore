# v0.30.2 External Capability Observation Contract

## Release Definition

- Release name: v0.30.2 External Capability Observation Contract
- Parent track: External Dominion Control Plane Foundation
- Previous: v0.30.1 External Target Inventory / Identity / Trust Boundary
- Next: v0.30.3 Digestion Candidate / Assimilation Decision Contract

v0.30.2 is capability observation contract/model only. It defines how externally claimed or described capabilities are represented as observation artifacts. It does not execute external targets, scan external systems, fetch source refs, invoke providers, call network, access credentials, execute commands, automate browsers, control RPA, send gateway messages, delegate to external agents, create internal skills, create dominion targets, or grant authority.

## Relationship To v0.30.0 And v0.30.1

v0.30.0 defined the External Dominion Contract and capped v0.30 grantability at `D3_SIMULATE`. v0.30.1 defined external target inventory, identity, availability, and trust-boundary records. v0.30.2 consumes only provided inventory records and declared capability material. It preserves all v0.30.0 and v0.30.1 boundaries.

## Capability Observation Definition

Capability observation records what a target claims or describes it can do, where that claim came from, which evidence refs support the claim, which effect and boundary surfaces may be implicated, and which risk signals should be preserved.

Capability observation is not invocation. Capability observation is not certification. Capability observation is not permission. A capability observation report is not an internal skill, not a dominion target, and not execution readiness.

Capability kind is descriptive only and does not imply permission.

## Observation Status Definitions

- `unknown`: conservative default.
- `declared`: claimed but not verified.
- `source_described`: source reference exists, not runtime validation.
- `evidence_linked`: evidence refs exist.
- `observed_by_manifest`: manifest-described observation only.
- `observed_by_documentation`: documentation-described observation only.
- `conflict_detected`: conflicting claims exist and conflict notes are required.
- `blocked`: prevents later eligibility in v0.30.2.
- `future_track`: intentionally unavailable now.

## CapabilityObservationReport Structure

A report records:

- report id and target id;
- optional inventory id;
- source kind and source ref as references only;
- capability descriptors;
- aggregate effect surfaces;
- aggregate boundary surfaces;
- aggregate risk signals;
- observed surface flags for credential, network, command, provider, browser, RPA, gateway, memory, and delegation;
- conservative confidence;
- evidence refs;
- withdrawal conditions;
- blocked reasons;
- metadata.

The report does not grant DominionLevel, does not create DominionDecision, does not create internal skill, and does not create DominionTarget.

## Aggregation Rules

Risk signals can infer descriptive effect surfaces. Effect surfaces can infer descriptive boundary surfaces. Aggregation combines surfaces and signals across capability descriptors. Dangerous surfaces require review; they do not permit execution.

Dangerous surfaces include network, credential, command, provider, browser, RPA, gateway, delegation, memory mutation, write, unknown, and external side effect surfaces.

## Safe Defaults

- unknown capability kind remains conservative;
- declared capabilities remain declared only;
- missing evidence keeps confidence conservative;
- metadata is not evidence unless explicitly included in `evidence_refs`;
- source refs are stored but not fetched;
- source descriptions are not runtime validation;
- dangerous surfaces trigger dominion review requirement only;
- digestion review eligibility is not digestion;
- dominion review requirement is not dominion target creation;
- D4-D9 Dominion levels remain future-track.

## Flow Gates

v0.30.2 represents this flow:

inventory record -> capability observation input -> capability descriptors -> observation report -> advisory summary -> v0.30.3 digestion candidate handoff

The flow does not create digestion candidates yet. It does not create dominion targets. It does not create external delegation packets.

## Hard Prohibitions

v0.30.2 does not implement:

- external target execution
- live external observation scanning
- source ref fetching
- external agent harness invocation
- provider SDK invocation
- provider invocation
- network calls or outbound requests
- credential or secret access
- subprocess, shell, or command execution
- RPA/A360/Brity/UiPath runtime control
- browser automation
- gateway or channel message sending
- raw external output memory persistence
- automatic retry
- rollback execution
- live-runtime-ready target marking
- v0.29 provider adapter gate bypass
- grantability above `D3_SIMULATE`
- conversion of observed capability into permission
- conversion of observed capability into internal skill
- conversion of observed capability into DominionAuthority

## Gate Inheritance

v0.30.2 inherits:

- v0.29 provider adapter gates: no provider invocation, no provider SDK bypass, no network access bypass, no credential access bypass, no command execution bypass, no raw provider output persistence, no retry, and no rollback execution without later gate;
- v0.30.0 contract gates: D4-D9 remain future-track and max grantability remains `D3_SIMULATE`;
- v0.30.1 inventory gates: inventory is not execution, identity is not permission, trust is not authority, and source refs are not fetched.

## Acceptance Criteria

- Capability kind, observation status, effect surface, and risk signal taxonomies exist.
- Capability descriptors validate ids, names, list fields, evidence refs, and conflict notes.
- Observation inputs store source refs without fetching them.
- Observation reports validate target-id consistency and aggregate surfaces/signals.
- Dangerous surfaces are detected conservatively.
- Advisory summary recommendations are not permissions.
- Build-from-inventory uses only inventory and passed-in data.
- No DominionDecision, DominionAuthority, internal skill, or dominion target is created.
- v0.30.0 and v0.30.1 focused tests still pass.

## Withdrawal Conditions

Withdraw v0.30.2 validity if any of the following appear:

- source refs are fetched;
- live external scan appears;
- observed capability is invoked;
- observed capability becomes permission, certification, internal skill, dominion target, or authority;
- provider/network/credential/command/browser/RPA/gateway/delegation runtime appears;
- raw external output is persisted as memory;
- D4-D9 become grantable or inferred from capability observation;
- v0.29, v0.30.0, or v0.30.1 gates are bypassed.

## Handoff

The next version is v0.30.3 Digestion Candidate / Assimilation Decision Contract.

v0.30.3 may define how capability observation reports become digestion candidates or are denied/deferred. It must still not create internal skills, dominion targets, external delegation packets, or execution readiness without later explicit gates.
