# v0.30.5 External Delegation Candidate / Dry-Run Plan

Parent track: External Dominion Control Plane Foundation

## Relationship To v0.30.0-v0.30.4

v0.30.0 defines the External Dominion contract and keeps v0.30 grantability capped at `D3_SIMULATE`.

v0.30.1 defines External Target Inventory, Identity, Availability, and Trust Boundary. Inventory is not permission.

v0.30.2 defines `CapabilityObservationReport`. Observation is not certification, permission, or runtime readiness.

v0.30.3 defines digestion candidates and assimilation decisions. A digestion candidate is not an active internal artifact, and an assimilation decision is not implementation.

v0.30.4 defines Dominion authority request, scope, evaluation, decision, and review requirement. Authority decision is not execution, approval required is not approval granted, and D4-D9 remain future-track.

v0.30.5 consumes the v0.30.4 authority vocabulary to model possible future external delegation without performing it.

## External Delegation Candidate

External delegation candidate means ChantaCore may later delegate a bounded task to an external agent, harness, runtime, provider, gateway, browser, RPA system, MCP surface, or private overlay. In v0.30.5, it only records planning artifacts.

Delegation candidate is not delegation. It does not grant `D8_DELEGATE_AGENT`, create a live parent-child session, contact an external system, or create runtime readiness.

## Packet Preview And Input Envelope

`ExternalDelegationInputEnvelope` is a local planning envelope. It describes context, structured inputs, allowed references, prohibited capabilities, object references, evidence refs, data-boundary notes, and redaction requirements.

Delegation packet preview is not packet send. Input envelope is not external input submission.

## Expected Output Schema

`ExternalDelegationExpectedOutputSchema` describes what a future external result would need to contain. It defaults to evidence-required, result-envelope-required, and raw-output-forbidden.

Expected output schema is not external result. External result envelope is schema-only in v0.30.5.

## Effect Preview And Risk Preview

`ExternalDelegationEffectPreview` records expected effect, boundary, and risk surfaces. If dangerous effects are possible, `side_effect_free_expected` must be false.

`ExternalDelegationRiskPreview` records risk signals, blocked reasons, required reviews, future gate requirements, human approval requirements, no-op recommendation, rationale, and evidence refs.

Human approval required is not approval granted. Risk preview does not execute or block by itself.

## Dry-Run Plan Versus Dry-Run Execution

`ExternalDelegationDryRunPlan` contains simulated steps only. Every step must keep external runtime, network, credential, command, browser, RPA, and gateway usage false. The plan carries no-execution, no-external-contact, no-credential-use, no-network-use, and no-command-use guarantees.

Dry-run plan is not dry-run execution.

`ExternalDelegationDryRunReport` is derived from the plan only. Dry-run report is not external output.

## No-Op Plan

`ExternalDelegationNoOpPlan` records a valid no-action path, blocked reasons, and safer alternatives. No-op is valid and is not failure.

## Handoff Preview

`ExternalDelegationHandoffPreview` points to `v0.30.6 Approval/Audit/Rollback Boundary`, `no_op`, or `future_track`.

Handoff preview is not handoff execution. `ready_for_execution` remains false.

## D8 Future-Track Rule

`D8_DELEGATE_AGENT` remains future-track in v0.30.5. D8 may appear as a requested level in governance records, but it must not be granted and must not create external delegation runtime.

## Safe Defaults

- Unknown intent is conservative.
- Draft packet is not packet transfer.
- Dry-run planned is not execution.
- Dry-run reported is not external runtime output.
- Blocked stops progression.
- Future-track is not readiness.
- No-op is valid.
- Provider, gateway, RPA, browser, command, network, credential, private-data, and delegation surfaces require conservative risk preview and usually route to no-op, future gate, or v0.30.6 approval/audit boundary.

## Flow

`DominionAuthorityDecision -> ExternalDelegationIntent -> ExternalDelegationCandidate -> InputEnvelope -> ExpectedOutputSchema -> EffectPreview/RiskPreview -> DryRunPlan -> DryRunReport or NoOpPlan -> HandoffPreview -> v0.30.6 Approval/Audit/Rollback Boundary`

The flow creates contract artifacts only.

## Hard Prohibitions

v0.30.5 must not:

- execute external targets;
- invoke external agent harnesses;
- send delegation packets to external systems;
- spawn processes;
- call provider SDKs;
- perform network calls;
- access credentials or secrets;
- use subprocess, shell, or command execution;
- implement RPA runtime control;
- implement browser automation;
- implement gateway/channel message send;
- persist raw external output as memory;
- add automatic retry or rollback execution;
- mark any target as live-runtime-ready;
- bypass v0.29 provider adapter gates;
- raise v0.30 max grantability beyond `D3_SIMULATE`;
- grant `D8_DELEGATE_AGENT`;
- treat authority review as delegation permission;
- treat dry-run plan as dry-run execution;
- create actual parent-child session traces.

## Gate Inheritance

v0.30.5 inherits v0.29 provider adapter gates and all v0.30.0, v0.30.1, v0.30.2, v0.30.3, and v0.30.4 boundaries.

Runtime readiness flags remain false:

- `provider_invocation_runtime_ready = False`
- `limited_preview_execution_ready_now = False`
- `live_adapter_runtime_ready = False`
- `external_agent_dominion_runtime_ready = False`
- `network_access_ready = False`
- `credential_access_ready = False`
- `command_execution_ready = False`
- `rpa_runtime_control_ready = False`
- `gateway_control_ready = False`

No runtime readiness flag changes in v0.30.5.

## Acceptance Criteria

- External delegation intent model exists.
- External delegation candidate model exists.
- Input envelope model exists.
- Expected output schema model exists.
- Effect preview and risk preview models exist.
- Dry-run step, plan, and report models exist.
- No-op plan exists.
- Handoff preview exists.
- Helpers are pure.
- Packet preview is not packet send.
- Dry-run plan is not execution.
- Dry-run report is not external output.
- No-op is valid.
- Handoff preview is not handoff execution.
- D8 remains future-track.
- `ready_for_execution` remains false.
- Focused v0.30.0-v0.30.5 tests pass.

## Withdrawal Conditions

Withdraw this release interpretation if:

- any helper executes, fetches, invokes, registers, writes external data, accesses credentials, or contacts an external runtime;
- any packet preview is sent;
- any parent-child live delegation session is created;
- any dry-run plan performs runtime work;
- any report is treated as external output;
- `D8_DELEGATE_AGENT` is granted;
- `ready_for_execution` becomes true;
- runtime readiness flags change.

## Handoff To v0.30.6

v0.30.6 should define Approval / Audit / Rollback Boundary for external delegation and Dominion preview. v0.30.5 may hand off a preview to that boundary, but it does not execute handoff, approve delegation, or contact external systems.
