# v0.30.0 External Dominion Contract

## Release Definition

- Release name: v0.30.0 External Dominion Contract
- Parent track: External Dominion Control Plane Foundation
- Previous: v0.29.9 External Provider Adapter Foundation Consolidation
- Next: v0.30.1 External Target Inventory / Identity / Trust Boundary

v0.30.0 is contract-first only. It defines the initial vocabulary and validation rules for representing external targets and future dominion requests. It does not execute, delegate to, invoke, control, automate, or message any external target.

## Purpose

External world includes OpenCode, OpenClaw, Hermes, external skill packages, external provider runtimes, external gateways, MCP servers, browser automation runtimes, RPA/A360/Brity/UiPath, Schumpeter private overlay candidates, and unknown future agent harnesses.

The purpose of v0.30.0 is to model these as future observation, digestion, planning, and dominion objects under ChantaCore's OCEL-native process-intelligence substrate. The release only creates contract/model concepts.

## Architecture Boundary

- Core = OCEL-native process intelligence substrate.
- Internal = observation / digestion / dominion triad.
- External = external agent harnesses, skills, runtimes, providers, gateways, RPA, browser runtimes, MCP servers, and future external capability sources.

## Contract-Only Scope

The release defines:

- `ExternalTargetKind`
- `ExternalTargetStatus`
- `ExternalTargetRecord`
- `DominionLevel`
- `ExternalDominionContract`
- `DominionAuthorityRequest`
- `DominionDecision`
- pure grantability helpers

It does not define runtime service implementations.

## ExternalTarget Definition

An external target is a described external capability source, not a controlled runtime. Target registration records identity, source reference, declared capabilities, risk tags, and metadata. Registration does not imply trust, permission, availability, or execution.

`unknown` is allowed as a target kind or status, but unknown is not available and does not imply trust.

Declared capabilities are descriptions only. They are not permissions.

## DominionLevel Ladder

- `D0_OBSERVE`
- `D1_DESCRIBE`
- `D2_PLAN`
- `D3_SIMULATE`
- `D4_EXECUTE_READ`
- `D5_EXECUTE_WRITE_PROPOSAL`
- `D6_EXECUTE_SANDBOX`
- `D7_EXECUTE_NETWORK_PREVIEW`
- `D8_DELEGATE_AGENT`
- `D9_GATEWAY_CONTROL`

For v0.30.0, the maximum grantable level is `D3_SIMULATE`.

`D4_EXECUTE_READ` through `D9_GATEWAY_CONTROL` are future-track and are not grantable in v0.30.0.

## Required Principles

- External target registration is not execution.
- External target identity is not trust.
- External target trust is not permission.
- Capability declaration is not permission.
- Observation is not invocation.
- Digestion candidate is not internal skill.
- Dominion target is not controlled runtime.
- Dominion authority request is not authority grant.
- Dry-run plan is not execution.
- Preview eligibility is not preview execution.
- Approval candidate is not approval.
- Rollback plan is not rollback execution.
- No-op / deny / defer / future-track are valid outcomes.
- Unknown is not available.
- Readiness is not runtime enablement.

## Authority Requests And Decisions

A `DominionAuthorityRequest` may ask for any level D0-D9, but the request itself grants nothing.

A `DominionDecision` may grant only D0-D3 in v0.30.0. D4-D9 requests must be denied, deferred, no-op, observe-only, describe-only, dry-run-only, or future-track. `approval_required=True` records a later requirement and does not mean approval was granted.

## Acceptance Criteria

- External target kinds and statuses exist.
- Unknown target kind/status does not imply trust or availability.
- DominionLevel D0-D9 exists.
- D0-D3 are grantable in v0.30.0.
- D4-D9 are not grantable in v0.30.0 and remain future-track.
- ExternalTargetRecord validates nonblank target id and display name.
- ExternalDominionContract records v0.29 gate inheritance and all not-ready flags.
- DominionAuthorityRequest grants no authority.
- DominionDecision cannot grant D4+.
- Helpers are pure and do not execute external systems.
- Focused tests pass without network, credentials, subprocess, provider SDKs, RPA, browser, gateway, or external runtime.

## Withdrawal Conditions

Withdraw v0.30.0 validity if any of the following appear:

- external agent execution or external delegation execution
- external harness invocation
- provider invocation or provider SDK invocation
- network call or outbound request
- credential or secret access
- shell, subprocess, or command execution
- browser automation
- RPA/A360/Brity/UiPath runtime control
- gateway or channel send runtime
- raw external output persistence as memory
- retry or rollback execution
- D4+ level grantability in v0.30.0
- live runtime readiness flags becoming true
- v0.29 provider adapter gate bypass
- LLM judgment as sole trust/certification authority
