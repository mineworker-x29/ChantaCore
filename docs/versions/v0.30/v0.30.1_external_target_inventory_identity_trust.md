# v0.30.1 External Target Inventory / Identity / Trust Boundary

## Release Definition

- Release name: v0.30.1 External Target Inventory / Identity / Trust Boundary
- Parent track: External Dominion Control Plane Foundation
- Previous: v0.30.0 External Dominion Contract
- Next: v0.30.2 External Capability Observation Contract

v0.30.1 is inventory, identity, and trust-boundary only. It adds models for registering external targets and separating identity claims, identity evidence, availability status, trust boundaries, runtime boundaries, and dominion eligibility. It does not observe, digest, execute, delegate, invoke, control, automate, fetch, or message any external system.

## Relationship To v0.30.0

v0.30.0 created the External Dominion Contract and DominionLevel ladder. v0.30.1 preserves the v0.30.0 maximum grantable level of `D3_SIMULATE`. It does not raise grantability and does not change runtime readiness flags.

## ExternalTargetInventory Definition

`ExternalTargetInventory` is a pure in-memory inventory structure for external target records. It can register a provided `ExternalTargetRegistrationRequest`, return records by target id, list records, and answer whether a target id is known.

Inventory registration is not execution. Registration does not fetch `source_ref`, inspect an external system, call a provider, open a network connection, read credentials, execute commands, automate a browser, control RPA, send gateway messages, or create authority grants.

## Identity Claim / Evidence / Trust Distinction

- Identity claim is self-declared metadata.
- `claimed` is not verified identity.
- `source_described` means documentation or a source reference exists.
- `evidence_linked` means evidence refs were supplied.
- `reference_verified` means supplied references are internally consistent; it is still not trust and not permission.
- `conflict_detected` means identity claims conflict and requires conflict notes.
- `blocked` means identity cannot be used without explicit future override.

Identity is not permission. Verified reference is not trust.

## Availability Versus Invocation Readiness

Availability is informational only:

- `unknown` is not available.
- `unavailable` is not available.
- `documentation_only` is not callable.
- `local_reference_only` is not executable.
- `candidate` is not runtime-ready.
- `blocked` prevents normal eligibility in v0.30.1.
- `future_track` is intentionally reserved for a later version.

Availability is not invocation readiness.

## Trust Boundary Status

- `unknown` defaults to untrusted behavior.
- `untrusted` prevents trust-based progression.
- `observation_only` permits metadata-level observation only.
- `description_only` permits descriptive modeling only.
- `dry_run_candidate` is not dry-run execution.
- `blocked` prevents v0.30.1 flow beyond inventory.
- `future_track` is reserved for later versions.

Trust classification is not permission. Trust boundary is not runtime control.

## Boundary Surfaces

Boundary surfaces include:

- `data_boundary`
- `credential_boundary`
- `network_boundary`
- `command_boundary`
- `filesystem_boundary`
- `memory_boundary`
- `provider_boundary`
- `browser_boundary`
- `rpa_boundary`
- `gateway_boundary`
- `delegation_boundary`
- `unknown`

Unknown and untrusted targets prohibit dangerous surfaces by default: credential, network, command, browser, RPA, and gateway boundaries.

## Safe Defaults

Registration creates conservative defaults:

- unknown target kind remains untrusted and unavailable;
- known target kind with source evidence becomes a candidate record only;
- source references are stored but not fetched;
- declared capabilities remain descriptive;
- risk tags remain descriptive;
- default trust boundary is at most `D1_DESCRIBE`;
- blocked trust boundary uses `D0_OBSERVE`;
- all D4-D9 execution, network preview, delegation, and gateway levels remain future-track.

## Flow Gates

v0.30.1 represents this flow:

target registration -> identity profile -> trust boundary -> observation / digestion / dominion eligibility

The flow does not produce execution permission.

- Observation eligibility allows only conservative metadata or descriptive flow.
- Digestion eligibility requires source description, linked evidence, or verified reference and a non-blocked state.
- Dominion eligibility means candidate-level reasoning only. It is not runtime control, authority grant, or execution.

## Semantic Rules

- Inventory registration is not execution.
- Identity claim is not verified identity.
- Verified identity is not trust.
- Trust classification is not permission.
- Availability is not invocation readiness.
- Capability declaration is not permission.
- Trust boundary is not runtime control.
- Dry-run eligibility is not dry-run execution.
- Unknown means not trusted and not available.
- Blocked means no later gate should proceed without explicit override in a future version.
- Future-track means intentionally not available in v0.30.1.
- No-op / deny / defer / future-track remain valid outcomes.

## Hard Prohibitions

v0.30.1 does not implement:

- external target execution
- external agent harness invocation
- provider SDK invocation
- provider invocation
- network calls or outbound requests
- credential or secret access
- subprocess, shell, or command execution
- RPA/A360/Brity/UiPath runtime control
- browser automation
- gateway or channel message sending
- source reference fetching
- raw external output persistence as memory
- automatic retry
- rollback execution
- live-runtime-ready target marking
- v0.29 provider adapter gate bypass
- grantability beyond `D3_SIMULATE`

## v0.29 And v0.30.0 Gate Inheritance

v0.30.1 inherits all v0.29 provider adapter boundaries:

- no provider invocation bypass
- no network access bypass
- no credential access bypass
- no command execution bypass
- no provider SDK bypass
- no external side-effect bypass
- no raw provider output persistence
- no automatic retry
- no rollback execution without later gate

v0.30.1 also inherits v0.30.0:

- External Dominion Contract remains contract-first.
- D4-D9 remain future-track.
- runtime readiness flags remain false.
- authority requests grant nothing by themselves.
- dominion decisions cannot grant D4+.

## Runtime Readiness Flags

No runtime readiness flag changes in v0.30.1:

- `provider_invocation_runtime_ready = False`
- `limited_preview_execution_ready_now = False`
- `live_adapter_runtime_ready = False`
- `external_agent_dominion_runtime_ready = False`
- `network_access_ready = False`
- `credential_access_ready = False`
- `command_execution_ready = False`
- `rpa_runtime_control_ready = False`
- `gateway_control_ready = False`

## Acceptance Criteria

- External identity status, availability status, trust boundary status, and boundary surfaces exist.
- Identity profile validates target id, evidence refs, reference verification, and conflicts.
- Trust boundary profile cannot exceed `D3_SIMULATE`.
- Blocked trust boundary uses the safest observable ceiling.
- Inventory record requires matching target/identity/trust target ids.
- Registration creates conservative defaults.
- Duplicate target id is rejected.
- Registration does not create DominionDecision or DominionAuthority.
- Declared capabilities do not become permissions.
- Candidate availability is not invocation readiness.
- Dry-run candidate is not dry-run execution.
- Dangerous boundaries are prohibited by default for unknown or untrusted targets.
- Helper flow checks remain conservative and pure.
- v0.30.0 focused tests still pass.

## Withdrawal Conditions

Withdraw v0.30.1 validity if any of the following appear:

- source refs are fetched;
- external targets are executed or invoked;
- external agent delegation runtime appears;
- provider SDKs or providers are invoked;
- network, credential, secret, shell, subprocess, browser, RPA, gateway, retry, or rollback runtime appears;
- raw external output is persisted as memory;
- inventory, identity, availability, or trust is treated as permission;
- dry-run candidate is treated as dry-run execution;
- D4-D9 become grantable;
- v0.29 provider adapter gates are bypassed.

## Handoff

The next version is v0.30.2 External Capability Observation Contract.

v0.30.2 may define how externally declared capabilities are observed as metadata. It must still not execute external targets, invoke providers, call network, access credentials, control browser/RPA/gateway surfaces, or delegate to external agents.
