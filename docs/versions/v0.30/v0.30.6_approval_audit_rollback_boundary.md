# v0.30.6 Approval / Audit / Rollback Boundary

Parent track: External Dominion Control Plane Foundation

## Relationship To v0.30.0 Through v0.30.5

v0.30.0 defines the External Dominion contract and keeps v0.30 grantability capped at `D3_SIMULATE`.

v0.30.1 defines External Target Inventory, Identity, Availability, and Trust Boundary. Inventory is not permission.

v0.30.2 defines capability observation. Observation is not certification, permission, or runtime readiness.

v0.30.3 defines digestion candidates and assimilation decisions. A digestion candidate is not an active internal artifact, and an assimilation decision is not implementation or Dominion authority.

v0.30.4 defines Dominion authority request, scope, evaluation, decision, and review requirement. Authority decision is not execution, approval required is not approval granted, and D4-D9 remain future-track.

v0.30.5 defines external delegation candidates, input envelopes, expected output schemas, dry-run plans, dry-run reports, no-op plans, and handoff previews. Delegation candidate is not delegation, and dry-run plan/report is not execution or external output.

v0.30.6 consumes the v0.30.5 handoff preview vocabulary and defines approval, audit, result boundary, rollback plan, no-op boundary, failure classification, retry deferral, and v0.30.7 certification handoff contracts.

## Approval Requirement

`ExternalDominionApprovalRequirement` records what evidence, reviews, boundary checks, and result boundary actions would be required before any later certification review.

Approval requirement is not approval. `required`, `evidence_required`, `human_review_required`, and `approval_candidate_ready` are governance statuses only. D4-D9 requests require `requires_future_gate=True`.

## Approval Candidate

`ExternalDominionApprovalCandidate` packages a requirement for a decision record. It may be ready for a decision record, but it is not approval granted.

`ready_for_execution` remains false in v0.30.6. D4-D9 requested levels must not become execution-ready.

## Approval Decision Record

`ExternalDominionApprovalDecisionRecord` records one of:

- `deny`
- `defer`
- `no_op`
- `require_more_evidence`
- `require_human_review`
- `allow_certification_review`
- `future_track`
- `blocked`

Approval decision record is not execution. `allow_certification_review` means v0.30.7 certification matrix review only. Approval for certification review is not execution approval. `approved_for_execution` remains false.

## Audit Policy And Audit Trail Plan

`ExternalDominionAuditPolicy` defines required audit event kinds, object references, evidence refs, OCEL visibility, append-only requirements, redaction logging, and raw-payload/secret logging prohibitions.

`ExternalDominionAuditTrailPlan` plans what would be logged as audit references. Audit plan is not audit execution. Audit trail plan does not emit events and must guarantee no raw payload or secret logging.

## OCEL Trace Plan

`ExternalDominionOCELTracePlan` describes planned event types, object types, and relation types for future visibility.

OCEL trace plan is not event emission. It does not write OCEL events.

## Result Boundary Policy

`ExternalDominionResultBoundaryPolicy` constrains any future result with default actions:

- reject raw output
- require result envelope
- require evidence refs
- require no memory persistence
- allow schema only

Result boundary policy is not result ingestion. Raw external output must not become memory. Result envelope requirement does not mean result exists.

## Rollback Plan

`ExternalDominionRollbackPlan` records whether rollback is conceptually available, descriptive rollback steps, limitations, and fallback no-op requirements.

Rollback plan is not rollback execution. `rollback_available=True` does not execute rollback.

## No-Op Boundary

`ExternalDominionNoOpBoundary` records a valid no-action boundary with safe alternatives and blocked reasons.

No-op is valid. No-op is not failure and executes nothing.

## Failure Classification

`ExternalDominionFailureClassification` records governance failure classes including insufficient evidence, trust boundary violation, approval missing, result boundary violation, unsafe external surfaces, credential/network/command/provider/browser/RPA/gateway/delegation risks, memory contamination risk, raw output risk, future gate required, and policy block.

Failure classification is not retry execution.

## Retry Deferral Policy

`ExternalDominionRetryDeferralPolicy` defaults to `retry_allowed_now=False` and `max_retry_count=0`.

Retry deferral policy is not retry.

## v0.30.7 Certification Handoff

`ExternalDominionV0307CertificationHandoff` can mark a package ready for the v0.30.7 certification matrix only when approval decision, audit policy, result boundary policy, and rollback-or-no-op artifacts exist.

Handoff to v0.30.7 is not certification. `ready_for_execution` remains false.

## Safe Defaults

- Unknown statuses are conservative.
- Approval candidate ready is not approval granted.
- Human review required is not human approval.
- Approval required is not approval granted.
- Certification review approval is not execution approval.
- Audit policy is not audit execution.
- Audit trail plan does not emit events.
- OCEL trace plan does not emit events.
- Result boundary rejects raw output and memory persistence by default.
- Rollback plans do not execute rollback.
- Retry deferral does not retry.
- Blocked stops progression.
- Future-track is not readiness.
- D4-D9 remain future-track in v0.30.6.

## Flow

`ExternalDelegationHandoffPreview -> ApprovalRequirement -> ApprovalCandidate -> ApprovalDecisionRecord -> AuditPolicy -> OCELTracePlan -> ResultBoundaryPolicy -> RollbackPlan or NoOpBoundary -> FailureClassification/RetryDeferral as needed -> V0307CertificationHandoff`

The flow creates contract artifacts only.

## Hard Prohibitions

v0.30.6 must not:

- execute external targets;
- approve real execution;
- execute rollback;
- invoke external agent harnesses;
- transfer delegation packets to external systems;
- spawn processes;
- call provider SDKs;
- perform network calls;
- access credentials or secrets;
- use subprocess, shell, or command execution;
- implement RPA runtime control;
- implement browser automation;
- implement gateway/channel message control;
- persist raw external output as memory;
- add automatic retry execution;
- mark any target as live-runtime-ready;
- bypass v0.29 provider adapter gates;
- raise v0.30 max grantability beyond `D3_SIMULATE`;
- grant `D7_EXECUTE_NETWORK_PREVIEW`;
- grant `D8_DELEGATE_AGENT`;
- grant `D9_GATEWAY_CONTROL`;
- set `ready_for_execution=True`;
- treat approval boundary readiness as execution approval;
- treat rollback plan as rollback execution;
- treat audit plan as audit already performed.

## Gate Inheritance

v0.30.6 inherits v0.29 provider adapter gates and all v0.30.0-v0.30.5 boundaries.

Runtime readiness flags remain false:

- `provider_invocation_runtime_ready = False`
- `limited_preview_execution_ready_now = False`
- `live_adapter_runtime_ready = False`
- `external_agent_dominion_runtime_ready = False`
- `network_access_ready = False`
- `credential_access_ready = False`
- `command_execution_ready = False`
- `rpa_runtime_control_ready = False`
- `gateway_control_ready = False`

No runtime readiness flag changes in v0.30.6.

## Withdrawal Conditions

Withdraw this release interpretation if:

- any helper executes, fetches, invokes, registers, writes external data, accesses credentials, or contacts an external runtime;
- any helper approves real execution;
- `approved_for_execution` becomes true;
- `ready_for_execution` becomes true;
- an audit trail plan emits events;
- an OCEL trace plan emits events;
- a result boundary ingests raw external output;
- raw external output is persisted as memory;
- rollback plan executes rollback;
- retry deferral performs retry;
- D7, D8, or D9 are granted;
- runtime readiness flags change.

## Handoff To v0.30.7

v0.30.6 is handoff-ready for v0.30.7 External Dominion Certification / Dominion Matrix only. It does not complete certification, approve execution, perform audit, execute rollback, retry, or contact external systems.
