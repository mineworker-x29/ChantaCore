# v0.30.7 External Dominion Certification / Dominion Matrix

Parent track: External Dominion Control Plane Foundation

## Relationship To v0.30.0 Through v0.30.6

v0.30.0 defines the External Dominion contract and keeps v0.30 grantability capped at `D3_SIMULATE`.

v0.30.1 defines External Target Inventory, Identity, Availability, and Trust Boundary. Inventory is not permission.

v0.30.2 defines capability observation. Observation is not certification, permission, or runtime readiness.

v0.30.3 defines digestion candidates and assimilation decisions. A digestion candidate is not an active internal artifact, and an assimilation decision is not implementation or Dominion authority.

v0.30.4 defines Dominion authority request, scope, evaluation, decision, and review requirement. Authority decision is not execution, approval required is not approval granted, and D4-D9 remain future-track.

v0.30.5 defines external delegation candidates and dry-run plans. Delegation candidate is not delegation, and dry-run plan/report is not execution or external output.

v0.30.6 defines approval, audit, result-boundary, rollback-plan, no-op, failure-classification, retry-deferral, and v0.30.7 handoff contracts. Approval for certification review is not execution approval.

v0.30.7 consumes v0.30.6 handoff artifacts and creates certification/governance matrix artifacts for v0.30.8 limited preview gate review only.

## Certification Policy

`ExternalDominionCertificationPolicy` defines the required case kinds, blocking case kinds, minimum status, and mandatory proof flags.

Certification policy is not certification result. Its defaults require no execution, no network, no credential, no command, result boundary, OCEL visibility plan, provider gate inheritance, and `ready_for_execution=False`.

## Certification Case

`ExternalDominionCertificationCase` is a descriptive case in the matrix. Case kinds include observation coverage, identity/trust coverage, capability observation coverage, digestion decision coverage, authority boundary coverage, delegation dry-run coverage, approval/audit/rollback coverage, no-execution, no-network, no-credential, no-command, no-provider-bypass, no-RPA, no-browser, no-gateway, no raw output persistence, no secret logging, result boundary, rollback/no-op, OCEL visibility, provider gate inheritance, D4-D9 future-track, and ready-for-execution-false.

Certification case is not execution. No-* cases verify contract artifacts only, not live environment behavior.

## Certification Case Result

`ExternalDominionCertificationCaseResult` records status, pass flag, limitations, failed reasons, evidence refs, and reviewers.

Certification pass is not execution approval. `passed_with_limitations` requires limitations. Failed and blocked results require failed reasons. A failed blocking result prevents matrix readiness for v0.30.8 gate review.

## Certification Matrix

`ExternalDominionCertificationMatrix` aggregates cases and results. It may set `ready_for_v0308_limited_preview_gate_review=True` only when blocking cases pass and the aggregate decision is pass or pass-with-limitations for limited preview gate review.

Certification matrix is not production certification. It is not live adapter certification and does not grant execution. `ready_for_execution`, `production_certified`, and `live_adapter_certified` remain false.

## Certification Report

`ExternalDominionCertificationReport` summarizes the matrix, limitations, failures, followups, and v0.30.8 gate review eligibility.

Certification report is not runtime enablement, execution permission, production certification, or live adapter certification.

## Specialized Reports

Specialized reports include:

- `ObservationCertificationReport`
- `DigestionCertificationReport`
- `AuthorityCertificationReport`
- `DelegationDryRunCertificationReport`
- `ApprovalAuditRollbackCertificationReport`
- `NoExecutionCertificationReport`
- `NoNetworkCertificationReport`
- `NoCredentialCertificationReport`
- `NoCommandCertificationReport`
- `NoProviderBypassCertificationReport`
- `NoRPACertificationReport`
- `NoBrowserCertificationReport`
- `NoGatewayCertificationReport`
- `OCELVisibilityCertificationReport`
- `ResultBoundaryCertificationReport`
- `RollbackNoOpCertificationReport`
- `ProviderGateInheritanceCertificationReport`

No-network certification does not allow network. No-credential certification does not allow credentials. No-command certification does not allow commands. No-provider-bypass certification does not allow provider invocation. No-RPA, no-browser, and no-gateway certification do not allow those runtimes.

## v0.30.8 Preview Gate Handoff

`ExternalDominionV0308PreviewGateHandoff` carries matrix and report refs into v0.30.8 limited preview gate review.

`ready_for_v0308_limited_preview_gate_review` is not `ready_for_limited_preview_execution`. The handoff is not limited preview execution and is not production certification. `ready_for_limited_preview_execution`, `ready_for_execution`, `production_certified`, and `live_adapter_certified` remain false.

## D4-D9 Future-Track Rule

D4-D9 remain future-track in v0.30.7. Certification cannot grant `D7_EXECUTE_NETWORK_PREVIEW`, `D8_DELEGATE_AGENT`, or `D9_GATEWAY_CONTROL`.

## Safe Defaults

- Certification policy is not certification result.
- Certification case is not execution.
- Certification matrix is not production certification.
- Certification report is not runtime enablement.
- Certification pass is not execution approval.
- Certification pass only means eligible for later v0.30.8 limited preview gate review.
- Limited preview gate handoff is not limited preview execution.
- No-network, no-credential, and no-command reports do not authorize those capabilities.
- OCEL visibility plan is not OCEL event emission.
- Result boundary certification is not result ingestion.
- Rollback/no-op certification is not rollback execution.
- Provider gate inheritance certification is not provider invocation.
- Future-track is not readiness.

## Flow

`V0307CertificationHandoff -> CertificationPolicy -> CertificationCases -> CertificationCaseResults -> CertificationMatrix -> CertificationReport -> V0308PreviewGateHandoff`

The flow creates governance contract artifacts only.

## Hard Prohibitions

v0.30.7 must not:

- execute external targets;
- approve real execution;
- certify production readiness;
- certify live runtime readiness;
- execute rollback;
- retry;
- invoke external agent harnesses;
- transfer delegation packets;
- spawn processes;
- call provider SDKs;
- perform network calls;
- access credentials or secrets;
- use subprocess, shell, or command execution;
- implement RPA runtime control;
- implement browser automation;
- implement gateway/channel message control;
- persist raw external output as memory;
- mark any target as live-runtime-ready;
- bypass v0.29 provider adapter gates;
- raise v0.30 grantability beyond `D3_SIMULATE`;
- grant `D7_EXECUTE_NETWORK_PREVIEW`, `D8_DELEGATE_AGENT`, or `D9_GATEWAY_CONTROL`;
- set `ready_for_execution=True`;
- set `ready_for_limited_preview_execution=True`;
- set `production_certified=True`;
- set `live_adapter_certified=True`;
- treat certification matrix pass as execution permission.

## Withdrawal Conditions

Withdraw this release interpretation if:

- any helper executes, fetches, invokes, registers, writes external data, accesses credentials, or contacts an external runtime;
- any helper approves execution;
- any helper emits OCEL events;
- any helper executes rollback or retry;
- certification pass is treated as execution permission;
- no-network/no-credential/no-command/no-provider-bypass/no-RPA/no-browser/no-gateway reports are treated as runtime permission;
- `ready_for_execution`, `ready_for_limited_preview_execution`, `production_certified`, or `live_adapter_certified` becomes true;
- D7, D8, or D9 are granted;
- runtime readiness flags change.
