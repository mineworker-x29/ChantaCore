# v0.32.1 OpenCode-style Harness Observation Profile

Parent track: External Harness Observation & Digestion Pipeline.

v0.32.1 builds on v0.32.0 External Harness Profile Contract + Reference Corpus Static Observation Contract and inherits v0.31.9 Internal Triad Skill Foundation v1 boundaries. It defines an OpenCode-style static observation profile for coding-agent and coding-harness surfaces. This is a style profile, not a product clone and not an assumption about any specific OpenCode product API.

## Release Boundary

OpenCode-style profile is not OpenCode execution. It does not execute OpenCode, execute external harnesses, execute `references/` code, import reference code as runtime, install dependencies, run reference project tests, live-scan external systems, fetch `source_ref`, write workspace files, edit code, apply patches, invoke tools, register tools, load plugins, invoke providers, access network, access credentials, read secret files, execute commands, run shell, send packets, mutate registry or memory, emit OCEL events, or implement UI runtime.

`references/OpenCode` is read-only static corpus only when present. Reference corpus static observation is not live harness scanning.

## Definition

An OpenCode-style harness is a coding-agent or coding-harness style surface that may expose workspace, file, tool, plugin, provider, command, shell, LSP/editor, approval, audit, result envelope, or OCEL trace relevance surfaces. v0.32.1 models those surfaces as observation contracts only.

Workspace observation is not workspace access. Code-edit surface observation is not code edit permission. Patch observation is not patch application. Tool registry surface observation is not tool invocation or registration. Plugin surface observation is not plugin loading. Provider hook surface observation is not provider invocation. Command risk boundary is not command execution.

## Observation Focus Taxonomy

Observation focus values include workspace structure, file operation model, code edit model, patch model, search model, tool registry model, plugin model, external plugin model, provider hook model, command execution boundary, shell boundary, LSP/editor boundary, dependency manifest, configuration manifest, approval boundary, audit boundary, result envelope, OCEL trace relevance, digestion relevance, dominion relevance, and unknown.

Digestion relevance is not a digestion candidate. Dominion relevance is not a dominion target.

## Capability Taxonomy

Capabilities are descriptive only: read workspace, read file, write file, edit code, apply patch, search workspace, invoke tool, register tool, load plugin, load external plugin, invoke provider, execute command, execute shell, editor integration, dependency resolution, config loading, approval gate, audit trace, result envelope, and unknown. High-risk capability kinds do not imply permission.

## Risk Signal Taxonomy

Risk signals include workspace write risk, code edit risk, patch application risk, command execution risk, shell execution risk, provider invocation risk, network access risk, credential access risk, plugin loading risk, external plugin risk, dependency install risk, tool registry mutation risk, tool invocation risk, LSP/editor runtime risk, raw output persistence risk, secret file read risk, memory mutation risk, registry mutation risk, OCEL emission risk, and unknown. A risk signal is advisory and not proof of exploitability.

## Reference Source Ref

`OpenCodeReferenceSourceRef` points to v0.32.0 reference source, inventory, entry ids, or a local path ref string. It is not source fetch and not execution. Local path refs remain path references only.

## Surface Observations

`OpenCodeSurfaceObservation` records a surface kind, focus kind, capability kind, source refs, evidence quality, risk signal kinds, boundary notes, prohibited runtime actions, assumptions, and limitations. High-risk capabilities require prohibited runtime actions.

`OpenCodeWorkspaceSurfaceObservation` summarizes static file tree candidates and detected high-risk surfaces. It is not workspace access or workspace write.

`OpenCodeToolRegistrySurfaceObservation` records possible tool manifest, registry, and invocation paths. It is not tool registration, registry mutation, or tool invocation.

`OpenCodePluginSurfaceObservation` records possible plugin manifest/runtime/external plugin paths. It is not plugin loading.

`OpenCodeProviderHookSurfaceObservation` records possible provider config and provider hook paths. It is not provider invocation, network access, or credential access.

`OpenCodeCommandRiskBoundary` records possible command and shell paths and required boundaries. It is not command or shell execution.

`OpenCodeConfigManifestObservation` records possible package manifests, config paths, lockfiles, scripts, and dependency entries. Config manifest observation is not dependency installation or script execution.

## Static Observation Input And Output

`OpenCodeStaticObservationInput` consumes v0.32.0 `ExternalHarnessProfile`, `ReferenceCorpusSnapshot`, and `ReferenceFileInventory` identifiers or source refs. It is not an execution request.

`OpenCodeObservationFinding` records static findings. `OpenCodeRiskSignal` records advisory risk signals. `OpenCodeDigestionHint` is not `InternalSkillCandidate`. `OpenCodeDominionHint` is not `DominionTarget`.

`OpenCodeStyleObservationProfile` and `OpenCodeObservationOutput` aggregate observations, findings, risk signals, digestion hints, and dominion hints. Output is not manifest extraction execution, not digestion candidate generation, and not dominion target creation.

## No-Execution Guarantee

`OpenCodeObservationRunPreview` and `OpenCodeNoExecutionGuarantee` explicitly preserve no OpenCode execution, no reference code execution, no dependency install, no runtime import, no workspace write, no code edit, no patch application, no tool invocation, no tool registration, no plugin loading, no external plugin loading, no provider invocation, no command or shell execution, no network, no credential, no secret file read, no registry mutation, no memory mutation, and no OCEL emission.

## Readiness Report

`V0321ReadinessReport` can set design-stage readiness for v0.32.4 manifest extraction and v0.32.5 risk classification only. It cannot mark execution, OpenCode execution, reference code execution, workspace write, code edit, command execution, provider invocation, network access, or credential access as ready.

## Optional Local Integration Test Policy

Optional integration tests may exist under `tests/integration/test_opencode_reference_static_observation.py`, but they must skip unless `CHANTA_ENABLE_REFERENCE_CORPUS_TESTS=1`. They must use `CHANTA_REFERENCES_ROOT` or skip safely, locate OpenCode reference folders by conventional names only if present, skip when absent, and remain read-only. They must not execute files, import files, install dependencies, run package scripts or tests, read `.env`/secret/key/token/credential files, or be required for default test pass.

## Flow

ExternalHarnessProfile / ReferenceCorpusSnapshot / ReferenceFileInventory -> OpenCodeStaticObservationInput -> OpenCode surface observations -> OpenCodeObservationFinding / RiskSignal -> DigestionHint / DominionHint -> OpenCodeObservationOutput -> V0321ReadinessReport -> v0.32.4 External Skill Manifest Extraction.

## Safe Defaults

All execution and runtime readiness flags remain false. `ready_for_v0324_manifest_extraction` and `ready_for_v0325_risk_classification` mean next design-stage handoff only.

## Hard Prohibitions

v0.32.1 does not execute OpenCode, external harnesses, or reference code; does not import reference code; does not install dependencies; does not run reference tests; does not live-scan; does not fetch source refs or URLs; does not write workspace files; does not edit code; does not apply patches; does not invoke tools; does not register tools; does not load plugins; does not invoke providers; does not access network, credentials, or secrets; does not execute commands or shell; does not implement browser/RPA/gateway runtime; does not send packets; does not mutate registry or memory; does not emit OCEL events; and does not implement UI runtime.

## Gate Inheritance

v0.32.1 inherits v0.32.0 gates and v0.31.9 consolidation gates. v0.32.0 established that reference corpus operations are read-only static observation contracts only. v0.32.1 preserves that boundary for OpenCode-style harness observation.

## Acceptance Criteria

- OpenCode surface, observation focus, capability, risk signal, status, and evidence taxonomies exist.
- Reference source refs, surface observations, workspace/tool/plugin/provider/command/config observations, input, findings, risk signals, digestion hints, dominion hints, profile, output, run preview, no-execution guarantee, and readiness report exist.
- Helpers classify inventory metadata without executing, importing, installing, scanning, writing, or reading prohibited content.
- All runtime readiness flags remain false.
- Focused tests pass.

## Withdrawal Conditions

Withdraw v0.32.1 readiness if implementation executes OpenCode or reference code, imports reference code, installs dependencies, runs commands, scans live systems, fetches sources, writes workspace files, edits code, applies patches, invokes tools/providers/plugins, accesses network or credentials, reads secret files, mutates registry or memory, emits OCEL events, creates `InternalSkillCandidate`, creates `DominionTarget`, or sets runtime readiness flags true.

## Handoff

v0.32.1 hands off to v0.32.4 External Skill Manifest Extraction and v0.32.5 Risk Classification as design-stage work only. It is not execution readiness.
