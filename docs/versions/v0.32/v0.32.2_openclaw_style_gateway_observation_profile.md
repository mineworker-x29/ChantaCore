# v0.32.2 OpenClaw-style Gateway Observation Profile

Parent track: External Harness Observation & Digestion Pipeline.

v0.32.2 builds on v0.32.0 External Harness Profile Contract + Reference Corpus Static Observation Contract and inherits v0.31.9 Internal Triad Skill Foundation v1 boundaries. It also coexists with v0.32.1 OpenCode-style Harness Observation Profile without depending on OpenCode-specific internals.

## Release Boundary

OpenClaw-style profile is not OpenClaw execution. It is a gateway/action harness observation contract only. It does not execute OpenClaw, execute external harnesses, execute or import `references/` code, install dependencies, run reference project tests, live-scan external systems, fetch `source_ref`, connect to gateways, access channels, receive or send messages, access email/calendar/contacts, access attachments or private data, access credentials, call webhooks, access network, invoke providers, execute commands, send packets, mutate registry or memory, emit OCEL events, or implement UI runtime.

`references/OpenClaw` is read-only static corpus only when present. Reference corpus static observation is not live gateway scanning.

## Definition

An OpenClaw-style harness is a gateway/action harness style surface that may expose gateway, channel, message, app action, webhook, email, calendar, contact, notification, attachment, private data, credential, network, approval, audit, automation trigger, result envelope, or OCEL trace relevance surfaces. v0.32.2 models those surfaces as observation contracts only.

Gateway observation is not gateway connection. Channel observation is not channel access. Message boundary is not message send. Action surface observation is not action execution. Private data boundary is not private data access. Credential/network boundary is not credential or network access. Approval boundary is not approval granted.

## Observation Focus Taxonomy

Observation focus values include gateway model, channel model, message flow model, action model, webhook model, email boundary, calendar boundary, contact boundary, notification boundary, file attachment boundary, private data boundary, credential boundary, network boundary, approval boundary, audit boundary, automation trigger boundary, result envelope, OCEL trace relevance, digestion relevance, dominion relevance, and unknown.

Digestion relevance is not a digestion candidate. Dominion relevance is not a dominion target.

## Capability Taxonomy

Capabilities are descriptive only: observe gateway, observe channel, receive message, send message, invoke app action, call webhook, read email, send email, read calendar, modify calendar, read contacts, send notification, read attachment, access private data, use credential, access network, request approval, record audit, emit result envelope, and unknown. High-risk capability kinds do not imply permission.

## Risk Signal Taxonomy

Risk signals include gateway control risk, channel access risk, message send risk, email send risk, calendar write risk, contact access risk, notification send risk, attachment access risk, private data exposure risk, credential access risk, network access risk, webhook call risk, app control risk, external side effect risk, approval bypass risk, audit gap risk, raw output persistence risk, memory mutation risk, registry mutation risk, OCEL emission risk, and unknown. A risk signal is advisory and not proof of exploitability.

## Reference Source Ref

`OpenClawReferenceSourceRef` points to v0.32.0 reference source, inventory, entry ids, or a local path ref string. It is not source fetch and not execution. Local path refs remain path references only.

## Surface Observations

`OpenClawSurfaceObservation` records a surface kind, focus kind, capability kind, source refs, evidence quality, risk signal kinds, boundary notes, prohibited runtime actions, assumptions, and limitations. High-risk capabilities require prohibited runtime actions.

`OpenClawGatewaySurfaceObservation` records possible gateway manifests, runtime paths, configs, and risk flags. It is not gateway connection or gateway runtime.

`OpenClawChannelSurfaceObservation` records possible channel manifests, runtime paths, and message surfaces. It is not channel access, message receive, or message send.

`OpenClawActionSurfaceObservation` records possible action manifests, runtime paths, triggers, and side-effect risk. It is not action execution or external action invocation.

`OpenClawMessageSurfaceBoundary` records possible message/send/receive paths and required boundaries. It is not message operation.

`OpenClawPrivateDataBoundaryObservation` records possible private data, attachment, email, calendar, and contact surfaces. It is not private data access or attachment access.

`OpenClawCredentialNetworkBoundaryObservation` records possible credential, network, and webhook surfaces. It is not credential access, network access, or webhook call.

`OpenClawApprovalBoundaryRequirement` records approval and audit requirements. Approval boundary is not approval granted.

`OpenClawConfigManifestObservation` records possible package, config, action, channel, gateway, script, and dependency entries. Config manifest observation is not dependency installation or script execution.

## Static Observation Input And Output

`OpenClawStaticObservationInput` consumes v0.32.0 `ExternalHarnessProfile`, `ReferenceCorpusSnapshot`, and `ReferenceFileInventory` identifiers or source refs. It is not an execution request.

`OpenClawObservationFinding` records static findings. `OpenClawRiskSignal` records advisory risk signals. `OpenClawDigestionHint` is not `InternalSkillCandidate`. `OpenClawDominionHint` is not `DominionTarget`.

`OpenClawStyleObservationProfile` and `OpenClawObservationOutput` aggregate observations, findings, risk signals, digestion hints, and dominion hints. Output is not manifest extraction execution, not digestion candidate generation, and not dominion target creation.

## No-Execution Guarantee

`OpenClawObservationRunPreview` and `OpenClawNoExecutionGuarantee` explicitly preserve no OpenClaw execution, no reference code execution, no dependency install, no runtime import, no gateway connection or control, no channel access, no message receive or send, no email access or send, no calendar access or write, no contact access, no notification send, no attachment access, no private data access, no credential access, no webhook call, no network, no command execution, no provider invocation, no registry mutation, no memory mutation, and no OCEL emission.

## Readiness Report

`V0322ReadinessReport` can set design-stage readiness for v0.32.4 manifest extraction and v0.32.5 risk classification only. It cannot mark execution, OpenClaw execution, reference code execution, gateway connection, channel access, message send, private data access, credential access, network access, or webhook call as ready.

## Optional Local Integration Test Policy

Optional integration tests may exist under `tests/integration/test_openclaw_reference_static_observation.py`, but they must skip unless `CHANTA_ENABLE_REFERENCE_CORPUS_TESTS=1`. They must use `CHANTA_REFERENCES_ROOT` or skip safely, locate OpenClaw reference folders by conventional names only if present, skip when absent, and remain read-only. They must not execute files, import files, install dependencies, run package scripts or tests, read `.env`/secret/key/token/credential files, or be required for default test pass.

## Flow

ExternalHarnessProfile / ReferenceCorpusSnapshot / ReferenceFileInventory -> OpenClawStaticObservationInput -> OpenClaw surface observations -> OpenClawObservationFinding / RiskSignal -> DigestionHint / DominionHint -> OpenClawObservationOutput -> V0322ReadinessReport -> v0.32.4 External Skill Manifest Extraction.

## Safe Defaults

All execution and runtime readiness flags remain false. `ready_for_v0324_manifest_extraction` and `ready_for_v0325_risk_classification` mean next design-stage handoff only.

## Hard Prohibitions

v0.32.2 does not execute OpenClaw, external harnesses, or reference code; does not import reference code; does not install dependencies; does not run reference tests; does not live-scan; does not fetch source refs or URLs; does not connect to gateways or channels; does not receive or send messages; does not access email, calendar, contacts, attachments, private data, or credentials; does not call webhooks; does not access network; does not invoke providers; does not execute commands or shell; does not implement browser/RPA/gateway runtime; does not send packets; does not mutate registry or memory; does not emit OCEL events; and does not implement UI runtime.

## Gate Inheritance

v0.32.2 inherits v0.32.0 gates and v0.31.9 consolidation gates. v0.32.0 established that reference corpus operations are read-only static observation contracts only. v0.32.2 preserves that boundary for OpenClaw-style gateway/action observation.

## Acceptance Criteria

- OpenClaw surface, observation focus, capability, risk signal, status, and evidence taxonomies exist.
- Reference source refs, surface observations, gateway/channel/action/message/private-data/credential-network/approval/config observations, input, findings, risk signals, digestion hints, dominion hints, profile, output, run preview, no-execution guarantee, and readiness report exist.
- Helpers classify inventory metadata without executing, importing, installing, scanning, connecting, sending, calling, writing, or reading prohibited content.
- All runtime readiness flags remain false.
- Focused tests pass.

## Withdrawal Conditions

Withdraw v0.32.2 readiness if implementation executes OpenClaw or reference code, imports reference code, installs dependencies, runs commands, scans live systems, fetches sources, connects to gateways or channels, receives or sends messages, accesses email/calendar/contacts/private data/attachments/credentials, calls webhooks, invokes providers, accesses network, mutates registry or memory, emits OCEL events, creates `InternalSkillCandidate`, creates `DominionTarget`, or sets runtime readiness flags true.

## Handoff

v0.32.2 hands off to v0.32.4 External Skill Manifest Extraction and v0.32.5 Risk Classification as design-stage work only. It is not execution readiness.
