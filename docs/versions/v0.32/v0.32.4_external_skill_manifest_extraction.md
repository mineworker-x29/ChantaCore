# v0.32.4 External Skill Manifest Extraction

Parent track: External Harness Observation & Digestion Pipeline.

v0.32.4 adds a static manifest candidate extraction contract on top of v0.32.0 External Harness Profile Contract and Reference Corpus Static Observation Contract. It consumes, but does not execute, v0.32.1 OpenCode-style observation output, v0.32.2 OpenClaw-style observation output, v0.32.3 Hermes-style observation output, ReferenceFileInventory, and ReferenceCorpusSnapshot artifacts. It preserves the v0.31.9 Internal Triad Skill Foundation boundary: extraction findings are not certification, digestion candidates, InternalSkillCandidate objects, or DominionTarget objects.

## Definition

External Skill Manifest Extraction identifies manifest-like candidate artifacts from already available static observations and safe inventory metadata. It emits structured manifest candidate records for later review.

Manifest extraction is not runtime discovery. Manifest candidate is not active manifest. Reference path is not execution or import permission.

## Boundary Rules

- Skill manifest candidate is not active skill.
- Tool manifest candidate is not registered tool and not tool invocation.
- Plugin manifest candidate is not plugin loading.
- External plugin manifest candidate is not external plugin loading.
- Mission manifest candidate is not mission installation or mission execution.
- Gateway manifest candidate is not gateway connection, message send, or action execution.
- Provider manifest candidate is not provider invocation, network access, or credential access.
- Profile manifest candidate is not profile activation, memory access, or private data access.
- Candidate set is not registry.
- Extraction rule is not scanner runtime.
- Extraction finding is not certification and not activation.
- ready_for_v0325_risk_classification and ready_for_v0326_digestion_candidate_generation are design-stage handoff flags only.
- All execution/readiness flags remain false.

## Reference Corpus Support

references/OpenCode, references/OpenClaw, references/Hermes, or similar directories may be represented only through v0.32.0 read-only reference corpus artifacts. v0.32.4 prefers ReferenceFileInventory metadata, ReferenceCorpusSnapshot metadata, and observation outputs. Secret files are not read. Source refs are not fetched. URLs are not fetched. No live scan is performed.

Optional local integration tests, if added later, must be disabled by default with CHANTA_ENABLE_REFERENCE_CORPUS_TESTS=1, use CHANTA_REFERENCES_ROOT or skip safely, and perform read-only inventory/metadata checks only. They must never execute files, import files, install dependencies, run package scripts or tests, or read .env, secret, key, token, or credential files.

## Contract Artifacts

ExternalManifestSourceRef records source kind, source IDs, reference entry IDs, manifest path refs, and evidence refs. It is not source fetch, file execution, or runtime import.

ExternalManifestFieldObservation records static field names, summaries, observed string values, inferred declarative capabilities, effect surfaces, risk surfaces, evidence refs, and limitations. It is not manifest activation.

ExternalManifestExtractionRule records non-runtime path patterns, file extensions, detected kinds, and prohibited file patterns. It produces no runtime effects and is not scanner runtime.

Manifest candidates:

- ExternalSkillManifestCandidate: inactive skill design artifact.
- ExternalToolManifestCandidate: inactive tool manifest candidate; not registered and not invoked.
- ExternalPluginManifestCandidate: inactive plugin manifest candidate; not loaded.
- ExternalMissionManifestCandidate: inactive mission manifest candidate; not installed or executed.
- ExternalGatewayManifestCandidate: inactive gateway manifest candidate; no gateway connection, message send, or action execution.
- ExternalProviderManifestCandidate: inactive provider manifest candidate; no provider invocation, network access, or credential access.
- ExternalProfileManifestCandidate: inactive profile manifest candidate; no profile activation, memory access, or private data access.

ExternalManifestCandidateSet groups candidates for later design-stage review. It is not a registry. ExternalManifestExtractionInput is not an execution request. ExternalManifestExtractionFinding is not certification. ExternalManifestExtractionReport is not runtime extraction.

ExternalManifestExtractionRunPreview and ExternalManifestNoRuntimeGuarantee preserve no-runtime guarantees. V0324ReadinessReport keeps ready_for_manifest_activation, ready_for_execution, ready_for_harness_execution, ready_for_reference_code_execution, ready_for_plugin_loading, ready_for_tool_registration, ready_for_tool_invocation, ready_for_mission_installation, ready_for_provider_invocation, ready_for_gateway_connection, ready_for_network_access, ready_for_credential_access, and ready_for_command_execution false.

## Flow

OpenCode/OpenClaw/Hermes ObservationOutput + ReferenceFileInventory + ReferenceCorpusSnapshot -> ExternalManifestExtractionInput -> ExternalManifestFieldObservation / Finding -> Manifest Candidates -> ExternalManifestCandidateSet -> ExternalManifestExtractionReport -> V0324ReadinessReport -> v0.32.5 External Capability Risk Classification.

v0.32.6 Digestion Candidate Generator may consume the later reviewed output as design-stage input only. v0.32.4 itself does not create InternalSkillCandidate objects.

## Hard Prohibitions

v0.32.4 must not execute external harnesses, execute OpenCode/OpenClaw/Hermes, execute references/ code, import references/ code, install dependencies, run tests inside references/, load plugins, load external plugins, register tools, invoke tools, install missions, execute missions, activate profiles, access live memory, write memory, invoke providers, connect to gateways, access channels, send messages, call webhooks, perform workspace writes, edit code, apply patches, access credentials or secrets, read .env/secret/key/token/credential files as data, perform network calls, invoke browser automation, invoke RPA, send packets, fetch source_ref, fetch URLs, live-scan external targets, spawn processes, use subprocess/shell/command execution, mutate registries, activate policies, emit OCEL events, persist OCEL objects or relations, write runtime logs, write database records, implement UI runtime, execute rollback, or retry.

## Gate Inheritance

v0.32.4 inherits v0.32.0-v0.32.3 gates:

- ExternalHarnessProfile remains non-executable.
- Reference corpus remains read-only static observation material.
- OpenCode-style profile remains not OpenCode execution.
- OpenClaw-style profile remains not gateway/action execution.
- Hermes-style profile remains not Hermes runtime.

## Acceptance Criteria

- Source, candidate, status, capability, effect, risk, and evidence taxonomies exist.
- Source refs, field observations, extraction rules, specialized manifest candidates, candidate set, input, finding, report, run preview, no-runtime guarantee, and readiness report exist.
- Helpers are pure and classify only from provided metadata such as path, file name, extension, and detected kind.
- No plugin loading, tool registration/invocation, mission installation/execution, provider invocation, gateway connection, network, credential, command, secret file read, reference code import/execution, dependency install, registry mutation, memory mutation, OCEL emission, or UI runtime is introduced.

## Withdrawal Conditions

Withdraw v0.32.4 readiness if any implementation path loads manifests at runtime, loads plugins, registers or invokes tools, installs or executes missions, invokes providers, connects gateways, reads secrets, fetches source refs or URLs, scans live systems, executes commands, imports or executes reference code, installs dependencies, mutates registry or memory, emits OCEL events, or sets any runtime readiness flag true.

## Handoff

v0.32.4 is ready for v0.32.5 Risk Classification and v0.32.6 Digestion Candidate Generator only when tests pass and runtime boundary checks remain clean. It is not execution readiness.
