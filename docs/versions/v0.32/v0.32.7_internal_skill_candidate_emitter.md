# v0.32.7 Internal Skill Candidate Emitter

Release name: v0.32.7 Internal Skill Candidate Emitter.

Parent track: External Harness Observation & Digestion Pipeline.

v0.32.7 builds on v0.32.6 Digestion Candidate Generator by consuming
`ExternalHarnessDigestionCandidate`, `ExternalToInternalPatternMap`,
`ExternalDigestionCandidateSet`, and digestion generation report metadata. It
also inherits the static observation, manifest extraction, risk classification,
and no-runtime gates from v0.32.0 through v0.32.6.

v0.32.7 also relates to the older v0.31.4 digestion candidate and
internalization planning concepts, but it does not create an
internalization plan. It emits external-derived internal candidate design
artifacts only.

## Definition

Internal candidate emission is the design-stage conversion of external
digestion patterns into reviewable internal candidate records:

- `EmittedInternalSkillCandidate`
- `EmittedInternalToolContractCandidate`
- `EmittedInternalMissionCandidate`
- `EmittedInternalPolicyCandidate`
- `EmittedInternalMemorySchemaCandidate`
- `EmittedInternalPromptPatternCandidate`
- `EmittedInternalTraceEventPatternCandidate`
- `EmittedInternalResultEnvelopeCandidate`
- `EmittedInternalApprovalBoundaryCandidate`
- `EmittedInternalProfilePatternCandidate`

These records are not active artifacts. They are not implementation files,
runtime handlers, registry entries, memory writers, mission installations, or
permission grants.

## Non-Runtime Semantics

An emitted internal candidate is a design-stage artifact only. It is not an
active skill, registered tool, installed mission, active policy, memory writer,
prompt injection, OCEL event emission, result ingestion, approval grant,
implementation, internalization plan, or execution permission.

`EmittedInternalSkillCandidate` is not an active skill.
`EmittedInternalToolContractCandidate` is not a registered tool.
`EmittedInternalMissionCandidate` is not an installed mission.
`EmittedInternalPolicyCandidate` is not an active policy.
`EmittedInternalMemorySchemaCandidate` is not a memory writer.
`EmittedInternalPromptPatternCandidate` is not prompt injection.
`EmittedInternalTraceEventPatternCandidate` is not OCEL event emission.
`EmittedInternalResultEnvelopeCandidate` is not result ingestion.
`EmittedInternalApprovalBoundaryCandidate` is not approval granted.
`EmittedInternalProfilePatternCandidate` is not profile activation or private
data access.

`ExternalHarnessInternalCandidateSet` is not a registry. An
`InternalCandidateEmissionReport` is not a runtime result. Candidate emission is
not implementation and is not an `InternalizationPlan`.

`ready_for_v0328` and `ready_for_v0329` are next design-stage handoff signals
only. They are not execution readiness. `ready_for_candidate_activation`,
`ready_for_registry_mutation`, `ready_for_internalization`, and
`ready_for_execution` remain false.

## Taxonomies

`InternalCandidateEmissionKind` describes emitted candidate record shape:
skill, tool contract, mission, policy, memory schema, prompt pattern, trace
event pattern, result envelope, approval boundary, profile pattern, no-op,
future track, and unknown. The kind is descriptive and does not activate or
register anything.

`InternalCandidateEmissionStatus` describes draft, emitted,
emitted-with-gaps, review, blocked, deferred, rejected, future-track, no-op,
and unknown states. `emitted` is not active. `requires_review` is not approval.

`InternalCandidateEmissionRoute` describes handoff routing:
`emit_for_v0329_consolidation`, `send_to_v0328_dominion_emitter`, review,
defer, reject, block, future-track, no-op, and unknown. The route is not
execution and does not create a dominion artifact.

`InternalCandidateEmissionSourceKind` describes source metadata such as
external digestion candidates, pattern maps, candidate sets, generation
reports, digestibility reports, manifest candidates, risk classifications,
OpenCode/OpenClaw/Hermes observation outputs, reference inventories, reference
corpus snapshots, manual review, and unknown. Source kinds do not fetch, scan,
import, or execute.

`InternalCandidateEmissionEvidenceQuality` describes evidence strength:
unknown, none, weak, partial, sufficient for candidate emission, sufficient for
v0.32.9 review, conflicting, and blocked. Evidence quality is not activation
evidence or runtime trust.

`InternalCandidateEmissionReviewRequirementKind` covers evidence, contract,
schema, validation, test, documentation, safety, boundary, OCEL trace, human,
future-gate, and unknown reviews. A review requirement is not approval, and a
human review requirement is not approval granted.

`InternalCandidateEmissionBlockerKind` describes diagnostic blockers:
insufficient evidence, conflicting evidence, missing external digestion
candidate, missing pattern map, blocked digestibility assessment, dominion
route, future gate, unsafe runtime surface, missing contract summaries,
incompatibility with the internal triad, duplicate candidate risk, and unknown.
A blocker does not remediate.

## Model Layer

Source refs identify static source metadata and evidence references. They are
not source fetches and are not execution.

Evidence refs summarize static evidence quality, limitations, and conflicts.
They are not runtime trust.

Review requirements preserve approval as false and default to blocking
activation and registry mutation. They are not approval.

Blockers describe why a candidate cannot proceed cleanly to later review. They
do not execute remediation.

The emitted candidate base enforces the shared invariant that activation,
skill activation, registry mutation, memory mutation, internalization, and
execution readiness remain false. Specialized emitted candidates add the
proposed contract fields for skills, tools, missions, policies, memory schemas,
prompt patterns, trace event patterns, result envelopes, approval boundaries,
and profile patterns.

`ExternalHarnessInternalCandidateSet` groups emitted design artifacts by
candidate family and records deferred, rejected, blocked, dominion-required,
future-track, and no-op source refs. It is not a registry and does not activate
any candidate.

`InternalCandidateEmissionInput`, `InternalCandidateEmissionFinding`, and
`InternalCandidateEmissionReport` capture the static input, item-level finding,
and report layer. The report is not a runtime result and not implementation.

`InternalCandidateEmissionRunPreview` lists planned static steps and artifacts
while explicitly guaranteeing no harness execution, reference code execution,
install, import runtime, plugin loading, tool registration or invocation,
mission installation or execution, provider invocation, gateway connection,
network, credentials, command execution, secret file reads, candidate
activation, skill activation, registry mutation, memory mutation,
internalization, or implementation creation.

`InternalCandidateEmissionNoRuntimeGuarantee` records the same no-runtime
guarantees as contract metadata. `V0327ReadinessReport` records only design
handoff readiness to v0.32.8 or v0.32.9 and keeps all runtime, activation,
registry, memory, internalization, implementation, plugin, tool, mission,
provider, gateway, network, credential, and command flags false.

## Flow

`ExternalHarnessDigestionCandidate` / `ExternalToInternalPatternMap` /
`ExternalDigestionCandidateSet` / `GenerationReport`
-> `InternalCandidateEmissionInput`
-> emitted internal candidate artifacts
-> `ExternalHarnessInternalCandidateSet`
-> `InternalCandidateEmissionReport`
-> `V0327ReadinessReport`
-> v0.32.8 External Dominion Candidate Emitter or v0.32.9 Consolidation.

## Optional Local Integration Test Policy

Any optional reference-corpus integration test must skip unless
`CHANTA_ENABLE_REFERENCE_CORPUS_TESTS=1`. It must use `CHANTA_REFERENCES_ROOT`
or skip safely, skip if the reference root is absent, and inspect only
read-only inventory or metadata permitted by the reference inventory policy.
It must never execute files, import files, install dependencies, run package
scripts or tests, read `.env`/secret/key/token/credential files, or become
required for the default test pass.

## Safe Defaults

All emitted candidates default to inactive design status. Activation,
registration, mutation, internalization, implementation, and execution flags
are false. Approval is false. Raw output and memory persistence are false for
result-envelope candidates.

## Hard Prohibitions

v0.32.7 does not execute external harnesses, OpenCode, OpenClaw, Hermes, or
reference code. It does not import reference code, install dependencies, run
reference tests, live-scan external systems, fetch source refs or URLs, read
secret files, load plugins, register or invoke tools, install or execute
missions, invoke providers, connect gateways, access channels, send messages,
call webhooks, use network, access credentials, execute commands, use browser
automation, control RPA, send packets, mutate registries, mutate memory, emit
OCEL events, persist runtime records, create implementation files, implement UI
runtime, execute rollback, retry, activate emitted candidates, create active
internal runtime artifacts, create an `InternalizationPlan`, create
`DominionTarget`, or create `DominionDecision`.

## Gate Inheritance

v0.32.7 inherits the v0.32.0 reference-corpus read-only contract, v0.32.1
OpenCode observation non-runtime profile, v0.32.2 OpenClaw gateway observation
non-runtime profile, v0.32.3 Hermes runtime observation non-runtime profile,
v0.32.4 inactive manifest candidate contract, v0.32.5 risk classification
non-permission contract, and v0.32.6 digestion candidate design-stage
contract. It also preserves the v0.31.9 internal triad foundation boundary.

## Acceptance Criteria

- Emission taxonomies exist.
- Source refs, evidence refs, blockers, review requirements, emitted candidate
  models, candidate set, input, finding, report, run preview, no-runtime
  guarantee, and readiness report exist.
- Helper functions are pure and create contract artifacts only.
- Emitted candidates remain inactive design artifacts.
- Candidate set is not registry.
- Candidate emission is not implementation and not internalization.
- Runtime, activation, registry, memory, internalization, implementation,
  plugin, tool, mission, provider, gateway, network, credential, and command
  readiness flags remain false.

## Withdrawal Conditions

Withdraw the v0.32.7 readiness claim if any implementation path executes
external or reference code, imports reference code as runtime, installs
dependencies, performs network or credential access, reads secret files,
registers or invokes tools, installs or executes missions, invokes providers,
connects gateways, mutates registries or memory, emits OCEL runtime events,
creates implementation files, activates candidates, creates an
internalization plan, creates dominion artifacts, or sets a prohibited
readiness flag true.

## Handoff

v0.32.7 can hand off to v0.32.8 External Dominion Candidate Emitter for
design-stage dominion review and to v0.32.9 Consolidation for external
observation/digestion consolidation. That handoff is not execution readiness,
candidate activation, registry mutation, internalization, implementation, or
dominion target creation.
