# v0.32.6 Digestion Candidate Generator

Parent track: External Harness Observation & Digestion Pipeline.

v0.32.6 builds on v0.32.5 External Capability Risk Classification, v0.32.4 External Skill Manifest Extraction, v0.32.0-v0.32.3 observation/profile/reference artifacts, and v0.31.9 Internal Triad Skill Foundation.

## Definition

External digestion candidate generation is a contract-layer process that selects digestible external patterns from static risk classifications and manifest candidates. It emits design-stage `ExternalHarnessDigestionCandidate` and `ExternalToInternalPatternMap` artifacts.

It does not emit internal candidates. It does not create active skills, tools, missions, policies, memory schemas, internalization plans, dominion targets, or runtime objects.

## Boundary Statements

`ExternalHarnessDigestionCandidate` is not `InternalSkillCandidate`.

`ExternalHarnessDigestionCandidate` is not active internalization.

`ExternalToInternalPatternMap` is not `InternalizationPlan`.

Pattern signal is not generated code.

Digestibility assessment is not approval.

Harness pattern digestibility report is not runtime certification.

Candidate set is not registry.

`ready_for_v0327_internal_candidate_emitter` and `ready_for_v0328_dominion_candidate_emitter` mean next design-stage handoff only. They are not execution readiness.

`ready_for_internal_candidate_emission`, `ready_for_internal_candidate_creation`, `ready_for_internalization`, and `ready_for_execution` remain false in v0.32.6.

v0.32.6 does not emit internal candidates and does not create `DominionTarget`.

## Contract Artifacts

`ExternalDigestionSourceRef` references static upstream artifacts only.

`ExternalDigestionPatternSignal` records a pattern signal and optional suggested internal artifact kind. The suggestion is not artifact creation.

`ExternalDigestibilityAssessment` records posture, route, blockers, evidence, and gaps. It is not approval or internalization.

`ExternalDigestionBlocker` records why an item is blocked, routed to dominion, or future gated. It does not execute remediation.

`ExternalToInternalPatternMap` maps external patterns to suggested internal contract summaries. It is not `InternalSkillCandidate` and not `InternalizationPlan`.

`ExternalHarnessDigestionCandidate` groups source refs, pattern signals, assessment, optional pattern map, evidence, blockers, status, and route. It is a design-stage artifact only.

`ExternalDigestionCandidateSet` groups external digestion candidates. It is not a registry and does not emit internal candidates.

`HarnessPatternDigestibilityReport` summarizes pattern counts, candidate counts, blockers, gaps, and design-stage handoffs. It is not runtime certification.

`ExternalDigestionCandidateGenerationInput`, `ExternalDigestionCandidateGenerationFinding`, and `ExternalDigestionCandidateGenerationReport` define the generator contract. Findings are not internal candidates or certification. Reports are not runtime generation.

`ExternalDigestionCandidateRunPreview` and `ExternalDigestionCandidateNoRuntimeGuarantee` preserve the no-runtime boundary.

`V0326ReadinessReport` is a design-stage readiness report only. It is not runtime enablement.

## Safe Flow

`ExternalCapabilityRiskClassificationReport` / `ExternalCapabilityRiskMap` / `ExternalCapabilityBoundaryMap` / `ExternalManifestCandidateSet` / ObservationOutputs / `ReferenceFileInventory`
-> `ExternalDigestionCandidateGenerationInput`
-> `ExternalDigestionPatternSignal` / `ExternalDigestibilityAssessment` / `ExternalDigestionBlocker`
-> `ExternalToInternalPatternMap`
-> `ExternalHarnessDigestionCandidate`
-> `ExternalDigestionCandidateSet`
-> `HarnessPatternDigestibilityReport`
-> `ExternalDigestionCandidateGenerationReport`
-> `V0326ReadinessReport`
-> v0.32.7 Internal Skill Candidate Emitter.

The flow is contract-artifact handoff only. It is not execution, not internal candidate emission, and not internalization.

## Reference Corpus Support

Reference material under `references/` may be represented through v0.32.0 `ReferenceFileInventory` and `ReferenceCorpusSnapshot` metadata. v0.32.6 should prefer those static artifacts and prior reports over direct file inspection.

Optional local integration tests, if added later, must skip unless `CHANTA_ENABLE_REFERENCE_CORPUS_TESTS=1`, use `CHANTA_REFERENCES_ROOT` safely, skip if references are absent, and perform read-only metadata checks only.

Secret-like files are not read. Reference paths are not execution or import permission.

## Hard Prohibitions

v0.32.6 does not execute external harnesses, OpenCode, OpenClaw, Hermes, or reference code. It does not import reference code, install dependencies, run package scripts, run tests inside references, load plugins, register or invoke tools, install or execute missions, activate profiles, access or write live memory, invoke providers, connect gateways, access channels, send messages, call webhooks, perform network calls, read credentials or secret files, execute commands, start browser automation, control RPA runtimes, send packets, mutate registries, mutate memory, emit OCEL events, write runtime logs, write database records, implement UI runtime, retry, or rollback.

v0.32.6 does not create `InternalSkillCandidate`, `InternalToolContractCandidate`, `InternalMissionCandidate`, `InternalPolicyCandidate`, `InternalMemorySchemaCandidate`, `InternalPromptPatternCandidate`, `InternalTraceEventPatternCandidate`, `InternalResultEnvelopeCandidate`, `InternalCandidateSet`, `InternalizationPlan`, `DominionTarget`, or `DominionDecision`.

## Gate Inheritance

v0.32.6 inherits v0.32.0-v0.32.5 gates:
- reference corpus operations remain read-only static metadata usage.
- observation profiles remain non-runtime.
- manifest candidates remain inactive.
- risk classifications remain non-permission and non-candidate-creating.
- all execution, internalization, registry, memory, plugin, tool, mission, provider, gateway, network, credential, and command readiness flags stay false.

## Acceptance Criteria

- Digestion candidate, pattern, route, posture, evidence, blocker, and source taxonomies exist.
- Source refs, pattern signals, assessments, blockers, pattern maps, external digestion candidates, candidate sets, reports, inputs, findings, run previews, guarantees, and readiness reports validate non-runtime semantics.
- Helper functions are pure and produce only v0.32.6 external digestion contract artifacts.
- `ExternalHarnessDigestionCandidate` remains design-stage only.
- `ExternalToInternalPatternMap` remains a pattern map only.
- v0.32.7 and v0.32.8 handoffs are design-stage only.

## Withdrawal Conditions

Withdraw v0.32.6 readiness if any implementation executes harnesses, executes or imports reference code, installs dependencies, performs live scanning, fetches source refs or URLs, reads secret files, loads plugins, registers or invokes tools, installs or executes missions, invokes providers, connects gateways, uses network or credentials, executes commands, mutates registry or memory, emits OCEL events, creates internal candidates, creates an internalization plan, creates dominion artifacts, or sets runtime/internalization readiness true.

## Handoff

v0.32.6 may hand off only to v0.32.7 Internal Skill Candidate Emitter and v0.32.8 External Dominion Candidate Emitter as design-stage follow-up work. It does not enable execution.
