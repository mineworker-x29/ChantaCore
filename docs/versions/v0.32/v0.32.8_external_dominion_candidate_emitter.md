# v0.32.8 External Dominion Candidate Emitter

Release name: v0.32.8 External Dominion Candidate Emitter.

Parent track: External Harness Observation & Digestion Pipeline.

v0.32.8 builds on v0.32.5 External Capability Risk Classification,
v0.32.6 Digestion Candidate Generator, and v0.32.7 Internal Skill Candidate
Emitter. It consumes dominion-required routes, dominion-required digestion
blockers, and dominion-required internal candidate source refs as metadata.

v0.32.8 also relates to v0.31.6 Dominion Target / Dominion Decision concepts,
but it does not create an active target or decision. It emits governance and
design-stage dominion candidate artifacts only.

## Definition

External dominion candidate emission is the static conversion of high-control
external risk surfaces into reviewable dominion governance candidates:

- `ExternalHarnessDominionCandidate`
- `EmittedInternalDominionTargetCandidate`
- `EmittedDominionControlBoundaryCandidate`
- `EmittedDominionFutureGateItem`
- `EmittedDominionNoOpDecision`
- `ExternalHarnessDominionCandidateSet`

These artifacts are not external control, authority grants, runtime
permissions, runtime registry entries, active dominion targets, or dominion
decisions.

## Non-Runtime Semantics

`ExternalHarnessDominionCandidate` is not external control.
`EmittedInternalDominionTargetCandidate` is not active
`InternalDominionTarget`.
`EmittedDominionControlBoundaryCandidate` is not permission and not runtime
enforcement.
`EmittedDominionFutureGateItem` is not readiness.
`EmittedDominionNoOpDecision` is valid and not failure.
`ExternalHarnessDominionCandidateSet` is not runtime registry.
`ExternalDominionCandidateEmissionReport` is not runtime result.

Authority posture is descriptive governance metadata, not an authority grant.
Candidate emission is not `DominionDecision` creation and does not grant
authority. D4-D9 remain future-track. `ready_for_v0329` means next
design-stage handoff only. `ready_for_external_control`,
`ready_for_authority_grant`, `ready_for_active_dominion_target_creation`,
`ready_for_dominion_decision_creation`, and `ready_for_execution` remain false.

v0.32.8 does not control external runtimes.

## Taxonomies

`ExternalDominionCandidateKind` describes candidate surfaces such as external
runtime control, provider invocation, network access, credential access,
command execution, browser runtime, RPA control, gateway connection, channel
access, message send, webhook calls, plugin loading, tool invocation, mission
execution, delegation execution, memory mutation, registry mutation, private
data access, raw output persistence, OCEL emission, approval and audit
boundaries, future-track, no-op, and unknown. Candidate kind is descriptive
and does not enable external control.

`ExternalDominionControlSurfaceKind` describes control surfaces including
external runtime, provider, network, credential, command, browser, RPA,
gateway, channel, message, webhook, plugin, external plugin, tool, mission,
delegation, memory, registry, private data, raw output, OCEL trace, approval,
audit, and unknown. Control surface kind does not grant permission.

`ExternalDominionRiskSurfaceKind` describes advisory risk surfaces including
external side effects, private data exposure, credential exposure, network side
effects, command execution, provider invocation, browser automation, RPA
control, gateway control, channel access, message send, webhook call, plugin
loading, tool invocation, mission execution, delegation execution, memory
contamination, raw output persistence, registry mutation, OCEL schema drift,
OCEL emission, approval bypass, audit gap, and unknown. Risk surface is not
proof of exploitability.

`ExternalDominionEmissionRoute` covers consolidation handoff, review, future
gate, defer, reject, block, future-track, no-op, and unknown. Route is not
execution and not external control.

`ExternalDominionCandidateStatus` covers draft, emitted, emitted-with-gaps,
review, blocked, deferred, rejected, future-track, no-op, and unknown.
`emitted` is not active, review is not approval, and no-op is valid.

`ExternalDominionAuthorityPosture` covers no authority, descriptive only,
boundary only, future gate only, no-op, blocked, future-track, and unknown. No
value grants authority. Boundary-only is not permission. Future-gate-only is
not readiness.

`ExternalDominionEvidenceQuality` covers unknown, none, weak, partial,
sufficient for dominion candidate, sufficient for v0.32.9 review, conflicting,
and blocked. Evidence quality is not authority evidence and not approval.

`ExternalDominionReviewRequirementKind` covers evidence, boundary, safety,
security, privacy, credential, network, command, provider, plugin, gateway,
memory, registry, OCEL trace, approval, audit, human, future gate, and unknown
reviews. Review requirement is not approval.

`ExternalDominionBlockerKind` covers insufficient evidence, conflicting
evidence, missing risk classification, missing boundary map, missing digestion
candidate, missing internal candidate emission context, unsafe runtime surface,
external control surface, authority grant required, future gate required,
plugin/tool/mission/provider/gateway/credential/network/command/memory/
registry/private-data surfaces, dominion governance incompatibility, and
unknown. A blocker is diagnostic and does not remediate. Authority-grant
required is a blocker/future-track signal, not authority grant.

`ExternalDominionSourceKind` describes risk classifications, risk maps,
boundary maps, digestion candidates, digestion candidate sets, generation
reports, internal candidate emission reports, internal candidate sets,
manifest candidates, OpenCode/OpenClaw/Hermes observation outputs, reference
inventories, reference snapshots, manual dominion review, and unknown. Source
kind does not fetch, scan, import, or execute.

## Model Layer

Source refs identify static source metadata. They are not source fetches and
not execution.

Evidence refs summarize static evidence quality, limitations, and conflicts.
They are not runtime trust and not authority grants.

Risk signals map candidate kinds, control surfaces, and advisory risk surfaces
to recommended boundaries and reviews. High, critical, or blocked severity
requires a boundary or review. Risk signals are not proof and do not grant
authority.

Control boundary candidates define blocked runtime actions, required reviews,
and future gates. They default to blocking execution, external control, and
authority grants. They are not permission and not runtime enforcement.

Review requirements preserve `approval_granted = False` and default to
blocking external control and authority grants. They are not approval.

Blockers describe why a dominion candidate cannot proceed cleanly. They do not
execute remediation.

Emitted dominion target candidates are inactive design artifacts. Their
`max_allowed_level` may be omitted or limited to descriptive levels no higher
than D3_SIMULATE. D4-D9 cannot be granted.

Emitted boundary candidates are not permission. Future gate items are not
readiness. No-op decisions are valid and not failure.

Candidate set, emission input, finding, report, run preview, no-runtime
guarantee, and readiness report preserve the same non-runtime boundary.

## Flow

`ExternalCapabilityRiskClassification` / `RiskMap` / `BoundaryMap` /
`ExternalDigestionCandidateSet` / `InternalCandidateEmissionReport`
-> `ExternalDominionCandidateEmissionInput`
-> `RiskSignal` / `BoundaryCandidate` / `FutureGate` / `NoOp`
-> `EmittedInternalDominionTargetCandidate`
-> `ExternalHarnessDominionCandidate`
-> `CandidateSet`
-> `ExternalDominionCandidateEmissionReport`
-> `V0328ReadinessReport`
-> v0.32.9 External Observation/Digestion Consolidation.

## Optional Local Integration Test Policy

Any optional reference-corpus integration test must skip unless
`CHANTA_ENABLE_REFERENCE_CORPUS_TESTS=1`. It must use `CHANTA_REFERENCES_ROOT`
or skip safely, skip if the reference root is absent, and inspect only
read-only inventory or metadata permitted by the reference inventory policy.
It must never execute files, import files, install dependencies, run package
scripts or tests, read `.env`/secret/key/token/credential files, or become
required for the default test pass.

## Safe Defaults

All runtime, control, authority, active target, decision, provider, gateway,
network, credential, command, browser, RPA, packet, registry, and memory
readiness flags default to false. All no-runtime guarantee fields default to
true. D4-D9 are prohibited as grants and remain future-track.

## Hard Prohibitions

v0.32.8 does not execute external harnesses, OpenCode, OpenClaw, Hermes, or
reference code. It does not import reference code, install dependencies, run
reference tests, live-scan external systems, fetch source refs or URLs, read
secret files, load plugins, register or invoke tools, install or execute
missions, invoke providers, connect gateways, access channels, send messages,
call webhooks, use network, access credentials, execute commands, use browser
automation, control RPA, send packets, mutate registries, mutate memory, emit
OCEL events, persist runtime records, write runtime logs, write database
records, implement UI runtime, execute rollback, retry, create active
`InternalDominionTarget`, create `DominionDecision`, grant authority, grant
D4-D9, or control external runtimes.

## Acceptance Criteria

- Dominion taxonomies exist.
- Source refs, evidence refs, risk signals, control boundary candidates,
  review requirements, blockers, target candidates, boundary candidates,
  future gates, no-op decisions, candidate set, input, finding, report, run
  preview, no-runtime guarantee, and readiness report exist.
- Helper functions are pure and create contract artifacts only.
- Candidate emission does not create active targets, decisions, authority
  grants, D4-D9 grants, external control, or runtime readiness.
- v0.32.0-v0.32.7 gates remain preserved.

## Withdrawal Conditions

Withdraw the v0.32.8 readiness claim if any implementation path executes
external or reference code, imports reference code as runtime, installs
dependencies, performs network or credential access, reads secret files,
registers or invokes tools, installs or executes missions, invokes providers,
connects gateways, mutates registries or memory, emits OCEL runtime events,
creates active dominion targets, creates dominion decisions, grants authority,
grants D4-D9, enables external control, or sets a prohibited readiness flag
true.

## Handoff

v0.32.8 hands off only to v0.32.9 External Observation/Digestion
Consolidation. That handoff is not execution readiness, external control,
authority grant, active dominion target creation, or dominion decision
creation.
