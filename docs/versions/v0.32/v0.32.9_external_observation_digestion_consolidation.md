# v0.32.9 External Observation / Digestion Consolidation

Release name: v0.32.9 External Observation / Digestion Consolidation.

Foundation release name: External Harness Observation & Digestion Pipeline v1.

Parent track: External Harness Observation & Digestion Pipeline.

v0.32.9 consolidates v0.32.0 through v0.32.8 and records the design-stage
handoff to v0.33 Internal General Agent Runtime MVP. It also inherits the
v0.30.9 External Dominion Control Plane Foundation and v0.31.9 Internal Triad
Skill Foundation gates.

## Scope

v0.32.9 is consolidation/control-plane only. Consolidation is not runtime
enablement. Pipeline readiness is not execution readiness. The v0.33 handoff
is not runtime execution readiness. The release manifest is not production
release. Coverage matrix records are not certification.

## What v0.32.x Accomplished

v0.32.0 defined the external harness profile contract and read-only reference
corpus static observation contract.

v0.32.1 modeled OpenCode-style static observation. v0.32.2 modeled
OpenClaw-style gateway observation. v0.32.3 modeled Hermes-style runtime
observation. These profiles remain non-runtime and do not execute harnesses.

v0.32.4 defined inactive external skill manifest extraction. v0.32.5 defined
capability risk classification as governance metadata, not permission.
v0.32.6 defined external digestion candidate generation as design-stage
pattern mapping, not internal candidate emission. v0.32.7 defined emitted
internal candidates as inactive design artifacts. v0.32.8 defined external
dominion candidates as governance/design-stage artifacts, not external control
or authority grants.

## What v0.32.x Did Not Accomplish

v0.32.x did not execute external harnesses, OpenCode, OpenClaw, Hermes, or
reference code. It did not import reference code as runtime modules, install
reference dependencies, live-scan external systems, fetch source refs or URLs,
read secrets, load plugins, register or invoke tools, install or execute
missions, invoke providers, connect gateways, access credentials, execute
commands, control browser/RPA/gateway runtimes, send packets, mutate registries
or memory, emit OCEL runtime events, persist runtime traces, implement UI
runtime, activate candidates, internalize candidates, grant authority, grant
D4-D9, or enable external control.

Read-only local tool execution remains false in v0.32.9. Runtime adapter
loading remains false in v0.32.9. Runtime OCEL emission remains false.

## Track Summaries

Reference Corpus Static Observation Track: read-only local corpus metadata can
be represented through inventory and snapshot artifacts. Reference corpus
coverage is not permission to read arbitrary files.

OpenCode-style profile: captures static command, provider, plugin, tool
registry, workspace, and manifest surfaces without execution.

OpenClaw-style profile: captures static gateway, action, channel, message,
approval, credential, network, and private-data surfaces without connection.

Hermes-style profile: captures static skill, tool, mission, memory, provider,
profile, delegation, and runtime-isolation surfaces without runtime activation.

Manifest extraction: identifies inactive external manifest candidates.

Risk classification: classifies candidates into safe descriptive, digestible,
review, dominion, blocked, future-track, no-op, or unknown governance routes.

Digestion candidate generation: emits external digestion candidates and
external-to-internal pattern maps as design artifacts only.

Internal candidate emission: emits inactive internal candidate records only.

External dominion candidate emission: emits governance/design-stage dominion
candidates only. D4-D9 remain future-track.

## Consolidation Artifacts

Release flags record that the pipeline foundation can be ready for v0.33
design handoff while all runtime readiness flags remain false.

Pipeline snapshot records included v0.32.0-v0.32.8 versions, artifact groups,
release flags, status, readiness level, evidence, gaps, risks, and withdrawal
conditions. It is not runtime enablement.

Coverage matrices record artifact, test, and documentation coverage for
profiles, reference corpus, OpenCode, OpenClaw, Hermes, manifest extraction,
risk classification, digestion candidates, internal candidate emission, and
dominion candidate emission. Coverage is not certification.

Gap register records blocking gaps, non-blocking gaps, future-track items,
recommended v0.33 items, and later work. A gap register is not failure by
itself unless blocking gaps exist.

Risk register records known risks, high-risk surfaces, prohibited runtime
surfaces, mitigations, unresolved risks, and evidence. Risk register entries
are not proof of exploitability, and mitigations are not permissions.

Release manifest records included modules, docs, tests, optional integration
tests, release flags, test commands, gaps, risks, and next handoff. Optional
integration tests are disabled by default.

Consolidation audit trail records reviewed artifacts, tests, docs, boundary
checks, negative runtime checks, and explicit no-runtime confirmations. It is
metadata, not runtime audit execution.

v0.33 handoff packet records reusable profile, reference corpus, manifest,
risk, digestion, internal candidate, and dominion candidate references. The
handoff packet is not implementation. It may recommend future read-only
runtime work, but does not enable it.

## v0.33 Handoff

The handoff target is v0.33 Internal General Agent Runtime MVP. Handoff focus
items include profile runtime, prompt assembly, session runtime, safe read-only
tool registry, safe workspace inspection, agent step runner, runtime OCEL trace
emitter, and CLI surface. These are future design-stage handoff topics only.
They are not enabled by v0.32.9.

## Hard Prohibitions

v0.32.9 does not implement runtime execution, external harness execution,
reference code execution, read-only local tool execution, runtime adapter
loading, runtime OCEL emission, runtime trace persistence, UI runtime,
external control, authority grant, D4-D9 grant, provider invocation, gateway
connection, network access, credential access, command execution, browser
automation, RPA control, packet send, registry mutation, memory mutation,
plugin loading, tool registration or invocation, mission installation or
execution, source-ref fetch, URL fetch, dependency install, runtime import, or
secret file reads.

## Gate Inheritance

v0.32.9 preserves v0.30.9, v0.31.9, and every v0.32.0-v0.32.8 gate. It does
not bypass any previous no-runtime, no-permission, no-activation, no-control,
or no-authority boundary.

## Acceptance Criteria

- Consolidation status and readiness taxonomies exist.
- Release flags, pipeline snapshot, coverage matrices, gap/risk registers,
  release manifest, audit trail, v0.33 handoff packet, and consolidation report
  exist.
- v0.32.0-v0.32.8 are included in snapshot and manifest.
- Runtime/control/authority/certification flags remain false.
- D4-D9 remain future-track.
- v0.33 handoff is clear and design-stage only.

## Withdrawal Conditions

Withdraw the v0.32.9 consolidation readiness claim if implementation code
executes harnesses or reference code, imports reference code as runtime,
installs dependencies, live-scans, fetches source refs, reads secrets, loads
plugins, registers or invokes tools, installs or executes missions, invokes
providers, connects gateways, performs network/credential/command/browser/RPA
runtime actions, sends packets, mutates registries or memory, emits OCEL
runtime events, persists runtime traces, implements UI runtime, enables
external control, grants authority, grants D4-D9, sets production/live adapter
certification true, or sets any prohibited readiness flag true.
