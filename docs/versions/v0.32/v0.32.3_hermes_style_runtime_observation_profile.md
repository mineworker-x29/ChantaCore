# v0.32.3 Hermes-style Runtime Observation Profile

Parent track: External Harness Observation & Digestion Pipeline.

v0.32.3 extends the v0.32.0 External Harness Profile Contract and Reference Corpus Static Observation Contract with a Hermes-style runtime/profile/mission/skill observation profile. It also follows the v0.31.9 Internal Triad Skill Foundation boundary: digestion hints and Dominion hints are signals only, not InternalSkillCandidate or DominionTarget objects. v0.32.3 stands beside v0.32.1 OpenCode-style and v0.32.2 OpenClaw-style profiles without depending on their internals.

## Definition

Hermes-style means a persistent assistant or runtime harness style surface. The profile describes profile, memory, mission, skill, tool, delegation, provider routing, runtime/container/isolation, authorization, approval, audit, result envelope, and OCEL trace relevance surfaces.

This is a style profile, not a product clone. It does not assume a Hermes product API. Hermes-style profile is not Hermes execution. Hermes runtime is never started.

## Boundary Rules

- Profile observation is not profile activation.
- Memory surface observation is not memory access, memory write, or memory mutation.
- Mission surface observation is not mission installation or mission execution.
- Skill surface observation is not skill registration or skill execution.
- Tool surface observation is not tool registration or tool invocation.
- Delegation boundary observation is not delegation execution.
- Provider routing boundary is not provider invocation.
- Runtime isolation boundary is not runtime start or container start.
- Approval/audit boundary requirement is not approval execution, approval granted, or audit event emission.
- Config manifest observation is not dependency installation or script execution.
- Risk signal is advisory and is not proof of exploitability.
- Digestion hint is not InternalSkillCandidate.
- Dominion hint is not DominionTarget.
- ready_for_v0324_manifest_extraction and ready_for_v0325_risk_classification are design-stage handoff flags only, not execution readiness.
- All execution and runtime readiness flags remain false.

## Reference Corpus Support

references/Hermes, if present, is read-only static corpus only. Reference code is never executed. Reference code is never imported as runtime. Dependencies are never installed. v0.32.3 consumes v0.32.0 ReferenceFileInventory, ReferenceCorpusSnapshot, and ExternalHarnessProfile artifacts as contract input. A local path such as references/Hermes may be represented only as local_path_ref.

Optional local reference inspection, if added in a later test skeleton, must be disabled by default with CHANTA_ENABLE_REFERENCE_CORPUS_TESTS=1 and must only inspect read-only inventory metadata permitted by v0.32.0 policy. It must never run files, import files, install dependencies, start Hermes runtime, run package scripts or tests, or read .env, secret, key, token, or credential files.

## Contract Artifacts

HermesReferenceSourceRef records source_ref_id, reference source IDs, inventory IDs, entry IDs, local_path_ref, label, evidence refs, and metadata. It is not source fetch and not execution.

Surface and boundary observations:

- HermesProfileSurfaceObservation: profile paths and activation/private-data risks; not profile activation.
- HermesMemorySurfaceObservation: memory schema/store/writer paths and memory/private-data risks; not memory access or mutation.
- HermesMissionSurfaceObservation: mission manifests, runtime paths, scheduler paths, installation/execution risks; not mission install or execution.
- HermesSkillSurfaceObservation: skill manifests, registry paths, runtime paths, registration/execution risks; not skill registration or execution.
- HermesToolSurfaceObservation: tool manifests, registry paths, runtime paths, command risk; not tool registration, invocation, or command execution.
- HermesDelegationBoundaryObservation: delegation manifests, runtime paths, agent routes; not delegation execution or external agent runtime.
- HermesProviderRoutingBoundaryObservation: provider config/router paths, provider/network/credential risks; not provider routing, provider invocation, network access, or credential access.
- HermesRuntimeIsolationBoundaryObservation: runtime, container, and isolation config paths; not Hermes runtime start, container runtime, or command execution.
- HermesApprovalAuditBoundaryRequirement: approval, authorization, and audit paths; approval_granted remains false and no audit event is emitted.
- HermesConfigManifestObservation: package, config, profile, memory, mission, skill, and tool manifest candidates; not dependency install or script execution.

HermesStaticObservationInput is a contract input, not an execution request. HermesObservationFinding is not permission. HermesRiskSignal does not grant authority. HermesDigestionHint and HermesDominionHint are routing hints for later design stages only.

HermesStyleObservationProfile, HermesObservationOutput, HermesObservationRunPreview, HermesNoExecutionGuarantee, and V0323ReadinessReport preserve no-execution guarantees. ready_for_execution, ready_for_hermes_execution, ready_for_hermes_runtime, ready_for_reference_code_execution, ready_for_profile_activation, ready_for_memory_access, ready_for_memory_write, ready_for_mission_execution, ready_for_skill_execution, ready_for_tool_invocation, ready_for_delegation_execution, ready_for_provider_invocation, ready_for_network_access, and ready_for_credential_access remain false.

## Flow

ExternalHarnessProfile / ReferenceCorpusSnapshot / ReferenceFileInventory -> HermesStaticObservationInput -> Hermes surface observations -> HermesObservationFinding / HermesRiskSignal -> HermesDigestionHint / HermesDominionHint -> HermesObservationOutput -> V0323ReadinessReport -> v0.32.4 External Skill Manifest Extraction.

This flow is design-stage only. Handoff to v0.32.4 Manifest Extraction and v0.32.5 Risk Classification does not authorize execution.

## v0.32.0 Gate Inheritance

v0.32.3 inherits v0.32.0 gates: external harness profiles remain non-executable, reference corpus material remains read-only static observation material, and reference file inventory is not runtime file access.

## Acceptance Criteria

- Hermes-style surface, focus, capability, risk, status, and evidence taxonomies exist.
- Hermes reference refs, surface observations, boundary observations, static input, findings, risks, hints, profile, output, run preview, no-execution guarantee, and readiness report exist.
- Helpers are pure and derive observations only from provided metadata.
- No Hermes execution, runtime start, reference execution/import, install, live scan, provider invocation, command, network, credential access, memory access/write, mission/skill/tool execution, delegation, container runtime, approval execution, registry mutation, memory mutation, OCEL emission, or UI runtime is introduced.
- v0.32.0 readiness gates remain false for runtime behavior.

## Withdrawal Conditions

Withdraw v0.32.3 readiness if any implementation path executes Hermes, starts runtime, imports reference code, installs dependencies, reads live memory, executes missions/skills/tools, delegates to agents, invokes providers, starts containers, grants approvals, accesses credentials or network, executes commands, mutates registry or memory, emits OCEL events, or sets any runtime readiness flag true.

## Handoff

v0.32.3 is ready for v0.32.4 Manifest Extraction and v0.32.5 Risk Classification only when the tests pass and the runtime boundary search finds no executable Hermes behavior. It is not execution readiness.
