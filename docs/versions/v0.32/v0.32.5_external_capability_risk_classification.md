# v0.32.5 External Capability Risk Classification

Parent track: External Harness Observation & Digestion Pipeline.

v0.32.5 builds on v0.32.4 External Skill Manifest Extraction and the v0.32.1-v0.32.3 OpenCode, OpenClaw, and Hermes observation profiles. It also inherits the v0.32.0 external harness profile and reference corpus static observation gates.

## Definition

External capability risk classification is a governance and design-stage contract layer. It consumes static manifest candidate metadata, extraction reports, prior observation outputs, and reference inventory metadata, then emits advisory classification artifacts.

It distinguishes descriptive capability surfaces from candidates that require review, future gates, or later dominion routing. It does not grant permission. It does not certify runtime behavior. It does not create active internal or external control objects.

## Boundary Statements

Risk classification is not permission.

Risk classification is not runtime certification.

Risk severity is advisory and is not proof of exploitability. High, critical, blocked, or future-track classifications identify conservative handling requirements only.

`digestible_pattern` is not a DigestionCandidate. It is only a possible v0.32.6 design-stage route.

`dominion_required` is not a DominionTarget. It is only a possible v0.32.8 design-stage route.

Boundary requirement is not runtime enforcement by itself.

Review requirement is not approval. `human_review` is still not approval granted.

Future gate is not readiness. A future gate records missing later artifacts or reviews.

No-op is valid and not failure.

`ready_for_v0326_digestion_candidate_generation` and `ready_for_v0328_dominion_candidate_emitter` mean next design-stage handoff only. All execution, permission, certification, candidate creation, dominion creation, plugin, tool, mission, gateway, provider, network, credential, and command readiness flags remain false.

## Contract Artifacts

`ExternalCapabilityRiskSourceRef` references static source artifacts only. Source refs do not fetch, scan, import, or execute anything.

`ExternalCapabilityRiskFactor` records an advisory risk surface, risk class, severity, boundary kinds, review requirements, and evidence. A risk factor is not proof of exploitability and is not permission.

`ExternalCapabilityBoundaryRequirement` records required governance boundaries. It blocks execution, activation, and runtime registration by default, but it is not runtime enforcement.

`ExternalCapabilityReviewRequirement` records review needs. `approval_granted` remains false.

`ExternalCapabilityRiskClassification` groups source refs, risk factors, boundary requirements, review requirements, evidence, gaps, and route signals. It is not permission, not a DigestionCandidate, and not a DominionTarget.

`ExternalCapabilityRiskMap` groups classification ids by class and severity. It is not a permission map.

`ExternalCapabilityBoundaryMap` groups boundaries, reviews, prohibited surfaces, future gates, and blocked items. It is not runtime enforcement.

`ExternalCapabilityNoOpRecommendation` records an intentional no-op route and safe alternatives. It executes nothing.

`ExternalCapabilityFutureGateItem` records missing later artifacts or reviews. It is not readiness.

`ExternalCapabilityRiskClassificationInput`, `ExternalCapabilityRiskClassificationFinding`, and `ExternalCapabilityRiskClassificationReport` define the input, finding, and report layer. Findings are not certification or permission. Reports are not runtime classification.

`ExternalCapabilityRiskClassificationRunPreview` and `ExternalCapabilityRiskNoRuntimeGuarantee` make the no-runtime boundary explicit.

`V0325ReadinessReport` is a design-stage readiness report only. It is not runtime enablement.

## Safe Flow

`ExternalManifestCandidateSet` / `ExternalManifestExtractionReport` / OpenCode, OpenClaw, Hermes outputs / `ReferenceFileInventory`
-> `ExternalCapabilityRiskClassificationInput`
-> `ExternalCapabilityRiskFactor` / `ExternalCapabilityBoundaryRequirement` / `ExternalCapabilityReviewRequirement`
-> `ExternalCapabilityRiskClassification`
-> `ExternalCapabilityRiskMap` / `ExternalCapabilityBoundaryMap` / `ExternalCapabilityNoOpRecommendation` / `ExternalCapabilityFutureGateItem`
-> `ExternalCapabilityRiskClassificationReport`
-> `V0325ReadinessReport`
-> v0.32.6 Digestion Candidate Generator or v0.32.8 External Dominion Candidate Emitter.

The arrows are contract-artifact handoffs. They are not runtime execution.

## Reference Corpus Support

Reference material under `references/` may be represented through v0.32.0 `ReferenceFileInventory` and `ReferenceCorpusSnapshot` metadata. v0.32.5 prefers those already-built static artifacts over direct file inspection.

Optional local integration tests, if added later, must skip unless `CHANTA_ENABLE_REFERENCE_CORPUS_TESTS=1`, use `CHANTA_REFERENCES_ROOT` safely, skip if references are absent, and perform read-only metadata checks only.

Secret-like files are not read. Reference paths are not execution or import permission.

## Hard Prohibitions

v0.32.5 does not execute external harnesses, OpenCode, OpenClaw, Hermes, or reference code. It does not import reference code, install dependencies, run package scripts, run tests inside references, load plugins, register or invoke tools, install or execute missions, activate profiles, access or write live memory, invoke providers, connect gateways, access channels, send messages, call webhooks, perform network calls, read credentials or secret files, execute commands, start browser automation, control RPA runtimes, send packets, mutate registries, mutate memory, emit OCEL events, write runtime logs, write database records, implement UI runtime, retry, or rollback.

v0.32.5 does not create DigestionCandidate, InternalSkillCandidate, DominionTarget, or DominionDecision.

## Gate Inheritance

v0.32.5 inherits v0.32.0-v0.32.4 gates:
- reference corpus operations remain read-only static metadata usage.
- observation profiles remain non-runtime.
- manifest candidates remain inactive.
- candidate sets remain non-registries.
- all execution/readiness flags stay false unless explicitly defined as next design-stage handoff and still non-execution.

## Acceptance Criteria

- Risk class, severity, route, boundary, review, evidence, and source taxonomies exist.
- Risk source refs, risk factors, boundary requirements, review requirements, classifications, maps, no-op recommendations, future gates, inputs, findings, reports, run previews, guarantees, and readiness reports validate non-runtime semantics.
- Helper functions are pure and create only v0.32.5 contract artifacts.
- `digestible_pattern` remains a route signal only.
- `dominion_required` remains a route signal only.
- Runtime, permission, certification, candidate creation, dominion creation, plugin, tool, mission, provider, gateway, network, credential, and command readiness flags remain false.

## Withdrawal Conditions

Withdraw v0.32.5 readiness if any implementation executes harnesses, executes or imports reference code, installs dependencies, performs live scanning, fetches source refs or URLs, reads secret files, loads plugins, registers or invokes tools, installs or executes missions, invokes providers, connects gateways, uses network or credentials, executes commands, mutates registry or memory, emits OCEL events, creates active digestion/internal/dominion artifacts, or sets runtime readiness true.

## Handoff

v0.32.5 may hand off only to v0.32.6 Digestion Candidate Generator and v0.32.8 External Dominion Candidate Emitter as design-stage follow-up work. It does not enable execution.
