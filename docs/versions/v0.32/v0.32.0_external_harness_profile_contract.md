# v0.32.0 External Harness Profile Contract + Reference Corpus Static Observation Contract

Parent track: External Harness Observation & Digestion Pipeline.

v0.32.0 follows v0.31.9 Internal Triad Skill Foundation v1 Consolidation. The v0.31 line established internal triad contracts for observation, digestion, dominion, OCEL trace planning, workbench display contracts, and foundation consolidation. v0.32.0 starts the next track by defining how external agent harnesses and local reference corpora can be represented as contract artifacts.

## Release Boundary

v0.32.0 defines external harness profile and reference corpus contracts only. It does not execute external harnesses, execute reference code, import reference code as runtime modules, install dependencies, fetch `source_ref`, fetch URLs, live-scan external systems, invoke providers, access credentials, read secret file contents, execute commands, send packets, mutate registries, write memory, emit OCEL events, persist runtime trace data, or implement UI runtime.

ExternalHarnessProfile is not harness execution. A harness surface is not permission. An observation boundary is not runtime enforcement by itself. A local path ref is not execution permission. The `references/` tree is treated only as a read-only static corpus.

## Status, Observation Mode, Risk, Evidence

Profile status values are `unknown`, `draft`, `profile_ready`, `profile_ready_with_gaps`, `blocked`, `deferred`, `future_track`, and `no_op`. `profile_ready` is not runtime readiness.

Observation modes are `contract_only`, `manual_profile`, `reference_static_observation`, `manifest_only_observation`, `documentation_only_observation`, `future_live_scan`, `future_runtime`, and `unknown`. v0.32.0 may actively use only contract/manual/reference-static/manifest/documentation modes. `future_live_scan` and `future_runtime` are vocabulary for later gates, not active modes.

Risk posture is advisory and does not prove exploitability. Evidence quality is not runtime trust. `sufficient_for_profile`, `sufficient_for_static_observation`, and `sufficient_for_next_stage_review` are not approvals for execution.

## Reference Corpus Static Observation

The repository may contain downloaded external harness references under `references/`. v0.32.0 treats this content as local read-only reference material only. Reference code is never executed. Reference code is never imported as runtime. Reference dependencies are never installed. Static observation is not live scan. Manifest candidate extraction is not plugin loading.

Reference source kinds are:

- `local_reference_directory`
- `local_reference_file`
- `local_reference_manifest`
- `local_reference_documentation`
- `local_reference_config`
- `sanitized_manifest`
- `manual_reference`
- `unknown`

`ReferenceHarnessSource.local_path_ref` is a path reference string only. It does not grant permission to load, import, execute, install, or scan.

## Read-Only Corpus Policy

`ReferenceHarnessCorpusPolicy` allows only contract-level static operations:

- file tree inventory modeling
- text file read policy modeling
- manifest parse policy modeling
- optional hash field modeling

The policy prohibits execution, installation, runtime import, network access, credentials, command execution, provider invocation, browser automation, RPA control, gateway control, packet send, and secret file read. Secret-like patterns such as `.env`, `*secret*`, `*key*`, `*token*`, and `*credential*` remain prohibited.

## Reference File Inventory Policy

`ReferenceFileInventoryPolicy` models read-only inventory metadata. `ReferenceFileInventoryEntry` represents a file path, size, extension, optional hash, optional detected kind, optional text preview, or skipped reason. Secret/prohibited files should be represented with `skipped_reason`, not read as content. Inventory is not runtime file access.

## Reference Corpus Snapshot

`ReferenceCorpusSnapshot` summarizes source ids, inventory ids, profile seed ids, gaps, blocked reasons, and design-stage handoff readiness. The snapshot is not persistence and is not runtime state.

## No-Execution Guarantees

`ExternalHarnessProfileNoExecutionGuarantee` and `ReferenceCorpusNoExecutionGuarantee` record explicit no-execution/no-import/no-install/no-network/no-credential/no-command/no-provider/no-browser/no-RPA/no-gateway/no-packet/no-registry/no-memory guarantees. These are contract metadata, not runtime enforcement by themselves.

## Readiness Report

`V0320ReadinessReport` can mark readiness for v0.32.1, v0.32.2, v0.32.3, and v0.32.4 design work only. It cannot mark execution, external harness execution, reference code execution, or live scan readiness.

## Flow

V032HandoffPacket / manual config / references root -> ExternalHarnessProfile / ReferenceHarnessCorpus -> ReferenceFileInventory -> ReferenceCorpusSnapshot -> V0320ReadinessReport -> v0.32.1 OpenCode-style Harness Observation Profile.

## Optional Local Integration Test Policy

Optional local reference corpus tests may be added under `tests/integration/`, but they must skip unless `CHANTA_ENABLE_REFERENCE_CORPUS_TESTS=1`. They must use `CHANTA_REFERENCES_ROOT` or skip safely, must skip when the root is absent, and must remain read-only. They must not execute files, import files, install dependencies, read `.env`/secret/key/token/credential files, or be required for the default test pass.

## Hard Prohibitions

v0.32.0 does not execute external harnesses, execute `references/` code, import `references/` code, install dependencies, run tests inside references, fetch `source_ref`, fetch URLs, live-scan external systems, implement OpenCode/OpenClaw/Hermes runtime, invoke providers, perform network calls, access credentials or secrets, read secret files as content, use subprocess or shell execution, implement browser automation, implement RPA control, implement gateway send, send packets, mutate active registries, write memory, emit OCEL events, persist OCEL objects or relations, write runtime logs, write database records, implement UI runtime, execute rollback, retry, or mark a target live-runtime-ready.

## Gate Inheritance

v0.32.0 inherits v0.30 and v0.31 gates. v0.31.9 allowed only design-stage handoff to the External Harness Observation & Digestion Pipeline. v0.32.0 preserves that boundary and does not convert it into runtime capability.

## Acceptance Criteria

- External harness profile taxonomy exists.
- Harness surface taxonomy exists.
- Observation mode, risk posture, and evidence quality taxonomies exist.
- Runtime readiness flags remain false.
- ExternalHarnessSurfaceDescriptor, ExternalHarnessObservationBoundary, ExternalHarnessProfile, ExternalHarnessProfileSet, and no-execution guarantees exist.
- ReferenceHarnessSourceKind, ReferenceHarnessCorpusPolicy, ReferenceHarnessSource, ReferenceHarnessCorpus, inventory models, ReferenceCorpusSnapshot, ReferenceCorpusNoExecutionGuarantee, and V0320ReadinessReport exist.
- `references/` is represented as read-only static corpus only.
- Tests verify no execution, import, install, network, credential, command, provider, browser, RPA, gateway, packet, registry, memory, OCEL, or UI runtime readiness.

## Withdrawal Conditions

Withdraw v0.32.0 readiness if any implementation executes harnesses, executes reference code, imports reference code as runtime, installs dependencies, fetches source refs or URLs, live-scans external systems, reads secrets, invokes network/provider/browser/RPA/gateway/packet surfaces, mutates registry or memory, emits OCEL events, implements UI runtime, or sets runtime readiness flags true.

## Handoff

v0.32.0 hands off to v0.32.1 OpenCode-style Harness Observation Profile, v0.32.2 OpenClaw-style Harness Observation Profile, and v0.32.3 Hermes-style Harness Observation Profile as design-stage work only. v0.32.0 does not enable OpenCode, OpenClaw, or Hermes runtime.
