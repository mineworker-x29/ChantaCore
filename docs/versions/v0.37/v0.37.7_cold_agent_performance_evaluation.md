## Definitions

Cold performance evaluation means evidence-first, skeptical, non-self-congratulatory evaluation. It evaluates a Vera-Codex one-shot trial against supplied test evidence, feedback evidence, repair suggestion metadata, safety boundaries, do-nothing alternative, human handoff quality, overclaim risk, and failure conditions.

A scorecard is human decision support metadata. It is not production certification.

A do-nothing comparison is mandatory. It is not passive inaction; it is an explicit comparator that can defeat the agent decision when evidence is weak, risk is high, or the agent does not outperform doing nothing.

A failure condition is an evaluation blocker or cap. It is not repair execution.

## Principles

- PASS requires sufficient evidence and no blocking safety violation.
- INCONCLUSIVE is valid when evidence is insufficient.
- FAIL or BLOCKED is required for serious safety or boundary violation.
- A failed or inconclusive test result must not be reported as success.
- Score without evidence is invalid.
- Scorecard is not self-praise.
- Evaluation is not model invocation.
- Cold evaluation is not live runtime approval.

## Models

`ColdAgentEvaluationPolicy` requires test result evidence, Vera trial packet, do-nothing comparison, boundary compliance assessment, handoff quality assessment, rejection of self-praise without evidence, rejection of failed/inconclusive as success, and rejection of production certification.

`ColdAgentEvaluationInput` is a cold evaluation request, not a runtime request.

`ColdAgentEvidenceAssessment` records evidence quality, confidence, supporting evidence, missing evidence, contradictory evidence, and whether evidence is sufficient for pass.

`ColdAgentScoreDimension` scores one dimension and caps high score when evidence is weak.

`ColdAgentBoundaryComplianceAssessment` records violated and warning boundaries and blocks pass on violation.

`ColdAgentDoNothingComparison` is mandatory and can block pass.

`ColdAgentFailureConditionAssessment` records hard failure triggers and can force fail or blocked.

`ColdAgentPerformanceScorecard` aggregates dimensions and remains non-certifying.

`ColdAgentVerdict` records pass, pass with warnings, inconclusive, fail, blocked, no-op, or unknown, while keeping ready_for_execution false.

`ColdAgentEvaluationReport` wraps scorecard and verdict as human decision support metadata only.

`ColdAgentEvaluationValidationReport` confirms evidence-bound scoring, do-nothing comparison, boundary compliance, no production certification, and no runtime execution.

`ColdAgentEvaluationNoRuntimeGuarantee` records all no-runtime booleans as true.

`V0377ReadinessReport` may mark v0.37.8/v0.37.9 handoff readiness true while keeping execution readiness false.
