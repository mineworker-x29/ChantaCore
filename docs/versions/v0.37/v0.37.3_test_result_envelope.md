# v0.37.3 Test Result Envelope & Output Classifier

Parent track: Controlled Sandbox Test Runner & Vera-Codex Agent Evaluation.

v0.37.3 builds on:

- v0.37.2 Sandbox Test Execution Engine: supplies `SandboxTestExecutionResult`, `ControlledTestSubprocessResult`, and bounded output capture metadata.

- v0.36.9 Human-approved Patch Apply Sandbox v1: remains patch-apply context only; result classification does not apply patches.

## Release Definition

v0.37.3 creates test result envelope and output classification metadata only.

A test result envelope is a bounded evidence structure assembled from supplied v0.37.2 execution metadata. It records process classification, output signals, evidence snippets, failure class, outcome class, future feedback evidence, and future evaluation evidence.

An output classifier is a pure text classifier over supplied stdout/stderr. It detects common signals such as pytest pass/fail summaries, unittest summaries, tracebacks, assertion errors, import errors, syntax errors, missing dependency messages, timeout messages, command-blocked messages, no-tests-collected messages, and secret-like output.

## Boundary Distinctions

- Test result envelope is not test execution.
- Output classifier is not a test runner.
- Process classification is not full semantic diagnosis.
- Output classification is not failure diagnosis.
- Failure classification is not repair permission.
- Missing dependency classification is not dependency install permission.
- Timeout classification is not retry permission.
- Process exit zero is not production certification.
- Passed test process is not overall task success.
- Failed test process is not an automatic repair trigger.
- Future Vera-Codex evidence is not Vera-Codex trial execution.
- Future cold evaluation evidence is not cold performance evaluation execution.

Confidence must remain evidence-based. If the supplied process metadata or output text is insufficient, `inconclusive` is valid and preferred over overclaiming.

## Models

`SandboxTestResultFlagSet` records that result envelope, process classification, output classifier, failure classification, and evidence snippet extraction are available. It may mark future feedback and future evaluation evidence readiness true. It must keep all execution, subprocess, shell, install, network, repair, provider, external-agent, Dominion, persistent trace, UI, authority, and production flags false.

`SandboxTestResultSourceRef` points at supplied metadata. It does not fetch, read, write, execute, or shell out.

`SandboxTestResultEnvelopePolicy` allows classification and evidence extraction only. It requires bounded output, redaction, and confidence levels. It blocks test execution, subprocess execution, shell, dependency install, network access, automatic repair, Vera-Codex trial execution, cold performance evaluation, external agent execution, and Dominion runtime.

`SandboxTestResultEnvelopeInput` is a classifier request, not an execution request. It contains supplied stdout/stderr, return code, timeout status, source refs, and prohibited runtime actions.

`SandboxTestProcessClassification` records process-level facts such as exit zero, nonzero, timeout, blocked before execution, not executed, or unknown. It is not production certification.

`SandboxTestOutputSignal` records bounded output signals. Signals do not trigger repair.

`SandboxTestEvidenceSnippet` records bounded and redacted excerpts. Secret-like content forces redaction metadata.

`SandboxTestFailureClassification` records likely failure class with confidence. Repair, retry, and dependency install are always false.

`SandboxTestOutcomeClassification` records passed, failed, or inconclusive outcome metadata. Production certification is always false.

`SandboxTestResultEnvelope` combines the supplied input, process classification, output signals, evidence snippets, failure classification, and outcome classification. It may be eligible for future feedback, future Vera-Codex trial evidence, and future cold evaluation evidence. It does not perform execution and cannot certify production readiness.

Validation reports, classifier reports, run previews, no-execution guarantees, and `V0373ReadinessReport` keep the same boundary: metadata and design-stage handoff only.

- Bounded stdout, stderr, and evidence snippet limits.
- Redaction required.
- Confidence level required.
- Process classification enabled.
- Output classification enabled.
- Evidence snippet extraction enabled.
- Future feedback and future Vera-Codex evidence metadata enabled.
- Test execution disabled.
- Subprocess disabled.
- Shell disabled.
- Dependency install disabled.
- Network disabled.
- Automatic repair disabled.
- Vera-Codex trial execution disabled.
- Cold performance evaluation disabled.
- External agent execution disabled.
- Dominion runtime disabled.

The default no-execution guarantee sets all `no_*` fields true.

## Acceptance Criteria

- Result envelope taxonomies and dataclasses exist.
- Output signal detection works on supplied text only.
- Evidence snippets are bounded and redacted.
- Missing dependency class does not allow install.
- Timeout class does not allow retry.
- Failure class does not allow repair.
- Process exit zero does not certify production.
- v0.37.2 execution metadata can be converted into a v0.37.3 envelope.
- No helper reads files, writes files, invokes subprocess, shells out, installs dependencies, accesses network, invokes providers, invokes external agents, runs Vera-Codex evaluation, runs cold evaluation, or implements Dominion runtime.

## Withdrawal Conditions

Withdraw v0.37.3 readiness if any implementation path:

- Executes tests or subprocesses.
- Introduces shell, command execution, package manager execution, dependency install, or network access.
- Reads credentials, secrets, token files, or environment credentials.
- Writes live workspace files, patches, traces, logs, databases, JSONL, or OCEL files.
- Treats exit zero as production certification.
- Treats failure classification as repair permission.
- Treats missing dependency as install permission.
- Treats timeout as retry permission.
- Runs Vera-Codex trial or cold performance evaluation.
- Invokes external coding agents or Dominion runtime.
- Sets unsafe readiness flags true.
