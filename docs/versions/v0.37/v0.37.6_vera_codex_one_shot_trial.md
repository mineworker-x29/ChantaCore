# v0.37.6 Vera-Codex One-shot Agent Operation Trial

Parent track: Controlled Sandbox Test Runner & Vera-Codex Agent Evaluation.

v0.37.6 creates Vera-Codex one-shot trial metadata only. It does not call model providers, invoke Codex CLI, invoke Claude Code, invoke external agents, execute tools, execute tests, run subprocess, execute shell/subprocess/commands, install dependencies, access network, generate repair patches/diffs/hunks, edit files, apply patches, perform automatic repair, implement retry loops, implement multi-cycle loops, run cold performance evaluation, or implement Dominion runtime.

## Relationship

- v0.37.5 Repair Suggestion Metadata supplies repair suggestion envelopes and reports. v0.37.6 reviews them as future-gated metadata only.
- v0.37.4 Test Feedback Report supplies feedback and diagnosis metadata. v0.37.6 uses it as evidence, not as repair permission.
- v0.37.3 Test Result Envelope supplies result and outcome metadata. v0.37.6 must not report failed or inconclusive evidence as success.
- v0.37.2 Sandbox Test Execution Engine may supply execution metadata. v0.37.6 does not execute tests.
- v0.36.9 Human-approved Patch Apply Sandbox v1 remains patch-apply boundary context only. v0.37.6 does not apply patches.

## Definition

A Vera-Codex one-shot trial is a bounded metadata evaluation of supplied ChantaCore artifacts:

1. one explicit evaluation task,
2. one supplied evidence bundle,
3. one deterministic operator/evaluator decision,
4. one bounded decision trace,
5. one stop reason,
6. one mandatory human handoff memo.

The trial count is exactly 1. The max cycle count is exactly 1. Automatic retry and automatic repair are disabled.

## Vera Judgment

In this stage, Vera's judgment means a bounded ChantaCore operation adequacy assessment. It asks whether the v0.37 evidence chain is coherent enough to hand off to v0.37.7/v0.37.8, or whether human review, do-nothing, or block is safer.

The allowed outcomes are:

- evidence supports future cold evaluation input,
- evidence supports human review,
- evidence supports do-nothing,
- evidence supports a future repair proposal gate,
- evidence is insufficient,
- safety boundary blocks,
- no action.

This is not a model provider invocation. It is not a live Vera agent session. It is not a Codex CLI invocation. It is not autonomous runtime.

## Distinctions

- Vera-Codex trial is not model provider invocation.
- One-shot operator decision is not autonomous agent runtime.
- Decision trace is bounded summary metadata, not hidden chain-of-thought.
- Handoff memo is not action execution.
- Repair suggestion review is not repair execution.
- Trial result is not cold performance scorecard.
- Trial result is not production certification.
- Do-nothing alternative must be represented.
- Human handoff is mandatory.

## Models

`VeraCodexTrialPolicy` fixes max trial count and max cycle count to exactly 1 and blocks provider/tool/test/subprocess/shell/install/network/repair/retry/multi-cycle/external-agent/Dominion.

`VeraCodexOperatorProfile` records a bounded evidence evaluator. It must not be autonomous, external, Dominion, provider-enabled, or tool-enabled.

`VeraCodexTrialInput` is a one-shot evaluation request, not an autonomous request.

`VeraCodexEvidenceItem` and `VeraCodexEvidenceBundle` record supplied evidence. Required evidence must be present for trial sufficiency.

`VeraCodexTrialTask` requires do-nothing representation and human handoff.

`VeraCodexOperationConstraintSet` fixes one trial, one cycle, no retry, no repair, no provider, no tools, no external agents, no Dominion, no tests, no patch generation, and no file writes.

`VeraCodexDecisionTraceStep` and `VeraCodexDecisionTrace` are bounded summaries. They do not contain chain-of-thought and do not execute actions.

`VeraCodexDoNothingAssessment` is always represented.

`VeraCodexRepairSuggestionAssessment` may support future repair proposal input, but `supports_repair_now` remains false.

`VeraCodexTaskHandlingAssessment` records operation adequacy metadata. It is not production certification.

`VeraCodexTrialDecision` is metadata only and never allows action, repair, retry, model invocation, tool execution, or external agent use.

`VeraCodexStopReason` is mandatory and blocks continuation, retry, and repair.

`VeraCodexHandoffMemo` is mandatory and executes no action.

`VeraCodexTrialSafetyReport` is static metadata validation and keeps `ready_for_execution` false.

`VeraCodexOneShotTrialPacket` ties the profile, input, task, constraints, evidence bundle, trace, assessments, decision, safety report, stop reason, and handoff memo into one metadata packet.

`VeraCodexTrialValidationReport` confirms one-shot, evidence bundle, no runtime execution, no chain-of-thought, human handoff, and stop reason.

`VeraCodexNoAutonomyGuarantee` records all no-autonomy booleans as true.

`V0376ReadinessReport` may mark v0.37.7/v0.37.8 handoff readiness true while keeping execution readiness false.
