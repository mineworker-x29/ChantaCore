# v0.23.5 Dominion Static Safety Check

Korean name: 지배 정적 안전성 검사

Track: Internal Dominion Foundation

## Version Definition

v0.23.5 introduces deterministic Dominion static safety checks over the v0.23.4 DominionControlPlan. It reads the control plan and related read models, evaluates rule results and findings, and produces a DominionStaticSafetyReport.

This is Public ChantaCore work. v0.23.x remains Internal Dominion Foundation. Self-Execution Safety remains reclassified to v0.24.x Local Runtime Provider. External vendors remain future provider adapters. GrowthKernel remains future_consumer_not_dependency.

## Objective

The objective is to decide whether a DominionControlPlan is eligible to proceed to the future runtime preflight stage. Static safety pass only permits moving to runtime preflight.

## Core Boundaries

- Static safety check is not preflight.
- Static safety check is not provider API call.
- Static safety check is not external runtime touch.
- Static safety check is not authorization.
- Static safety check is not dispatch.
- Static safety pass only permits moving to runtime preflight.

## Status Rules

Static safety is `passed` when required bindings and policies exist and no blocked or failed conditions appear. It is `warning` for acceptable future-stage uncertainty such as production requiring a future strong gate. It is `failed` for missing required bindings or policies. It is `blocked` for credential exposure, raw secret output, provider API call, runtime touch, dispatch attempt, already dispatched plan, or vendor hardcoding in the internal/core path.

`eligible_for_preflight` may be true only for `passed` or acceptable `warning`. `safe_to_dispatch` is always false in v0.23.5.

## OCEL Mapping

Object types include static safety request, rule, rule result, category result, finding, report, no-action candidate, and needs-more-input candidate. Event types include request, source load, rule load, category checks, report creation, warning, failed, and blocked events. Relation types include checks_dominion_static_safety, uses_control_plan, evaluates_dominion_static_rule, produces rule results/findings/reports, eligible_for_runtime_preflight, not_safe_to_dispatch, and the continued no-runtime-touch/no-provider-call/no-dispatch relations.

Effect types remain read_only_observation and state_candidate_created. OCEL remains the canonical substrate.

## CLI

- `python -m chantacore.cli dominion static-safety check --plan-id <id>`
- `python -m chantacore.cli dominion static-safety report --report-id <id>`
- `python -m chantacore.cli dominion static-safety findings --report-id <id>`
- `python -m chantacore.cli dominion static-safety rules`
- `python -m chantacore.cli dominion static-safety summary --plan-id <id>`

CLI output shows report_id, plan_id, static_safety_status, eligible_for_preflight, safe_to_dispatch=false, preflight_required=true, human_gate_required=true, authorization_required=true, disabled dispatch/runtime/provider/credential flags, and the next required step. It does not print credential values, raw secrets, or private full paths.

## Acceptance Criteria

- DominionStaticSafetyReport can be created from DominionControlPlan.
- Static rule categories run over lifecycle/provider/runtime/capability/environment/input/output/status/idempotency/cancel/migration.
- Static safety may mark eligible_for_preflight but never safe_to_dispatch.
- No preflight, gate, authorization, dispatch, provider API call, or external runtime touch occurs.
- No credential value is stored or output.
- No LLM judge is used.
- OCEL/PIG/OCPX visibility exists.
- v0.23.4, v0.23.3, v0.23.2, v0.23.1, v0.23.0, v0.22.x, v0.21.x, and v0.20.x tests remain valid.

## Withdrawal Conditions

Withdraw the v0.23.5 readiness claim if static safety performs runtime preflight, dispatch, provider API calls, runtime touch, credential materialization, authorization, human gate creation, LLM judging, or vendor-specific core behavior.

## Validity Horizon

Valid until control plan, inventory, capability policy, provider registry, or Dominion policy changes.

Next version: v0.23.6 Runtime Preflight / Reachability Check.
