# v0.23.1 Runtime / Agent / System Inventory

Version: v0.23.1
Version name: Runtime / Agent / System Inventory
Korean name: 런타임·에이전트·시스템 인벤토리
Track: Internal Dominion Foundation
Layer: internal_dominion

## Objective

v0.23.1 implements the declarative inventory stage for Internal Dominion. It creates a sanitized, vendor-neutral, OCEL-visible map of possible Dominion targets:

- external runtimes
- agents
- tools
- systems
- control surfaces
- provider refs
- credential boundaries
- environment boundaries

This inventory is declared, static, or operator-provided only.

## Relationship To v0.23.0

v0.23.0 introduced the OCEL-native Internal Dominion Contract. v0.23.1 keeps that contract and implements only `skill:dominion_runtime_inventory` as a read-only declarative inventory view.

`skill:dominion_contract_view` remains read-only. Remaining Dominion seed skills remain contract-only stubs and non-dispatching.

## Inventory Status Rules

Passed or declared status requires sanitized sources, no credential values, no provider API calls, no external runtime touch, provider-neutral core entries, and no dispatch enabled.

Warning status is used for unknown runtime type, unknown environment, production runtime declarations, missing credential boundary without secret exposure, missing status tracking, or missing outcome mapping.

Failed status is used for provider-specific logic in internal/core inventory, early dispatch enablement, early provider API call enablement, active v0.23 self-execution references, or GrowthKernel active dependency language.

Blocked status is used for credential value exposure, raw secret output, external runtime touch, provider API call, network/MCP/plugin/shell execution, or similar forbidden effects.
