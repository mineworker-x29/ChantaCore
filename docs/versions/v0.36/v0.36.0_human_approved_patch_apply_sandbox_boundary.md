Parent track: Human-approved Patch Apply Sandbox.

v0.36.0 is boundary foundation only. It starts the v0.36 track after v0.35.9 Controlled Patch Proposal Layer v1, but it does not perform dry-run apply simulation, create a sandbox workspace, perform sandbox patch apply, apply patches, write/edit files, call apply_patch, call git apply, execute tests, execute shell/subprocess/commands, execute/import OpenCode/Hermes/OpenClaw, invoke Claude Code/Codex CLI/external coding agents, implement Dominion runtime, or implement infinite agent loops.

v0.35.9 consolidated proposal artifacts, risk reports, human review metadata, trace packets, and CLI preview surfaces. v0.36.0 consumes that handoff as design metadata only and defines the boundary for future human-approved patch apply sandbox work.

Patch proposal artifacts are not patch application. Review metadata is not apply approval. Trace metadata is not persistent trace write. CLI preview is not shell execution.

## Definitions

Human-approved Patch Apply Sandbox means a future bounded apply environment where patch application must pass explicit human approval, scope gate, sandbox policy, no-secret/no-shell/no-network checks, rollback planning, and audit trace.

Patch apply sandbox is not live patch apply. Sandbox write is not live workspace write. Dry-run simulation is not filesystem mutation. Human approval contract is not model self-approval. Review metadata is not apply approval.

In v0.36, agentic operation means bounded function/task execution: one explicit task, one bounded operation cycle, human-approved, sandbox-only, no live workspace mutation, no autonomous retry, no self-directed multi-step loop, no external agent orchestration, and no Dominion runtime. Agentic function/task execution is not an independent autonomous agent.

Digestion-first / Dominion-fallback remains metadata only. Dominion awareness is not Dominion runtime.
