# v0.36.8 CLI Sandbox Apply & Agentic Task Surface

Parent track: Human-approved Patch Apply Sandbox.

v0.36.8 builds on v0.36.7 Patch Apply Sandbox OCEL Trace Packet, v0.36.6 Bounded Agentic Function/Task Operation Cycle, v0.36.5 Sandbox Post-Apply Validation, v0.36.4 Sandbox Patch Apply Engine, v0.36.3 Sandbox Workspace / Overlay Policy, v0.36.2 Dry-run Patch Apply Simulation, v0.36.1 Apply Candidate & Human Approval Contract, and v0.35.8 CLI Patch Proposal Surface. It creates a bounded CLI-facing surface for previewing and dispatching human-approved sandbox patch apply metadata flows.

## Definition

The CLI Sandbox Apply & Agentic Task Surface is a pure Python command contract and handler layer. It may parse CLI-like argv lists, classify safe and denied commands, build invocation and decision artifacts, dispatch allowed requests to bounded v0.36 helpers, and return bounded in-memory output artifacts.

The CLI surface is not shell execution. argv parsing is not OS command execution. sandbox-apply-run is not live apply. Sandbox file write is not live workspace write. agentic-task-run-once is not autonomous agent runtime. sandbox-trace-preview is not trace persistence. A validation command is not test execution. A denied command is a safe outcome, not remediation execution.

## Relationships

v0.36.7 supplies returned/in-memory trace packet helpers. v0.36.8 may call trace preview helpers to return packet metadata, but it does not persist traces.

v0.36.6 supplies bounded single-cycle agentic operation helpers. v0.36.8 may run agentic-task-run-once through those helpers, but it does not start independent autonomous runtime, multi-cycle loops, automatic retry, automatic repair, or external agent orchestration.

v0.36.5 supplies sandbox post-apply validation helpers. v0.36.8 may return validation and reconciliation metadata, but validation is not test execution and is not production certification.

v0.36.4 supplies sandbox patch apply helpers. v0.36.8 may dispatch sandbox-apply-run only through those helpers and only under an explicit validated sandbox root. It does not apply patches to the live workspace.

v0.36.3, v0.36.2, and v0.36.1 supply sandbox workspace, dry-run, candidate, and human approval metadata used by preview commands.

## Digestion And Dominion Boundary

The CLI policy follows digestion-first behavior: observe, summarize, model, and adapt static patterns safely before any future-gated Dominion discussion. Dominion fallback remains future-gated and blocked.

Commands or arguments attempting run-claude-code, run-codex, run-opencode, run-hermes, run-openclaw, dominion, external-agent-loop, infinite-loop, recursive-agent, agent-chain, harness-execute, auto-repair, repair-loop, retry-loop, or multi-cycle are denied, blocked, no-op, or future-gated outputs only. They are never executed.

OpenCode, Hermes, and OpenClaw references may be represented only as rejected or future-gated metadata. v0.36.8 does not execute, import, install, or test reference harness code.

## Surface Policy

CLISandboxApplySurfacePolicy allows safe preview commands and sandbox-apply-run as sandbox-scoped dispatch only. It allows sandbox file write and sandbox patch apply readiness only through v0.36.4 sandbox helpers under a validated sandbox root.

The policy blocks shell, subprocess, command execution, live workspace write, unrestricted patch application, live workspace write, live code edit, apply_patch, git_apply, test execution, dependency installation, reference execution/import, external agent execution, Claude Code invocation, Codex CLI invocation, Dominion runtime, infinite loops, automatic retry, automatic repair, multi-cycle loops, provider invocation, network access, credential access, secret reads, persistent trace writes, and UI runtime.

Argument specs bound values by length and prohibited patterns. Source refs, command specs, invocations, decisions, denied commands, runtime contexts, results, outputs, reports, previews, guarantees, and readiness reports are metadata artifacts only.

## Runtime Context

CLISandboxApplyRuntimeContext records whether supplied v0.36 artifacts are available and whether an explicit sandbox root is configured. Sandbox apply run may be allowed only when the context has a sandbox root and sandbox-scoped permissions. Live workspace write, unrestricted patch application, workspace write, code edit, shell, test execution, external agent execution, and Dominion runtime permissions remain false.

Runtime context metadata must not contain credentials. The CLI layer does not read environment variables, .env files, API keys, token files, credential files, or secret files.

## Invocation Flow

argv list + CLI policy + runtime_context -> CLISandboxApplyInvocation -> CLISandboxApplyInvocationDecision -> optional bounded v0.36 helper dispatch -> CLISandboxApplyRunOutput -> CLISandboxApplyRunReport -> V0368ReadinessReport -> v0.36.9 Consolidation.

Safe preview commands return bounded in-memory outputs. sandbox-apply-run dispatches to v0.36.4 sandbox helpers only. sandbox-validate and sandbox-reconcile dispatch to v0.36.5 validation helpers only. agentic-task-run-once dispatches to v0.36.6 single-cycle helpers only. sandbox-trace-preview dispatches to v0.36.7 returned trace packet helpers only.

## No-External-Side-Effect Guarantee
