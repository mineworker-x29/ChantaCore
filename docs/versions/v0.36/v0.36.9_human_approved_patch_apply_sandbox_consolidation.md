# v0.36.9 Human-approved Patch Apply Sandbox Consolidation

Foundation release name: Human-approved Patch Apply Sandbox v1.

Parent track: Human-approved Patch Apply Sandbox.

v0.36.9 consolidates v0.36.0 through v0.36.8 and preserves the v0.35.9 Controlled Patch Proposal Layer v1 boundary. It is consolidation only. It records release readiness, coverage, boundaries, risks, gaps, audit facts, bounded agentic task status, Digestion/Dominion status, external-agent-control status, and the v0.37 design-stage handoff.

Human-approved Patch Apply Sandbox v1 is a bounded foundation, not production runtime. Consolidation ready is not production ready. Sandbox apply workflow readiness is not live apply readiness.

## Relationship To Prior Stages

v0.36.0 established the Human-approved Patch Apply Sandbox boundary foundation. It kept live workspace write, live patch apply, test execution, shell execution, external agent execution, and Dominion runtime blocked.

v0.36.1 added Apply Candidate Envelope and Human Approval Contract metadata. It validated operator-supplied approval evidence and rejected model-generated self-approval.

v0.36.2 added Dry-run Patch Apply Simulation, hunk alignment, conflict detection, and in-memory simulated file deltas. Dry-run simulation is not patch application.

v0.36.3 added Sandbox Workspace / Overlay Policy, sandbox manifest, workspace plan, overlay policy, and file-map metadata. It preserved the live workspace write block.

v0.36.4 added Sandbox Patch Apply Engine. It allows sandbox directory materialization, sandbox file write, sandbox-only patch apply, sandbox write records, and sandbox apply results under a validated sandbox root. v0.36.x enables sandbox-only patch apply, not live patch application. v0.36.x enables sandbox file writes, not live workspace writes.

v0.36.5 added Sandbox Post-Apply Validation and Reconciliation. It added static sandbox validation, expected-vs-actual reconciliation, safety regression scan, and scope validation. Successful sandbox validation is not test success and not production certification.

v0.36.6 added Bounded Agentic Function/Task Operation Cycle. It supports bounded single-cycle function/task operation, one run packet, step records, stop reason, and mandatory human handoff. It is not autonomous agent runtime.

v0.36.7 added Patch Apply Sandbox OCEL-style returned/in-memory trace packets. Trace packet readiness is not persistent trace write readiness. OCEL-style packet creation is not OCEL file write.

v0.36.8 added CLI Sandbox Apply & Agentic Task Surface. CLI sandbox apply readiness is not shell readiness. CLI argv parsing is not OS command execution. CLI sandbox-apply-run delegates only to v0.36.4 sandbox helper boundaries. CLI agentic-task-run-once delegates only to v0.36.6 single-cycle boundaries.

## Digestion And Dominion Consolidation

Digestion first remains the policy: read, observe, summarize, model, adapt, and internalize static patterns safely. Dominion fallback remains future-gated. Dominion requires explicit future gate, strict boundary, audit trace, and human approval. v0.36.9 does not implement Dominion runtime.

External-agent-control patterns are consolidated only as blocked/future-gated metadata. v0.36.9 does not implement Codex-to-Claude-Code loops, OpenCode/Hermes execution loops, OpenClaw execution loops, recursive agent self-invocation, external agent orchestration, infinite loops, or automatic repair loops.

## Recommended v0.37 Track

v0.37 should be Controlled Sandbox Test Runner & Feedback Loop.

The recommended first step is a controlled sandbox test runner boundary. v0.37 should begin with allowlisted sandbox-only test command policy, no dependency install by default, timeout and resource limits, bounded output, no network, no arbitrary shell command, test result envelope, feedback report, repair suggestion metadata only, and a human checkpoint before any repair loop.

v0.37 should not immediately open general shell/test authority. Test execution must be sandbox-only, allowlisted, bounded, timeout-controlled, no-network, non-live, fully traced, and initially feedback-only.
