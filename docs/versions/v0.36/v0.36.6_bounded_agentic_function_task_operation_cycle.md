## Definition

Bounded agentic function/task execution means one explicit task, one bounded operation packet, one pass through supplied stage artifacts, fixed maximum step count, mandatory stop reason, and mandatory human handoff after the cycle.

It is not an independent autonomous agent. It is not a general agent runtime. It is not a self-directed control loop. It is not a multi-cycle loop. It does not retry automatically, repair automatically, invoke model providers, execute tools, run tests, or write live workspace files.

## Distinctions

- A bounded agentic task is not independent autonomous agent operation.
- A single-cycle operation is not a multi-cycle loop.
- A step record is not tool execution.
- A run packet is not a self-directed control loop.
- A stop reason prevents implicit continuation.
- A successful cycle is not production certification.
- A successful cycle is not live workspace readiness.

## Digestion / Dominion Posture

ChantaCore remains digestion-first. Static artifacts are read, modeled, summarized, adapted, and bounded before any future authority can be considered.

Dominion remains blocked in v0.36.6. Dominion fallback requires an explicit future gate, strict boundary, audit trace, and human approval. v0.36.6 records the policy as metadata only:

- `digestion_first_policy_applied`
- `dominion_runtime_blocked`
- `external_agent_execution_blocked`
- `infinite_agent_loop_blocked`
- `recursive_self_invocation_blocked`
- `automatic_repair_loop_blocked`
- `bounded_agentic_task_only`
- `no_independent_autonomous_agent_runtime`
- `mandatory_human_handoff_after_cycle`

External agent orchestration is prohibited. v0.36.6 does not invoke Claude Code, Codex CLI, OpenCode, Hermes, OpenClaw, or any external coding agent.

## Artifacts

`AgenticOperationPolicy` fixes `max_cycle_count` at exactly 1 and blocks automatic retry, automatic repair, multi-cycle loop, recursive self-invocation, independent agent runtime, external-agent orchestration, Dominion runtime, model invocation, general tool execution, live workspace write, patch application, test execution, shell, and dependency install.

`AgenticOperationInput` is a bounded task request, not an autonomous agent request.

`AgenticOperationIntent` requires `single_cycle_only=True`, `human_handoff_required=True`, `automatic_retry_allowed=False`, and `automatic_repair_allowed=False`.

`AgenticOperationStageArtifactRef` records supplied v0.36 artifacts. Required missing artifacts block or require review.

`AgenticOperationStepRecord` records stage-level metadata. It never records external tool execution, shell execution, test execution, live workspace write, or repair as performed.

`AgenticOperationStepSequence` is single-cycle only and bounded by `max_step_count`.

`AgenticOperationSafetyReport` is static metadata and keeps `ready_for_execution=False`.

`AgenticOperationStopReason` is mandatory and prevents implicit continuation, retry, and repair.

`AgenticOperationResult` may mark one cycle complete, but it is not production certification and not live workspace readiness.

`AgenticOperationRunPacket` has `max_cycle_count=1`, `automatic_retry_allowed=False`, `automatic_repair_allowed=False`, `human_handoff_required=True`, a stop reason through its result, and `ready_for_execution=False`.

`AgenticOperationNoAutonomyGuarantee` keeps all no-autonomy/no-loop/no-tool/no-test/no-live-write guarantees true.

`V0366ReadinessReport` may mark bounded operation and design handoff readiness true while preserving all autonomous/execution flags false.

## Safe Defaults

The default policy is single-cycle and fail-closed:

- `max_cycle_count` is exactly 1.
- Human handoff is mandatory.
- Unknown or missing required artifacts block or require review.
- Failed post-apply validation blocks successful completion.
- Blocking safety findings block successful completion.
- Successful operation result is future trace/CLI input only.

## Acceptance Criteria

- Required taxonomies, contracts, helpers, reports, guarantees, and readiness report exist.
- Run packet is bounded to exactly one cycle.
- Stop reason is mandatory.
- Human handoff is mandatory.
- Successful fake post-apply validation can produce a completed single-cycle packet.
- Failed validation, missing artifact, or blocking safety finding blocks or requires review.
- No helper reads/writes files, applies patches, calls providers, executes tools, shells out, runs tests, invokes external agents, or grants Dominion runtime.

## Withdrawal Conditions

Withdraw v0.36.6 readiness if any helper starts a second cycle, retries automatically, repairs automatically, performs recursive self-invocation, calls a model provider, executes tools, writes live workspace files, applies patches, runs tests, shells out, invokes external agents, imports/executes references, implements Dominion runtime, or sets any autonomous/execution readiness flag true.
