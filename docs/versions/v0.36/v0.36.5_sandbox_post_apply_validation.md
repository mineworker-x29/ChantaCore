## Definition

Sandbox post-apply validation is a bounded, sandbox-read-only validation layer. It may read files under a validated sandbox root, compare actual sandbox snapshots to expected dry-run after-content, check write-record scope, scan bounded text for static safety regressions, and produce validation/reconciliation reports.

It is not test execution. Static validation is not pytest. A reconciliation report is not remediation execution. A sandbox file read is not live workspace access. Successful validation is not live apply permission and is not production certification.

## Bounded Agentic Interpretation

In v0.36, agentic means bounded function/task execution metadata and future-gated single-cycle operation. v0.36.5 may validate the result of a bounded sandbox apply function, but it does not perform automatic repair, does not start a multi-cycle loop, and does not implement independent autonomous agent runtime.

## Models

`SandboxPostApplyValidationPolicy` allows sandbox file read, write-record validation, expected/actual comparison, static safety scan, reconciliation report, and future agentic task input. It blocks sandbox write, live write, patch application, workspace write, code edit, `apply_patch`, `git apply`, test execution, shell, dependency install, external agent execution, Dominion runtime, automatic repair, and multi-cycle repair loops.

`SandboxPostApplyValidationInput` is a validation request, not a write/apply request. Its prohibited runtime actions include write, apply, `apply_patch`, `git apply`, shell, test, install, external agent, Dominion, and repair loop.

`SandboxExpectedFileState` is expected metadata only. `SandboxActualFileState` is sandbox read metadata only. `SandboxAppliedFileObservation` is not write/apply readiness. `SandboxFileContentComparison` compares hashes and metadata; it does not repair mismatches.

Reports:

- `SandboxProposalReconciliationReport` compares expected and actual sandbox states and never triggers repair.
- `SandboxStaticValidationReport` performs static checks and does not run tests.
- `SandboxSafetyRegressionReport` detects unsafe patterns as metadata.
- `SandboxScopeValidationReport` confirms no outside/live/reference paths.
- `SandboxPostApplyValidationReport` may be successful, but it is not test success, live apply permission, or production certification.
- `SandboxPostApplyNoLiveWriteNoTestGuarantee` preserves no-write/no-test guarantees while allowing sandbox reads.
- `V0365ReadinessReport` may mark validation/reconciliation readiness and design-stage handoff only.

## Flow

`SandboxPatchApplyResult + WriteRecords + DryRunApplySimulationResult + SandboxWorkspaceManifest -> ValidationInput -> SandboxFileObservations -> ContentComparisons -> ReconciliationReport -> StaticValidationReport -> PostApplyValidationReport -> V0365ReadinessReport -> v0.36.6 Bounded Agentic Function/Task Operation Cycle`

## Safe Defaults

v0.36.5 fails closed on missing or unsafe sandbox roots. Every sandbox read validates path containment before reading. Validation blocks outside-root, live-root, reference-root, traversal, absolute path, and symlink escape surfaces.

Previews are bounded and redacted. Findings are metadata. Unsafe findings produce block/review outcomes, not automatic repair.

## Acceptance Criteria

- Required taxonomies, models, reports, guarantees, and readiness report exist.
- Sandbox file reads work only under validated sandbox root.
- Expected vs actual comparison works.
- Scope validation detects outside/live/reference path risks.
- Static safety scan detects unsafe readiness, provider/network, credential/secret, shell/test/install, external-agent, Dominion, persistent trace, live-write, and repair-loop openings as metadata.
- Reports preserve no write, no apply, no repair, no test execution, no shell, no external agent, and no Dominion runtime.

## Withdrawal Conditions

Withdraw v0.36.5 readiness if any helper writes files, applies additional patches, performs automatic repair, shells out, runs tests, reads outside the sandbox root, reads live workspace paths, imports/executes references, invokes external agents, grants Dominion authority, or sets execution/live-write/repair readiness true.

## Handoff

v0.36.5 hands off to v0.36.6 and v0.36.7 only as design-stage input. Successful validation is future bounded agentic task input only. It is not test success, not live apply permission, and not production certification.
