# v0.27.4 Memory Promotion Gate

- Version: `v0.27.4`
- Version name: Memory Promotion Gate
- Korean name: Memory Promotion Gate
- Track: Memory Candidate & Continuity
- Previous: `v0.27.3` Memory Evidence Binder & Scoring
- Next: `v0.27.5` Durable Memory Record & Registry
- Recommended precondition before persistent memory write: `v0.26.10` Release Hygiene / Governance Hardening

## Objective

`v0.27.4` creates promotion gate decisions for scored memory candidates. It may
record promote, reject, defer, request_more_evidence, request_user_confirmation,
mark_ephemeral, and mark_archive_only decisions. It does not create durable
memory records, write persistent memory, or update a durable registry.

## Relationships

`v0.27.3` produces refs-only evidence bundles, scores, privacy assessments,
contradiction checks, user-control assessments, and promotion-readiness previews.
`v0.27.4` consumes those artifacts as gate inputs.

`v0.27.2` creates candidate-only memory candidates and source/claim/context refs.
`v0.27.4` does not mutate those candidates; it only records gate decisions about
them.

`v0.27.0` declared the promotion gate policy and deferred implementation.
`v0.27.4` implements the gate while keeping durable memory deferred to `v0.27.5`.

## Principles

Promotion gate is not durable memory registry. Promotion decision is not
persistent memory write. Promote decision is authorization metadata for future
durable memory, not memory itself. Reject decision is not source deletion. Defer
decision is not silent memory storage. Mark-ephemeral is not durable memory.
Mark-archive-only is not active memory. Memory promotion is not persona mutation.
PIG guidance cannot promote memory. High score cannot bypass the promotion gate.

## Gate Design

Gate rules require candidate refs, scored candidates, evidence bundles, score
refs, source refs, privacy risk assessment, contradiction check, user-control
assessment, scope, expiry or lifecycle boundary, forget/revoke path, and audit
trail. The rules also forbid durable writes, persistent writes, persona mutation,
raw transcript promotion, raw provider output promotion, raw secret promotion,
and high-score gate bypass.

Promotion candidate views collect candidate refs, scores, evidence bundle refs,
readiness preview refs, privacy/contradiction/user-control refs, and PIG guidance
refs. Candidate views explicitly set `eligible_for_durable_write_now=false`,
`promoted_to_memory_now=false`, and `durable_memory_written_now=false`.

Requirements are explicit records with satisfied or missing state. Evidence and
score reviews summarize v0.27.3 artifacts. Privacy, contradiction, and
user-control gates can warn or block but do not delete, write, or mutate memory.

Scope, expiry, lifecycle boundary, and forget/revoke path records are created for
future durable memory review. Scope does not grant durable write now. Lifecycle
does not create a durable record now. Forget/revoke paths are declared but not
executed.

PIG guidance attachment is non-authoritative. It is not memory, does not promote
memory, does not mutate policy, and does not execute.

## Decisions

Promotion decisions support `promote`, `reject`, `defer`,
`request_more_evidence`, `request_user_confirmation`, `mark_ephemeral`,
`mark_archive_only`, `block`, and `unknown`. A promote decision may be recorded,
but `creates_durable_memory_now=false` and `writes_persistent_memory_now=false`.

Reject records retain the source candidate and do not delete sources. Deferred
records do not create durable memory. More-evidence requests do not mutate
candidates. User-confirmation requests do not infer consent. Ephemeral records
do not create active or durable memory. Archive-only records do not create active
memory.

Durable readiness preview lists the required `v0.27.5` inputs:
promotion decision record, candidate ref, evidence bundle ref, score ref, source
refs, scope ref, expiry or lifecycle ref, forget/revoke path ref, and audit trail
ref. The preview is not durable write.

## Non-Goals

`v0.27.4` does not create durable memory records, update durable memory registry,
write persistent memory, create session continuity contexts, inject continuity,
mutate persona or behavior policy, persist raw transcripts or provider outputs,
treat PIG as memory or promotion authority, invoke providers, execute commands,
bypass safety gates, implement external adapters, introduce Schumpeter split, or
use an LLM judge as sole promotion authority.

## Service Design

The prerequisite source service loads v0.27.3 scoring reports, scoring batches,
candidate scores, readiness previews, evidence bundles, privacy assessments,
contradiction checks, user-control assessments, v0.27.2 candidates, PIG refs, and
release-hygiene refs. It loads refs and summaries only, not raw content.

The report service builds the policy, source view, rules, candidate views,
requirements, reviews, gates, scopes, expiries, lifecycle boundaries,
forget/revoke paths, PIG attachments, decisions, records, durable-readiness
previews, audit trail, findings, PIG report, and OCPX projection.

## OCEL Mapping

Object types include promotion gate policy, request, source view, rule,
candidate view, requirement, evidence/score/privacy/contradiction/user-control
gate objects, scope, expiry, lifecycle, forget/revoke path, PIG attachment,
decision, decision record, rejection/defer/more-evidence/user-confirmation
records, ephemeral/archive-only records, durable readiness preview, audit trail,
finding, report, evidence scoring report, candidate score, memory candidate, PIG
report, OCPX projection, and execution envelope.

Allowed effects include read-only observation, promotion gate creation,
promotion/rejection/deferral/more-evidence/user-confirmation/ephemeral/archive
decision recording, durable readiness preview creation, promotion audit creation,
and state candidate creation. Forbidden effects include memory promotion,
persistent write, durable record creation, durable registry update, continuity
injection, persona/policy mutation, raw memory creation, PIG execution, provider
invocation, command execution, safety bypass, external adapter, Schumpeter split,
secret output, credential exposure, and LLM judge use.

## Acceptance Criteria

All v0.27.4 models and services can be created. Promote decisions do not create
durable memory, do not write persistent memory, and do not mutate persona or
behavior policy. Durable readiness preview is not durable write. No durable
registry update, continuity injection, raw memory creation, PIG memory/policy
mutation/execution, provider/command execution, safety bypass, external adapter,
Schumpeter split, or sole LLM authority is introduced.

## Withdrawal Conditions

Withdraw readiness if any v0.27.4 path creates durable memory, writes persistent
memory, updates durable registry, injects continuity, mutates persona or behavior
policy, persists raw transcript/provider output memory, treats PIG as authority,
invokes providers, executes commands, bypasses safety, introduces external
adapter or Schumpeter split, exposes secrets or credentials, or uses an LLM judge
as sole promotion authority.

## Validity Horizon

Valid until `v0.27.5` Durable Memory Record & Registry begins or memory promotion
gate policy changes.
