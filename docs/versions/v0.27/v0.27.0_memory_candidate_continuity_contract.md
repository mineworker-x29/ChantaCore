# v0.27.0 Memory Candidate & Continuity Contract

- Version: `v0.27.0`
- Version name: Memory Candidate & Continuity Contract
- Korean name: Memory Candidate·Continuity 계약
- Track: Memory Candidate & Continuity
- Previous: `v0.26.9` Workspace Agent Workbench Consolidation
- Recommended prerequisite before persistent memory write: `v0.26.10` Release Hygiene / Governance Hardening
- Next: `v0.27.1` Memory Source / Ref Boundary

## Objective

`v0.27.0` declares the contract for the Memory Candidate & Continuity track. It
starts from the `v0.26.9` Workbench Foundation v1 handoff and defines the
source, candidate, evidence, scoring, promotion, durable memory, session
continuity, injection, audit, privacy, governance, PIG guidance, and safety
boundaries for later `v0.27.x` releases.

This version is contract-only. It creates no memory candidates, performs no
memory scoring, promotes no memory, writes no durable memory, and injects no
continuity into runtime behavior.

## Relationship To v0.26.9

`v0.26.9` produces the Workspace Agent Workbench Foundation v1 consolidation,
the `WorkbenchV027ReadinessReport`, and the refs-only
`WorkbenchMemoryCandidateHandoffPacket`. `v0.27.0` treats those as source refs
for a contract. It does not reinterpret raw transcripts or raw provider output
as memory material.

## v0.26.10 Prerequisite

Third-party review raised release hygiene, governance, package, and data hygiene
concerns. `v0.27.0` can still be created because it is non-mutating. Persistent
memory write and durable registry work are deferred and require or strongly
prefer `v0.26.10` Release Hygiene / Governance Hardening.

## Core Principles

Memory candidate is not memory. Memory promotion is not persona mutation. PIG
guidance is not memory. Raw transcript is not default memory source. Session
continuity is not raw transcript replay. Continuity injection is guidance, not
behavior override. Durable memory must be gated, scoped, auditable, revocable,
and forgettable. Explicit user instruction outranks stale memory. Safety gate
must not be bypassed by memory.

## Source Boundary

Allowed future source categories are refs-only Workbench snapshot refs, OCEL
export package refs, session context refs, trace summary refs, evidence summary
refs, PIG guidance refs, approval decision refs, command candidate refs, failure
cause refs, human intervention refs, and event quality report refs.

Raw transcripts, raw provider outputs, raw secrets, credentials, private full
paths, and unredacted file content are forbidden as default memory sources.

## Candidate, Evidence, Scoring, And Promotion

Candidate extraction is deferred to `v0.27.2`. Evidence binding and scoring are
deferred to `v0.27.3`. Scores are not promotion decisions. Promotion gates are
deferred to `v0.27.4` and must require source refs, evidence bundles, scores,
privacy risk assessment, scope, lifecycle policy, audit trail, and a forget or
revoke path.

## Durable Memory And Continuity

Durable memory registry work is deferred to `v0.27.5` and must require
provenance, scope, evidence indexes, lifecycle policy, revocation, and forgetting.
Session continuity context construction is deferred to `v0.27.6`. Continuity
injection is deferred to `v0.27.7` and must remain previewable, traceable,
guidance-only, and subordinate to active safety and permission gates.

## Audit, Privacy, And Governance

Audit is required for promotion, update, revoke, and forget operations. Silent
memory overwrite and unlogged deletion are forbidden. Sensitive memory requires a
stricter gate. Raw secret, credential, private full path, and unscoped sensitive
personal attribute memory are blocked by default. Schema versioning and migration
plans are required before durable registry changes.

## Service Design

The contract service reads refs and summaries only. It provides prerequisite
source loading, roadmap creation, policy creation, contract creation, findings,
report rendering, PIG report generation, and OCPX projection generation. It does
not read raw transcripts, dump raw provider output, access credentials, invoke
providers, execute commands, write memory, or mutate runtime behavior.

## OCEL Mapping

eligibility, candidate, evidence, scoring, promotion gate, durable memory,

## Acceptance Criteria

The contract, roadmap, and all required policies can be created. Source
boundaries allow only refs-only Workbench/OCEL/PIG sources and forbid raw
transcripts, raw provider outputs, raw secrets, credentials, private paths, and
unredacted content. Candidate policy states candidates are not memory. Scoring is
not promotion. Promotion requires gates. Persistent memory write is forbidden in
`v0.27.0`. PIG guidance is not memory, policy mutation, or execution.

`ready_for_v0_27_1` may be true. `ready_for_v0_28` remains false.

## Withdrawal Conditions

Withdraw this contract if `v0.27.0` extracts or scores memory candidates,
promotes memory, writes persistent or durable memory, creates continuity
contexts, injects continuity, mutates persona or behavior policy, persists raw
transcripts or provider outputs as memory, treats PIG as memory or authority,
invokes providers, executes commands, bypasses safety gates, implements external
adapters, introduces Schumpeter split, exposes secrets or credentials, or uses an
LLM judge.

## Validity Horizon

Valid until `v0.27.1` Memory Source / Ref Boundary begins or memory policy
changes.
