# v0.27.7 Continuity Injection Boundary

## Version Definition

- Version: `v0.27.7`
- Name: Continuity Injection Boundary
- Korean name: Continuity Injection Boundary
- Track: Memory Candidate & Continuity
- Previous: `v0.27.6` Session Continuity Context Builder
- Next: `v0.27.8` Memory Audit / Update / Revoke / Forget

## Objective

`v0.27.7` defines and validates the continuity injection boundary. It may create
injection policies, source views, target surface catalogs, compatibility rules,
eligibility evaluations, priority rules, safety and permission boundary rules,
context item bindings, injection bundles, previews, decision records, rejected
and deferred records, boundary traces, audit trails, and boundary reports.

## Relationships

`v0.27.6` precedes this version by creating refs-only session continuity context
packs. `v0.27.7` reads those context packs, context items, staleness warnings,
conflict reports, and privacy filters as source refs. It does not mutate or
inject those artifacts into runtime.

`v0.27.5` supplies durable registry refs through `v0.27.6`; `v0.27.7` does not
create durable memory records, update durable registries, or write persistent
memory.

`v0.27.0` remains the contract source for the continuity injection policy,
privacy boundary, PIG boundary, safety boundary, and the full `v0.27.x` roadmap.

## Core Principles

- Continuity injection boundary is not runtime injection.
- Injection preview is not applied context.
- Injection bundle is not behavior override.
- Memory context is guidance, not authority.
- Explicit user instruction outranks memory.
- Safety gate and permission boundary must remain active.
- Contradictory memory must be surfaced.
- Stale memory must be warned.
- Privacy-filtered memory must remain filtered.
- Memory must not trigger provider invocation or command execution.
- PIG guidance may inform relevance, but PIG guidance is not authority.

## Implemented Surface

The target surface catalog includes `default_agent_context_future`,
`decision_service_guidance_future`, `skill_proposal_router_context_future`,
`route_candidate_context_future`, `provider_selection_context_future`,
`safety_gate_context_future`, and `workbench_display_context`. All surfaces
accept refs-only preview bundles, but none allow runtime mutation, provider
invocation, or command execution in `v0.27.7`.

Compatibility rules preserve refs-only context, target preview compatibility,
runtime mutation prohibition, user-instruction priority, safety and permission
boundaries, stale warning preservation, contradiction report preservation,
privacy filter preservation, raw replay prohibition, provider and command
prohibition, and the PIG non-authority boundary.

Eligibility evaluations can mark target surfaces eligible for preview or future
handoff bundle creation. Runtime injection and mutation remain false.

Priority policy orders safety policy, permission policy, explicit user
instruction, current task context, active memory context, and stale/archive
memory context. Safety rules prevent memory from bypassing or lowering safety.
Permission rules prevent memory from granting permission or expanding scope.

Context item bindings preserve warning, conflict, and privacy refs. Injection
bundles are refs-only future handoff artifacts. Injection previews are not
applied context. Decision records are OCEL-visible and do not apply runtime
injection.

## Non-Goals

`v0.27.7` does not perform runtime continuity injection, mutate Default Agent
context, mutate DecisionService, mutate SkillProposalRouter, mutate SafetyGate,
mutate PermissionPolicy, update memory, revoke memory, forget memory, mutate
persona, mutate behavior policy, replay raw transcripts, replay raw provider
outputs, use PIG guidance as authority, invoke providers, execute commands,
mutate files, bypass safety or permission boundaries, implement external
adapters, introduce Schumpeter split, or use an LLM judge as sole injection
authority.

## Service Design

The prerequisite source service loads `v0.27.6` continuity context build reports,
session continuity context, context packs, context items, conflict reports,
staleness warnings, privacy filters, PIG refs, and target surface contract refs.
It loads refs and summaries only, never raw transcripts, provider outputs,
secrets, or credentials.

Builder services create policy, source view, target catalog, compatibility
rules, eligibility evaluations, priority policy/rules, safety and permission
boundary rules, bindings, bundles, previews, decisions, records, boundary
traces, audit trails, findings, and the report.

## Acceptance Criteria

- Injection policy, source view, target surface catalog, compatibility rules,
  eligibility evaluations, priority policy, safety and permission boundary
  rules, bindings, bundle, preview, decision records, boundary trace, audit
  trail, and boundary report can be created.
- Runtime injection does not occur.
- Default Agent, DecisionService, SkillRouter, SafetyGate, and PermissionPolicy
  mutation does not occur.
- Memory update, revoke, and forget do not occur.
- Persona and behavior policy mutation do not occur.
- Provider, command, and file execution do not occur.
- Safety and permission bypass do not occur.
- PIG guidance is not authority.
- External adapter and Schumpeter split are not introduced.
- LLM judge is not used as sole authority.

## Withdrawal Conditions

Withdraw readiness if any runtime context is mutated, if any target surface
applies the bundle now, if memory changes are executed, if raw transcript or
provider output is replayed, if PIG guidance becomes authority, if provider,
command, or file execution occurs, or if safety/permission boundaries are
bypassed.

## Validity Horizon

Valid until `v0.27.8` Memory Audit / Update / Revoke / Forget begins or the
continuity injection boundary policy changes.
