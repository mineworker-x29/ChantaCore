# v0.27.3 Memory Evidence Binder & Scoring

- Version: `v0.27.3`
- Version name: Memory Evidence Binder & Scoring
- Korean name: Memory Evidence Binder·Scoring
- Track: Memory Candidate & Continuity
- Previous: `v0.27.2` Memory Candidate Extraction
- Recommended prerequisite before persistent memory write: `v0.26.10` Release Hygiene / Governance Hardening
- Next: `v0.27.4` Memory Promotion Gate

## Objective

`v0.27.3` binds evidence to memory candidates and creates scoring artifacts. It
reads `v0.27.2` candidate extraction reports, candidate batches, candidate refs,
claims, source links, PIG signals, `v0.27.1` source quality/source boundary refs,
and available event quality, trace coverage, redaction, PIG, and OCPX refs.

This version may score candidates. It must not promote candidates, write
persistent or durable memory, update a durable registry, create session
continuity contexts, inject continuity, mutate persona or behavior policy,
invoke providers, execute commands, implement external adapters, introduce
Schumpeter split, or use an LLM judge as sole scoring authority.

## Relationship To v0.27.2

`v0.27.2` creates candidate-only artifacts. `v0.27.3` binds refs-only evidence to
those candidates and creates score breakdowns, candidate scores, scoring
decisions, and promotion-readiness previews. A scored candidate is still not
memory.

## Relationship To v0.27.1

`v0.27.1` source quality, redaction, trace coverage, and event quality refs may
inform scoring. Those refs remain refs-only inputs. Raw transcript, raw provider
output, raw secrets, credentials, and private full paths remain forbidden
evidence.

## Relationship To v0.27.0

`v0.27.0` declared scoring as policy but deferred implementation. `v0.27.3`
implements the scoring slice while preserving the original contract: scoring is
not promotion, and persistent memory write remains deferred.

## Core Principles

Evidence binding is not promotion. Scoring is not promotion. High score is not
automatic memory. Score is evidence for future promotion gate, not a gate
result. Evidence strength is not truth. Contradiction check is not automatic
deletion. Privacy risk is not automatic rejection by itself. PIG scoring signal
is not memory. PIG guidance cannot promote memory. LLM judge must not be sole
scoring authority.

## Evidence Bundle

Each `MemoryCandidateEvidenceBundle` groups refs-only evidence items, support
links, claim support assessments, source refs, PIG guidance refs, event quality
refs, trace coverage refs, and redaction refs for one candidate. Bundles do not
include raw transcript, raw provider output, raw secrets, credentials, or private
full paths.

## Evidence Item And Support Link

`MemoryEvidenceItem` records sanitized evidence summaries and support strength.
`MemoryEvidenceSupportLink` connects a candidate claim to an evidence item.
Support strength does not assert truth.

## Claim Support Assessment

`MemoryClaimSupportAssessment` records supported, weakly supported,
unsupported, contradicted, blocked, or unknown support. It keeps
`claim_asserted_as_truth=False`.

## Assessment Models

The implementation creates evidence strength, source quality, recency,
stability, reuse value, specificity, privacy risk, contradiction, and user
control requirement assessments. These are scoring signals only. Contradiction
checks do not delete or overwrite candidates. Privacy risk does not automatically
reject or promote memory.

## Score Breakdown

Required dimensions are evidence strength, source quality, recency, stability,
reuse value, specificity, risk level, privacy sensitivity, contradiction risk,
and user control requirement. `MemoryCandidateScoreBreakdown` keeps positive,
negative, and blocking signal refs. The breakdown is explainable and refs-only.

## Candidate Score

`MemoryCandidateScore` creates an overall score and score band. It sets
`score_is_promotion=False`, `high_score_auto_promotes=False`,
`persistent_memory_written=False`, and `durable_memory_record_created=False`.

## Scoring Decision And Batch

`MemoryCandidateScoringDecision` records whether a score was created and whether
the candidate may be ready for the future `v0.27.4` promotion gate. It does not
promote memory and does not write persistent memory. `MemoryCandidateScoringBatch`
aggregates scored, partial, deferred, blocked, and ready-for-gate-review counts.

## Promotion Readiness Preview

`MemoryPromotionReadinessPreview` is a preview only. It has
`preview_is_not_promotion=True` and `promotion_performed_now=False`. Actual
promotion remains deferred to `v0.27.4`.

## Audit Trail

`MemoryEvidenceScoringAuditTrail` records scoring request refs, candidate refs,
evidence bundle refs, assessment refs, score refs, scoring decision refs, and
promotion readiness preview refs. It keeps raw content excluded.

## Non-Goals

This version does not promote memory, write persistent memory, write durable
memory, update durable memory registry, create session continuity context,
inject continuity, mutate persona, mutate behavior policy, persist raw
transcripts or provider outputs as memory, treat PIG as memory/policy/execution,
invoke providers, execute commands, bypass safety gates, add external adapters,
introduce Schumpeter split, expose raw secrets or credentials, or use an LLM
judge as sole scoring authority.

## Service Design

The service layer loads v0.27.2 candidate refs and summaries, builds policy,
source view, binding rules, evidence bundles, assessments, PIG scoring signals,
dimension values, score breakdowns, scores, scoring decisions, scoring batch,
readiness previews, audit trail, findings, report, PIG report, OCPX projection,
and CLI output. It does not call providers, execute commands, or use network or
external adapters.

## OCEL Mapping

Object types include evidence scoring policy, request, source view, binding
rule, evidence bundle, evidence item, support link, claim support assessment,
assessment models, PIG scoring signal, dimension value, score breakdown,
candidate score, scoring decision, scoring batch, readiness preview, audit
trail, finding, report, candidate, candidate batch, extraction report, PIG
report, OCPX projection, and execution envelope.

Allowed effects are `read_only_observation`, `memory_candidate_evidence_bound`,
`memory_candidate_scored`, `memory_candidate_score_breakdown_created`,
`memory_contradiction_check_created`,
`memory_privacy_risk_assessment_created`,
`memory_promotion_readiness_preview_created`,
`memory_evidence_scoring_audit_created`, and `state_candidate_created`.

Forbidden effects include memory promotion, persistent write, durable record or
registry update, session continuity context creation, continuity injection,
persona mutation, behavior policy mutation, raw transcript/provider output
memory, PIG memory/policy/execution, provider invocation, command execution,
safety bypass, external adapter implementation, Schumpeter split, raw secret
output, credential exposure, and LLM judge use.

## Acceptance Criteria

Policy, source view, binding rules, evidence bundles, evidence items, support
links, claim support assessments, required assessments, PIG scoring signals,
dimension values, score breakdowns, candidate scores, scoring decisions, scoring
batch, promotion-readiness previews, audit trail, and report can be created.

Scores do not promote memory. High scores do not auto-promote memory. No
persistent or durable memory write occurs. No durable registry update occurs. No
continuity injection occurs. No persona or behavior policy mutation occurs. No
raw transcript/provider output memory occurs. No PIG memory/policy/execution
occurs. No provider/command execution occurs. No safety bypass occurs. No
external adapter or Schumpeter split is introduced. No LLM judge is used as sole
authority.

`ready_for_v0_27_4` may be true. `ready_for_v0_28` remains false.

## Withdrawal Conditions

Withdraw this version if scoring promotes memory, high scores create memory,
promotion readiness preview performs promotion, persistent/durable memory is
written, a durable registry is updated, continuity is injected, persona or
behavior policy is mutated, raw transcript/provider output becomes memory, PIG
is treated as memory/policy/execution, providers or commands execute, safety
gates are bypassed, external adapters or Schumpeter split are introduced,
secrets or credentials are exposed, or an LLM judge becomes sole scoring
authority.

## Validity Horizon

Valid until `v0.27.4` Memory Promotion Gate begins or memory scoring policy
changes.
