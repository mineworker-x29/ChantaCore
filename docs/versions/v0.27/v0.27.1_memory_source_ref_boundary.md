# v0.27.1 Memory Source / Ref Boundary

- Version: `v0.27.1`
- Version name: Memory Source / Ref Boundary
- Korean name: Memory Source·Ref Boundary
- Track: Memory Candidate & Continuity
- Previous: `v0.27.0` Memory Candidate & Continuity Contract
- Recommended prerequisite before persistent memory write: `v0.26.10` Release Hygiene / Governance Hardening
- Next: `v0.27.2` Memory Candidate Extraction

## Objective

`v0.27.1` implements the refs-only source/ref boundary for future memory
candidate extraction. It reads the `v0.27.0` contract and the `v0.26.9`
Workbench handoff, then creates source policy views, category catalogs, source
refs, bundles, registry views, eligibility rules, eligibility evaluations,
eligibility decisions, redaction views, source quality reports, forbidden-source
reports, and candidate-readiness boundaries.

It does not create memory candidates, score memory, promote memory, write
persistent or durable memory, create session continuity contexts, inject
continuity, mutate persona or behavior policy, invoke providers, execute
commands, bypass safety gates, add external adapters, introduce Schumpeter split,
or use an LLM judge.

## Relationship To v0.27.0

`v0.27.0` declares the Memory Candidate & Continuity contract. `v0.27.1`
materializes the first non-mutating boundary under that contract: source refs can
be inspected and evaluated, but source eligibility is not candidate extraction.

## Relationship To v0.26.9 And v0.26.8

The Workbench Foundation v1 handoff provides candidate-ready input names such as
Workbench snapshot refs, OCEL export package refs, session context refs, trace
summary refs, evidence summary refs, PIG guidance refs, approval decision refs,
command candidate refs, failure cause refs, human intervention refs, and event
quality report refs. `v0.26.8` provides snapshot/export, quality, trace coverage,
and redaction refs. `v0.27.1` preserves those as refs and summaries only.

## Core Principles

Memory source is not memory. Memory source ref is not memory candidate. Source
eligibility is not candidate extraction. Source quality score is not memory
scoring. Source bundle is not memory registry. Redaction report is not data
deletion. PIG guidance ref is not memory. Raw transcript is not default memory
source. Raw provider output is not memory evidence.

## Allowed Source Categories

Allowed categories are `workbench_snapshot_refs`, `ocel_export_package_refs`,
`session_context_refs`, `trace_summary_refs`, `evidence_summary_refs`,
`pig_guidance_refs`, `approval_decision_refs`, `command_candidate_refs`,
`failure_cause_refs`, `human_intervention_refs`, `event_quality_report_refs`,
`trace_coverage_report_refs`, `redaction_report_refs`, and
`release_hygiene_status_refs`.

## Forbidden Source Categories

Forbidden categories are `raw_transcript`, `raw_provider_output`, `raw_secret`,
`credential`, `private_full_path`, `unredacted_file_content`,
`unscoped_personal_sensitive_attribute`, `uncited_inference`,
`unredacted_runtime_db`, `unredacted_bak_file`, and `external_adapter_output`.
Forbidden categories must not be accepted as eligible memory sources.

## Eligibility Rules

Required rules include source ref required, evidence ref required, refs-only
required, raw transcript forbidden, raw provider output forbidden, raw secret
forbidden, credential forbidden, private full path forbidden, redaction required,
source quality required, event quality required for candidate readiness, trace
coverage warning if missing, PIG guidance ref allowed but not memory, source
without ref blocked, and raw content source blocked.

## Redaction Policy

Redaction removes raw content from the source view and preserves refs and
sanitized summaries. It documents exclusion; it does not delete source data.

## Source Quality Policy

Source quality checks include event quality, trace coverage, evidence density,
provenance strength, redaction status, recency, stability, privacy risk,
contradiction warnings, and release hygiene status. These are source-quality
signals, not memory scoring.

## Candidate Readiness Boundary

The readiness boundary identifies refs that may be eligible for `v0.27.2`
candidate extraction. It does not create candidates and does not create memory.

## Non-Goals

This version does not perform memory candidate extraction, memory scoring,
memory promotion, persistent memory write, durable memory record creation,
durable memory registry update, session continuity context creation, continuity
injection, persona mutation, behavior policy mutation, raw transcript memory,
raw provider output memory, PIG memory/policy/execution, provider invocation,
command execution, safety gate bypass, external adapter work, Schumpeter split,
raw secret or credential exposure, or LLM judging.

## Service Design

The service layer loads contract, handoff, snapshot/export, quality, trace
coverage, redaction, PIG, OCPX, and release hygiene refs when available. It
builds category catalogs, source refs, bundles, registry views, rules,
evaluations, decisions, redaction reports, quality reports, risk flags,
forbidden-source reports, candidate-readiness boundaries, findings, PIG report,
OCPX projection, and CLI output. It never loads raw transcripts, raw provider
outputs, secrets, credentials, or private full paths.

## OCEL Mapping

Object types include source boundary policy view, request, catalog, source ref,
bundle, registry view, eligibility rule/evaluation/decision, redaction policy,
redaction view/report, quality policy/signal/report, risk flag, forbidden source
report, candidate readiness boundary, finding, report, memory contract, workbench
handoff, workbench snapshot, OCEL export package, PIG report, OCPX projection,
and execution envelope.

Allowed effects are `read_only_observation`, `memory_source_boundary_created`,
`memory_source_ref_created`, `memory_source_ref_bundle_created`,
`memory_source_eligibility_evaluated`, `memory_source_redaction_view_created`,
`memory_source_quality_report_created`,
`memory_candidate_readiness_boundary_created`, and `state_candidate_created`.

Forbidden effects include memory candidate creation/extraction/scoring, memory
scoring, promotion, persistent write, durable record/registry updates, continuity
context creation, injection bundles, persona or behavior policy mutation, raw
transcript/provider output memory, PIG memory/policy/execution, provider
invocation, command execution, safety bypass, external adapters, Schumpeter
split, raw secret output, credential exposure, and LLM judge use.

## Acceptance Criteria

Source boundary policy view, category catalog, source refs, source bundle,
registry view, eligibility rules/evaluations/decisions, redaction report, source
quality report, risk flags, forbidden-source report, candidate-readiness
boundary, and report can be created. Refs are refs-only. Raw transcript/provider
output/secret/credential/private path sources are not eligible. Candidate
readiness can be computed for `v0.27.2` without creating candidates.

`ready_for_v0_27_2` may be true. `ready_for_v0_28` remains false.

## Withdrawal Conditions

Withdraw this boundary if forbidden raw sources are accepted, source-without-ref
or source-without-evidence is accepted as eligible, memory candidates are
created, memory is scored/promoted/written, continuity is injected, persona or
behavior policy is mutated, providers or commands execute, safety gates are
bypassed, external adapters or Schumpeter split are introduced, secrets or
credentials are exposed, or an LLM judge is used.

## Validity Horizon

Valid until `v0.27.2` Memory Candidate Extraction begins or memory source
boundary policy changes.
