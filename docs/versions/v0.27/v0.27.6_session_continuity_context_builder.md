# v0.27.6 Session Continuity Context Builder

## Version Definition

- Version: `v0.27.6`
- Name: Session Continuity Context Builder
- Korean name: Session Continuity Context Builder
- Track: Memory Candidate & Continuity
- Previous: `v0.27.5` Durable Memory Record & Registry
- Next: `v0.27.7` Continuity Injection Boundary

## Objective

`v0.27.6` builds refs-only session continuity context artifacts from durable
memory registry refs and allowed workbench refs. It creates source views,
eligibility rules, continuity memory refs, ref bundles, relevance scores,
recency windows, staleness warnings, conflict reports, privacy filters, context
items, context packs, session continuity contexts, previews, build decisions,
audit trails, and build reports.

## Relationships

`v0.27.5` precedes this version by creating the gated durable memory record and
registry layer. `v0.27.6` may read `DurableMemoryRegistryReport`,
`DurableMemoryRegistry`, `DurableMemoryRecord`, registry entries, scope,
provenance, evidence index, lifecycle, privacy boundary, conflict marker,
forget/revoke binding, and audit refs. It does not create durable memory
records or update the durable registry.

`v0.27.4` supplies promotion gate decisions that made durable records possible
in `v0.27.5`. `v0.27.6` treats those as provenance through durable memory refs,
not as runtime instructions.

`v0.27.0` remains the contract layer. The source boundary, candidate boundary,
evidence/scoring boundary, promotion boundary, durable registry boundary,
continuity boundary, privacy boundary, PIG boundary, and safety boundary remain
in force.

## Core Principles

- Session continuity context is not raw transcript replay.
- Continuity context pack is not runtime injection.
- Continuity context item is not behavior override.
- Memory retrieval is not command execution.
- Memory relevance score is not authority.
- Stale memory must be surfaced, not silently applied.
- Contradictory memory must be surfaced, not silently resolved.
- Privacy-filtered memory must remain filtered.
- Revoked, forgotten, expired, and blocked memory must not be used as active continuity context.
- PIG guidance is not memory authority.

## Implemented Surface

The source view separates active, archived, revoked, forgotten, expired, and
blocked memory refs. Only active memory or allowed reference memory can enter a
context pack. Archived memory is background/archive reference only. Revoked,
forgotten, expired, and blocked memory cannot become active context.

Eligibility rules require refs-only context, scope alignment, privacy filtering,
conflict surfacing, staleness warnings, no raw replay, no runtime injection, no
persona mutation, no behavior policy mutation, no provider invocation, and no
command execution.

Continuity memory refs retain memory status, scope, provenance, evidence index,
lifecycle, privacy boundary, and conflict marker refs. The ref bundle separates
active context, background references, conflict warnings, stale warnings, and
blocked refs.

Relevance scoring ranks candidate refs for future context use, but relevance
score is not authority. Recency windows and staleness warnings surface stale or
expired memory. Conflict reports surface conflicts and do not automatically
resolve them. Privacy filters keep filtered memory filtered and do not include
raw sensitive content.

Context items and context packs are refs-only. They do not inject runtime state,
mutate persona, or mutate behavior policy. The session continuity context and
preview are built for `v0.27.7` review only.

## Non-Goals

`v0.27.6` does not inject continuity into runtime, mutate Default Agent context,
mutate DecisionService, mutate SkillProposalRouter, mutate SafetyGate, mutate
PermissionPolicy, update memory, revoke memory, forget memory, write persistent
memory, create durable memory records, update durable registries, mutate
persona, mutate behavior policy, replay raw transcripts, replay raw provider
outputs, use PIG guidance as authority, invoke providers, execute commands,
bypass safety gates, implement external adapters, introduce Schumpeter split, or
use an LLM judge as sole continuity authority.

## Service Design

The prerequisite source service loads refs and summaries only. It may load
durable registry reports, registries, records, registry entries, scope,
provenance, evidence index, lifecycle, privacy, conflict, forget/revoke, session
context, trace summary, and PIG refs. It must not load raw transcripts, raw
provider outputs, secrets, credentials, or private full paths.

Builder services create the policy, source view, eligibility rules, memory refs,
ref bundle, relevance policy, relevance scores, recency windows, staleness
warnings, conflict reports, privacy filters, context items, context pack,
session continuity context, build decision, preview, audit trail, findings, and
build report.

## Acceptance Criteria

- Session continuity policy, source view, eligibility rules, memory refs, ref
  bundle, relevance score, staleness warning, conflict report, privacy filter,
  context item, context pack, session continuity context, preview, audit trail,
  and build report can be created.
- Context pack is refs-only and is not runtime injection.
- Revoked, forgotten, expired, and blocked memory is not active context.
- Stale, conflicting, and private memory is surfaced or filtered.
- Default Agent, DecisionService, SkillRouter, SafetyGate, and PermissionPolicy
  are not mutated.
- Memory update, revoke, and forget are not executed.
- Provider invocation, command execution, safety bypass, persona mutation,
  behavior policy mutation, external adapter work, Schumpeter split, and sole
  LLM judge authority are absent.

## Withdrawal Conditions

Withdraw readiness if continuity context is injected before `v0.27.7`, if any
runtime or policy surface is mutated, if memory records are updated/revoked/
forgotten, if raw transcript or provider output is replayed, if sensitive raw
content is included, if PIG guidance becomes authority, or if provider/command/
safety/external adapter boundaries are crossed.

## Validity Horizon

Valid until `v0.27.7` Continuity Injection Boundary begins or the session
continuity context policy changes.
