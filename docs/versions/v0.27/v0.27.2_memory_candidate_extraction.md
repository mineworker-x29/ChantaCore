# v0.27.2 Memory Candidate Extraction

- Version: `v0.27.2`
- Version name: Memory Candidate Extraction
- Korean name: Memory Candidate 추출
- Track: Memory Candidate & Continuity
- Previous: `v0.27.1` Memory Source / Ref Boundary
- Recommended prerequisite before persistent memory write: `v0.26.10` Release Hygiene / Governance Hardening
- Next: `v0.27.3` Memory Evidence Binder & Scoring

## Objective

`v0.27.2` implements Memory Candidate Extraction. It reads the `v0.27.1`
refs-only source boundary and candidate-readiness boundary, then creates
candidate-only artifacts from eligible source refs.

This version creates memory candidates for the first time, but a memory
candidate is not memory. Candidate extraction is not scoring, promotion, durable
memory creation, persistent memory write, registry update, or session continuity
injection.

## Relationship To v0.27.1

`v0.27.1` creates source refs, source bundles, eligibility decisions, redaction
views, source quality reports, forbidden-source reports, and candidate-readiness
boundaries. `v0.27.2` consumes those refs and creates candidate-only artifacts.
It does not widen the source boundary and does not read raw transcript or raw
provider output.

## Relationship To v0.27.0

`v0.27.0` declares the Memory Candidate & Continuity contract. `v0.27.2`
implements only the candidate extraction slice under that contract. Scoring,
promotion, durable memory, session continuity, injection, audit/update/revoke,
and consolidation remain deferred to later `v0.27.x` versions.

## Relationship To v0.26.9 Handoff

The `v0.26.9` Workbench handoff remains a refs-only source surface. `v0.27.2`
may use Workbench snapshot, OCEL export, event quality, trace coverage,
redaction, PIG, and OCPX refs when available. It does not use raw transcripts or
raw provider output as default candidate sources.

## Core Principles

Memory candidate is not memory. Candidate extraction is not scoring. Candidate
extraction is not promotion. Candidate extraction is not persistent memory
write. Candidate source link is not evidence score. Candidate claim is not
asserted truth. Candidate context is not continuity injection. PIG guidance
signal is not memory. Raw transcript is not default candidate source.

## Allowed Candidate Types

Allowed candidate types are task preference, workflow preference, project state,
decision pattern, skill usage pattern, provider route pattern, approval
preference, failure pattern, context summary, long task continuity, user
instruction, system boundary, do-nothing preference, safety preference, and
unknown.

## Extraction Rules

Required rules include eligible source, source ref, evidence ref, provenance,
redaction, candidate type, ref-supported claim, sanitized summary, risk flags,
PIG guidance as signal only, no scoring, no promotion, and no persistent write.
Raw transcript, raw provider output, raw secret, credential, and private full
path sources are forbidden.

## Service Design

The service layer loads `v0.27.1` source-boundary artifacts, builds an extraction
policy, source view, deterministic type classifier, extraction rules,
candidate-only artifacts, decisions, a candidate batch, an audit trail, findings,
PIG report, OCPX projection, and CLI output. It uses refs and sanitized
summaries only. It does not invoke providers, execute commands, load external
adapters, call networks, score candidates, promote memory, write durable memory,
or mutate runtime behavior.

## Acceptance Criteria

The extraction policy, source view, extraction rules, type classifier, candidate
batch, memory candidate, source link, claim, context, provenance, PIG signal,
risk flag, redaction view, extraction decision, audit trail, and extraction
report can be created. Candidates are created only from eligible refs, remain
`candidate_only`, include source/evidence/provenance/risk/redaction context, and
do not assert claims as truth.

No scoring, promotion, persistent or durable memory write, durable registry
update, continuity injection, persona mutation, behavior policy mutation, raw
transcript/provider output memory, PIG memory/policy/execution, provider or
command execution, safety bypass, external adapter, Schumpeter split, raw secret
output, credential exposure, or LLM judge authority is introduced.

`ready_for_v0_27_3` may be true. `ready_for_v0_28` remains false.

## Withdrawal Conditions

Withdraw this version if candidates are created from forbidden raw sources, if a
candidate lacks source refs or evidence refs, if scoring or promotion occurs, if
persistent/durable memory is written, if a durable registry is updated, if
continuity is injected, if persona or behavior policy is mutated, if PIG is
treated as memory/policy/execution, if providers or commands execute, if safety
gates are bypassed, if external adapters or Schumpeter split are introduced, if
secrets or credentials are exposed, or if an LLM judge becomes sole authority.

## Validity Horizon

Valid until `v0.27.3` Memory Evidence Binder & Scoring begins or memory
candidate extraction policy changes.
