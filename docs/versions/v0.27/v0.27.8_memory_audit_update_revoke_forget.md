# v0.27.8 Memory Audit / Update / Revoke / Forget

## Version Definition

- Version: `v0.27.8`
- Name: Memory Audit / Update / Revoke / Forget
- Korean name: Memory Audit·Update·Revoke·Forget
- Track: Memory Candidate & Continuity
- Previous: `v0.27.7` Continuity Injection Boundary
- Next: `v0.27.9` Memory Candidate & Continuity Consolidation

## Objective

`v0.27.8` implements memory lifecycle control for durable memory records. It may
create review records, update candidates, update decisions and records,
supersede records, revoke records, forget records and tombstones, archive and
expiration records, conflict resolution records, no-op decisions, registry
update previews and records, lifecycle audit trails, and lifecycle reports.

## Relationships

`v0.27.7` precedes this version by defining injection boundaries. `v0.27.8`
reads those boundary reports as refs and does not inject continuity.

`v0.27.6` precedes the injection boundary by building refs-only continuity
context packs. Lifecycle control may reference those context reports, but does
not replay or mutate them.

`v0.27.5` supplies durable memory registry and durable memory record refs.
Lifecycle operations are scoped to those refs and are gated before update,
revoke, forget, archive, expiration, or registry update records are marked as
applied.

`v0.27.0` remains the governing Memory Candidate & Continuity contract.

## Core Principles

- Memory review is not mutation.
- Memory update candidate is not update execution.
- Revoke removes active use.
- Forget removes active use and recallable memory content.
- Forget is not source data deletion by default.
- Archive is not active memory.
- Expiration is not deletion.
- Conflict resolution must be recorded.
- No-op is a valid lifecycle decision.
- PIG guidance is not lifecycle authority.

## Implemented Surface

The lifecycle policy enables review, update candidates, update decisions,
conditional update records, supersede records, revoke, forget, archive,
expiration, conflict resolution, no-op decisions, lifecycle gates, and audit.
Source data deletion remains disabled now. Silent overwrite and unlogged
deletion are forbidden.

The lifecycle source view reads durable registry reports, registries, durable
memory record refs, registry entries, record status refs, lifecycle policies,
privacy boundaries, conflict markers, forget/revoke bindings, evidence indexes,
provenance refs, continuity context refs, injection boundary refs, and PIG refs.
It is refs-only and excludes raw transcripts, raw provider outputs, secrets, and
credentials.

The lifecycle gate records whether memory record refs exist, lifecycle policy
and forget/revoke binding allow the operation, evidence is available, privacy
and conflict checks pass, audit is ready, user confirmation is required or
present, source data deletion was requested, raw memory restoration is absent,
and persona or behavior policy mutation is absent.

Review requests and review records are non-mutating. Update candidates are not
update execution. Update decisions and update records are tied to lifecycle
gates and audit refs. Revoke removes active use. Forget removes active use and
recallable memory content, then creates an audit-only tombstone. Archive and
expiration remove active use without source deletion. Conflict resolution is
recorded and not silently resolved. No-op records no mutation.

Registry update previews are not registry updates. Registry update records are
OCEL-visible and gated.

## Non-Goals

`v0.27.8` does not silently overwrite memory, perform unlogged deletion, delete
source data by default, restore raw transcripts, restore raw provider outputs,
mutate persona, mutate behavior policy, use PIG guidance as lifecycle authority,
invoke providers, execute commands, bypass safety, inject continuity, implement
external adapters, introduce Schumpeter split, or use an LLM judge as sole
lifecycle authority.

## Service Design

The prerequisite source service loads `v0.27.7`, `v0.27.6`, and `v0.27.5`
reports and registry artifacts as refs and summaries only. Builder services
create the control policy, source view, operation gates, review records, update
candidates and decisions, supersede/revoke/forget/archive/expiration records,
conflict records, privacy/evidence/scope checks, PIG guidance attachments,
no-op decisions, registry update previews/records, audit trails, findings, and
the lifecycle report.

## OCEL Mapping

Object types include lifecycle policy, request, source view, operation gate,
review request/record, update candidate/decision/record, supersede record,
revoke request/decision/record, forget request/decision/record/tombstone,
archive and expiration decision/record, conflict resolution record, privacy
gate, evidence review, lifecycle scope, PIG guidance attachment, no-op
decision, lifecycle audit trail, registry update preview/record, finding,
report, durable memory record, durable memory registry, PIG report, OCPX
projection, and execution envelope.

Allowed effects include read-only observation, review record creation, update
candidate creation, update decision recording, update record creation,
supersede record creation, revoke record creation, forget record creation,
forget tombstone creation, archive/expiration record creation, conflict
resolution record creation, lifecycle registry update, lifecycle audit
creation, and state candidate creation.

Conditional effects are guarded by `MemoryLifecycleOperationGate`. Forbidden
effects include silent memory overwrite, unlogged deletion, source data
deletion, raw restoration, persona mutation, behavior policy mutation, PIG
authority/policy/execution, provider invocation, command execution, safety
bypass, continuity injection, external adapters, Schumpeter split, raw secret
output, credential exposure, and LLM judge use.

## Acceptance Criteria

- Lifecycle policy, source view, operation gate, review records, update
  candidates/decisions/records, revoke records, forget records, tombstones,
  archive and expiration records, conflict resolution records, no-op decisions,
  registry update previews/records, audit trail, and lifecycle report can be
  created.
- Forget does not delete source data by default.
- Tombstones contain no recallable memory content, raw source content, or
  secrets.
- No silent overwrite or unlogged deletion occurs.
- No persona or behavior policy mutation occurs.
- No PIG authority/policy/execution occurs.
- No provider/command execution, safety bypass, or continuity injection occurs.
- No external adapter or Schumpeter split is introduced.
- LLM judge is not used as sole lifecycle authority.

## Withdrawal Conditions

Withdraw readiness if any lifecycle operation bypasses the lifecycle gate, if
source data is deleted by default, if tombstones include recallable content, if
raw transcript or provider output is restored, if PIG guidance becomes
authority, if provider/command execution or safety bypass occurs, or if
continuity injection is performed.

## Validity Horizon

Valid until `v0.27.9` Memory Candidate & Continuity Consolidation begins or the
memory lifecycle policy changes.
