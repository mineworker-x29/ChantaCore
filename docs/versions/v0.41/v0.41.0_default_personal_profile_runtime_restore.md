Profile loading is not a security sandbox. The Default Personal profile is an identity/config/policy/runtime-context boundary, not OS-level isolation.

## Personal Runtime Root Contract

`PersonalRuntimeRoot` records root path, profile home path, existence status, optional writability status, and metadata-only posture.

Canonical v0.41.0 values:

- `metadata_only=True`
- `created_by_init=False`
- no root creation
- no directory creation
- no filesystem mutation

## Default Personal Profile Config Contract

`DefaultPersonalProfileConfig` represents the `default-personal` profile config.

Canonical v0.41.0 values:

- `profile_id="default-personal"`
- `profile_is_sandbox=False`
- `provider_invocation_allowed=False`
- `prompt_submission_allowed=False`
- `agent_loop_allowed=False`
- `skill_execution_allowed=False`
- `workspace_mutation_allowed=False`
- `shell_execution_allowed=False`
- `subagent_invocation_allowed=False`
- `production_certified=False`

## Default Personal Profile Paths Contract

`DefaultPersonalProfilePaths` normalizes profile paths for soul, role, domain, policy, provider config, session store, trace store, and restore context. Path object creation is metadata-only and must not create files or directories.

## Identity Files Contract

`DefaultPersonalIdentityFiles` tracks soul, role, domain, and core memory files. Missing files are reported and do not hard-fail v0.41.0. Missing identity files must be surfaced in status reports and v0.41.1 handoff. Identity files are not auto-created in v0.41.0.

## Policy Files Contract

`DefaultPersonalPolicyFiles` tracks policy, safety boundary, and restore context status. `unsafe_capabilities_closed=True` is required. Missing policy or safety boundary must not open runtime authority and blocks later runtime readiness.

## Profile Runtime State Contract

`DefaultPersonalRuntimeState` records metadata/config-read profile state only.

Canonical v0.41.0 values:

- `runtime_opened=False`
- `provider_ready=False`
- `prompt_assembly_ready=False`
- `session_store_ready=False`
- `agent_loop_ready=False`
- `skill_registry_ready=False`
- `trace_emission_ready=False`
- `user_test_ready=False`

## Profile Status Report Contract

`DefaultPersonalProfileStatusReport` reports whether root, config, identity files, policy files, and restore context are present. It keeps `runtime_opened=False`, `unsafe_capabilities_closed=True`, `next_recommended_version="v0.41.1"`, and recommends installable CLI bootstrap plus idempotent init command.

## Profile Safety Posture Contract

`DefaultPersonalProfileSafetyPosture` is deny-first and non-sandbox:

- `profile_is_sandbox=False`
- `deny_first=True`
- writes, shell, provider invocation, prompt submission, AgentLoop, skill execution, subagent invocation, autonomous loop, Dominion, and production certification are all false.
