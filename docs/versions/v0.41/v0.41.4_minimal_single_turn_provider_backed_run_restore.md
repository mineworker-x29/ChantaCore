## Scoped Provider Text Run Summary

v0.41.4 is the first v0.41 release that may send assembled prompt text to a
provider. This authority is scoped to `chanta-cli run` only.

Allowed scope:

- command is `run`
- provider config is explicit for real provider transport
- provider mode is `text_only`
- prompt is assembled through the v0.41.2 prompt assembly foundation
- provider request is text-only
- provider response is untrusted text
- session append writes only bounded user/assistant turns

## Provider Transport Contract

v0.41.4 supports text-only provider transport under the run command. Provider
responses are treated as untrusted text and are not trusted for memory,
execution, or process state.

Transport must not send:

- provider tool schemas
- active tools payload
- function calling payload
- subagent context
- arbitrary file attachments

## Mock Provider Transport Contract

The mock provider transport is deterministic and test-friendly. It returns:

```text
Mock provider response: <summary of user_input>
```

It does not access network, inspect files, use subprocess, call providers, print
secrets, or execute skills.

## OpenAI-compatible Transport Contract

OpenAI-compatible transport is bounded and may be used only from `run` when a
provider profile config explicitly sets text-only mode, base URL, and model.

It uses a configured endpoint path such as `/chat/completions` but does not
send tool schemas, active tools, function calling config, subagent context, or
file attachments. API keys may be read only from the configured environment
variable and must never be printed.

If config is missing, real provider run fails safely. Mock provider run remains
the required deterministic test path.

## Session Turn Append Contract

Successful run appends exactly two turn records:

- user turn
- assistant turn

Append path:

```text
<home>/profiles/default-personal/state/sessions/<session_id>/turns.jsonl
```

The append contract keeps:

- overwrite_allowed=False
- write_outside_home_allowed=False
- overwritten_files empty
- outside_home_paths empty
- provider response not written to memory
- provider response not trusted for execution or process state

## Assistant Response Rendering Contract

Rendered responses include a v0.41.4 single-turn marker, profile/session
metadata when available, trace notice, and untrusted provider text notice.

The trace notice must state that trace runtime remains closed until v0.41.5.
