This single integrated document is the v0.41.2 release document, restore document, prompt assembly contract document, session store contract document, and v0.41.3 handoff document.

## Prompt Assembly Summary

Prompt assembly is deterministic metadata construction. The assembled prompt preview is rendered locally and is not submitted to any model. Missing sources are recorded as warnings, not treated as runtime authority.

## Prompt Block Order Contract

Required order:

1. safety invariant
2. Soul
3. profile role
4. domain instruction
5. project instruction
6. read-only skill policy
7. restore summary
8. session context
9. user input

Project instructions and user input cannot override the safety invariant because the safety invariant is always first and explicitly closes runtime authority.

## Safety Invariant Block Contract

The safety invariant block states that v0.41.2 is preview/session only. Provider invocation, prompt submission to model, AgentLoop, skill execution, shell execution, test execution, subagent invocation, autonomous loop, Dominion runtime, and production certification are closed. The output is a preview, not a model response.

## Soul / Role / Domain Binding Contract

Soul, Role, and Domain sources are loaded from the selected default-personal profile when present. Missing identity files are recorded and the preview may continue with warnings. This binding is usable for preview only and not usable for provider-backed run.

## Project Instruction Contract

Project instruction references may include local files such as `AGENTS.md` and `README.md`. Loading is bounded and lower priority than the safety invariant. Project context must not weaken or bypass the safety block.

## Read-only Skill Policy Block Contract

The read-only skill policy block is prompt policy metadata only. v0.41.2 does not execute skills. File write, file edit, patch apply, shell execution, test execution, subagent invocation, and provider tool calling remain closed.

## Restore Summary Block Contract

The restore summary block references v0.40.9, v0.41.0, and v0.41.1 restore context when documents are present. Restore documents provide context only and do not grant runtime authority.

## Session Context Block Contract

Session context may include session id, created_at, profile id, prior turn count, and explicit no-provider-output/no-model-response statements. Prompt preview does not write session turns.

## Prompt Preview Contract

`chanta-cli prompt preview --profile default-personal --home <path> "<user input>"` renders a deterministic prompt preview. It does not call providers, submit prompts, start AgentLoop, execute skills, write a session, or emit trace runtime.

## Session Store Contract

The session store is append-only schema foundation. v0.41.2 defines `session.json`, `turns.jsonl`, and `events.jsonl` paths under the selected profile home. It does not append runtime turn records from prompt preview.

## Session New Contract

`chanta-cli session new --profile default-personal --home <path>` creates a bounded session directory under:

```text
<home>/profiles/default-personal/state/sessions/<session_id>/
```

It may create `session.json`, `turns.jsonl`, and `events.jsonl` only if missing. It does not overwrite existing files.

## Session List Contract

`chanta-cli session list --profile default-personal --home <path>` reads bounded session metadata from the selected profile home and returns session records. It is read-only and does not call providers, submit prompts, start AgentLoop, or execute skills.
