## Repository Baseline Assumptions

This restore assumes v0.40.9 through v0.41.4 are present. v0.41.5 builds on the default-personal profile store, prompt/session store, safe provider metadata, read-only skill registry metadata, and minimal single-turn provider-backed run.

## v0.41.4 Provider-backed Run Summary

v0.41.4 opened `chanta-cli run`, scoped prompt submission for run only, text-only provider invocation for run only, mock provider transport, session turn append, and assistant response rendering. v0.41.5 observes that path and records bounded trace events without changing the run authority model.

## Runtime Event Model Contract

Runtime events are PI-native metadata records with event kind, status, profile/session/run references, command name, object refs, message, redacted metadata, and explicit safety booleans.

In v0.41.5:

* shell execution is always false.
* subagent invocation is always false.
* production certification is always false.
* skill execution is always false.
* provider invocation and prompt submission are true only for scoped run provider text call events inherited from v0.41.4.

## Runtime Object Reference Contract

Runtime object refs identify profiles, sessions, runs, prompt bundles, provider configs, provider request/response metadata, skills, commands, trace stores, denials, safety gates, restore context, runtime reports, and OCEL projection candidates.

Object refs must not contain secret values. Paths are bounded to the explicit profile home or clearly represent metadata-only references.

## Trace Store Contract

The trace store is under:

```text
<home>/profiles/default-personal/state/traces/
```

The opened append-only files are:

```text
events.jsonl
runs.jsonl
denials.jsonl
reports.jsonl
```

Trace store paths are bounded to the explicit home path. Existing files are appended to, not overwritten.

## Trace Append Policy

Trace append is allowed only when bounded to the explicit profile home. The policy disallows overwrite, disallows writes outside home, creates missing trace directories under home, redacts secret-like metadata, and disables recursive trace-view events by default.

## Command Trace Envelope Contract

Command trace envelopes wrap command lifecycle events with start/complete/fail/denial event references and a trace append result. They do not execute commands. They only describe bounded command observation.

## Trace Recent Contract

`chanta-cli trace recent --profile default-personal --home <path> --limit 10` reads recent bounded trace events. The command is read-only by default and does not emit recursive trace-view noise.

It does not invoke providers, submit prompts, execute skills, execute shell, or invoke subagents.

## Trace Summary Contract

`chanta-cli trace summary --profile default-personal --home <path>` summarizes bounded trace events by event kind, status, and command name. It reports run count, denial count, provider call count, skill execution count, shell execution count, subagent invocation count, and production certification count.

For valid v0.41.5 traces, skill, shell, subagent, and production certification counts remain zero.

## Run Report Last Contract

`chanta-cli run-report last --profile default-personal --home <path>` reads trace/session information to summarize the latest run. It reports run id, session id, status, user input preview, assistant response preview, provider invocation status, prompt submission status, trace event count, and session turn count.

It does not call a provider, submit a prompt, execute skills, execute shell, or trust provider output for execution.

## Safety Check Command Contract

`chanta-cli safety check-command --profile default-personal --home <path> --command "<command text>"` inspects command text only. It never executes the command text.

Unsafe command patterns include recursive forced deletion, shell execution, arbitrary write/edit, patch application, runtime test execution, credential access, unbounded network command text, subagent invocation, Dominion runtime, and production certification.

## Denial Event Contract

Unsafe or unsupported requests produce denial records with `blocked=True` and `executed=False`. When a trace home is available, v0.41.5 appends a denial event to the bounded trace store.

## Trace Validation Contract

Trace validation inspects trace events for closed runtime capabilities. A valid v0.41.5 trace has no shell execution, workspace mutation outside bounded store append, skill execution, subagent invocation, or production certification.

Provider invocation flags are valid only on scoped run provider text call events.

## OCEL Projection Candidate Contract

v0.41.5 may create metadata-only OCEL projection candidates from runtime object/event refs. No OCEL export or import is performed.

`export_performed=False` is invariant in v0.41.5.

## Expected Test Interpretation

Pass means v0.41.5 trace/report capabilities are present and the v0.41.4 run boundary remains intact. A failed forbidden-runtime scan means the patch should be withdrawn or narrowed before commit.

## Known Limitations

v0.41.5 is observability hardening, not final release certification. The trace model is intentionally lightweight and append-only. OCEL support is metadata-only. Trace viewing commands avoid recursive trace events by default.

## Withdrawal Conditions

Withdraw v0.41.5 if it claims final v0.41.6 readiness, opens provider doctor completion, opens tool/function calling, starts a general AgentLoop, executes skills, executes shell, executes safety-check command text, invokes subagents, writes outside bounded profile/session/trace stores, prints credentials, or creates separate v0.41.5 restore/release documents.
