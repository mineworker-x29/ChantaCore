## Doctor Contract

`chanta-cli doctor` checks Python version, package import, CLI entrypoint, cwd, selected personal home, profile home, profile config, identity files, policy files, restore docs, v0.40.9 handoff doc, v0.41.0 profile runtime import, closed capabilities, and next handoff. It does not call providers, access credentials, submit prompts, start AgentLoop, execute skills, or mark v0.41.6 user-test ready.

## Init Default Personal Contract

`chanta-cli init default-personal --home <path>` may create only bounded profile directories and default files under the selected home path. It creates files only if missing, never overwrites existing files, and remains idempotent.

Allowed bootstrap shape:

```text
<home>/
profiles/
default-personal/
profile.json
soul/SOUL.md
profile/ROLE.json
profile/DOMAIN.json
profile/POLICY.json
memory/CORE_MEMORY.md
state/sessions/
state/traces/
runtime/
```

## Profile Status Contract

`chanta-cli profile status --profile default-personal --home <path>` reports profile/config/soul/role/domain/policy/core memory/sessions/traces existence. It keeps runtime, provider invocation, prompt submission, AgentLoop, skill execution, and production certification false.

## Unsupported Command Policy

Unsupported commands return blocked decisions with `executed=False`.

- run/ask: future-gated to v0.41.4.
- provider doctor: future-gated to v0.41.3.
- prompt/session: future-gated to v0.41.2.
- trace: future-gated to v0.41.5.
- write/shell/subagent/dominion/production-certify: denied or future-gated beyond v0.41.
