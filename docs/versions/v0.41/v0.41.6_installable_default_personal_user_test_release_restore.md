## Canonical User Test Flow

```powershell
py -m pip install -e .
chanta-cli --version
chanta-cli doctor
chanta-cli init default-personal --home "$env:LOCALAPPDATA\ChantaCore"
chanta-cli profile status --profile default-personal
chanta-cli provider doctor --profile default-personal --no-completion
chanta-cli run --profile default-personal "Summarize what ChantaCore is in three bullets."
chanta-cli trace recent --profile default-personal --limit 10
chanta-cli safety check-command --profile default-personal --command "Remove-Item -Recurse -Force C:\"
```

## Deterministic Mock Acceptance Flow

This flow is required for automated tests and does not require an external provider:

```powershell
chanta-cli --version
chanta-cli doctor
chanta-cli init default-personal --home "%TEMP%\chantacore-v0416-test"
chanta-cli profile status --profile default-personal --home "%TEMP%\chantacore-v0416-test"
chanta-cli provider doctor --profile default-personal --home "%TEMP%\chantacore-v0416-test" --no-completion
chanta-cli run --profile default-personal --home "%TEMP%\chantacore-v0416-test" --provider mock "Summarize what ChantaCore is in three bullets."
chanta-cli trace recent --profile default-personal --home "%TEMP%\chantacore-v0416-test" --limit 10
chanta-cli trace summary --profile default-personal --home "%TEMP%\chantacore-v0416-test"
chanta-cli run-report last --profile default-personal --home "%TEMP%\chantacore-v0416-test"
chanta-cli safety check-command --profile default-personal --home "%TEMP%\chantacore-v0416-test" --command "Remove-Item -Recurse -Force C:\"
```

Expected result: mock response rendered, two session turns appended, run trace events emitted, run report found, dangerous command denied and recorded.

## Configured Provider Acceptance Flow

This flow is supported if the user has configured an OpenAI-compatible or local provider through the default-personal provider metadata:

```powershell
chanta-cli provider doctor --profile default-personal --no-completion
chanta-cli run --profile default-personal "Summarize what ChantaCore is in three bullets."
chanta-cli trace recent --profile default-personal --limit 10
chanta-cli run-report last --profile default-personal
```

Configured-provider flow is environment-dependent and not required for CI. Provider doctor remains no-completion. Completion is allowed only inside `run`. Tool calling and function calling remain closed.

## Troubleshooting

* chanta-cli not found: run `py -m pip install -e .`.
* profile missing: run `chanta-cli init default-personal`.
* provider not configured: use `--provider mock` or configure provider metadata.
* run fails with provider unavailable: start the configured provider or use mock flow.
* trace recent empty: run the mock provider flow first.
* safety check-command does not emit denial trace: verify the same `--home` path is used.
* permission/path problem on Windows: use `$env:LOCALAPPDATA\ChantaCore` or another writable explicit home.

Windows user guide command block:

