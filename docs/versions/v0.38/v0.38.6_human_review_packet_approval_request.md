# v0.38.6 Human Review Packet & Approval Request Contract

Parent track: Bounded Repair Proposal Loop.

v0.38.6 builds on v0.38.5 Repair Proposal Safety & Static Validation, v0.38.4 Proposed Diff / Code Hunk Metadata Generation, v0.38.3 Repair Scope Planner & Change Intent Model, v0.38.2 Read-only Sandbox Source Context Snapshot, v0.38.1 Repair Proposal Evidence Contract, and v0.38.0 Bounded Repair Proposal Boundary Foundation.

## Definition

v0.38.6 creates human review packet and approval request contract metadata only. It packages evidence, source context summaries, repair scope, proposed patch metadata, and safety validation metadata into a bounded in-memory review packet preview.

A human review packet is not human approval. An approval request contract is not approval. Approval request metadata is not approval capture. A review checklist is not approval. Review questions do not collect responses in v0.38.6. Apply preconditions are not apply permission. Future v0.39 apply contract input is not sandbox repair apply. Future loop trial input is not loop execution. Future CLI surface input is not UI or runtime execution.

The layer defines:

## Policy

The human review policy allows review packet metadata, approval request contract metadata, checklist metadata, question metadata, apply precondition metadata, future loop trial input metadata, future CLI surface input metadata, and future v0.39 apply contract input metadata.

The policy blocks approval capture, approval grant, apply permission, review packet file write, review packet external send, UI runtime, source read, source write, patch file write, file edit, patch application, apply_patch, git apply, repair execution, test execution, subprocess, shell, dependency install, network access, model provider invocation, external agent execution, and Dominion runtime.

Every packet must include an evidence summary, patch summary, safety summary, do-nothing review comparison, approval-not-granted state, apply-not-allowed state, human-review-required-before-any-apply state, and future apply preconditions as metadata only.

v0.38.6 does not collect human approval. It does not grant approval. It does not grant apply permission. It does not write review packet files. It does not send review packets externally. It does not create UI runtime. It does not read source files. It does not read sandbox source files. It does not read live workspace files. It does not read reference corpus files. It does not write files. It does not write patch files. It does not edit files. It does not apply patches. It does not call apply_patch. It does not call git apply. It does not execute repair. It does not execute tests. It does not run subprocess. It does not execute shell/subprocess/commands. It does not install dependencies. It does not access network. It does not invoke model providers. It does not invoke Codex CLI, Claude Code, or external agents. It does not implement Dominion runtime. It does not certify production readiness.

Digestion-first policy is recorded as metadata. Dominion runtime, external agent execution, infinite agent loops, recursive self-invocation, automatic repair loops, repair execution, model provider invocation, and tool execution remain blocked/future-gated metadata only. No independent autonomous agent runtime is introduced. Human review remains mandatory before any apply.

Reference corpus entries under references/OpenCode, references/Hermes, and references/OpenClaw are not executed, imported, installed, tested, or used as live adapters. Reference patterns may appear only as digested/adapted/rejected/future-gated metadata.
