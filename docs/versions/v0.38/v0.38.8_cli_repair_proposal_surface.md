# v0.38.8 CLI Repair Proposal Surface

Parent track: Bounded Repair Proposal Loop.

v0.38.8 builds on v0.38.7 Bounded Repair Proposal Loop Trial, v0.38.6 Human Review Packet & Approval Request Contract, v0.38.5 Repair Proposal Safety & Static Validation, v0.38.4 Proposed Diff / Code Hunk Metadata Generation, v0.38.3 Repair Scope Planner & Change Intent Model, v0.38.2 Read-only Sandbox Source Context Snapshot, v0.38.1 Repair Proposal Evidence Contract, and v0.38.0 Bounded Repair Proposal Boundary Foundation.

## Definition

The CLI repair proposal surface is a bounded metadata surface over existing v0.38 repair proposal artifacts. It parses CLI-like argv into structured invocation metadata, evaluates the command against a deny-by-default surface policy, and returns bounded in-memory preview output.

The CLI surface is not a shell. CLI argv parsing is not OS command execution. CLI command dispatch is not arbitrary command execution. CLI preview is not file export. `repair-patch-metadata-preview` is not patch application. `repair-safety-preview` is not safety validation execution. `repair-review-packet-preview` is not human approval. `repair-loop-packet-preview` is not loop execution. Handoff preview is not apply permission.

## Safe Commands

Safe preview commands are:

`repair-proposal-help`, `repair-proposal-status`, `repair-boundary-preview`, `repair-evidence-preview`, `repair-source-context-preview`, `repair-scope-preview`, `repair-patch-metadata-preview`, `repair-safety-preview`, `repair-review-packet-preview`, `repair-loop-packet-preview`, `repair-proposal-bundle-preview`, `repair-do-nothing-preview`, `repair-handoff-preview`, and `repair-no-op`.

Each safe command renders in-memory metadata only.

Examples: `shell`, `bash`, `powershell`, `cmd`, `python`, `pytest`, `npm`, `pip`, `curl`, `wget`, `git`, `git-apply`, `git apply`, `apply-patch`, `apply_patch`, `repair-apply`, `auto-repair`, `retry-loop`, `multi-cycle-loop`, `approve`, `capture-approval`, `run-codex`, `run-claude-code`, `run-opencode`, `run-hermes`, `run-openclaw`, `provider-call`, `model-call`, `external-agent`, `dominion`, `infinite-loop`, `recursive-agent`, and `unknown`.

Denied command handling is a safe outcome. It does not execute remediation.

## Models

`RepairCLIFlagSet` records CLI surface readiness while preserving all runtime, shell, subprocess, command, apply, test, repair, approval, export, send, UI, source-read, source-write, new-patch-generation, model, external-agent, Dominion, and production flags as false.

`RepairCLISourceRef` records supplied metadata references. It does not read files, write files, send packets, apply patches, or execute repair.

`RepairCLIArgumentSpec` bounds argument metadata. It is not execution.

`RepairCLICommandSpec` describes safe preview commands and denied command metadata. Dispatch targets are static renderer metadata only, not dynamic shell/import/runtime execution.

`RepairCLISurfacePolicy` allows only safe preview commands and blocks file export, external send, UI runtime, shell, subprocess, arbitrary command execution, dependency installation, network, source read/write, patch file write, file edit, patch application, apply_patch, git apply, repair execution, test execution, new patch generation, approval capture/grant, apply permission, model providers, external agents, and Dominion runtime.

`RepairCLISurface` is the in-memory registry and policy container. It is not a runtime shell.

`RepairCLIInvocation` stores argv as metadata only. argv is never passed to shell or subprocess.

`RepairCLIInvocationDecision` permits preview only for safe commands. All runtime booleans remain false.

`RepairCLIDeniedCommand` represents safe denial. It lists safe alternatives.

`RepairCLICommandResult` and `RepairCLIOutput` are bounded in-memory preview envelopes. They do not write files, export packets, send externally, invoke UI runtime, execute commands, read source, generate new patch metadata, apply patches, run tests, execute repair, capture approval, invoke models, invoke external agents, invoke Dominion runtime, certify production readiness, or become execution-ready.

`RepairCLIReport`, `RepairCLIRunPreview`, `RepairCLINoRuntimeGuarantee`, and `V0388ReadinessReport` preserve the same no-runtime guarantees and may mark future v0.38.9 consolidation input and future v0.39 handoff metadata only.

## Digestion And Reference Policy

The digestion-first policy is recorded as metadata. Dominion runtime, external agent execution, infinite agent loops, recursive self-invocation, automatic repair loops, repair execution, model provider invocation, and tool execution remain blocked.

Reference corpus patterns may be represented only as digested, adapted, rejected, or future-gated CLI metadata. v0.38.8 does not execute, import, install, test, run, or read OpenCode, Hermes, OpenClaw, or other reference code as repair source context.

## Acceptance Criteria

The release is acceptable when safe commands parse to preview-only decisions, unsafe and unknown commands deny safely, command results remain preview-only, CLI output remains bounded and in memory, no runtime booleans are enabled, no source-read or patch-generation helpers are called, and focused plus bounded regression tests pass.

## Withdrawal Conditions

Withdraw v0.38.8 readiness if any helper passes argv to shell/subprocess, executes command names, writes or exports files, sends externally, invokes UI runtime, reads source, generates new patch metadata, applies patches, runs tests, captures or grants approval, invokes providers/external agents, implements Dominion runtime, or sets execution readiness true.

## Handoff

v0.38.8 hands off only metadata to v0.38.9 Bounded Repair Proposal Loop Consolidation and future v0.39 human-approved sandbox repair apply work. These handoffs are not consolidation execution or sandbox apply.
