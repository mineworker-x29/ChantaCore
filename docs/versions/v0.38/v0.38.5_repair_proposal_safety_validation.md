Parent track: Bounded Repair Proposal Loop.

## Definition

v0.38.5 creates repair proposal safety/static validation metadata only. It consumes existing proposed patch metadata, scope metadata, source context metadata, evidence metadata, and boundary metadata. It may scan bounded in-memory proposed patch metadata text, including proposed diff text and proposed code hunk text already supplied by v0.38.4.

Static validation is not test execution. Safety pass is not apply permission. Safety pass is not correctness proof. Safety pass is not production certification. Future human review packet input is not approval. Future loop trial input is not loop execution.

## Policy

The safety policy allows static validation, target validation, content validation, boundary violation scan, unsafe operation detection, safety risk assessment, do-nothing safety comparison, future human review packet input, and future loop trial input.

The policy blocks source file read, sandbox source read, live workspace read, source file write, sandbox source write, patch file write, file edit, patch application, apply_patch, git apply, repair execution, test execution, subprocess, shell, dependency install, network access, model provider invocation, external agent execution, and Dominion runtime.

High or critical risk must block or require review. Critical unsafe operations block future loop trial input. Missing evidence/context/scope, missing do-nothing comparison, missing human review requirement, do-nothing preferred states, and boundary violations must block or require review.
