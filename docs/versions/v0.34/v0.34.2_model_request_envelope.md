# v0.34.2 Model Request Envelope

Parent track: Controlled Model Invocation Boundary.

Related foundations:
- v0.34.1 Provider Profile & Invocation Policy.
- v0.34.0 Controlled Model Invocation Boundary Foundation.
- v0.33.2 Prompt Assembly Pipeline.

v0.34.2 defines Model Request Envelope metadata and validation artifacts only. A request envelope is not model invocation. Provider-ready request metadata is not provider invocation. Provider binding is not provider call. Prompt payload ref is not raw prompt persistence.

## Explicit Distinctions

- ModelRequestEnvelope is not provider invocation, model invocation, or execution.
- ModelRequestEnvelopeInput is not a provider invocation request.
- ModelPromptPayloadRef is bounded/redacted and is not raw prompt persistence.
- Provider binding is metadata only and is not provider call.
- Token budget is not provider call.
- Stop policy is not execution.
- Timeout policy is not network access.
- Safety constraints are not runtime enforcement by themselves.

## Request Envelope Surface

The envelope records:
- prompt payload format and source refs
- bounded/redacted prompt preview
- token budget
- stop policy
- timeout policy
- output policy
- safety constraints
- provider profile binding metadata
- validation findings and reports

Payload formats are representations only: message_list, plain_text_prompt, structured_blocks, prompt_ref_only, redacted_preview_only, and test_fixture. Raw unbounded prompt payloads are not allowed.

Request sources are metadata refs only: v0.34.1 provider profile, v0.34.0 boundary, v0.33.2 PromptAssemblyOutput, session runtime, step runner, manual spec, test fixture, and safe reference-context refs.

Data sensitivity marks secret-like, credential-like, token-like, and unknown data as blocked or requiring redaction. Request envelopes must not contain secret/credential/token values.

## Policy Defaults

Token budget blocks unbounded prompts and responses.

Stop policy is bounded by sequence count and sequence length.

Timeout policy is metadata only. Streaming and retries remain false, and max_retries remains 0.

Output policy blocks raw response persistence, tool calls, function calls, patch output, command output, and untrusted actions. Response envelope, response sanitizer, and action quarantine are required.

Safety constraints require provider profile validation, prompt redaction, token budget, output policy, no tool calls, no function calls, no recursive calls, and no raw prompt persistence.

## Flow

`PromptAssemblyOutput / ProviderProfileDescriptor / ProviderInvocationPolicy / ControlledModelInvocationBoundary -> ModelPromptPayloadRef -> ModelRequestEnvelope -> ValidationReport -> V0342ReadinessReport -> v0.34.3 Model Response Envelope or v0.34.4 Existing Provider Boundary Adapter`

The conversion from PromptAssemblyOutput uses supplied in-memory prompt metadata only. It creates a bounded redacted preview and source refs. It does not read files, fetch refs, persist raw prompts, call providers, invoke existing provider boundary, or use network.

## Reference Corpus Policy

references/OpenCode, references/Hermes, and references/OpenClaw remain safe reference-context assets only. v0.34.2 does not execute them, import them, install their dependencies, run their tests or package scripts, or inject their content into request envelopes unless that content is already present as safe, bounded, redacted PromptAssemblyOutput metadata. Reference path refs may appear only as safe metadata refs.

## Hard Prohibitions

v0.34.2 does not call providers. It does not invoke existing provider boundary. It does not invoke provider SDKs. It does not use network. It does not read credentials. It does not read `.env`. It does not read secrets. It does not read env vars for credentials. It does not execute agent steps. It does not execute tools. It does not execute shell/subprocess/commands. It does not write/edit/patch. It does not persist raw prompts. It does not persist raw model request payloads. It does not persist traces. It does not implement UI runtime. It does not grant authority.

Tool calls, function calls, and recursive calls remain disabled.

## Acceptance Criteria

v0.34.2 is acceptable only if ModelRequestEnvelope, ModelPromptPayloadRef, token budget, stop policy, timeout policy, output policy, safety constraints, provider binding, validation report, run preview, no-invocation guarantee, and readiness report exist while every unsafe readiness flag remains false.

## Withdrawal Conditions

Withdraw v0.34.2 readiness if any model/provider/existing-boundary invocation, provider SDK import, network access, credential/env/secret read, endpoint construction, agent/tool execution, tool/function/recursive call, shell/subprocess/command execution, workspace write/edit/patch, reference execution/import/install, dependency install, raw prompt persistence, persistent trace write, UI runtime, external control, authority grant, or unsafe readiness flag is introduced.
