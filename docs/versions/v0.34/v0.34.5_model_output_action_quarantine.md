# v0.34.5 Model Output Action Quarantine

Parent track: Controlled Model Invocation Boundary.

Related foundations:
- v0.34.4 Existing Provider Boundary Adapter.
- v0.34.3 Model Response Envelope & Sanitizer.

v0.34.5 defines Model Output Action Quarantine metadata and validation artifacts only. Model output remains untrusted. An action candidate is not action execution. A quarantine decision is not execution permission. A quarantine packet is not an action queue. A safe route is not execution.

## Explicit Distinctions

- ModelResponseActionSignal is a v0.34.3 detection artifact.
- ModelOutputActionCandidate is a v0.34.5 quarantine artifact, not an action.
- Quarantine packet is not an action queue.
- Safe route is non-executing handoff metadata only.
- Patch-like candidate is future-track only and is not a patch proposal in v0.34.5.
- Final response, ask-user, and no-op candidates may be allowed only as non-executing output routes.

## Response Trust Model

Model output remains untrusted even after sanitization. Sanitized output can be reduced into candidates and route metadata, but it cannot execute tools, write files, call providers, inspect the workspace, or override policy.

## Candidate Extraction

The extraction layer consumes v0.34.3 ModelResponseEnvelope, ModelSanitizedResponsePayload, or ModelResponseActionSignal metadata. It emits bounded ModelOutputActionCandidate records with candidate kind, preview, source refs, trust level, risk kinds, signal strength, and proposed route.

The extraction policy may allow final response, ask-user, no-op, and safe workspace inspection candidates as metadata. It blocks patch, command, provider-reinvoke, credential, and external-harness candidate allowances.

## Risk And Routes

High-risk candidates must block. This includes shell command, command execution, workspace write, code edit, patch, patch application, dependency install, provider reinvoke, network access, credential access, secret read, external harness execution, OpenCode/Hermes/OpenClaw execution, reference code execution, browser/RPA gateway, and authority grant candidates.

Safe routes are non-executing. Future safe workspace inspection and future agent step runner routes are handoff metadata only. They do not execute v0.33.5 safe tools in v0.34.5.

Patch-like candidates are blocked/future-track only. Patch proposal generation and patch application are not part of v0.34.5.

## Reference Corpus Policy

references/OpenCode, references/Hermes, and references/OpenClaw remain safe reference-context assets only. Model output suggesting OpenCode, Hermes, OpenClaw, reference import, reference install, reference tests, or reference code execution is blocked. Quarantine does not read reference files.

## Flow

`ModelResponseEnvelope / SanitizedPayload / ActionSignals -> ActionCandidates -> RiskAssessment -> QuarantineDecision -> BlockedRecords/SafeRoutes -> QuarantinePacket -> ValidationReport -> V0345ReadinessReport -> v0.34.6 Agent Step Runner Model Integration`

v0.34.7 Model Invocation OCEL Trace Packet is also a design-stage handoff target.

## Hard Prohibitions

v0.34.5 does not execute actions. It does not execute tools. It does not inspect workspace files. It does not call providers. It does not invoke existing provider boundary. It does not use network. It does not read credentials. It does not read secrets. It does not execute agent steps. It does not execute shell/subprocess/commands. It does not write/edit/patch. It does not generate patch proposals. It does not apply patches. It does not persist raw responses or raw model output. It does not persist traces. It does not implement UI runtime. It does not grant authority.

## Acceptance Criteria

v0.34.5 is acceptable only if candidate extraction, risk assessment, route policy, quarantine decisions, blocked records, safe routes, quarantine packet, validation report, no-execution guarantee, and readiness report exist while every execution/tool/write/patch/provider/network/credential/agent/persistence/UI/control readiness flag remains false.

## Withdrawal Conditions

Withdraw v0.34.5 readiness if any action execution, tool execution, workspace inspection execution, provider invocation, existing boundary invocation, network access, credential/secret read, agent step execution, shell/subprocess/command execution, workspace write/edit/patch, patch proposal generation, patch application, reference execution/import/install, dependency install, raw response persistence, raw model output persistence, persistent trace write, UI runtime, external control, authority grant, or unsafe readiness flag is introduced.

## Handoff

v0.34.5 hands off only to v0.34.6 Agent Step Runner Model Integration and v0.34.7 Model Invocation OCEL Trace Packet as design-stage work. It does not enable action execution.
