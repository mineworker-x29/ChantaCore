## Boundary Definition

Controlled model invocation boundary is not model invocation. It is a policy and contract layer that defines how future stages may decide whether a model provider can be called through a strict gate.

Controlled model invocation is not arbitrary provider invocation, arbitrary network access, credential access, autonomous runtime, general tool execution, write/edit/patch, or external harness execution.

Existing provider/chat_service boundary refs are string/id references only in v0.34.0. An existing provider boundary ref is not a provider call. It does not import a provider SDK, read credentials, construct endpoints, or perform network access.

Credential posture is not credential access. Network posture is not network access. Provider surface policy is not provider invocation.

v0.34.0 creates no new provider SDK adapter, no credential reader, and no network adapter.

## Reference Corpus Policy

references/OpenCode and references/Hermes remain safe reference-context assets only. They may be inspected only through v0.33.5 bounded safe read-only path policy in later stages that explicitly route through that policy. v0.34.0 does not execute OpenCode, Hermes, or OpenClaw, does not import reference code, does not install dependencies, and does not run reference tests.

## Taxonomies

Track taxonomy covers v0.34.0 through v0.34.9 from boundary foundation to consolidation.

Surface taxonomy separates safe design-stage refs from prohibited surfaces: provider SDK direct, provider network endpoint, credential store, API key value, workspace write, code edit, patch application, shell command, external harness, reference code, persistent trace, and UI runtime.

Capability taxonomy separates boundary definition and future handoff from prohibited capabilities: call existing provider boundary, direct provider SDK call, provider network access, credential read, secret read, agent step execution, tool execution, workspace write, code edit, patch application, shell execution, external harness execution, and reference code execution.

Risk taxonomy records provider invocation, arbitrary network access, credential exposure, secret read, provider SDK bypass, endpoint construction, prompt leak, raw model output persistence, model output action authority, recursive model call, autonomous loop, general tool execution, workspace write, command execution, reference execution, external harness execution, persistent trace write, external control, and authority grant risks.

Decision taxonomy allows design-stage boundary definition and handoff only. Allow decisions are not model invocation.

Boundary status and readiness taxonomy distinguish boundary contract readiness from invocation readiness. `boundary_contract_ready` is not provider invocation readiness.

Provider boundary posture may be existing-boundary-ref-only or future-gated. Direct provider, provider SDK, and network endpoint postures are blocked.

Credential posture records no credential access, credential ref only, or future-gated existing boundary management. Direct secret/env reads are blocked.

Network posture records no network access, future-gated existing boundary management, arbitrary network blocked, and provider endpoint blocked.
