# v0.34.9 Controlled Model Invocation Boundary Consolidation

Foundation release name: Controlled Model Invocation Boundary v1.

Parent track: Controlled Model Invocation Boundary.

v0.34.9 is consolidation only. It records release readiness, coverage, known risks, explicit gaps, and the v0.35 design-stage handoff. Controlled Model Invocation Boundary v1 is a bounded foundation, not production runtime.

## Relationship To Prior Stages

v0.34.9 consolidates v0.34.0 through v0.34.8 and inherits the v0.30, v0.31, v0.32, v0.33, and v0.34 gates.

- v0.34.0 defined the controlled model invocation boundary.
- v0.34.1 defined provider profile metadata and invocation policy.
- v0.34.2 defined model request envelopes.
- v0.34.3 defined model response envelopes, sanitizer behavior, risk signals, and action-signal detection.
- v0.34.4 defined the existing provider boundary adapter route.
- v0.34.5 defined model output action quarantine.
- v0.34.6 integrated bounded model-backed steps with the v0.33.6 runner.
- v0.34.7 defined returned/in-memory model invocation OCEL-style trace packets.
- v0.34.8 defined bounded CLI-facing model-backed handlers.
- v0.33.9 remains the prior Internal General Agent Runtime MVP v1 consolidation layer.

## What v0.34.x Accomplished

v0.34.x enables controlled model-backed flow only through existing-boundary adapter and policy gates. Controlled model invocation is only through v0.34.4. Bounded model-backed step execution is only through v0.34.6. Model invocation trace packet creation is only through v0.34.7 and is not persistence. CLI model-backed surface is only through v0.34.8 and is not shell.

The track now has release flags, a snapshot, a capability matrix, coverage matrices, boundary/risk/gap registers, a release manifest, an audit trail, a v0.35 handoff packet, and a final v0.34 consolidation report.

## What v0.34.x Did Not Accomplish

v0.34.x does not enable direct provider SDK invocation. It does not enable direct network access. It does not enable credential, env, or secret reads. It does not enable autonomous general agent runtime, general tool execution, shell/subprocess/command execution, workspace write, code edit, patch proposal, or patch application.

v0.34.x does not execute OpenCode, Hermes, or OpenClaw. It does not import reference code, install reference dependencies, or run reference tests/package scripts. It does not persist trace files, database records, or runtime logs. It does not enable external trace sinks, UI runtime, external control, authority grants, or D4-D9.

## Reference Corpus Policy

references/OpenCode and references/Hermes remain safe reference-context assets only. Reference paths may appear as safe refs or safe inspection summaries from earlier bounded layers. v0.34.9 does not inspect, read, execute, import, install, or test reference code.

## Consolidation Artifacts

The consolidation module defines:

- Release flags for Controlled Model Invocation Boundary v1.
- A controlled model invocation snapshot covering v0.34.0 through v0.34.8.
- A capability matrix distinguishing controlled/bounded capabilities from prohibited capabilities.
- Coverage records for each v0.34 stage.
- Boundary, risk, and gap registers.
- A release manifest.
- An audit trail.
- A V035HandoffPacket.
- A V034ConsolidationReport.

Coverage matrices are not certification. The audit trail is metadata, not runtime audit execution. The release manifest is not a production release.

## Readiness Rules

ready_for_controlled_model_invocation_boundary_v1 may be true. ready_for_controlled_existing_boundary_invocation may be true only through v0.34.4. ready_for_bounded_model_backed_step_execution may be true only through v0.34.6. ready_for_cli_model_backed_surface may be true only through v0.34.8. ready_for_model_invocation_trace_packet_creation may be true only through v0.34.7.

ready_for_execution remains false. ready_for_direct_provider_invocation remains false. ready_for_provider_sdk_invocation remains false. ready_for_direct_network_access remains false. ready_for_network_access remains false. ready_for_credential_access remains false. ready_for_secret_read remains false. ready_for_general_agent_execution remains false. ready_for_autonomous_agent_runtime remains false. ready_for_general_tool_execution remains false. ready_for_workspace_write, ready_for_code_edit, ready_for_patch_proposal, and ready_for_patch_application remain false. ready_for_persistent_trace_write and ready_for_external_trace_sink remain false. production_certified remains false.

D4-D9 are future-track only.

## V035 Handoff

The recommended v0.35 track is Controlled Patch Proposal Layer. v0.35 should generate patch proposal artifacts only. v0.35 should not apply patches. Patch application belongs to a later human-approved sandbox track, likely v0.36.

The v0.35 handoff is design-stage only. It is not implementation, not production runtime, and not workspace write readiness.

## Hard Prohibitions

v0.34.9 must not add provider invocation behavior, import provider SDKs, use requests/httpx/urllib/aiohttp/socket, read credentials/secrets/env, execute shell/subprocess/commands, use eval/exec, execute tools, execute unquarantined actions, run autonomous loops, write/edit/patch files, generate patch proposals, apply patches, execute/import/install references, persist traces, write database/log records, implement UI runtime, mutate registry/memory, or grant authority.

## Acceptance Criteria

- Controlled Model Invocation Boundary v1 snapshot exists.
- Release flags preserve all unsafe readiness flags false.
- Capability matrix lists bounded v0.34 features and prohibited unsafe surfaces.
- Coverage exists for v0.34.0 through v0.34.8.
- Boundary, risk, and gap registers include prohibited unsafe surfaces.
- V035 handoff names Controlled Patch Proposal Layer as design-stage only.
- V034 consolidation report keeps ready_for_execution and production_certified false.

## Withdrawal Conditions

This consolidation should be withdrawn if a direct provider SDK/network/credential path is introduced, if shell/subprocess/command execution is introduced, if write/edit/patch or patch proposal is enabled in v0.34.9, if traces are persisted, if autonomous/general tool execution is introduced, if references are executed/imported/installed, or if any unsafe readiness flag or production_certified becomes true.
