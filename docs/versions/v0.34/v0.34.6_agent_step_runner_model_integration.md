# v0.34.6 Agent Step Runner Model Integration

Parent track: Controlled Model Invocation Boundary.

v0.34.6 connects the v0.34 controlled model invocation artifacts to the v0.33.6 Agent Step Runner MVP. It builds on v0.34.5 Model Output Action Quarantine, v0.34.4 Existing Provider Boundary Adapter, v0.34.3 Model Response Envelope & Sanitizer, and the v0.33.6 bounded step runner.

## Definition

A bounded model-backed agent step is a single v0.34.6 integration artifact that can route already sanitized and quarantined model output into the v0.33.6 Agent Step Runner. It is not autonomous general agent runtime. It is not direct provider invocation. It is not general tool execution. It is not write/edit/patch execution.

The model output trust policy remains conservative: model output must be wrapped by v0.34.3 and quarantined by v0.34.5 before step integration. A quarantined safe route is not action execution by itself. v0.34.6 may execute one bounded model-backed step only by converting that route into a v0.33.6 `AgentStepInput` and calling the bounded v0.33.6 runner.

## Boundary Rules

v0.34.6 may execute one bounded model-backed step. It does not enable autonomous general agent runtime. It does not directly call providers. It does not import provider SDKs. It does not use direct network libraries. It does not read credentials or secrets. It does not execute unquarantined model output. It does not execute general tools. It does not execute shell/subprocess/commands. It does not write/edit/patch. It does not generate patch proposals. It does not apply patches. It does not execute OpenCode, Hermes, or OpenClaw.

Provider/model invocation is allowed only through the v0.34.4 existing-boundary adapter when explicitly supplied and policy-approved. That adapter remains distinct from direct provider invocation and direct SDK/network integration.

Safe workspace inspection is allowed only through the v0.33.6/v0.33.5 safe path. The v0.33.6 safe tool bridge remains distinct from general tool execution. Reference path mentions may be inspected only through the existing safe read-only policy. Reference execution, import, dependency install, test execution, and package-script execution remain blocked.

## Quarantine Bridge

The quarantine bridge selects safe v0.34.5 routes:

- final response
- ask user
- no-op
- safe workspace inspection

Unsafe routes, missing quarantine packets, missing response envelopes, missing runners, and policy violations produce blocked, review, no-op, or future-gated outputs. Patch-like candidates remain blocked/future-track only. They are not patch proposals in v0.34.6.

## Step Runner Bridge

The step runner bridge converts the selected safe route into a v0.33.6-compatible supplied model output. The v0.33.6 runner then decides and performs only its bounded behavior: non-mutating final response, ask-user, no-op, or v0.33.5 safe workspace inspection. This is not a general action execution bridge.

## Taxonomies

v0.34.6 defines integration mode, status, source, decision, route, risk, outcome, and readiness taxonomies. Unknown, unsafe, missing, and policy-violating values block or future-gate.

The risk taxonomy includes unquarantined model output, unsafe action candidate, general tool execution, workspace write, code edit, patch proposal, patch application, shell/command execution, direct provider invocation, provider SDK bypass, direct network access, credential/secret access, reference/external harness execution, autonomous loop, and raw persistence risks.

## Models

The release defines model-backed step flags, source refs, integration policy, input, plan, decision, quarantine bridge, runner bridge, execution record, output, report, run preview, no-external-side-effect guarantee, and readiness report.

`ModelBackedStepOutput` is not persistent storage. `ModelBackedStepExecutionRecord` is not runtime log persistence. No raw prompt or raw response is persisted.

## Flow

`ModelRequestEnvelope / ExistingProviderBoundaryInvocationResult / ModelResponseEnvelope / QuarantinePacket -> ModelBackedStepInput -> Plan -> Decision -> QuarantineBridge -> v0.33.6 AgentStepRunner -> ModelBackedStepOutput -> V0346ReadinessReport -> v0.34.7 Model Invocation OCEL Trace Packet`

## Readiness

`ready_for_bounded_model_backed_step_execution` may be true. `ready_for_agent_step_runner_model_integration` may be true. `ready_for_v0347_model_invocation_ocel_trace_packet` and `ready_for_v0348_cli_model_backed_agent_step_surface` mean design-stage handoff only.

`ready_for_execution` remains false. `ready_for_general_agent_execution` remains false. `ready_for_autonomous_agent_runtime` remains false. `ready_for_direct_provider_invocation` remains false. `ready_for_provider_sdk_invocation` remains false. `ready_for_general_tool_execution` remains false. `ready_for_unquarantined_action_execution` remains false. `ready_for_workspace_write`, `ready_for_code_edit`, `ready_for_patch_proposal`, and `ready_for_patch_application` remain false. `production_certified` remains false.

## Acceptance Criteria

v0.34.6 is acceptable only if bounded model-backed step integration, quarantine bridge, v0.33.6 runner bridge, decisions, execution record, output, report, no-side-effect guarantee, and readiness report exist while direct provider SDK/network/credential/general tool/unquarantined action/write/patch/general runtime readiness stays false.

## Withdrawal Conditions

Withdraw v0.34.6 readiness if any direct provider SDK/network/credential path, general agent runtime, autonomous loop, general tool execution, unquarantined action execution, shell/subprocess/command execution, workspace write/edit/patch path, patch proposal generation, patch application, reference execution/import/install, raw prompt/response persistence, persistent trace write, UI runtime, external control, authority grant, or unsafe readiness flag is introduced.

## Handoff

v0.34.6 hands off only to v0.34.7 Model Invocation OCEL Trace Packet and v0.34.8 CLI Model-backed Agent Step Surface. It does not certify production execution.
