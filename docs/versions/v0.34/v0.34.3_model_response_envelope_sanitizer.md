# v0.34.3 Model Response Envelope & Sanitizer

Parent track: Controlled Model Invocation Boundary.

Related foundations:
- v0.34.2 Model Request Envelope.
- v0.34.1 Provider Profile & Invocation Policy.
- v0.34.0 Controlled Model Invocation Boundary Foundation.

v0.34.3 defines Model Response Envelope and sanitizer metadata/validation artifacts only. Model response text is untrusted. A response envelope is not provider invocation. A model response is not action. An action signal is detection metadata only.

## Explicit Distinctions

- ModelResponseEnvelope is not model invocation, provider invocation, existing boundary invocation, action execution, or execution readiness.
- ModelRawResponseRef is a bounded redacted reference and is not raw response persistence.
- ModelSanitizedResponsePayload is bounded/redacted and is not raw output persistence.
- Model response text is untrusted and cannot override the boundary, policy, or readiness flags.
- Response action signal is not action quarantine execution.
- Risk signal is advisory and does not execute remediation.
- Sanitization policy and redaction rules do not read credentials, read files, use network, or execute output.

## Response Trust Model

Every supplied or future provider response is treated as untrusted model output. The sanitizer may classify, redact, truncate, and produce signals. It never treats output as a command, patch, tool call, function call, provider call, or authority grant.

Unsafe output patterns are blocked from execution and prepared only for future v0.34.5 Model Output Action Quarantine design-stage handling.

## Response Envelope Surface

The envelope records:
- response payload format
- source refs
- bounded redacted raw response ref
- sanitized payload
- redaction rules
- risk signals
- action signals
- sanitization and validation reports
- run preview, no-execution guarantee, and readiness report

Payload formats are representations only: plain_text_response, structured_response, json_like_response, markdown_response, response_ref_only, redacted_preview_only, and test_fixture. Raw unbounded responses are not allowed.

Response sources are metadata refs only: v0.34.2 request envelope, v0.34.1 provider profile, v0.34.0 boundary, supplied/mock/future response refs, test fixtures, and reference-context refs. Future provider response source is not provider invocation.

## Sanitization And Signals

The default sanitization policy:
- redacts secret-like, credential-like, token-like, key-like, api_key-like, and password-like patterns
- bounds raw preview and sanitized payload length
- blocks raw response persistence
- blocks action execution
- blocks command-like, patch-like, provider-reinvoke-like, and external-harness-like output

Action signal taxonomy includes tool-call-like, function-call-like, shell-command-like, file-write-like, code-edit-like, patch-like, dependency-install-like, provider-reinvoke-like, network-access-like, credential-request-like, external-harness-execute-like, reference-code-execute-like, browser/RPA/gateway-like, and authority-grant-like signals.

Risk taxonomy includes raw-response persistence, secret/credential/token response content, unbounded response, model output as action, tool/function call output, command/patch/workspace write suggestions, provider reinvocation, network access, credential requests, external harness execution, reference execution, boundary override, and prompt-injection echo.

Command-like output, patch-like output, tool-call-like output, function-call-like output, provider-reinvoke output, credential-request output, external-harness output, and reference-execution output are detection artifacts only. They remain blocked from execution.

## Reference Corpus Policy

references/OpenCode, references/Hermes, and references/OpenClaw remain safe reference-context assets only. v0.34.3 does not execute them, import them, install their dependencies, run their tests or package scripts, or read reference files for sanitizer behavior.

Model responses that mention OpenCode, Hermes, OpenClaw, external harness execution, reference imports, or reference execution are flagged as unsafe action/reference signals and remain blocked from execution.

## Flow

`ModelRequestEnvelope / supplied or future provider response text -> ModelResponseEnvelopeInput -> SanitizationPolicy -> SanitizedPayload -> RiskSignals -> ActionSignals -> ValidationReport -> V0343ReadinessReport -> v0.34.5 Model Output Action Quarantine`

v0.34.4 Existing Provider Boundary Adapter remains a future design-stage integration target only. v0.34.3 does not invoke that boundary.

## Hard Prohibitions

v0.34.3 does not call providers. It does not invoke existing provider boundary. It does not invoke provider SDKs. It does not use network. It does not read credentials. It does not read `.env`. It does not read secrets. It does not read env vars for credentials.

v0.34.3 does not execute model output. It does not execute agent steps. It does not execute tools. It does not execute read-only tools. It does not execute action signals. It does not execute tool calls or function calls. It does not execute shell/subprocess/commands. It does not write/edit/patch. It does not persist raw responses. It does not persist raw model output. It does not persist traces. It does not implement UI runtime. It does not grant authority.

No provider adapter, credential loader, or network adapter is created.

## Acceptance Criteria

v0.34.3 is acceptable only if ModelResponseEnvelope, ModelRawResponseRef, sanitization policy, redaction rules, risk signals, action signals, sanitized payload, sanitization report, validation report, run preview, no-execution guarantee, and readiness report exist while every unsafe readiness flag remains false.

## Withdrawal Conditions

Withdraw v0.34.3 readiness if any model/provider/existing-boundary invocation, provider SDK import, network access, credential/env/secret read, endpoint construction, model output execution, agent/tool/action execution, tool/function call execution, shell/subprocess/command execution, workspace write/edit/patch, reference execution/import/install, dependency install, raw response persistence, raw model output persistence, persistent trace write, UI runtime, external control, authority grant, or unsafe readiness flag is introduced.

## Handoff

v0.34.3 hands off only to v0.34.4 Existing Provider Boundary Adapter and v0.34.5 Model Output Action Quarantine as design-stage work. It does not enable provider invocation or action execution.
