# v0.34.8 CLI Model-backed Agent Step Surface

Parent track: Controlled Model Invocation Boundary.

v0.34.8 adds bounded CLI-facing model-backed handlers. It builds on v0.34.7 Model Invocation OCEL Trace Packet, v0.34.6 Agent Step Runner Model Integration, v0.34.5 Model Output Action Quarantine, v0.34.4 Existing Provider Boundary Adapter, and v0.33.8 CLI Agent Run Surface patterns.

## Definition

The CLI Model-backed Agent Step Surface parses CLI-like argv lists into in-memory invocation artifacts, evaluates them against a conservative CLI policy, and dispatches only to existing bounded v0.34 helpers. CLI surface is not shell execution. CLI argv parsing is not OS command execution. CLI dispatch is not direct provider invocation.

Safe commands include model help/status/dry-run, request envelope preview, response sanitize preview, response quarantine preview, mock/supplied model-backed ask or step, controlled model step through runtime context, trace preview, and no-op. Unknown and unsafe commands are denied or blocked by default.

## Boundary

v0.34.8 may expose bounded CLI-facing model-backed handlers. It does not execute shell commands. It does not use subprocess. It does not directly call providers. It does not import provider SDKs. It does not use direct network libraries. It does not read credentials or secrets. It does not enable autonomous general agent runtime. It does not execute general tools. It does not write, edit, or patch files. It does not generate patch proposals. It does not persist trace files or database records.

CLI output rendering is returned in memory and is not file write. CLI trace preview uses v0.34.7 returned trace packet creation only and is not persistent trace write. Bounded model-backed step uses v0.34.6. Controlled model-backed CLI step uses v0.34.6 and, if needed, v0.34.4 only when runtime context and policy allow it.

## Runtime Context Policy

CLI args alone never create provider authority. A controlled model command requires an explicit runtime context containing v0.34.6 runner and v0.34.4 adapter/callable refs. The CLI surface does not construct provider endpoints, read credentials, or call provider SDKs.

## Prompt And Response Policy

Prompt and response arguments are bounded and redacted in output artifacts. They may be used only to build in-memory request envelope previews, response sanitizer envelopes, quarantine packets, or v0.34.6 bounded step inputs. Raw prompt, raw response, and raw model output are not persisted.

## Reference Policy

references/OpenCode and references/Hermes remain safe reference-context assets only. v0.34.8 does not execute OpenCode/Hermes/OpenClaw, import reference code, install dependencies, or run reference tests/package scripts. Commands asking to run reference harnesses or install dependencies are blocked.

## Flow

`argv list / CLI policy / runtime context -> CLIModelBackedInvocation -> CLIModelBackedInvocationDecision -> optional v0.34.2/v0.34.3/v0.34.5/v0.34.6/v0.34.7 bounded call -> CLIModelBackedRunOutput -> CLIModelBackedRunReport -> V0348ReadinessReport -> v0.34.9 Consolidation`

## Acceptance Criteria

v0.34.8 is acceptable only if CLI model-backed taxonomies, flags, specs, policy, surface, invocation, decision, runtime context, denied command, command result, run output, run report, preview, guarantee, readiness report, parser, evaluator, runner, renderer, and preservation helpers exist while every shell, provider, SDK, network, credential, general runtime, general tool, write, edit, patch, persistence, UI, control, and authority path remains blocked.

## Withdrawal Conditions

Withdraw v0.34.8 readiness if any CLI argv is passed to a shell/subprocess, any direct provider or provider SDK call appears, any direct network or credential path appears, any general agent/tool runtime appears, any workspace write/edit/patch or patch proposal appears, any persistent trace write appears, or any unsafe readiness flag becomes true.

## Handoff

v0.34.8 hands off only to v0.34.9 Controlled Model Invocation Boundary Consolidation. It does not enable shell, direct provider calls, autonomous runtime, general tools, write/edit/patch, patch proposals, or persistent trace storage.
