# v0.34.7 Model Invocation OCEL Trace Packet

Parent track: Controlled Model Invocation Boundary.

v0.34.7 adds bounded Model Invocation OCEL Trace Packet artifacts. It builds on v0.34.6 Agent Step Runner Model Integration, v0.34.5 Model Output Action Quarantine, v0.34.4 Existing Provider Boundary Adapter, v0.34.3 Model Response Envelope & Sanitizer, and v0.34.2 Model Request Envelope.

## Definition

A Model Invocation OCEL Trace Packet is a returned, bounded, in-memory trace artifact for controlled model invocation lifecycle metadata. It may describe request envelopes, provider-boundary result artifacts, response envelopes, quarantine packets, and model-backed step outputs that were supplied to it. It does not perform provider invocation, existing-boundary invocation, agent step execution, tool execution, workspace inspection, or action execution.

Model invocation trace is not model invocation. Provider boundary call record trace is not provider boundary invocation. Trace event creation is metadata construction, not an external side effect. Trace object and relation creation are metadata construction, not registry mutation. Runtime trace packet is not persistent storage.

## Persistence Boundary

v0.34.7 creates bounded returned/in-memory trace packets only. It does not write trace files, databases, or logs. It does not use external trace sinks. It does not persist raw prompts, raw responses, or raw model output. `returned_trace_packet` is the default sink. `in_memory_test_sink` is tests-only metadata. Persistent and external sinks remain blocked.

## Redaction Policy

Trace attributes are bounded, redacted, and truncated. Raw prompt text, raw response text, raw model output, credential values, secret values, API keys, tokens, PEM material, id_rsa-like values, full file contents, and unbounded tool output are blocked or redacted. Attribute values are summaries, ids, counts, statuses, route labels, outcome labels, and safe refs only.

Reference context paths such as references/OpenCode and references/Hermes may appear only as bounded refs. v0.34.7 does not read reference files, execute OpenCode/Hermes/OpenClaw, import reference code, install reference dependencies, or run reference tests/package scripts.

## Taxonomies

v0.34.7 defines event, object, relation, attribute, sink, status, decision, risk, source, and readiness taxonomies. Unknown values are conservative. Events, objects, and relations are trace artifacts only.

## Hard Prohibitions

v0.34.7 does not call providers. It does not invoke existing provider boundary. It does not execute agent steps. It does not execute tools. It does not inspect workspace files. It does not read files. It does not write files. It does not write trace files/database/logs. It does not use external trace sinks. It does not persist raw prompts. It does not persist raw responses.
